编辑消息安全性配置

要为在 Enterprise Server 中部署的 Web 服务端点启用消息安全性，必须指定要在服务器端默认使用的提供者。如果为消息安全性启用了默认提供者，您还需要启用要由 Enterprise Server 中部署的 Web 服务客户机所使用的提供者。启用应用程序客户机的消息安全性中介绍了有关启用客户机使用的提供者的信息。

要为源于已部署的端点的 Web 服务调用启用消息安全性，您必须指定默认客户机提供者。如果已为 Enterprise Server 启用了默认客户机提供者，则必须确保从 Enterprise Server 中部署的端点中调用的所有服务均已配置为与消息层安全性兼容。

  1. 在导航树中，展开“配置”节点。
  2. 展开“安全性”节点。
  3. 选择“消息安全性”节点。

    此时将打开“消息安全性配置”页面。

  4. 在“消息安全性配置”页面上，单击要编辑的配置的验证层名称。

    默认情况下，唯一配置的消息安全性配置为 SOAP。

    此时将打开“编辑消息安全性配置”页面。

  5. 在“编辑消息安全性配置”页面上，从“默认提供者”下拉式列表中，选择服务器端提供者。

    对于不标识特定服务器提供者的应用程序，默认服务器提供者（也称为默认提供者）会标识为其调用的服务器提供者。可用选项为 ServerProvider、XWS_ServerProvider 或 none。默认情况下，未选择提供者，因此 Enterprise Server 将不会在服务器端调用消息安全性提供者。通常，请选择 ServerProvider 作为默认提供者。

  6. 从“默认客户机提供者”下拉式列表中，选择客户机提供者。

    对于不标识特定客户机提供者的应用程序，默认客户机提供者可标识为其调用的客户机提供者。可用选项为 ClientProvider、XWS_ClientProvider 或 none。默认情况下，未选择提供者，因此 Enterprise Server 将不会在客户机端调用消息安全性提供者。通常，请为默认客户机提供者选择 "none"。要启用默认提供者和消息保护策略以应用到源于 Enterprise Server 上所部署的 Web 服务端点的 Web 服务调用，请选择 ClientProvider。

  7. 单击“保存”。
  8. 有关如何修改已启用的客户机或服务器提供者的消息保护策略的说明，请参见编辑消息安全性提供者配置。
