设置高级应用程序配置

使用“应用程序配置”页面可以设置一些属性，通过这些属性可帮助确保能够检测到对部署的应用程序所做的更改并重新装入已修改的类。

  1. 在导航树中，选择 "Application Server" 节点。

    将打开“常规信息”页面。

  2. 在“常规信息”页面上，单击“高级”选项卡。

    此时将打开“应用程序配置”页面。

  3. 选中“启用重新装入”复选框以启用应用程序的动态重新装入。

    如果已启用动态重新装入，则在更改应用程序或模块的代码或部署描述符时，无需重新对其进行部署。只需将经过更改的 JavaServer PagesTM (JSPTM) 或类文件复制到应用程序或模块的部署目录中。服务器将定期检查部署目录中的更改，并自动使用这些更改动态重新部署应用程序。这在开发环境中很有用，因为它允许快速测试代码更改。但在生产环境中，动态重新装入可能会使性能降低。此外，每当完成重新装入时，当时的会话都将无效。客户机必须重新启动该会话。

    默认情况下启用此选项。

  4. 在“重新装入轮询时间间隔”字段中，键入检查应用程序和模块的代码更改并动态重新装入时要采用的时间间隔（以秒为单位）。

    默认值为 2。

  5. 在“管理会话超时”字段中，键入管理控制台会话超时之前处于非活动状态的分钟数。

    默认值为 60。

  6. 选中“启用自动部署”复选框，以启用应用程序的自动部署。

    如果将应用程序或模块文件（JAR、WAR、RAR 或 EAR）复制到特殊目录中，然后启用自动部署，则 Enterprise Server 将自动部署应用程序或模块。默认情况下启用此选项。

  7. 在“自动部署轮询时间间隔”字段中，键入检查应用程序和模块的代码更改并动态重新装入时要采用的时间间隔（以秒为单位）。

    默认值为 2。更改轮询时间间隔不会影响部署应用程序或模块所需的时间。

  8. 在“自动部署重试超时”字段中，键入部分复制的文件在发生错误前可以保持大小不变的秒数。

    默认值为 4。

  9. 在“自动部署目录”字段中，键入要监视的目录的完整路径，以便对应用程序进行自动部署。

    默认值为 domain-dir/autodeploy。

  10. 从“XML 验证”下拉式列表中，选择要对标准部署描述符和 Enterprise Server 部署描述符执行的 XML 验证的类型。

    类型可以为以下任意一种：

    full

    如果 XML 验证失败，则部署将失败。该值为默认值。

    parsing

    报告 XML 验证错误，但会执行部署。

    none

    不执行 XML 验证。

  11. 选中“已启用验证器”复选框以验证部署描述符文件。

    验证器将在自动部署之前检查文件的结构和内容。如果验证失败，则部署将不会执行。大型应用程序的检验通常会很耗费时间。默认情况下禁用此选项。

  12. 选中“已启用预编译”复选框以预编译任何 JSP 文件。

    如果禁用此选项，则在运行时首次访问 JSP 文件时对其进行编译。默认情况下禁用此选项。

  13. （可选的）在“其他属性”部分中，指定其他属性。

    Enterprise Server 不定义应用程序的任何其他属性。

    • 要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。
    • 要修改属性，请编辑该属性的“值”字段。
    • 要删除属性，请选中位于您要删除属性的“名称”字段左边的复选框，然后单击“删除属性”按钮。
  14. 单击“保存”。

    提示 - 要返回到默认设置，请单击“装入默认值”按钮，然后单击“保存”。

  15. 重新启动服务器。
