配置 JVM 常规设置

Java 虚拟机 (Java Virtual Machine, JVM) 使 Java 字节码可以在物理计算机上运行。调整 JVM 设置可以提高性能和避免出现内存分配错误。

Java 虚拟机包含在 Java Platform, Standard Edition（Java SE 平台）中，该平台是 Enterprise Server 所需要的。不正确的 JVM 设置将导致服务器无法运行，因此在更改这些设置时应当谨慎。

注 - 术语“Java 虚拟机”和 "JVM" 表示用于 Java 平台的虚拟机。

  1. 在导航树中，选择“配置”节点。
  2. 在“配置”节点下，选择“JVM 设置”节点。

    此时将打开“JVM 常规设置”页面。

  3. 在“JVM 常规设置”页面的“Java 主目录”字段中，键入 Java SE 软件安装目录的名称。

    Enterprise Server 依赖于 Java SE 平台。要验证此发行版是否支持您指定的 Java SE 软件版本，请参见《Sun GlassFish Enterprise Server v3 Release Notes》。如果输入的目录名称不存在或输入不受支持的 Java SE 软件版本的目录，将无法启动 Enterprise Server。

  4. 在“Javac 选项”字段中，键入 Java 编程语言编译器的命令行选项。

    部署 Enterprise JavaBeansTM (EJBTM) 组件后，Enterprise Server 将运行编译器。

  5. 选中“启用调试”复选框，以便使用 Java 平台调试器体系结构 (Java Platform Debugger Architecture, JPDA) 启用调试。

    JPDA 供应用程序开发者使用。有关更多信息，请参见《Sun GlassFish Enterprise Server v3 Preview Application Development Guide》。

  6. 在“调试选项”字段中，键入要传递给 JVM 的 JPDA 选项。

    仅当在上一步中启用调试时，此字段才适用。

  7. 在“RMI 编译选项”字段中，键入 rmic 编译器的命令行选项。

    部署 EJB 组件后，Enterprise Server 将运行 rmic 编译器。

  8. 在“字节码预处理程序”字段中，键入类名列表。

    使用逗号分隔类名。每个类都必须实现 com.sun.appserv.BytecodePreprocessor 接口。将按指定顺序调用这些类。

  9. （可选的）在“其他属性”部分中，指定其他属性。

    要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。

    Enterprise Server 不定义 JVM 的任何其他属性。

  10. 单击“保存”。
  11. 重新启动服务器。
