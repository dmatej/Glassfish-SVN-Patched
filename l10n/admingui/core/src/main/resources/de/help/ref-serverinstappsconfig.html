Anwendungskonfiguration

Auf dieser Seite können Sie die Eigenschaften für die Anwendungskonfiguration festlegen, um sicherzustellen, dass Änderungen an bereitgestellten Anwendungen ermittelt und die geänderten Klassen neu geladen werden.

Die Seite Applications Configuration (Anwendungskonfiguration) umfasst die folgenden Optionen.

Reload (Neu laden)

Wenn diese Option aktiviert ist, werden geänderte Klassen dynamisch neu geladen. Der Server prüft das Bereitstellungsverzeichnis regelmäßig auf Änderungen und stellt die Anwendung automatisch und dynamisch, einschließlich der Änderungen, erneut bereit. Diese Funktion ist besonders in einer Entwicklungsumgebung sehr nützlich, da Codeänderungen schnell getestet werden können. In einer Produktionsumgebung hingegen kann das dynamische erneute Laden zu einer Beeinträchtigung der Leistung führen. Darüber hinaus werden bei jedem erneuten Laden die Sitzungen ungültig, die zum Zeitpunkt des Ladevorgangs bestehen. Der Client muss die Sitzung neu starten. Diese Option ist standardmäßig aktiviert.

Reload Poll Interval (Intervall für Neuladen des Abrufs)

Legt fest, wie häufig der Server bereitgestellte Anwendungen und Module auf Code-Änderungen überprüfen soll. Der Standardwert ist 2.

Admin Session Timeout (Zeitüberschreitung Admin-Sitzung)

Gibt in Minuten die Dauer der Inaktivität an, bevor Administration Console eine Zeitüberschreitung erfährt und Sie sich erneut anmelden müssen. Der Standardwert ist 60.

Auto Deploy (Automatisch bereitstellen)

Wenn Sie diese Option aktivieren, werden die Anwendungen im Verzeichnis für automatische Bereitstellung automatisch bereitgestellt. Diese Option ist standardmäßig aktiviert.

Auto Deploy Poll Interval (Abrufintervall für automatische Bereitstellung)

Legt fest, wie häufig der Server das Verzeichnis für automatische Bereitstellung auf Anwendungs- oder Moduländerungen überprüfen soll. Das Ändern des Abrufintervalls hat keine Auswirkungen auf die Bereitstellungszeit von Anwendungen oder Modulen. Der Standardwert ist 2.

Auto Deploy Retry Timeout (Zeitüberschreitung für Wiederholungen der automatischen Bereitstellung)

Gibt in Sekunden an, wie lange die Größe einer teilweise kopierten Datei unverändert bleiben kann, bevor ein Fehler auftritt. Der Standardwert ist 4.

Wenn eine Datei langsam in ein Verzeichnis für automatische Bereitstellung kopiert wird, kann die Datei angezeigt werden, bevor die gesamte Datei kopiert ist. Folglich schlägt der Versuch, die Anwendung automatisch bereitzustellen, fehl. Wenn ein Versuch eine Anwendung automatisch bereitzustellen aus diesem Grund fehlschlägt, versucht Enterprise Server die Anwendung erneut bereitzustellen. Wenn die Größe der teilweise kopierten Datei für die angegebene Zeitdauer unverändert bleibt, tritt ein Fehler auf.

Auto Deploy Directory (Verzeichnis für automatische Bereitstellung)

Gibt das Verzeichnis an, das für die automatische Bereitstellung von Anwendungen überwacht wird. Der Standardwert ist domain-dir/autodeploy.

XML-Validierung

Legt den Typ der XML-Validierung fest, der bei Standard und Enterprise Server Bereitstellungsbeschreibungen ausgeführt werden soll. Der Typ kann einen der folgenden Werte annehmen:

Vollständig

Wenn die XML-Validierung fehlschlägt, schlägt die Bereitstellung fehl. Dies ist der Standardwert.

Parsing

Fehler bei der XML-Validierung werden protokolliert, die Bereitstellung wird jedoch durchgeführt.

Keine

Es wird keine XML-Validierung durchgeführt.

Verifier

Wenn Sie diese Option aktivieren, wird der Verifier vor der automatischen Bereitstellung ausgeführt. Wenn die Verifizierung fehlschlägt, wird die Bereitstellung nicht ausgeführt. Der Verifier überprüft Struktur und Inhalt der Datei. Die Überprüfung von großen Anwendungen ist häufig sehr zeitaufwendig. Die Option ist standardmäßig deaktiviert.

Precompile (Vorkompilieren)

Wenn Sie diese Option aktivieren, werden JavaServer PagesTM (JSPTM)-Dateien vorkompiliert. Wenn Sie diese Option deaktivieren, werden die JSP-Dateien beim ersten Zugriff zur Laufzeit kompiliert. Diese Option ist standardmäßig deaktiviert.

Additional Properties (Weitere Eigenschaften)

Zusätzliche Eigenschaften für Anwendungen Der Enterprise Server definiert keine zusätzlichen Eigenschaften für Anwendungen.
