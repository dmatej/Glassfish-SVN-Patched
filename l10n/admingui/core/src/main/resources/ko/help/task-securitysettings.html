보안 설정을 구성하는 방법

관리 콘솔에서는 다양한 시스템 차원의 보안 설정을 구성할 수 있습니다.

  1. 탐색 트리에서 구성 노드를 선택합니다.
  2. 구성 노드에서 보안 노드를 선택합니다.

    보안 페이지가 열립니다.

  3. 보안 관리자 활성화 확인란을 선택하여 보안 관리자를 켭니다.

    이 옵션을 활성화하면 JVM 옵션 -Djava.security.manager가 Enterprise Server의 JVM 설정에 추가됩니다. JVM 옵션을 구성하는 방법을 참조하십시오. 이 변경 사항을 활성화하려면 서버를 다시 시작해야 합니다.

    모든 응용 프로그램에 올바른 권한을 부여했는지 확인하십시오. 성능 향상을 위해 보안 관리자를 비활성화할 수 있습니다. 이 옵션은 기본적으로 비활성화됩니다.

  4. 감사 로깅을 켜려면 감사 로깅 사용 가능 확인란을 선택합니다.

    이 옵션을 활성화하면 서버가 감사 모듈 목록에서 선택한 모든 감사 모듈을 로드 및 실행합니다. 이 옵션을 비활성화하면 서버가 감사 모듈에 액세스되지 않습니다. 이 옵션은 기본적으로 비활성화됩니다.

  5. 기본 영역 드롭다운 목록에서 서버가 인증을 위해 사용하는 활성 영역을 선택합니다.

    배포 설명자가 다른 영역을 지정하지 않는 한 응용 프로그램은 이 영역을 사용합니다. 구성된 모든 영역이 목록에 표시됩니다. 기본값은 file입니다.

  6. 기본 기본값 정보를 지정합니다.
    1. 기본 기본값 필드에서 기본 사용자 이름을 입력합니다.

      다른 기본값을 제공하지 않을 경우 서버는 이 사용자 이름을 사용합니다.

    2. 기본 기본값 비밀번호 필드에서, 기본 기본값 필드에서 지정한 기본 기본값의 비밀번호를 입력합니다.

    정상적인 서버 작동에 대해서는 기본 기본값이 필요하지 않습니다.

  7. JACC 드롭다운 목록에서 구성된 JACC 공급자의 이름을 선택합니다.

    기본 선택 항목은 default 및 simple입니다. 기본 옵션은 default입니다.

  8. 감사 모듈 드롭다운 목록에서 감사 로깅이 활성화된 경우 감사 하위 시스템이 사용할 감사 공급자 모듈을 선택합니다.

    기본적으로 서버는 이름이 default인 감사 모듈을 사용합니다. 새 감사 모듈 만들기에 대한 자세한 내용은 감사 모듈을 만드는 방법을 참조하십시오.

  9. 응용 프로그램별 매핑이 없는 응용 프로그램에 기본과 역할 사이의 기본 매핑을 적용하려면 역할 매핑에 대한 기본 기본값 사용 가능 확인란을 선택합니다.
  10. 기본과 역할 사이의 기본 매핑을 활성화한 경우 매핑된 기본 클래스 필드에 기본과 역할 사이의 기본 매핑에 사용될 사용자 정의된 java.security.Principal 구현 클래스의 이름을 입력합니다.
  11. (옵션) 추가 등록 정보 섹션에서 추가 등록 정보를 지정합니다.

    유효 등록 정보는 기본 영역 필드에서 선택한 영역 유형에 따라 달라집니다.

    • 파일 영역 등록 정보에 대한 설명은 FileRealm 클래스와 관련된 등록 정보를 참조하십시오.

    • 인증서 영역 등록 정보에 대한 설명은 CertificateRealm 클래스와 관련된 등록 정보를 참조하십시오.

    • JDBC 영역 등록 정보에 대한 설명은 JDBCRealm 클래스와 관련된 등록 정보를 참조하십시오.

    • LDAP 영역 등록 정보에 대한 설명은 LDAPRealm 클래스와 관련된 등록 정보를 참조하십시오.

    • Solaris 영역 등록 정보에 대한 설명은 SolarisRealm 클래스와 관련된 등록 정보를 참조하십시오.

    • 등록 정보를 추가하려면 등록 정보 추가 버튼을 누릅니다. 빈 행이 표시되면 이름 필드에 등록 정보 이름을 입력하고 값 필드에 등록 정보 값을 입력합니다.
    • 등록 정보를 수정하려면 등록 정보의 값 필드를 편집합니다.
    • 등록 정보를 삭제하려면 삭제할 등록 정보의 이름 필드 왼쪽에 있는 확인란을 선택한 다음 등록 정보 삭제 버튼을 누릅니다.
  12. 저장을 누릅니다.
