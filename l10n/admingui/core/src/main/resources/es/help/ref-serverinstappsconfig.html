Configuración de aplicaciones

Utilice la página Configuración de aplicaciones para establecer las propiedades que permiten garantizar que los cambios realizados en las aplicaciones implementadas se detecten y las clases modificadas se vuelven a cargar.

La página Configuración de aplicaciones contiene las siguientes opciones.

Recargar

Si esta opción está habilitada, las clases modificadas se recargarán de forma dinámica. El servidor comprueba periódicamente los cambios realizados en el directorio de implementación y vuelve a implementar la aplicación de forma automática y dinámica con los cambios. Esto es útil en un entorno de desarrollo ya que permite poner a prueba los cambios realizados en el código de forma rápida. En cambio, en un entorno de producción la recarga dinámica puede hacer que disminuya el rendimiento. Además, cuando ha finalizado la recarga, las sesiones del periodo de transición pierden su validez. El cliente debe reiniciar la sesión. Esta opción está habilitada por defecto.

Intervalo de sondeo de recarga

Identifica la frecuencia con que el servidor debería comprobar los cambios de código en las aplicaciones y módulos implementados. El valor predeterminado es 2.

Tiempo de espera de sesión de administración

Identifica el número de minutos de inactividad antes de que se agote el tiempo de espera de Consola de administración y se deba iniciar una sesión de nuevo. El valor predeterminado es 60.

Implementación automática

Si esta opción está habilitada, las aplicaciones del directorio de implementación automática se implementan automáticamente. Esta opción está habilitada por defecto.

Intervalo de sondeo de implementación automática

Especifica la frecuencia con que el servidor debería comprobar el directorio de implementación automática para comprobar los cambios del módulo o de la aplicación. Si cambia el intervalo de sondeo no quedará afectado el tiempo que se tarda en implementar una aplicación o un módulo. El valor predeterminado es 2.

Tiempo de espera de reintento de implementación automática

Especifica el número de segundos que el tamaño de un archivo copiado parcialmente puede permanecer sin cambios antes de que se produzca un error. El valor prededeterminado es 4.

Si se copia un archivo lentamente en el directorio de implementación automática, éste puede aparecer antes de que se copie el archivo completo. Como resultado, se produce un error al intentar implementar la aplicación de forma automática. Si se produce un error al intentar implementar una aplicación de forma automática por este motivo, Servidor empresarial intentará implementar de nuevo la aplicación. Si el tamaño del archivo copiado parcialmente permanece sin cambios durante el período de tiempo especificado, se producirá un error.

Directorio de implementación automática

Identifica el directorio para supervisar la implementación automática de las aplicaciones. El valor predeterminado es domain-dir/autodeploy.

Validación XML

Especifica el tipo de validación XML que se llevará a cabo en los descriptores de implementación estándar y de Servidor empresarial. El tipo puede ser cualquiera de los siguientes:

Completo

Si se produce un error en la validación XML, no se realizará la implementación. Éste es el valor predeterminado.

Análisis

Se informa acerca de errores en la validación XML, pero se realiza la implementación.

Ninguno

No se realiza ninguna implementación XML.

Verificador

Si se habilita esta opción, el verificador se ejecuta antes de la implementación automática. Si se produce un error en la verificación, no se realiza la implementación. El verificador examinará la estructura y contenido del archivo. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción se encuentra deshabilitada de forma predeterminada.

Precompilar

Si está habilitada esta opción, los archivos JavaServer PagesTM ( JSPTM) se precompilarán. Si se deshabilita esta opción, los archivos JSP se compilarán en el tiempo de ejecución cuando se acceda a los mismos por primera vez. Esta opción se encuentra deshabilitada de forma predeterminada.

Propiedades adicionales

Propiedades adicionales para las aplicaciones. Servidor empresarial no define ninguna propiedad adicional para las aplicaciones.
