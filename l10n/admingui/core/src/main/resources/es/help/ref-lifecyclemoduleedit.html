Editar módulo de ciclo de vida

Consulte la página Editar módulo de ciclo de vida para modificar la configuración del módulo de ciclo de vida.

La página Editar módulo de ciclo de vida contiene las opciones siguientes.

Nombre

Nombre del módulo del ciclo de vida. Éste campo es de sólo lectura. Únicamente puede especificar un nombre al crear un nuevo módulo de ciclo de vida.

Nombre de clase

Nombre completo del archivo de clase del módulo de ciclo de vida. La clase debe implementar la interfaz com.sun.appserv.server.LifecycleListener.

Ruta de clase

Ruta de clase para el módulo de ciclo de vida. La ruta de clase especifica la ubicación del módulo de ciclo de vida. El valor predeterminado es domain-dir/applications . Si el módulo ya se encuentra en este directorio (es decir, en la ruta de clase del servidor), este campo puede dejarse en blanco.

Orden de carga

Orden en el que este módulo de ciclo de vida se cargará en el arranque. Los módulos con valores de orden de carga inferiores se cargarán antes. El valor puede estar entre 101 y el MAXINT del sistema operativo. Los valores de 0 a 100 están reservados.

Estado

Si se selecciona esta opción, el módulo de ciclo de vida estará habilitado. Esta opción está habilitada por defecto.

Error en la carga

Si se selecciona esta opción, el servidor se apagará si se produce un error en la carga del módulo de ciclo de vida. Esta opción se encuentra deshabilitada de forma predeterminada.

Servidores virtuales

Servidor o servidores virtuales que se asociarán con el módulo de ciclo de vida. De manera predeterminada, no hay ningún servidor virtual seleccionado.

Descripción

Descripción del módulo de ciclo de vida.

Propiedades adicionales

Propiedades adicionales del módulo de ciclo de vida. Servidor empresarial no define ninguna propiedad adicional para los módulos de ciclo de vida.
