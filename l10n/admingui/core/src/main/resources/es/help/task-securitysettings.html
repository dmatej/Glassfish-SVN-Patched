Cómo configurar las opciones de seguridad

La Consola de administración permite establecer varios valores de seguridad en todo el sistema.

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Seguridad.

    Se abrirá la página Seguridad.

  3. Marque la casilla de verificación Administrador de seguridad activado para activar el administrador de seguridad.

    Si se activa esta opción, se agregará una opción de JVM, -Djava.security.manager , a los valores de configuración de JVM de Servidor empresarial. Consulte Para configurar las opciones de JVM. Debe reiniciar el servidor para habilitar este cambio.

    Asegúrese de que ha otorgado los permisos correctos para todas las aplicaciones. Puede deshabilitar el administrador de seguridad para mejorar el rendimiento. Esta opción se encuentra deshabilitada de forma predeterminada.

  4. Marque la casilla de verificación Registro de auditoría activado para habilitar el registro de auditoría.

    Si esta opción está activada, el servidor cargará y ejecutará todos los módulos de auditoría seleccionados de la lista Módulos de auditoría. Si esta opción está desactivada, el servidor no accederá a los módulos de auditoría. Esta opción se encuentra deshabilitada de forma predeterminada.

  5. En la lista desplegable Dominio predeterminado, seleccione un dominio activo que el servidor utilizará para la autenticación.

    Las aplicaciones utilizarán este dominio siempre que el descriptor de implementación no especifique un dominio diferente. Todos los dominios configurados aparecen en la lista. El valor predeterminado es file.

  6. Especifique la información de Principal predeterminado.
    1. En el campo Principal predeterminado, introduzca el nombre de usuario predeterminado.

      El servidor utilizará este nombre de usuario si no se proporciona otro principal.

    2. En el campo Contraseña de principal predeterminado, introduzca la contraseña del valor principal predeterminada especificada en el campo Principal predeterminado.

    No se requiere un principal predeterminado para el funcionamiento normal de un servidor.

  7. En la lista desplegable JACC, seleccione el nombre de un proveedor JACC configurado.

    Las opciones predeterminadas son default y simple. La opción predeterminada es default.

  8. En la lista desplegable Módulos de auditoría, seleccione los módulos del proveedor de auditoría que utilizará el subsistema de auditoría si está habilitado el registro de auditoría.

    Por defecto el servidor utiliza un módulo de auditoría llamado default. Para obtener información sobre la creación de nuevos módulos de auditoría, consulte Cómo crear un módulo de auditoría.

  9. Seleccione la casilla de verificación Asignación de principal a función predeterminada habilitada para aplicar la asignación de principal a función predeterminada a las aplicaciones que no tengan un sistema de asignación específico.
  10. Si ha habilitado Asignación de principal a función predeterminada, en el campo Clases principales asignadas, introduzca el nombre de una clase de implementación java.security.Principal personalizada que se utilizará en la asignación de principal a función predeterminada.
  11. (Opcional) En la sección Propiedades adicionales, especifique las propiedades adicionales.

    Las propiedades válidas dependen del tipo de dominio seleccionado en el campo Dominio predeterminado.

    • Para obtener una descripción de las propiedades del dominio de archivo, consulte Propiedades específicas de la clase FileRealm.

    • Para obtener una descripción de las propiedades del dominio de certificado, consulte Propiedades específicas de la clase CertificateRealm .

    • Para obtener una descripción de las propiedades del dominio JDBC, consulte Propiedades específicas de la clase JDBCRealm .

    • Para obtener una descripción de las propiedades del dominio LDAP, consulte Propiedades específicas de la clase LDAPRealm .

    • Para obtener una descripción de las propiedades del dominio Solaris, consulte Propiedades específicas de la clase SolarisRealm .

    • Para añadir una propiedad, haga clic en el botón Agregar propiedad. En la fila vacía escriba el nombre de la propiedad en el campo Nombre y el valor de la propiedad en el campo Valor.
    • Para modificar una propiedad, edite el campo Valor de la propiedad.
    • Para eliminar una propiedad, marque la casilla de verificación situada a la izquierda del campo Nombre de la propiedad que quiera eliminar y, a continuación, haga clic en el botón Eliminar propiedades.
  12. Haga clic en Guardar.
