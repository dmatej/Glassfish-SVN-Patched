Edición de configuración SSL para el conector JMX

Antes de empezar

Esta tarea es relevante únicamente en caso de que la seguridad esté habilitada para el conector JMX.

  1. En el árbol de navegación, expanda el nodo Configuración.
  2. En el nodo Configuración, seleccione el nodo Servicio de administración.

    Se abrirá la página Editar conector JMX.

  3. Seleccione la casilla de verificación Seguridad habilitada para que la comunicación JMX esté encriptada.

    Esta opción se encuentra deshabilitada de forma predeterminada.

  4. Haga clic en la ficha SSL.

    Se abrirá la página SSL.

  5. En la página SSL, seleccione la casilla de verificación SSL3 habilitado para habilitar SSL3.

    Esta opción está habilitada por defecto.

  6. Seleccione la casilla de verificación TLS habilitado para habilitar la seguridad de la capa de transporte (TLS).

    Esta opción está habilitada por defecto.

  7. Selecciona la casilla de verificación Autenticación de clientes habilitada para solicitar a los clientes que se identifiquen en el servidor con cada solicitud.

    Esta opción se encuentra deshabilitada de forma predeterminada.

  8. En el campo Sobrenombre del certificado, introduzca el sobrenombre del certificado del servidor en la base de datos de certificados o token PKCS#11.

    En el certificado, el formato del nombre es tokenname:nickname. Incluyendo el tokenname: parte del nombre de este atributo es opcional. El valor predeterminado es s1as.

  9. En el campo Almacén de claves, introduzca el nombre del archivo de almacén de claves (por ejemplo, keystore.jks).
  10. En el campo Algoritmo de confianza, introduzca el nombre del algoritmo de administración de confianza (por ejemplo, PKIX) que se utilizará para la validación de rutas de certificación.
  11. En el campo Número máximo de certificados, introduzca el número máximo de certificados intermedios no autoemitidos que puede haber en una ruta de certificación.

    Este campo se utiliza solamente en caso de que el campo Algoritmo de confianza esté configurado en PKIX. Un valor de 0 implica que la ruta únicamente puede contener un certificado. Un valor de 1 implica que la longitud de la ruta no está restringida (no hay un máximo). La configuración en un valor inferior a -1 inicia una excepción. El valor predeterminado es 5.

  12. En el campo Almacén de confianza, introduzca el nombre de un archivo del almacén de confianza (por ejemplo, cacerts.jks).
  13. En el área Conjuntos de cifrado, especifique los conjuntos de cifrado que se utilizarán.

    Si no añade ningún conjunto de cifrado, se utilizarán todos los conjuntos de cifrado.

  14. Haga clic en Aceptar.
