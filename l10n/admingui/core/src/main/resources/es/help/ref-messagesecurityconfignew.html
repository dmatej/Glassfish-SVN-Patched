Nueva configuración de seguridad de mensajes

Consulte la página Nueva configuración de seguridad de mensajes para crear una configuración HttpServlet para la seguridad de mensajes.

La página Nueva configuración de seguridad de mensajes contiene las siguientes opciones.

Capa de autenticación

Capa del mensaje en la que se lleva a cabo la autenticación. El valor debe ser SOAP o HttpServlet. Debido a que de manera predeterminada está disponible la configuración SOAP, la única opción es HttpServlet.

Id. de proveedor

Único identificador de la configuración del proveedor para esta configuración de seguridad de mensajes.

Proveedor predeterminado

Si se selecciona esta opción, este proveedor será el proveedor predeterminado. El proveedor predeterminado será invocado para cualquier aplicación que no identifique un proveedor específico. Si el proveedor es el proveedor de cliente predeterminado, el proveedor de servidor predeterminado o ambos, viene determinado por el Tipo de proveedor.

Tipo de proveedor

Especifica el tipo de proveedor. Las opciones disponibles son:

client

Especifica que el proveedor es el proveedor de autenticación de clientes.

server

Especifica que el proveedor es el proveedor de autenticación del servidor.

client-server

Especifica que el proveedor es tanto el proveedor de autenticación del servidor como del cliente.

Nombre de clase

Nombre de la implementación de Java del proveedor. Los proveedores de autenticación de clientes deben implementar la com.sun.enterprise.security.jauth.ClientAuthModule interfaz. Los proveedores de servidor deben implementar la com.sun.enterprise.security.jauth.ServerAuthModule interfaz . Los proveedores de servidor y cliente deben implementar ambas interfaces.
