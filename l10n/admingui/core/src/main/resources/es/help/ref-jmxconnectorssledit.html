SSL

Consulte la página SSL para modificar la configuración SSL para el conector JMX.

La configuración que aparece en esta página es únicamente relevante si está habilitada la seguridad para el conector JMX en la página Editar conector JMX.

La página SSL contiene las siguientes opciones.

SSL3

Si se selecciona esta casilla de verificación, se habilitará el protocolo SSL3 para el conector JMX. Esta opción está habilitada por defecto.

TLS

Si se selecciona esta casilla de verificación, se habilitará el protocolo TLS·para el conector JMX. Esta opción está habilitada por defecto.

Autenticación de cliente

Si esta casilla de verificación está seleccionada, los clientes deben identificarse en el servidor con cada solicitud. Esta opción se encuentra deshabilitada de forma predeterminada.

Sobrenombre del certificado

Sobrenombre del certificado del servidor en la base de datos de certificados o token PKCS#11. En el certificado, el formato del nombre es tokenname:nickname. Incluyendo la tokenname: parte del nombre de este atributo es opcional. El valor predeterminado es s1as.

Almacén de claves

Nombre de archivo del almacén de claves (por ejemplo, keystore.jks).

Algoritmo de confianza

Nombre del algoritmo de administración de confianza (por ejemplo, PKIX) que se utilizará para la validación de rutas de certificación.

Longitud máxima del certificado

Número máximo de certificados intermedios no autoemitidos que puede haber en una ruta de certificación. Este campo se utiliza solamente en caso de que el campo Algoritmo de confianza esté configurado en PKIX. Un valor de 0 implica que la ruta únicamente puede contener un certificado. Un valor de -1 implica que la longitud de la ruta no está restringida (no hay un máximo). La configuración en un valor inferior a -1 inicia una excepción. El valor predeterminado es 5.

Almacén de confianza

Nombre de un archivo del almacén de confianza (por ejemplo, cacerts.jks).

Conjuntos de cifrado

Área en la que puede añadir o eliminar conjuntos de cifrado. Si no añade ningún conjunto de cifrado, se utilizarán todos los conjuntos de cifrado.
