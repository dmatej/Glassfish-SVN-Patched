Para configurar as definições gerais de JVM

A JavaTM Virtual Machine (JVM) permite que códigos de bytes Java sejam executados em um computador físico. O ajuste das configurações de JVM melhora o desempenho e evita erros de alocação de memória.

O Java Virtual Machine está incluído no Java Platform, Standard Edition (plataforma Java SE), que é exigido pelo Enterprise Server. Configurações de JVM incorretas impedirão que o servidor seja executado, portanto, tenha cuidado ao alterar essas configurações.

Observação - Os termos “Java Virtual Machine” e “JVM” significam uma Máquina Virtual para a plataforma Java.

  1. Na árvore de navegação, selecione o nó Configuração.
  2. Sob o nó Configuração, selecione o nó Configurações de JVM.

    A página Configurações Gerais de JVM é exibida.

  3. Na página Configurações Gerais de JVM, no campo Java Home, digite o nome do diretório de instalação do software Java SE.

    O Enterprise Server depende da plataforma Java SE. Para verificar se a versão do software Java SE especificada é compatível com esta versão, consulte Sun GlassFish Enterprise Server v3 Preview Release Notes . Se você especificar um nome de diretório inexistente ou se a versão do diretório não for compatível com o software Java SE, o Enterprise Server não será iniciado.

  4. No campo Opções Javac, digite as opções de linha de comando para o compilador de linguagem de programação Java.

    O Enterprise Server executa o compilador quando os componentes Enterprise JavaBeans TM (EJBTM) são implantados.

  5. Marque a caixa de seleção Depuração Ativada para ativar a depuração com o Java Platform Debugger Architecture (JPDA).

    O JPDA é utilizado pelos desenvolvedores de aplicativos. Para obter mais informações, consulte Sun GlassFish Enterprise Server v3 Preview Application Development Guide .

  6. No campo Opções de Depuração, digite as opções do JPDA a serem transmitidas para o JVM.

    Esse campo só será aplicável se a depuração tiver sido ativada na etapa precedente.

  7. No campo Opções de Compilação RMI, digite as opções de linha de comando para o compilador rmic.

    O Enterprise Server executa o compilador rmic quando os componentes EJB são implantados.

  8. No campo Pré-processador de Código de Byte, digite uma lista de nomes de classes.

    Use vírgulas para separar os nomes de classes. Cada classe deve implementar a interface com.sun.appserv.BytecodePreprocessor . As classes são chamadas na ordem especificada.

  9. (Opcional) Na seção Propriedades Adicionais, especifique essas propriedades.

    Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que é exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    O Enterprise Server não define propriedades adicionais para o JVM.

  10. Clique em Salvar.
  11. Reinicie o servidor.
