SSL

Use a página SSL para modificar as configurações SSL do conector JMX.

As configurações nesta página tem significado apenas se a segurança estiver ativada para o conector JMX na página Editar conector JMX.

A página SSL contém estas opções.

SSL3

Se a caixa de seleção estiver selecionada, o protocolo SSL3 está disponível para o conector JMX. Por padrão, a opção está ativada.

TLS

Se a caixa de seleção estiver selecionada, o protocolo TLS está disponível para o conector JMX. Por padrão, a opção está ativada.

Autenticação do Cliente

Se a caixa de seleção estiver selecionada, os clientes deverão identificar-se eles mesmos para o servidor a cada solicitação. Por padrão, a opção está desativada.

Nickname Certificado

O apelido do certificado do servidor no banco de dados de certificados ou o símbolo PKCS#11. No certificado, o formato do nome é tokenname:nickname. Incluindo tokenname: parte do nome nesse atributo é opcional. O valor padrão é s1as.

Armazenar senha

O nome do arquivo keystore (por exemplo, keystore.jks ).

Algoritmo Confiável

O nome do algoritmo de gerenciamento confiável (por exemplo, PKIX) a ser usado para validação do caminho de certificação.

Comprimento máx certificado

O número máximo de certificados intermediários não auto-emitidos que podem existir em um caminho de certificação. Este campo é usado apenas se o campo Algoritmo Confiável estiver definido como PKIX. O valor 0 indica que o caminho pode conter somente um certificado. O valor -1 indica que o comprimento do caminho não é restringido (não existe máximo). Definir um valor valor menor que -1 causa uma exceção a ser lançada. O valor padrão é 5.

Trust Store

O nome do arquivo truststore (por exemplo, cacerts.jks ).

Conjuntos Codificados

Uma área onde você pode adicionar ou remover conjuntos codificados. Se você não adicionar quaisquer conjuntos codificados, todos os conjuntos codificados serão usados.
