Segurança

Use a página Segurança para definir as propriedades de segurança do Enterprise Server.

A página Segurança contém estas opções.

Gerenciador de Segurança

Se esta opção for selecionada, o gerenciador de segurança do domínio será ativado. Por padrão, a opção está desativada.

Quando esta opção está ativada, a opção -Djava.security.manager, do JVM, é adicionada á configuração JVM do Enterprise Server. Consulte Para configurar as opções de JVM. É preciso reiniciar o servidor para ativar a alteração.

Verifique se você tem as permissões corretas para todos os aplicativos. Para melhorar o desempenho, desative o gerenciador de segurança.

Log de Auditoria

Se esta opção for ativada, o servidor carregará e executará todos os módulos de auditoria especificados na configuração Módulos de Auditoria. Se a opção for desativada, o servidor não acessará os módulos de auditoria. Por padrão, a opção está desativada.

Domínio Padrão

O domínio (padrão) ativo que o servidor usa para autenticação. Os aplicativos usam esse domínio, a menos que o descritor de implantação especifique um domínio diferente. Todos os domínios configurados são exibidos na lista. O valor padrão é arquivo.

Principal Padrão

Especifica o nome de usuário padrão. O servidor usa esse nome de usuário quando nenhum outro principal é fornecido. Se você digitar um valor nesse campo, digite a senha correspondente no campo Senha do Principal Padrão.

Um principal padrão não é exigido para operação normal do servidor.

Senha do Principal Padrão

A senha do principal padrão especificada no campo Principal Padrão. Um principal padrão não é exigido para operação normal do servidor.

JACC

O nome da classe de um provedor JACC configurado. O valor padrão é default.

Módulos de Auditoria

Os módulos do provedor de auditoria que serão usados pelo subsistema de auditoria se o log de auditoria estiver ativado. Por padrão, o servidor usa um módulo de auditoria chamado padrão. Para obter informações sobre a criação de novos módulos de auditoria, consulte Para criar um módulo de auditoria.

Mapeamento Principal Padrão para Função Ativado

Se esta opção for selecionada, o mapeamento principal-para-função será aplicado ao aplicativo que não tiver um mapeamento específico do aplicativo.

Classe de Principal Mapeada

Especifica uma implementação personalizada da classe java.security.Principal usada no mapeamento principal-para-função padrão, se o mapeamento principal-para-função padrão estiver ativado.

Propriedades Adicionais

Propriedades de segurança adicionais do servidor.

As propriedades válidas dependem do tipo de domínio selecionado no campo Domínio Padrão e, geralmente, são especificadas quando você edita um domínio. Para obter detalhes, consulte estes tópicos:

  • Propriedades específicas da classe FileRealm

  • Propriedades específicas da classe CertificateRealm

  • Propriedades específicas para a classe JDBCRealm

  • Propriedades específicas da classe LDAPRealm

  • Propriedades específicas da classe SolarisRealm
