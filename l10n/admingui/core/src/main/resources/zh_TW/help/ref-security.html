安全性

使用 [安全性] 頁面可設定 Enterprise Server 的安全性特性。

[安全性] 頁面包含下列選項。

安全性管理員

若選取此選項，則會啟用網域的安全性管理員。此選項預設為停用。

啟用此選項時，JVM 選項 -Djava.security.manager 會增加到 Enterprise Server 的 JVM 設定。請參閱配置 JVM 選項。您必須重新啟動伺服器才能啟用此變更。

確保所有應用程式都已授予正確的權限。 您可以關閉安全性管理員以提昇效能。

稽核記錄

若啟用此選項，伺服器將會載入並執行所有在 [稽核模組] 設定中指定的稽核模組。若停用此選項，伺服器將無法存取稽核模組。此選項預設為停用。

預設範圍

伺服器用於認證的使用中 (預設) 範圍。除非應用程式的部署描述元指定了不同的範圍，否則應用程式會使用此範圍。所有已配置的範圍都會顯示在清單中。預設值為 file。

預設主體

指定預設使用者名稱。若未提供任何其他主體，伺服器會使用此使用者名稱。如果在此欄位中鍵入值，請在 [預設主體密碼] 欄位中鍵入對應的密碼。

一般伺服器作業不需要預設主體。

預設主體密碼

在 [預設主體] 欄位中指定的預設主體之密碼。一般伺服器作業不需要預設主體。

JACC

已配置的 JACC 提供者的類別名稱。預設值為 default。

稽核模組

啟用稽核記錄時，由稽核子系統使用的稽核提供者模組。依預設，伺服器使用名為 default 的稽核模組。如需有關建立新稽核模組的資訊，請參閱建立稽核模組。

啟用預設主體至角色的對映

若選取此選項，即可將預設的主體至角色對映，套用於不具有應用程式專屬對映的應用程式。

對映的主體類別

若啟用預設主體至角色的對映，則指定預設主體至角色的對映中使用之 java.security.Principal 類別的自訂實作。

其他特性

伺服器的其他安全性特性。

有效的特性取決於 [預設範圍] 欄位中選取的範圍類型，並且通常是在編輯範圍時指定。如需詳細資訊，請參閱下列主題：

  • 專屬於 FileRealm 類別的特性

  • 專屬於 CertificateRealm 類別的特性

  • 專屬於 JDBCRealm 類別的特性

  • LDAPRealm 類別特有的特性

  • SolarisRealm 類別特有的特性
