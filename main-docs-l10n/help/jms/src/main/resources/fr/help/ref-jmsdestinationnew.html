Nouvelle ressource de destination JMS

La page Nouvelle ressource de destination JMS permet de créer une ressource de destination JMS.

Elle contient les options ci-après.

Nom JNDI

Nom unique identifiant la ressource de destination.

Il est recommandé d'utiliser le préfixe de dénomination de sous-contexte jms/ pour les ressources JMS. Par exemple : jms/Queue.

Nom de la destination physique

Nom de la destination physique à associer à cette ressource.

GlassFish Server crée la destination physique automatiquement lorsque cela est nécessaire et la supprime lorsque vous supprimez la ressource de destination.

Type de ressource

Type de la ressource de destination. Les options disponibles sont javax.jms.Topic et javax.jms.Queue.

Description

Description de la ressource de destination.

Statut

Statut de la ressource de destination. La ressource de destination peut être activée ou désactivée. Cette option est activée par défaut.

Autres propriétés

Propriétés supplémentaires de la ressource de destination. GlassFish Server ne définit aucune autre propriété pour les ressources de destination JMS.

Cibles

Clusters et instances autonomes de la ressource de destination. Déplacez les cibles souhaitées dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Enlever et Tout enlever.

Cette option apparaît uniquement si des clusters ou des instances autonomes ont été créés dans le domaine.
