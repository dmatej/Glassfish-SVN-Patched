Création d'une destination physique JMS

Vous n'avez généralement pas besoin de créer de destinations physiques, car vous spécifiez un nom de destination physique lorsque vous créez une ressource de destination JMS, et GlassFish Server crée automatiquement la destination physique si nécessaire.

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez l'instance à configurer :

    • Pour configurer une instance spécifique, sélectionnez le noeud de configuration de l'instance. Par exemple, pour l'instance par défaut, server, sélectionnez le noeud server-config.

    • Pour configurer les paramètres par défaut des futures instances utilisant une copie de default-config, sélectionnez le noeud default-config.

  3. Sous le noeud Configuration, développez le noeud Service JMS.

  4. Sous le noeud Service JMS, sélectionnez le noeud Destinations physiques.

    La page Destinations physiques apparaît.

  5. Sur la page Destinations physiques, cliquez sur Nouveau.

    La page Nouvelle destination physique apparaît.

  6. Dans le champ Nom, saisissez le nom de la destination physique.

  7. Dans la liste déroulante Type, sélectionnez javax.jms.Topic ou javax.jms.Queue.

  8. Dans le champ Nombre maximal de messages non distribués à un destinataire, saisissez le nombre maximal autorisé de messages non distribués pour la destination.

    La valeur -1 signifie que le nombre de messages non distribués est illimité. La valeur par défaut est -1.

  9. Dans le champ Taille maximale d'un message, saisissez la taille maximale, en octets, d'un message.

    La valeur -1 indique une taille de message illimitée. La valeur par défaut est -1.

  10. Dans le champ Mémoire totale maximale, saisissez la taille totale maximale de la mémoire, en octets, pour les messages non distribués.

    La valeur -1 signifie que la taille de la mémoire pour les messages non distribués est illimitée. La valeur par défaut est -1.

  11. Dans la liste déroulante Comportement aux limites de seuil, sélectionnez le comportement que le broker doit suivre lorsque le seuil limite de la mémoire est atteint.

    • Rejeter les derniers messages en date indique au broker de rejeter les messages les plus récents et d'en notifier le client émetteur à l'aide d'une exception, uniquement si le message est persistant.

      Rejeter les derniers messages en date est la valeur par défaut.

    • Ralentir les émetteurs indique au broker de réduire la fréquence à laquelle les émetteurs de messages envoient des messages.

    • Rejeter les éléments les plus anciens indique au broker de supprimer les messages les plus anciens.

    • Rejeter les messages dont la priorité est la plus basse indique au broker de supprimer les messages dont le niveau de priorité est le plus faible.

    Si la valeur est Rejeter les éléments les plus anciens ou Rejeter les messages dont la priorité est la plus basse et que le paramètre Utiliser la file d'attente des messages bloqués a la valeur True, les messages en trop sont transférés vers la file d'attente des messages bloqués.

  12. Dans le champ Nombre maximal d'émetteurs, saisissez le nombre maximal d'émetteurs de messages pouvant être associés à cette destination physique.

    Lorsque cette limite est atteinte, aucun émetteur supplémentaire ne peut être créé. La valeur -1 indique un nombre illimité d'émetteurs. La valeur par défaut est 100.

  13. Dans le champ Nombre maximal de messages lors de la distribution par batch, saisissez le nombre maximal de messages pouvant être transmis aux destinataires en un seul batch.

    La valeur -1 indique un nombre illimité de messages. La valeur par défaut est 1000.

  14. Dans la liste déroulante Utiliser la file d'attente des messages bloqués, sélectionnez True pour placer les messages bloqués dans la file d'attente des messages bloqués.

    Si la valeur est False, les messages bloqués sont supprimés. Cette option a la valeur True par défaut.

  15. Dans la liste déroulante Valider le schéma XML, sélectionnez True pour activer la validation de schéma XML pour la destination.

    Si la valeur est True, le programme client Message Queue tentera de valider un message XML par rapport aux définitions de schéma XML spécifiées (ou par rapport à la DTD si aucun fichier XSD n'est spécifié) avant de l'envoyer au broker. Si le schéma spécifié est introuvable ou que le message ne peut pas être validé, celui-ci n'est pas envoyé et une exception est générée. Cette option a la valeur False par défaut.

  16. Dans le champ Liste des URI du schéma XML, entrez la liste des chaînes d'URI de définition de schéma XML (XSD) séparées par des espaces.

    Si la validation de schéma XML est activée, les URI pointent vers l'emplacement des fichiers XSD à utiliser pour la validation de schéma XML.
