新規JMS物理宛先

新規JMS物理宛先を作成するには、「新規JMS物理宛先」ページを使用します。

通常は、JMS宛先リソースの作成時に物理宛先名を指定すると、GlassFish Serverが必要なときに物理宛先を自動的に作成するため、物理宛先を作成する必要はありません。

「新規JMS物理宛先」ページには、次のオプションが含まれます。

名前

物理宛先の名前。

タイプ

物理宛先のタイプ。選択可能な値は、javax.jms.Topicとjavax.jms.Queueです。

消費していないメッセージの最大数

宛先で許可されている未消費メッセージの最大数。-1を指定すると、未消費メッセージの数は制限されません。デフォルト値は-1です。

最大メッセージ・サイズ

単一メッセージの最大サイズ(バイト単位)。-1を指定すると、メッセージのサイズは制限されません。デフォルト値は-1です。

最大合計メモリー

未消費メッセージの最大合計メモリー(バイト単位)。-1を指定すると、未消費メッセージのメモリー・サイズは制限されません。デフォルト値は-1です。

しきい値制限の動作

メモリー制限のしきい値に達したときのブローカの動作。選択可能な値は次のとおりです:

最新メッセージの拒否

最新のメッセージを拒否し、メッセージを保持する場合のみ、メッセージを生成しているクライアントに例外を通知します。この値がデフォルト値です。

プロデューサのスロー・ダウン

メッセージ・プロデューサがメッセージを送信する頻度を下げます。

最も古いものをスロー

最も古いメッセージを破棄します。

最も優先度が低いメッセージをスロー

優先度の最も低いメッセージを破棄します。

値が「最も古いものをスロー」または「最も優先度が低いメッセージをスロー」で、「デッド・メッセージ・キューを使用」がtrueに設定されている場合、超過したメッセージはデッド・メッセージ・キューに送られます。

プロデューサの最大数

この物理宛先に関連付けることができるメッセージ・プロデューサの最大数。この制限に達すると、新しいプロデューサを作成できなくなります。-1を指定すると、プロデューサの数は制限されません。デフォルト値は100です。

バッチ配信の最大メッセージ数

単一のバッチでコンシューマに配送可能なメッセージの最大数。-1を指定すると、メッセージの数は制限されません。デフォルト値は1000です。

デッド・メッセージ・キューを使用

値をtrueに設定すると、デッド・メッセージがデッド・メッセージ・キューに送られます。値をfalseに設定すると、デッド・メッセージは破棄されます。デフォルトでは、このオプションはtrueに設定されます。

XMLスキーマの検証

値をtrueに設定すると、Message Queueクライアント・ランタイムはXMLメッセージをブローカに送信する前に、指定したXML Schema Definition (XSDが指定されていない場合はDTD)を使用してメッセージの検証を行います。指定したスキーマが見つからないかメッセージを検証できない場合、メッセージは送信されず例外がスローされます。デフォルトでは、このオプションはfalseに設定されます。

XMLスキーマURIリスト

XML Schema Definition (XSD) URI文字列の空白区切りのリスト。XMLスキーマ検証が有効な場合、URIはXMLスキーマ検証に使用する1つまたは複数のXSDの場所を指します。
