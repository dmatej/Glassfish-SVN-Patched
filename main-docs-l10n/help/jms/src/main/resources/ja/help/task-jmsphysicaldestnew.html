JMS物理宛先を作成するには

通常は、JMS宛先リソースの作成時に物理宛先名を指定すると、GlassFish Serverが必要なときに物理宛先を自動的に作成するため、物理宛先を作成する必要はありません。

  1. ナビゲーション・ツリーで、「構成」ノードを展開します。

  2. 構成するインスタンスを選択します:

    • 特定のインスタンスを構成するには、そのインスタンスの構成ノードを選択します。たとえば、デフォルト・インスタンスserverの場合は、server-configノードを選択します。

    • default-configのコピーを使用する将来のインスタンスのためにデフォルトの設定値を構成するには、default-configノードを選択します。

  3. 「構成」ノードで、「Javaメッセージ・サービス」ノードを展開します。

  4. 「Javaメッセージ・サービス」ノードで、「物理宛先」ノードを選択します。

    「物理宛先」ページが開きます。

  5. 「物理宛先」ページで、「新規」をクリックします。

    「新規物理宛先」ページが開きます。

  6. 「名前」フィールドに、物理宛先の名前を入力します。

  7. 「タイプ」ドロップダウン・リストから、javax.jms.Topicまたはjavax.jms.Queueのいずれかを選択します。

  8. 「消費していないメッセージの最大数」フィールドに、宛先で許可されている未消費メッセージの最大数を入力します。

    -1を指定すると、未消費メッセージの数は制限されません。デフォルト値は-1です。

  9. 「最大メッセージ・サイズ」フィールドに、単一メッセージの最大サイズをバイト単位で入力します。

    -1を指定すると、メッセージのサイズは制限されません。デフォルト値は-1です。

  10. 「最大総メモリー」フィールドに、未消費メッセージの最大合計メモリーをバイト単位で入力します。

    -1を指定すると、未消費メッセージのメモリー・サイズは制限されません。デフォルト値は-1です。

  11. 「しきい値制限の動作」ドロップダウン・リストから、メモリー制限のしきい値に達したときのブローカの動作を選択します。

    • Reject newest messagesは、最新のメッセージを拒否し、メッセージを保持する場合のみ、メッセージを生成しているクライアントに例外を通知するようにブローカに指示します。

      Reject newest messagesはデフォルト値です。

    • Slow down producersは、メッセージ・プロデューサがメッセージを送信する頻度を下げるようにブローカに指示します。

    • Throw out oldestは、最も古いメッセージを破棄するようにブローカに指示します。

    • Throw out lowest-priority messagesは、優先度の最も低いメッセージを破棄するようにブローカに指示します。

    値がThrow out oldestまたはThrow out lowest-priority messagesで、「デッド・メッセージ・キューを使用」がtrueに設定されている場合、超過したメッセージはデッド・メッセージ・キューに送られます。

  12. 「プロデューサの最大数」フィールドに、この物理宛先に関連付けることができるメッセージ・プロデューサの最大数を入力します。

    この制限に達すると、新しいプロデューサを作成できなくなります。-1を指定すると、プロデューサの数は制限されません。デフォルト値は100です。

  13. 「バッチ配信の最大メッセージ数」フィールドに、単一のバッチでコンシューマに配送可能なメッセージの最大数を入力します。

    -1を指定すると、メッセージの数は制限されません。デフォルト値は1000です。

  14. 「デッド・メッセージ・キューを使用」ドロップダウン・リストからtrueを選択して、デッド・メッセージをデッド・メッセージ・キューに送ります。

    値をfalseに設定すると、デッド・メッセージは破棄されます。デフォルトでは、このオプションはtrueに設定されます。

  15. 「XMLスキーマの検証」の「有効」ドロップダウン・リストからtrueを選択して、宛先でのXMLスキーマの妥当性検査を有効にします。

    値をtrueに設定すると、Message Queueクライアント・ランタイムはXMLメッセージをブローカに送信する前に、指定したXML Schema Definition (XSDが指定されていない場合はDTD)を使用してメッセージの検証を行います。指定したスキーマが見つからないかメッセージを検証できない場合、メッセージは送信されず例外がスローされます。デフォルトでは、このオプションはfalseに設定されます。

  16. 「XMLスキーマURIリスト」フィールドに、XML Schema Definition(XSD)URI文字列を空白区切りのリストで入力します。

    XMLスキーマ検証が有効な場合、URIはXMLスキーマ検証に使用する1つまたは複数のXSDの場所を指します。
