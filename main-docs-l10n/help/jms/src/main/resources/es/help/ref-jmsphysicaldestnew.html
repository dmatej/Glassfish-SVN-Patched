Nuevo Destino Físico JMS

Utilice la página Nuevo Destino Físico JMS para crear un nuevo destino físico JMS.

Normalmente, no es necesario crear destinos físicos, ya que se especifica un nombre de destino físico al crear un recurso de destino JMS y GlassFish Server crea automáticamente el destino físico cuando hace falta.

La página Nuevo Destino Físico JMS contiene las siguientes opciones.

Nombre

Nombre del destino físico.

Tipo

Tipo del destino físico. Las opciones disponibles son javax.jms.Topic y javax.jms.Queue.

Número Máximo de Mensajes No Consumidos:

Número máximo de mensajes no consumidos permitido para el destino. Un valor de -1 indica que no hay límite para el número de mensajes no consumidos. El valor por defecto es -1.

Tamaño Máximo de Mensaje

Tamaño máximo de un mensaje, en bytes. Un valor de -1 indica un tamaño de mensaje ilimitado. El valor por defecto es -1.

Memoria Máxima Total

Total máximo de memoria para mensajes no consumidos, en bytes. Un valor de -1 indica que no hay límite para el tamaño de memoria destinado a los mensajes no consumidos. El valor por defecto es -1.

Comportamiento Límite de Umbral

Comportamiento que debe seguir el broker cuando se alcanza el umbral del límite de memoria. Las opciones disponibles son:

Rechazar los mensajes más recientes

Rechazar los mensajes más recientes y notificar al cliente que los produce con una excepción sólo si el mensaje es persistente. Éste es el valor por defecto.

Ralentizar productores

Ralentizar la frecuencia con que los productores de mensajes los envían.

Rechazar los más antiguos

Descartar los mensajes más antiguos.

Rechazar los mensajes con prioridad más baja

Descartar los mensajes con la prioridad más baja.

Si el valor es Rechazar los más antiguos o Rechazar los mensajes con prioridad más baja y Usar cola de mensajes inoperativos se define como True, los mensajes que superen el límite se mueven a la cola de mensajes inoperativos.

Número Máximo de Productores

Número máximo de productores de mensajes que se pueden asociar a este destino físico. Cuando se alcanza este límite, no se pueden crear nuevos productores. El valor -1 indica un número ilimitado de productores. El valor por defecto es 100.

Recuento máximo de mensajes de entrega de lotes

Número máximo de mensajes que pueden entregarse a los consumidores en un solo lote. Un valor de -1 indica un número ilimitado de mensajes. El valor por defecto es 1.000.

Usar cola de mensajes inoperativos

Si el valor se define en "True", los mensajes inoperativos se colocan en la cola de mensajes inoperativos. Si el valor se define en "False", los mensajes inoperativos se descartarán. Por defecto, esta opción se define en "True".

Validar Esquema XML

Si el valor se define en "True", el tiempo de ejecución del cliente de Message Queue intentará validar un mensaje de XML en comparación con las definiciones del esquema XML especificadas (o comparándolo con el DTD, si es que no se especifica XSD) antes de enviarlo al broker. Si el esquema especificado no se puede localizar o no es posible validar el mensaje, éste último no se enviará y aparecerá una excepción. Por defecto, esta opción se define en "False".

Lista URI de Esquema XML

Una lista separada por espacios de cadenas URI (XSD) de definición de esquema XML. Si está activada la validación del esquema XML, las URI indican la ubicación de uno o más XSD que se emplearán para la validación del esquema XML.
