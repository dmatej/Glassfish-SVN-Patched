新建 JMS 目的地資源

使用「新建 JMS 目的地資源」頁面可建立新的 JMS 目的地資源。

「新建 JMS 目的地資源」頁面包含下列選項。

JNDI 名稱

可識別目的地資源的唯一名稱。

建議的作業是使用 JMS 資源的命名子環境字首 jms/。例如︰ jms/Queue。

實體目的地名稱

要與此資源關聯的實體目的地的名稱。

GlassFish Server 會在需要時自動建立實體目的地，並在您刪除目的地資源時將其刪除。

資源類型

目的地資源的類型。可用的選項包括 javax.jms.Topic 和 javax.jms.Queue。

描述

目的地資源的描述。

狀態

目的地資源的狀態。您可啟用或停用目的地資源。此選項預設為啟用。

其他特性

目的地資源的其他特性。GlassFish Server 未定義 JMS 目的地資源的任何其他特性。

目標

(已在網域中建立叢集或獨立執行處理時才會顯示此選項。)

資源的叢集和獨立執行處理。可使用「加入」與「全部加入」按鈕，將所需的目標移至「選取的目標」欄。可使用「移除」與「全部移除」按鈕，將任何不需要的目標移至「可用的目標」欄。
