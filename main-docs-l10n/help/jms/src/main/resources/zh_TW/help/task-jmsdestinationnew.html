建立 JMS 目的地資源

當您建立 JMS 目的地資源時，GlassFish Server 會針對目的地資源建立管理物件資源。您可以在「連線器」節點下檢視此資源。

  1. 在瀏覽樹狀結構中，展開「資源」節點。

  2. 在「資源」節點下，展開「JMS 資源」節點。

  3. 在「JMS 資源」節點下，選取「JMS 目的地資源」節點。

    將開啟「JMS 目的地資源」頁面。

  4. 在「JMS 目的地資源」頁面上，按一下「新建」。

    將開啟「新建 JMS 目的地資源」頁面。

  5. 在「JNDI 名稱」欄位中，鍵入資源的 JNDI 名稱。

    建議的作業是使用 JMS 資源的命名子環境字首 jms/。例如︰ jms/Queue。

  6. 在「實體目的地名稱」欄位中，鍵入要與此資源關聯的實體目的地名稱。

    GlassFish Server 會在需要時自動建立實體目的地，並在您刪除目的地資源時將其刪除。

  7. 從「資源類型」下拉式清單中，選取「javax.jms.Topic」或「javax.jms.Queue」。

  8. 在「描述」欄位中，鍵入目的地資源的描述。

  9. 選取「啟用狀態」核取方塊以在執行階段啟用該資源。

    此選項預設為啟用。

  10. 在「其他特性」區域中，指定目的地資源的其他特性。

    GlassFish Server 未定義 JMS 目的地資源的任何其他特性。

    若要增加特性，請按一下「增加特性」按鈕。在出現的空白列中，於「名稱」欄位中鍵入特性名稱，然後在「值」欄位中鍵入特性值。

  11. 在「目標」區域中，執行下列動作：

    1. 從「可用」欄中，選取一或多個目標以建置使用資源的應用程式。

      可用的目標包含可用的叢集與伺服器執行處理，以及預設的伺服器執行處理 server。

    2. 按一下「增加」，將目標移至「已選取」欄。

  12. 按一下「確定」。
