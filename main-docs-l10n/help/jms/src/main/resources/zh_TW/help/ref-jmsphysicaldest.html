JMS 實體目的地

Java 訊息服務 (JMS) 實體目的地物件由 Message Queue 代理程式維護。名為 mq.sys.dmq 的佇列是系統目的地，過期的和無法傳送的訊息都會重新導向至該目的地。

使用「JMS 實體目的地」頁面可組態 Message Queue 實體目的地。

對於每個實體目的地，都將提供以下資訊。

名稱

實體目的地名稱。

類型

實體目的地類型。

統計

實體目的地的統計。

目的地表格也包含下列選項。

新建

建立新實體目的地的按鈕。

刪除

刪除一或多個所選取之實體目的地的按鈕。

刪除所有訊息

從一或多個所選取之實體目的地移除訊息的按鈕。
