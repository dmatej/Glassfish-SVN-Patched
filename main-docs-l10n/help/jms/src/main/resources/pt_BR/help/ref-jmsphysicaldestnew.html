Novo Destino Físico JMS

Utilize a página Novo Destino Físico JMS para criar um novo destino físico JMS.

Normalmente não é preciso criar destinos físicos, porque você especifica um nome de destino físico quando cria um recurso de destino JMS e o GlassFish Server cria o destino físico automaticamente quando é necessário.

A página Novo Destino Físico JMS contém as opções a seguir.

Nome

O nome do destino físico.

Tipo

O tipo do destino físico. As escolhas disponíveis são javax.jms.Topic e javax.jms.Queue.

Número Máximo de Mensagens Não Consumidas

O número máximo de mensagens não consumidas permitido para o destino. Um valor de -1 indica que não há limite para o número de mensagens não consumidas. O valor default é -1.

Tamanho Máximo da Mensagem

O tamanho máximo, em bytes, de uma única mensagem. Um valor de -1 significa um tamanho de mensagem ilimitado. O valor default é -1.

Total Máximo de Memória

O total máximo de memória, em bytes, para mensagens não consumidas. Um valor de -1 indica que não há limite para o tamanho da memória de mensagens não consumidas. O valor default é -1.

Comportamento para Limite

O comportamento que o broker deve seguir quando o limite de memória é atingido. As opções disponíveis são:

Rejeitar mensagens mais recentes

Rejeita as mensagens mais recentes e notifica o cliente de produção com uma exceção somente se a mensagem for persistente. Este é o valor default.

Diminuir a taxa de produtores

Diminui a taxa com que produtores de mensagem enviam mensagens.

Rejeitar mais antiga

Descarta as mensagens mais antigas.

Rejeitar mensagens de prioridade mais baixa

Descarta as mensagens com a prioridade mais baixa.

Se o valor for Rejeitar mais antiga ou Rejeitar mensagens de prioridade mais baixa e se a opção Usar Fila de Mensagens Inativas estiver definida como Verdadeira, as mensagens em excesso serão movidas para a fila de mensagens inativas.

Número Máximo de Produtores

O número máximo de produtores de mensagens que pode ser associado a este destino físico. Quando esse limite é atingido, nenhum novo produtor pode ser criado. Um valor de -1 denota um número ilimitado de produtores. O valor default é 100.

Contagem Máxima Mensagens para Entrega Batch

O número máximo de mensagens que podem ser entregues para consumidores em um único batch. Um valor de -1 denota um número ilimitado de mensagens. O valor default é 1000.

Usar Fila de Mensagens Inativas

Se o valor for definido como Verdadeiro, as mensagens inativas serão colocadas na fila de mensagens inativas. Se o valor for definido como Falso, mensagens inativas serão descartadas. Por default, essa opção é definida como Verdadeiro.

Validar Esquema XML

Se o valor for definido como Verdadeiro, o runtime do cliente do Message Queue tentará validar uma mensagem XML contra as Definições de esquema XML especificadas (ou contra o DTD, se nenhuma XSD estiver especificado) antes de enviá-la para o broker. Se o esquema especificado não pode ser localizado ou a mensagem não pode ser validada, a mensagem não é enviada e uma exceção é lançada. Por default, essa opção é definida como Falso.

Lista de URIs de Esquema XML

Uma lista separada por espaços de strings de URI de Definição de Esquema XML (XSD). Se a validação de esquema XML estiver ativada, os URIs apontam para o local de uma ou mais XSDs a serem usados para a validação do esquema XML.
