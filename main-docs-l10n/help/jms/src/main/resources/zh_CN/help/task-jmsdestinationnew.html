创建 JMS 目标资源

创建 JMS 目标资源时，GlassFish Server 会为该目标资源创建“管理对象资源”。您可以在“连接器”节点下查看此资源。

  1. 在导航树中，展开“资源”节点。

  2. 在“资源”节点下，展开“JMS 资源”节点。

  3. 在“JMS 资源”节点下，选择“JMS 目标资源”节点。

    此时将打开“JMS 目标资源”页。

  4. 在“JMS 目标资源”页中，单击“新建”。

    此时将打开“新建 JMS 目标资源”页。

  5. 在“JNDI 名称”字段中，键入资源的 JNDI 名称。

    建议的做法是使用 JMS 资源的命名子上下文前缀 jms/。例如：jms/Queue。

  6. 在“物理目标名称”字段中，键入要与此资源关联的物理目标的名称。

    GlassFish Server 在需要时自动创建物理目标，并在您删除目标资源时删除物理目标。

  7. 从“资源类型”下拉列表中，选择 javax.jms.Topic 或 javax.jms.Queue。

  8. 在“说明”字段中，键入目标资源的说明。

  9. 选择“启用状态”复选框以在运行时启用该资源。

    默认情况下启用此选项。

  10. 在“其他属性”区域中，指定目标资源的其他属性。

    GlassFish Server 不定义 JMS 目标资源的任何其他属性。

    要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。

  11. 在“目标”区域中，执行以下操作：

    1. 从“可用”列中，选择要将使用资源的应用程序部署到的目标。

      可用目标包括可用集群和服务器实例，还有默认的服务器实例 server。

    2. 单击“添加”将目标移动到“所选”列。

  12. 单击“确定”。
