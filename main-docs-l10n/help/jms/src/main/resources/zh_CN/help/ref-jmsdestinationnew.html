新建 JMS 目标资源

使用“新建 JMS 目标资源”页可以创建新的 JMS 目标资源。

“新建 JMS 目标资源”页包含以下选项。

JNDI 名称

标识目标资源的唯一名称。

建议的做法是使用 JMS 资源的命名子上下文前缀 jms/。例如：jms/Queue。

物理目标名称

要与此资源关联的物理目标的名称。

GlassFish Server 在需要时自动创建物理目标，并在您删除目标资源时删除物理目标。

资源类型

目标资源的类型。可用选项为 javax.jms.Topic 和 javax.jms.Queue。

说明

目标资源的说明。

状态

目标资源的状态。可以启用或禁用目标资源。默认情况下启用此选项。

其他属性

目标资源的其他属性。GlassFish Server 不定义 JMS 目标资源的任何其他属性。

目标

目标资源的集群和独立实例。使用“添加”和“全部添加”按钮将所需目标移动到“所选目标”列。使用“删除”和“全部删除”按钮将任何不需要的目标移动到“可用目标”列。

仅当在域中已创建集群或独立实例时才会显示此选项。
