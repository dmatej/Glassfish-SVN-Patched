JMS 접속 팩토리 리소스를 편집하는 방법

  1. 탐색 트리에서 [리소스] 노드를 확장합니다.

  2. [리소스] 노드에서 [JMS 리소스] 노드를 확장합니다.

  3. [JMS 리소스] 노드에서 [JMS 접속 팩토리] 노드를 선택합니다.

    [JMS 접속 팩토리] 페이지가 열립니다.

  4. [JMS 접속 팩토리] 페이지에서 편집할 접속 팩토리의 이름을 누릅니다.

    [JMS 접속 팩토리 편집] 페이지가 열립니다.

  5. [설명] 필드에 접속 팩토리에 대한 설명을 입력합니다.

  6. 런타임에 리소스를 사용으로 설정하려면 [상태 사용] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용으로 설정되어 있습니다.

  7. [초기 및 최소 풀 크기] 필드에 풀의 최소 접속 수를 지정합니다.

    기본값은 8입니다.

  8. [최대 풀 크기] 필드에 풀의 최대 접속 수를 지정합니다.

    기본값은 32입니다.

  9. [풀 크기 조정 개수] 필드에 풀 유휴 시간 초과가 만료될 때 제거할 접속 수를 지정합니다.

    기본값은 2입니다.

  10. [유휴 시간 초과] 필드에 풀에서 접속이 유휴 상태로 유지될 수 있는 최대 시간(초)을 지정합니다.

    기본값은 300입니다.

  11. [최대 대기 시간] 필드에 접속을 요청하는 응용 프로그램이 접속 시간 초과까지 대기하는 시간을 지정합니다.

    기본값은 60000입니다.

  12. [실패 시 모든 접속 닫기] 체크 박스를 선택하여 단일 접속이 실패할 경우 GlassFish Server에서 풀의 모든 접속을 닫은 후 재설정하도록 합니다.

    이 체크 박스를 선택하지 않으면 개별 접속을 사용할 경우에만 접속이 다시 설정됩니다.

  13. JMS 접속 팩토리 리소스에 대해 트랜잭션 지원 값을 다음과 같이 지정합니다.

    • 트랜잭션 범위 내에서 둘 이상의 리소스 사용과 관련되는 트랜잭션에 사용할 수 있는 리소스에 대해 XATransaction(기본값)을 지정합니다.

      예를 들어 이 리소스와 JDBC 리소스, 커넥터 리소스 또는 다른 JMS 접속 팩토리 리소스입니다. 이 값은 가장 높은 유연성을 제공합니다. XATransaction으로 구성된 리소스는 2단계 커밋 작업에 참여합니다.

    • 트랜잭션 범위 내에서 하나의 리소스만 관여하는 트랜잭션에 사용되거나 둘 이상의 XA 리소스가 관여하는 분산된 트랜잭션에서 마지막 에이전트로 사용될 수 있는 리소스에 대해 LocalTransaction을 지정합니다.

      이 값은 훨씬 높은 성능을 제공합니다. LocalTransaction으로 구성된 리소스는 2단계 커밋 작업에 사용되지 않습니다.

    • 트랜잭션에 절대로 참여하지 않을 리소스에 대해 NoTransaction을 지정합니다.

      이 설정은 JMS 응용 프로그램에서 제한적으로 사용됩니다.

  14. [접속 검증] 체크 박스를 선택하여 접속 검증을 사용하도록 설정합니다.

    이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

  15. [추가 속성] 테이블에서 접속 팩토리에 대한 추가 속성을 지정합니다.

    사용 가능한 속성에 대한 자세한 내용은 JMS 접속 팩토리 관련 속성을 참조하십시오.

    • 속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

    • 속성을 수정하려면 속성의 [값] 필드를 편집합니다.

    • 속성을 삭제하려면 삭제할 속성의 [이름] 필드 왼쪽에 있는 체크 박스를 선택한 다음 [속성 삭제] 단추를 누릅니다.

  16. 선택적으로 [대상] 탭을 눌러 [JMS 접속 팩토리 리소스 대상] 페이지로 이동합니다. 이 페이지에서 다음을 수행합니다.

    1. [대상 관리]를 눌러 [리소스 대상 관리] 페이지를 엽니다.

      이 페이지에서 [사용 가능] 열과 [선택됨] 열 간에 대상을 이동합니다. 리소스를 사용하는 응용 프로그램이 배치될 대상을 [선택됨] 열에 배치해야 합니다. 사용 가능한 대상에는 기본 서버 인스턴스인 서버뿐 아니라 사용 가능한 클러스터 및 서버 인스턴스가 포함됩니다. [확인]을 눌러 변경 사항을 저장합니다.

    2. 대상에 대한 체크 박스를 선택한 다음 [사용] 또는 [사용 안함]을 눌러 해당 대상에 대한 리소스를 사용 또는 사용 안함으로 설정합니다.

  17. [저장]을 누릅니다.

    팁:

    기본 설정으로 돌아가려면 [기본값 로드] 단추를 누른 다음 [저장]을 누릅니다.
