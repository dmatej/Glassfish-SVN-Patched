JMS 접속 팩토리 편집

[JMS 접속 팩토리 편집] 페이지에서는 JMS 접속 팩토리에 대한 설정을 수정할 수 있습니다.

[JMS 접속 팩토리 편집] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값을 사용하는 설정을 기본값으로 복원하는 단추입니다. 기본값을 사용하지 않는 설정은 변경되지 않습니다.

JNDI 이름

리소스를 식별하는 고유한 JNDI 이름입니다. 관례상 이 이름은 리소스 유형 지시자 및 슬래시로 시작됩니다(예: concurrent/, jdbc/ 또는 jms/). [JNDI 이름]은 읽기 전용 필드입니다. 새 리소스를 생성할 때만 JNDI 이름을 지정할 수 있습니다

논리적 JNDI 이름

리소스의 논리적 JNDI 이름입니다. 리소스에 액세스할 때 사용하는 Java EE 표준 이름입니다. [논리적 JNDI 이름]은 읽기 전용 필드입니다. 자세한 내용은 논리적 JNDI 이름을 참조하십시오.

리소스 유형

접속 팩토리의 유형입니다. 리소스 유형은 읽기 전용 필드입니다. 새 JMS 접속 팩토리를 생성할 때만 리소스 유형을 지정할 수 있습니다.

설명

접속 팩토리에 대한 설명입니다.

상태

런타임에 리소스를 사용할 수 있는지 여부입니다.

기본 서버 인스턴스인 server가 도메인에 존재하는 경우에만 리소스의 런타임 가용성을 사용 또는 사용 안함으로 설정할 수 있습니다. 런타임 가용성은 기본적으로 사용으로 설정됩니다.

도메인에 클러스터 또는 독립형 인스턴스가 생성된 경우 [대상] 탭을 사용하여 해당 대상에서 리소스의 런타임 가용성을 제어할 수 있습니다.

초기 및 최소 풀 크기

접속 팩토리에 대한 풀의 최소 접속 수입니다. 기본값은 8입니다.

최대 풀 크기

풀의 최대 접속 수입니다. 기본값은 32입니다.

풀 크기 조정 개수

풀 유휴 시간 초과가 만료될 때 제거할 접속 수입니다. 기본값은 2입니다.

유휴 시간 초과

풀에서 접속이 유휴 상태로 있을 수 있는 최대 시간(초)입니다. 기본값은 300입니다.

최대 대기 시간

접속을 요청한 응용 프로그램이 접속 시간 초과까지 대기하는 시간입니다. 기본값은 60000입니다.

실패 시

[모든 접속 닫기] 체크 박스가 선택된 경우 접속이 하나라도 실패하면 GlassFish Server에서 풀의 모든 접속을 닫은 후 재설정합니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

트랜잭션 지원

다음과 같은 값을 사용할 수 있습니다.

XATransaction

트랜잭션 범위 내에서 둘 이상의 리소스 사용과 관련된 트랜잭션에 리소스를 사용할 수 있습니다. 이 값은 JMS 접속 팩토리의 기본값입니다.

예를 들어 트랜잭션에는 이 리소스와 JDBC 리소스, 커넥터 리소스 또는 다른 JMS 접속 팩토리 리소스가 포함될 수 있습니다. 이 값은 가장 높은 유연성을 제공합니다. XATransaction으로 구성된 리소스는 2단계 커밋 작업에 참여합니다.

LocalTransaction

리소스는 트랜잭션 범위 내에서 하나의 리소스만 관여하는 트랜잭션에 사용되거나 둘 이상의 XA 리소스가 관여하는 분산된 트랜잭션에서 마지막 에이전트로 사용될 수 있습니다.

이 값은 훨씬 높은 성능을 제공합니다. LocalTransaction으로 구성된 리소스는 2단계 커밋 작업에 사용되지 않습니다.

NoTransaction

리소스는 트랜잭션에 참여할 수 없습니다. 이 설정은 JMS 응용 프로그램에서 제한적으로 사용됩니다.

접속 검증

[필요] 체크 박스가 선택된 경우 접속을 응용 프로그램에 전달하기 전에 먼저 검증합니다. 리소스의 검증이 실패하면 리소스가 삭제되고, 새 리소스가 생성된 후 반환됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

추가 속성

접속 팩토리에 대한 추가 속성입니다.

사용 가능한 속성에 대한 자세한 내용은 JMS 접속 팩토리 관련 속성을 참조하십시오.
