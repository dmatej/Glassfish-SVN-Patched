JDBC 리소스 편집

[JDBC 리소스 편집] 페이지에서는 기존 JDBC 리소스를 수정할 수 있습니다.

[JDBC 리소스 편집] 페이지에는 다음 옵션이 포함되어 있습니다.

JNDI 이름

JDBC 리소스를 식별하는 고유 이름입니다. [JNDI 이름]은 읽기 전용 필드입니다. 새 JDBC 리소스를 생성할 때만 JNDI 이름을 지정할 수 있습니다.

풀 이름

JDBC 리소스와 연관된 접속 풀입니다.

설명

JDBC 리소스에 대한 간단한 설명입니다.

상태

이 옵션을 사용으로 설정하면 런타임 시 리소스를 사용할 수 있습니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

추가 속성

JDBC 리소스에 대한 추가 속성입니다. GlassFish Server에서는 JDBC 리소스에 대한 추가 속성을 정의하지 않습니다.

관련 작업

  • 데이터베이스 액세스를 설정하는 방법

  • JDBC 드라이버를 통합하는 방법

  • JDBC 리소스를 생성하는 방법

  • JDBC 리소스를 편집하는 방법

  • JDBC 리소스를 삭제하는 방법

관련 asadmin 명령

  • add-resources(1)

  • create-jdbc-resource(1)

  • delete-jdbc-resource(1)

  • list-jdbc-resources(1)

  • start-database(1)

  • stop-database(1)
