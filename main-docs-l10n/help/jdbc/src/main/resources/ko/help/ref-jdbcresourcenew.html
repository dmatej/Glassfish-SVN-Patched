새 JDBC 리소스

[새 JDBC 리소스] 페이지에서는 새 JDBC 리소스를 생성할 수 있습니다.

[새 JDBC 리소스] 페이지에는 다음 옵션이 포함되어 있습니다.

JNDI 이름

JDBC 리소스를 식별하는 고유 이름입니다. JNDI 이름은 카드 카탈로그가 도서관 내 책의 위치를 조직화하여 나타내는 것과 유사한 방식으로 분산 컴퓨팅 환경 내에서 구성 요소를 조직화하고 구성 요소의 위치를 찾아냅니다. 이에 따라 JNDI 이름은 JDBC 리소스에 액세스하는 중요한 방법이 됩니다. 일반적으로 이름은 jdbc/ 문자열로 시작합니다(예: jdbc/payrolldb). 반드시 슬래시를 입력합니다. 이름에는 영숫자, 밑줄, 대시 또는 점 문자만 사용해야 합니다.

풀 이름

JDBC 리소스와 연관된 접속 풀입니다.

설명

JDBC 리소스에 대한 간단한 설명입니다.

상태

이 옵션을 사용으로 설정하면 런타임 시 리소스를 사용할 수 있습니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

추가 속성

JDBC 리소스에 대한 추가 속성입니다. GlassFish Server에서는 JDBC 리소스에 대한 추가 속성을 정의하지 않습니다.

대상

JDBC 리소스에 대한 클러스터 및 독립형 인스턴스입니다. 원하는 대상을 [선택한 대상] 열로 이동하려면 [추가] 및 [모두 추가] 단추를 사용합니다. 필요하지 않은 대상을 [사용 가능한 대상] 열로 이동하려면 [제거] 및 [모두 제거] 단추를 사용합니다.

이 옵션은 클러스터나 독립형 인스턴스가 도메인에 생성된 경우에만 표시됩니다.
