編輯連線集區

使用「編輯連線集區」頁面可編輯 JDBC 連線集區的一般特性。

「編輯連線集區」頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原為設定值的按鈕。沒有預設值的設定則保持不變。

清除

在連線集區中重新初始化連線的按鈕。按下此按鈕可銷毀任何現有的連線、重新建立已針對集區建立的連線，並將集區復原為其初始與最小的集區大小。

Ping

驗證連線集區是否可用的按鈕。若您按一下此按鈕時出現錯誤訊息，請檢查資料庫伺服器是否已啟動。

集區名稱

連線集區的名稱。此為唯讀欄位。您僅能在建立 JDBC 連線集區時指定集區名稱。

資源類型

選項包括 javax.sql.XADataSource (全域作業事件)、java.sql.ConnectionPoolDataSource (本機作業事件，可能會提昇效能)、javax.sql.DataSource (僅限於本機作業事件) 以及java.sql.Driver。

資料來源類別名稱

實行資料來源之供應商專屬類別名稱。

驅動程式類別名稱

供應商提供的 JDBC 驅動程式類別名稱。若您選取了 java.sql.Driver 資源類型，此欄位通常會自動填入與您所選的驅動程式和資料庫供應商相關聯的類別名稱。若您選取了其中一個資料來源資源類型，會停用此欄位。

Ping

若選取了此選項，將會在集區建立或重新組態期間自動 Ping 集區，以識別並警告任何錯誤的屬性值。此選項預設為停用。

描述

連線集區的簡短描述。

集區的初始大小和最小大小

集區中連線的最小數目。該值還可決定首次建立集區或 GlassFish Server 啟動時，置於集區中的連線數目。預設值為 8。

集區的最大大小

集區中連線的最大數目。預設值為 32。

集區設定大小數量

當集區分別向最大集區大小擴充以及向最小集區大小收縮時，將成批調整大小。此值確定批次中的連線數目。將此值設置過大會延遲連線建立和資源回收；而將該值設置過小則會導致效率降低。預設值為 2。

閒置逾時

連線在集區中可保持閒置狀態的最長時間 (以秒為單位)。一旦超過此時間，即從集區中移除該連線。預設值為 300。

最大等待時間

連線逾時前，請求連線的應用程式等待的最長時間 (以毫秒為單位)。預設值為 60,000。

非作業事件連線

若啟用此選項，則會傳回非作業事件連線。依預設，不會指定此選項。

作業事件隔離

若指定作業事件隔離層級，則此集區中的連線會以指定的層級進行作業。否則，連線將使用 JDBC 驅動程式提供的預設隔離層級進行作業。依預設，不會指定此選項。

隔離層級

若選取了「受保證」核取方塊，則從集區中獲取的所有連線都具有相同的隔離層級。例如，若上次使用連線時以程式設計方式 (使用 con.setTransactionIsolation) 變更了連線的隔離層級，則此機制會將狀態變更回指定的隔離層級。

只有已指定作業事件隔離層級時，才可使用此欄位。
