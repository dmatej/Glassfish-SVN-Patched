JDBC 連線集區特定的特性

大多數的 JDBC 驅動程式均允許使用標準特性清單來指定使用者、密碼，以及其他資源組態資訊。對於 GlassFish Server 而言，雖然這些是選擇性特性，但是某些特性卻可能是大多數資料庫的必要特性。如需詳細資訊，請參閱 JDBC 4.0 Standard Extension API。

變更 JDBC 驅動程式特性後，需要重新啟動伺服器。

下列是許多 JDBC 供應商共用的標準特性。如需 JDBC 驅動程式可用特性的相關資訊，請參閱供應商的文件。

user

指定用於連線至資料庫的使用者名稱。

password

指定用於連線至資料庫的密碼。

databaseName

為此連線集區指定資料庫。

serverName

為此連線集區指定資料庫伺服器。

port

指定資料庫伺服器監聽請求時所在的連接埠。

networkProtocol

指定協定。

roleName

指定初始 SQL 角色名稱。

datasourceName

指定基礎 XADataSource 或 ConnectionPoolDataSource (如果已完成連線集區管理作業)。

description

指定文字描述。

url

指定此連線集區的 URL。雖然這不是標準特性，不過很常使用。
