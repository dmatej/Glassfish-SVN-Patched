Proprietà specifiche dei connection pool JDBC

La maggior parte dei driver JDBC consente l'utilizzo di liste di proprietà standard per specificare l'utente, la password e altre informazioni di configurazione delle risorse. Sebbene le proprietà siano facoltative per GlassFish Server, alcune risultano necessarie per la maggior parte dei database. Per informazioni dettagliate, vedere l'interfaccia API JDBC 4.0 Standard Extension.

La modifica delle proprietà del driver JDBC richiede il riavvio del server.

Le proprietà standard riportate di seguito sono comuni a molti fornitori di driver JDBC. Per informazioni sulle proprietà disponibili per il driver JDBC in uso, consultare la documentazione del fornitore.

user

Specifica il nome utente per la connessione al database.

password

Specifica la password per la connessione al database.

databaseName

Specifica il database per il connection pool corrente.

serverName

Specifica il database server per il connection pool corrente.

port

Specifica la porta utilizzata dal database server per l'ascolto delle richieste.

networkProtocol

Specifica il protocollo di comunicazione.

roleName

Specifica il nome del ruolo SQL iniziale.

datasourceName

Specifica una classe XADataSource di base oppure una classe ConnectionPoolDataSource se viene implementato il connection pooling.

description

Specifica una descrizione di testo.

url

Specifica l'URL per il connection pool corrente. Questa proprietà non è standard, tuttavia è utilizzata di frequente.
