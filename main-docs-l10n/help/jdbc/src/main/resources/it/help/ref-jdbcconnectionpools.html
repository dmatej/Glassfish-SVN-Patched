Connection pool

Un connection pool JDBC contiene un gruppo di connessioni JDBC create durante la registrazione del connection pool. I connection pool utilizzano un driver JDBC per creare connessioni fisiche al database. L'applicazione prende in prestito una connessione dal pool, la utilizza e quindi la chiude restituendola al pool. Per creare una risorsa JDBC, specificare il connection pool associato. È possibile specificare un singolo connection pool per più risorse JDBC.

Un connection pool JDBC è un gruppo di connessioni riutilizzabili per un determinato database. Poiché la creazione di ogni nuova connessione fisica richiede tempo, il server gestisce un pool di connessioni disponibili per ottimizzare le prestazioni. Quando un'applicazione necessita di una connessione, può ottenerne una dal pool. Quando un'applicazione chiude una connessione, la connessione viene restituita al pool.

Quando si crea un connection pool con Console di amministrazione, si definiscono gli aspetti di una connessione a un database specifico.

Utilizzare la pagina Connection pool JDBC per configurare i connection pool JDBC.

Per ogni connection pool vengono fornite le informazioni riportate di seguito.

Nome pool

Il nome del connection pool.

Tipo di risorsa

Il tipo di risorsa del connection pool.

Nome classe

Il nome della classe specifica del fornitore che implementa l'interfaccia API javax.sql.XADataSource, java.sql.ConnectionPoolDataSource, javax.sql.DataSource o java.sql.Driver.

Descrizione

Una breve descrizione del connection pool.

La tabella Pool contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare un nuovo connection pool.

Elimina

Pulsante che consente di eliminare uno o più connection pool selezionati.
