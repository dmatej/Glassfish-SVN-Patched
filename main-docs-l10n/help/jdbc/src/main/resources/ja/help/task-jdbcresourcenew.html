JDBCリソースを作成するには

Java DataBase Connectivity (JDBC)リソース(データ・ソース)は、データベースへの接続手段をアプリケーションに提供します。

開始前

JDBCリソースを作成するには、まずJDBC接続プールを作成する必要があります。JDBC接続プールを作成するにはを参照してください。

  1. ナビゲーション・ツリーで、「リソース」ノードを展開します。

  2. 「リソース」ノードで、「JDBC」ノードを展開します。

  3. 「JDBC」ノードで、「JDBCリソース」ノードを選択します。

    「JDBCリソース」ページが開きます。

  4. 「JDBCリソース」ページで、「新規」をクリックします。

    「新規JDBCリソース」ページが開きます。

  5. 「JNDI名」フィールドに、一意のリソース名を入力します。

    カード式の目録によって図書館の蔵書の分類と保管場所の管理を行う場合と同様の方法で、分散コンピューティング環境に配置されたコンポーネントの編成と場所の管理がJNDI名によって行われます。そのため、JNDI名はJDBCリソースへの重要なアクセス方法となります。規則により、名前はjdbc/文字列で始まります。たとえば、jdbc/payrolldbなどです。スラッシュを忘れないでください。

    名前に含めることができるのは、英数字、アンダースコア、ダッシュ、またはドットの各文字のみです。

  6. 「プール名」ドロップダウン・リストから、新規JDBCリソースに関連付ける接続プールを選択します。

  7. 「説明」フィールドで、新規JDBCリソースの簡単な説明を入力します。

  8. 「ステータス」の「有効」チェック・ボックスを選択して、リソースを実行時に使用できるようにします。

    このオプションはデフォルトで有効になっています。

  9. 「追加プロパティ」セクションで、追加プロパティを指定します。

    GlassFish Serverでは、JDBCリソースには追加プロパティは定義されません。

    プロパティを追加するには、「プロパティの追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティ名を入力し、「値」フィールドにプロパティ値を入力します。

  10. 使用可能なターゲットを選択してターゲットを追加するか、「選択したターゲット」リストからターゲットを除去します。

  11. 「OK」をクリックします。
