JDBCリソース

Java DataBase Connectivity (JDBC)リソース(データ・ソース)は、データベースへの接続手段をアプリケーションに提供します。通常、管理者はドメイン内にデプロイされたアプリケーションがアクセスするデータベースごとに1つのJDBCリソースを作成します。ただし、1つのデータベースに対して複数のJDBCリソースを作成できます。

アプリケーションは、Java Naming and Directory Interface (JNDI)APIツリーのデータソースを検索して接続プールからデータベース接続を取得してから、接続をリクエストします。データソースに関連付けられている接続プールが、アプリケーションへの接続を提供します。

「JDBCリソース」ページを使用して、JDBCリソースを構成します。

リソースごとに、次の情報が表示されます。

JNDI名

JDBCリソースを識別する一意の名前。

有効

リソースが有効な場合はtrue、無効な場合はfalseです。

接続プール

リソースに関連付けられたJDBC接続プール。

説明

リソースの簡単な説明。

「リソース」の表には、次のオプションも表示されます。

新規

新規リソースを作成するボタン。

削除

選択した1つまたは複数のリソースを削除するボタン。

有効

選択した1つまたは複数のリソースを有効にするボタン。

無効

選択した1つまたは複数のリソースを無効にするボタン。
