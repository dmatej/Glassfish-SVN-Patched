Propriedades Específicas para os Pools de Conexões JDBC

A maioria dos drivers JDBC permitem o uso de listas de propriedades padrão para especificar usuário, senha e outras informações de configuração de recursos. Embora as propriedades sejam opcionais no GlassFish Server, algumas delas podem ser necessárias para a maioria dos bancos de dados. Para obter detalhes, consulte a API JDBC 4.0 Standard Extension.

A alteração de propriedades do driver JDBC exige que o servidor seja reiniciado.

As seguintes propriedades padrão são comuns a muitos fornecedores de JDBC. Para obter informações sobre as propriedades disponíveis para o seu driver JDBC, consulte a documentação do fornecedor.

user

Especifica o nome de usuário para conexão ao banco de dados.

password

Especifica a senha para conexão ao banco de dados.

databaseName

Especifica o banco de dados para esse pool de conexões.

serverName

Especifica o servidor de banco de dados para esse pool de conexões.

port

Especifica a porta em que o servidor de banco de dados receberá as solicitações.

networkProtocol

Especifica o protocolo de comunicação.

roleName

Especifica o nome da atribuição SQL inicial.

datasourceName

Especifica um XADataSource adjacentes ou um ConnectionPoolDataSource se o pool de conexões estiver concluído.

description

Especifica uma descrição de texto.

url

Especifica o URL para esse pool de conexões. Embora essa não seja uma propriedade padrão, ela é de uso comum.
