Para Configurar o Serviço de Timer EJB

O serviço de timer é um serviço de notificação persistente e transacional que é fornecido pelo contêiner do enterprise bean e utilizado para programar notificações ou eventos utilizados por enterprise beans. Todos os enterprise beans, exceto os beans de sessão stateful, podem receber notificações do serviço de timer. Os timers persistentes definidos pelo serviço não são destruídos quando o servidor é submetido a shutdown ou reiniciado.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Selecione o nó Contêiner EJB.

    A página Enterprise Java Beans (EJB) é aberta.

  3. Clique na guia Serviço de Timer EJB.

    A página Serviço de Timer EJB será aberta.

  4. No campo Intervalo Mínimo de Entrega, digite o número mínimo de milissegundos permitidos antes que a próxima expiração para um determinado timer ocorra.

    A definição deste intervalo com um valor muito baixo pode causar a sobrecarga do servidor. O valor default é 1000.

  5. No campo Máximo de Tentativas de Entrega, digite o número máximo de tentativas feitas pelo serviço de timer para entregar uma expiração de timer em decorrência de uma exceção ou reversão.

    O valor default é 1.

  6. No campo Intervalo para Nova Tentativa de Entrega, digite o intervalo, em segundos, entre a as tentativas de nova entrega.

    O valor default é 5.000.

  7. No campo Fonte de Dados do Timer, digite o nome JNDI do recurso JDBC que será utilizado como a fonte de dados do timer.

    Caso seja especificado, esse valor substitui o valor default de jdbc/__TimerPool para a aplicação do sistema do serviço do timer.

  8. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.

  9. Reinicie o GlassFish Server.
