Serviço de Timer EJB

Use a página Serviço de Timer EJB para configurar o serviço de timer do EJB.

O serviço de timer é um serviço de notificação persistente e transacional que é fornecido pelo contêiner do enterprise bean e utilizado para programar notificações ou eventos utilizados por enterprise beans. Todos os enterprise beans, exceto os beans de sessão stateful, podem receber notificações do serviço de timer. Os timers persistentes definidos pelo serviço não são destruídos quando o servidor é submetido a shutdown ou reiniciado.

A página Serviço de Timer EJB contém a seguintes opções:

Carregar Defaults

Botão para restaurar definições para seus valores default. Definições que não tiveram seus valores default alterados.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Intervalo Mínimo de Entrega

O número mínimo de milissegundos permitidos antes que a próxima expiração para determinado timer ocorra. A definição deste intervalo com um valor muito baixo pode causar a sobrecarga do servidor. O valor default é 1000.

Máximo de Tentativas de Entrega

O número máximo de tentativas feitas pelo serviço de timer para entregar uma expiração de timer de em decorrência de uma exceção ou reversão. O valor default é 1.

Intervalo para Nova Tentativa de Entrega

O intervalo em milissegundos entre as novas tentativas de entrega. O valor default é 5.000.

Fonte de Dados do Timer

O nome JNDI do recurso JDBC que será usado como a fonte de dados do timer. Caso seja especificado, esse valor substitui o valor default de jdbc/__TimerPool para a aplicação do sistema do serviço do timer.
