EJB 타이머 서비스

[EJB 타이머 서비스] 페이지에서는 EJB 타이머 서비스를 구성할 수 있습니다.

타이머 서비스는 엔터프라이즈 Bean 컨테이너에서 제공하며 엔터프라이즈 Bean에서 사용하는 알림이나 이벤트의 일정을 잡는 데 사용하는 영구적인 트랜잭션 알림 서비스입니다. Stateful Session Bean을 제외한 모든 엔터프라이즈 Bean에서는 타이머 서비스로부터 알림을 수신할 수 있습니다. 서비스에 의해 설정된 영구 타이머는 서버가 종료되거나 재시작되어도 삭제되지 않습니다.

[EJB 타이머 서비스] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값을 사용하는 설정을 기본값으로 복원하는 단추입니다. 기본값을 사용하지 않는 설정은 변경되지 않습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

최소 전달 간격

특정 타이머에 대한 다음 타이머 만료가 발생하기 전에 허용되는 최소 시간(밀리초)입니다. 이 간격을 너무 낮게 설정하면 서버 오버헤드가 발생할 수 있습니다. 기본값은 1000입니다.

최대 재전송

타이머 서비스가 예외 사항 또는 롤백에 대한 타이머 만료 기한을 전달하기 위해 시도하는 최대 횟수입니다. 기본값은 1입니다.

재전송 간격

재전송 시도 간격(밀리초)입니다. 기본값은 5000입니다.

타이머 데이터 소스

타이머 데이터 소스로 사용될 JDBC 리소스의 JNDI 이름입니다. 이 값을 지정하면 타이머 서비스 시스템 응용 프로그램에 대한 기본값인 jdbc/__TimerPool을 대체합니다.
