So wird der EJB-Timerservice konfiguriert

Der Timerservice ist ein vom Enterprise Bean-Container bereitgestellter, persistenter Transaktionsbenachrichtigungsservice, mit dem von Enterprise Beans verwendete Benachrichtigungen oder Ereignisse geplant werden. Alle Enterprise Beans akzeptieren Stateful Session-Beans und können Benachrichtigungen vom Timerservice empfangen. Durch den Service gesetzte persistente Timer werden nicht endgültig gelöscht, wenn der Server heruntergefahren oder neu gestartet wird.

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Wählen Sie den Knoten "EJB-Container".

    Die Seite "Enterprise Java Beans (EJB)" wird geöffnet.

  3. Klicken Sie auf die Registerkarte "EJB-Timerservice".

    Die Seite "EJB-Timerservice" wird geöffnet.

  4. Geben Sie im Feld "Minimales Zustellungsintervall" die Mindestanzahl Millisekunden an, die verstreichen dürfen, bis der nächste Timerablauf für einen bestimmten Timer auftreten kann.

    Ein zu niedrig gewählter Wert für diese Einstellung kann zu einer Serverüberlastung führen. Der Standardwert ist 1000.

  5. Geben Sie im Feld "Maximale erneute Zustellungen" die maximale Anzahl an Versuchen ein, die der Timerservice unternimmt, um einen Timerablauf aufgrund einer Ausnahme oder einer Zurücksetzung zuzustellen.

    Der Standardwert ist 1.

  6. Geben Sie im Feld "Intervall für erneute Zustellung" die Zeit zwischen erneuten Zustellungsversuchen in Millisekunden ein.

    Der Standardwert ist 5000.

  7. Geben Sie im Feld "Timerdatenquelle" den JNDI-Namen der JDBC-Ressource ein, die als Timerdatenquelle verwendet werden soll.

    Wenn dieser Wert angegeben ist, überschreibt er den Standardwert von jdbc/__TimerPool für die Timerservice-Systemanwendung.

  8. Klicken Sie auf "Speichern".

    Tipp:

    Um zu den Standardeinstellungen zurückzukehren, klicken Sie auf die Schaltfläche "Standardwerte laden" und anschließend auf "Speichern".

  9. Starten Sie GlassFish Server neu.
