EJB-Containerverfügbarkeit

Auf der Seite "EJB-Containerverfügbarkeit" können Sie Verfügbarkeitseinstellungen für den EJB-Container konfigurieren.

Die Seite "EJB-Containerverfügbarkeit" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zur Wiederherstellung der Standardwerte von Einstellungen, die Standardwerte haben. Einstellungen, die keine Standardwerte haben, werden nicht geändert.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Verfügbarkeitsservice

Wenn das Kontrollkästchen "Aktiviert" gewählt ist, wird die Verfügbarkeit für den EJB-Container aktiviert.

HA-Persistenztyp

Das Persistenzverfahren, das verwendet wird, wenn "Verfügbarkeitsservice aktiviert" gewählt ist. Der Standardwert ist repliziert. Er stellt Persistenz basierend auf Failover in einem Cluster bereit. Ändern Sie diesen Wert nicht.

SFSB-Persistenztyp

Das Persistenzverfahren, das verwendet wird, wenn "Verfügbarkeitsservice aktiviert" nicht gewählt ist. Der Standardwert ist Datei. Er stellt Persistenz basierend auf dem Dateisystem bereit. Ändern Sie diesen Wert nicht.

Das Feld "Sessionspeicherverzeichnis" bestimmt, wo SFSB-Fixpunktzustände beibehalten werden. Informationen zum Verzeichnis des Sessionspeichers finden Sie unter So werden EJB-Einstellungen konfiguriert.

Weitere Eigenschaften

Weitere Eigenschaften für die EJB-Containerverfügbarkeit.

Bei GlassFish Server sind keine weiteren Eigenschaften für die EJB-Containerverfügbarkeit definiert.
