EJBタイマー・サービスを構成するには

タイマー・サービスは、エンタープライズBeanコンテナによって提供され、エンタープライズBeanが使用する通知やイベントのスケジュールに使用される、永続的なトランザクション通知サービスです。ステートフル・セッションBeans以外のエンタープライズBeansはすべて、タイマー・サービスからの通知を受信できます。このサービスによって設定された永続タイマーは、サーバーの停止や再起動では破棄されません。

  1. ナビゲーション・ツリーで、「構成」ノードを展開します。

  2. 「EJBコンテナ」ノードを選択します。

    「Enterprise Java Beans (EJB)」ページが開きます。

  3. 「EJBタイマー・サービス」タブをクリックします。

    「EJBタイマー・サービス」ページが開きます。

  4. 「最小配信間隔」フィールドに、次のタイマーの有効期限が切れて特定のタイマーの発生が可能になるまでの最短時間をミリ秒単位で入力します。

    この間隔の設定が短すぎると、サーバーがオーバーロードする可能性があります。デフォルト値は1000です。

  5. 「最大再配信回数」フィールドに、例外またはロール・バックによるタイマーの有効期限切れの配信を、タイマー・サービスが試行する最大数を入力します。

    デフォルト値は1です。

  6. 「再配信間隔」フィールドに、再配信を実行する間隔をミリ秒単位で入力します。

    デフォルト値は5000です。

  7. 「タイマー・データソース」フィールドに、タイマー・データソースとして使用するJDBCリソースのJNDI名を入力します。

    指定すると、この値はタイマー・サービス・システム・アプリケーションのjdbc/__TimerPoolのデフォルト値をオーバーライドします。

  8. 「保存」をクリックします。

    ヒント:

    デフォルト設定に返すには、「デフォルトのロード」ボタンをクリックし、「保存」をクリックします。

  9. GlassFish Serverを再起動します。
