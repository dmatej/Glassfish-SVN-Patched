컨텍스트 서비스를 생성하는 방법

컨텍스트 서비스는 XXX를 지정합니다.

  1. 탐색 트리에서 [리소스] 노드를 확장합니다.

  2. [리소스] 노드에서 [동시 리소스] 노드를 확장합니다.

  3. [동시 리소스] 노드에서 [컨텍스트 서비스]를 선택합니다.

    [컨텍스트 서비스] 페이지가 열립니다.

  4. [컨텍스트 서비스] 페이지에서 [새로 만들기]를 누릅니다.

    [새 컨텍스트 서비스] 페이지가 열립니다.

  5. [JNDI 이름] 필드에 고유한 이름을 입력합니다.

    이 이름은 JNDI 이름 지정 서비스에 등록됩니다. JNDI 이름은 카드 카탈로그가 도서관 내 책의 위치를 조직화하여 나타내는 것과 유사한 방식으로 분산 컴퓨팅 환경 내에서 구성 요소를 조직화하고 구성 요소의 위치를 찾아냅니다. 이에 따라 JNDI 이름은 리소스에 액세스하는 중요한 방법이 됩니다.

  6. XXX

  7. [설명] 필드에 리소스에 대한 간단한 설명을 입력합니다.

    이 설명은 최대 250자가 포함될 수 있는 문자열 값입니다.

  8. 리소스를 사용으로 설정하려면 [상태 사용] 체크 박스를 선택합니다.

    기본적으로 리소스는 사용으로 설정되어 있습니다. 즉, 생성하는 즉시 사용할 수 있습니다.

  9. [추가 속성] 섹션에서 리소스에 대한 추가 속성을 지정합니다.

    GlassFish Server에서는 컨텍스트 서비스 리소스에 대한 추가 속성을 정의하지 않습니다.

    속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

  10. [확인]을 누릅니다.
