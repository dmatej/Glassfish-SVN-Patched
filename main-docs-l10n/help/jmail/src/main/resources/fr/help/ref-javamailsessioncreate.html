Nouvelle session JavaMail

La page Nouvelle session JavaMail permet de créer une ressource de session JavaMail.

Elle contient les options ci-après.

Nom JNDI

Nom unique identifiant la session de messagerie. Utilisez le préfixe de dénomination de sous-contexte mail/ pour les ressources JavaMail. Par exemple : mail/MySession. Le nom ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

Hôte de messagerie

Nom d'hôte du serveur de messagerie par défaut. Les méthodes de connexion des objets Store et Transport utilisent cette valeur si aucune propriété d'hôte propre à un protocole n'est indiquée. Le nom doit pouvoir être résolu en un nom d'hôte réel.

Utilisateur par défaut

Nom utilisateur à indiquer lors de la connexion à un serveur de messagerie. Les méthodes de connexion des objets Store et Transport utilisent cette valeur si aucune propriété de nom utilisateur propre à un protocole n'est fournie. Le nom utilisateur ne peut contenir que des caractères alphanumériques, des traits de soulignement, des tirets ou des points.

Adresse de l'expéditeur par défaut

Adresse électronique de l'utilisateur par défaut. Le format est nom utilisateur@hôte.domaine.

Description

Description de cette session de messagerie.

Statut

Statut de la session de messagerie. La session de messagerie peut être activée ou désactivée. La valeur par défaut est Activé.

Protocole de stockage

Le service de protocole de stockage, qui se connecte à un serveur de messagerie, extrait les messages puis les enregistre dans des dossiers. Les valeurs autorisées sont les suivantes : imap, pop3, imaps et pop3s. La valeur par défaut est imap.

Classe de protocole de stockage

Classe d'implémentation du fournisseur de services de stockage. Les valeurs autorisées sont les suivantes :

  • com.sun.mail.imap.IMAPStore

  • com.sun.mail.pop3.POP3Store

  • com.sun.mail.imap.IMAPSSLStore

  • com.sun.mail.pop3.POP3SSLStore

La valeur par défaut est com.sun.mail.imap.IMAPStore.

Protocole de transport

Service de protocole de transport, qui envoie des messages. Les valeurs autorisées sont les suivantes : smtp et smtps. La valeur par défaut est smtp.

Classe de protocole de transport

Classe d'implémentation du fournisseur de services de transport. Les valeurs autorisées sont les suivantes :

  • com.sun.mail.smtp.SMTPTransport

  • com.sun.mail.smtp.SMTPSSLTransport

La valeur par défaut est com.sun.mail.smtp.SMTPTransport.

Déboguer

Lorsque cette option est sélectionnée, le débogage de cette ressource est activé. Si le niveau de journalisation JavaMail a la valeur FINE ou une valeur plus fine, la sortie de débogage est créée et incluse dans le fichier journal du système. Pour plus d'informations sur la définition du niveau de journalisation, reportez-vous à Configuration des niveaux de journalisation.

Autres propriétés

Propriétés supplémentaires de la session JavaMail. Pour obtenir la liste des propriétés disponibles, reportez-vous à la documentation de l'API JavaMail (http://java.sun.com/products/javamail/javadocs/index.html). Les propriétés supplémentaires doivent commencer par mail- et des traits d'union doivent être utilisés comme séparateurs à la place des points. Par exemple, mail-debug est correct mais mail.debug ne l'est pas. Le back-end GlassFish Server convertit les traits d'union en points, attendus par l'API JavaMail.

Cibles

Clusters et instances autonomes de la session JavaMail. Déplacez les cibles souhaitées dans la colonne Cibles sélectionnées à l'aide des boutons Ajouter et Tout ajouter. Déplacez toutes les cibles inutiles dans la colonne Cibles disponibles à l'aide des boutons Enlever et Tout enlever.

Cette option apparaît uniquement si des clusters ou des instances autonomes ont été créés dans le domaine.
