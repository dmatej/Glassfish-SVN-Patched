So wird eine JavaMail-Session erstellt

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Ressourcen" ein.

  2. Wählen Sie unter dem Ressourcenknoten den Knoten "JavaMail-Sessions" aus.

    Die Seite "JavaMail-Sessions" wird geöffnet.

  3. Klicken Sie in der Tabelle "Sessions" auf "Neu".

    Die Seite "Neue JavaMail-Session" wird geöffnet.

  4. Geben Sie im Feld "JNDI-Name" einen eindeutigen JNDI-Namen für die Mailsessionressource ein.

    Verwenden Sie das sub-context-Namenspräfix mail/ für JavaMail-Ressourcen. Beispiel: mail/MySession. Der Name darf ausschließlich alphanumerische Zeichen, Unterstriche, Gedankenstriche oder Punkte enthalten.

  5. Geben Sie im Feld "Mailhost" den Hostnamen des standardmäßigen Mailservers ein.

    Die Verbindungsmethoden der Speicher- und Transportobjekte verwenden diesen Wert, wenn keine protokollspezifische Hosteigenschaft angegeben ist. Der Name muss in einen tatsächlichen Hostnamen auflösbar sein.

  6. Geben Sie im Feld "Standardbenutzer" den Benutzernamen ein, der bei der Verbindungsherstellung mit einem Mailserver angegeben werden soll.

    Die Verbindungsmethode der Speicher- und Transportobjekte verwendet diesen Wert, wenn keine protokollspezifische Benutzernamenseigenschaft angegeben ist. Der Benutzername darf nur alphanumerische Zeichen, Unterstriche, Gedankenstriche oder Punkte enthalten.

  7. Geben Sie im Feld "Standardmäßige Absenderadresse" die E-Mail-Adresse des Standardbenutzers ein.

    Das Format ist username@host.domain.

  8. Geben Sie im Feld "Beschreibung" eine Beschreibung der Mailsession ein.

  9. Aktivieren Sie das Kontrollkästchen "Status", um die Mailsession zu aktivieren.

    Der Standardwert ist "Aktiviert".

  10. Definieren Sie im Feld "Speicherprotokoll" den zu verwendenden Speicherprotokollservice.

    Der Speicherprotokollservice stellt eine Verbindung zu einem Mailserver her, ruft Nachrichten ab und speichert Nachrichten in einem oder mehreren Ordnern. Zulässige Werte sind imap, pop3, imapsund pop3s. Der Standardwert ist imap.

  11. Geben Sie im Feld "Speicherprotokollklasse" die Serviceprovider-Implementierungsklasse für die Speicherung ein.

    Zulässige Werte sind:

    • com.sun.mail.imap.IMAPStore

    • com.sun.mail.pop3.POP3Store

    • com.sun.mail.imap.IMAPSSLStore

    • com.sun.mail.pop3.POP3SSLStore

    Der Standardwert ist com.sun.mail.imap.IMAPStore.

  12. Geben Sie im Feld "Transportprotokoll" die Kommunikationsmethode für den Transport an.

    Der Transportprotokollservice sendet Nachrichten. Zulässige Werte sind smtp und smtps. Standardmäßig wird das Transportprotokoll smtp verwendet.

  13. Geben Sie im Feld "Transportprotokollklasse" die Serviceprovider-Implementierungsklasse für den Transport ein.

    Der Standardwert ist com.sun.mail.smtp.SMTPTransport.

  14. Aktivieren Sie das Kontrollkästchen "Debug", um eine zusätzliche Debugging-Ausgabe einschließlich einem Protokoll-Tracing für diese Mailsession zu aktivieren.

    Wenn die JavaMail-Logebene auf FINE (Genau) oder höher festgelegt ist, wird eine Debugging-Ausgabe generiert und in die Systemlogdatei geschrieben. Weitere Informationen zur Festlegung der Logebene finden Sie unter So konfigurieren Sie Logebenen. Diese Option ist standardmäßig deaktiviert.

  15. Geben Sie in der Tabelle "Weitere Eigenschaften" zusätzliche Eigenschaften an, die von Anwendungen benötigt werden, wie z.B. eine Eigenschaft für einen protokollspezifischen Host oder Benutzernamen.

    Eine Liste der verfügbaren Eigenschaften finden Sie in der JavaMail-API-Dokumentation (http://java.sun.com/products/javamail/javadocs/index.html). Zusätzliche Eigenschaften müssen mit mail- beginnen und Bindestriche anstelle von Punkten als Trennzeichen verwenden. Beispiel: Richtig ist mail-debug, während mail.debug falsch ist. Das GlassFish Server-Backend wandelt die Bindestriche in die von der JavaMail-API erwarteten Punkte um.

    Um eine Eigenschaft hinzuzufügen, klicken Sie auf die Schaltfläche "Eigenschaft hinzufügen". Geben Sie in die daraufhin angezeigte leere Zeile den Eigenschaftsnamen in das Feld "Name" und den Eigenschaftswert in das Feld "Wert" ein.

  16. Wählen Sie aus der Liste der verfügbaren Ziele das Ziel oder die Ziele, für die die Mail-Session konfiguriert werden soll, und klicken Sie auf "Hinzufügen".

    Ziele können Cluster oder Standalone-Serverinstanzen sein. Wenn Sie kein Ziel auswählen, wird die Session für die standardmäßige Serverinstanz server konfiguriert.

  17. Klicken Sie auf "OK".
