Externe Ressource bearbeiten

Auf der Seite "Externe Ressource bearbeiten" können Sie die Einstellungen einer externen JNDI-Ressource ändern.

Die Seite "Externe Ressource bearbeiten" enthält die folgenden Optionen.

JNDI-Name

Ein eindeutiger Name, der die Ressource identifiziert. Der JNDI-Name ist ein schreibgeschütztes Feld. Sie können den JNDI-Namen nur festlegen, wenn Sie eine neue externe Ressource erstellen.

Ressourcentyp

Die vollständig angegebene Typdefinition für die Ressource.

Factory-Klasse

Der benutzerdefinierte Name für die Factory-Klasse. Diese Klasse implementiert die Schnittstelle javax.naming.spi.InitialContextFactory.

JNDI-Suche

Der JNDI-Wert, nach dem im externen Repository gesucht werden soll.

Beispiel: Wenn Sie eine externe Ressource für die Verbindung mit einem externen Repository erstellen, kann die JNDI-Suche zum Testen einer Bean-Klasse wie folgt aussehen: cn=testmybean.

Deployment-Reihenfolge

Die Deployment-Reihenfolge der externen Ressource.

Beschreibung

Eine Beschreibung der externen Ressource.

Status

Gibt an, ob die Ressource zur Laufzeit verfügbar ist.

Wenn in der Domain nur die Standardserverinstanz server vorhanden ist, kann die Laufzeitverfügbarkeit der Ressource aktiviert oder deaktiviert werden. Laufzeitverfügbarkeit ist standardmäßig aktiviert.

Wurden in der Domain Cluster oder Standalone-Instanzen erstellt, können Sie die Laufzeitverfügbarkeit der Ressource auf ihren Zielen über die Registerkarte "Ziel" steuern.

Weitere Eigenschaften

Weitere Eigenschaften für die externe Ressource. In GlassFish Server werden keine weiteren Eigenschaften für externe Ressourcen definiert.

Verwandte Aufgaben

  • So wird eine externe JNDI-Ressource erstellt

  • So wird eine externe JNDI-Ressource bearbeitet

  • So wird eine externe JNDI-Ressource gelöscht
