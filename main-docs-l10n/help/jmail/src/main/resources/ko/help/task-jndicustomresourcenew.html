사용자 정의 JNDI 리소스를 생성하는 방법

사용자 정의 JNDI 리소스는 사용자 정의 서버 차원의 리소스 객체 팩토리를 지정합니다.

  1. 탐색 트리에서 [리소스] 노드를 확장합니다.

  2. [리소스] 노드 아래에서 JNDI 노드를 확장합니다.

  3. JNDI 노드에서 [사용자 정의 리소스]를 선택합니다.

    [사용자 정의 리소스] 페이지가 열립니다.

  4. [사용자 정의 리소스] 페이지에서 [새로 만들기]를 누릅니다.

    [새 사용자 정의 리소스] 페이지가 열립니다.

  5. [JNDI 이름] 필드에 고유한 이름을 입력합니다.

    이 이름은 JNDI 이름 지정 서비스에 등록됩니다. JNDI 이름은 카드 카탈로그가 도서관 내 책의 위치를 조직화하여 나타내는 것과 유사한 방식으로 분산 컴퓨팅 환경 내에서 구성 요소를 조직화하고 구성 요소의 위치를 찾아냅니다. 이에 따라 JNDI 이름은 사용자 정의 리소스에 액세스하는 중요한 방법이 됩니다.

  6. [리소스 유형] 필드에서 다음 중 하나를 수행합니다.

    • 드롭다운 목록에서 유형 정의를 선택합니다.

    • 두번째 라디오 단추를 선택하고 xxx.xxx 형식의 정규화된 유형 정의를 입력합니다.

  7. [팩토리 클래스] 필드에 생성하려는 사용자 정의 리소스에 대한 팩토리 클래스 이름을 입력합니다.

    팩토리 클래스는 팩토리 클래스에 대한 사용자 지정 이름입니다. 이 클래스는 javax.naming.spi.ObjectFactory 인터페이스를 구현합니다.

  8. [설명] 필드에 리소스에 대한 간단한 설명을 입력합니다.

    이 설명은 최대 250자가 포함될 수 있는 문자열 값입니다.

  9. 리소스를 사용으로 설정하려면 [상태 사용] 체크 박스를 선택합니다.

    기본적으로 리소스는 사용으로 설정되어 있습니다. 즉, 생성하는 즉시 사용할 수 있습니다.

  10. [추가 속성] 섹션에서 사용자 정의 리소스에 대한 추가 속성을 지정합니다.

    GlassFish Server에서는 사용자 정의 리소스에 대한 추가 속성을 정의하지 않습니다.

    속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

  11. [확인]을 누릅니다.
