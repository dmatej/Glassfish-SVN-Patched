カスタムJNDIリソースを編集するには

  1. ナビゲーション・ツリーで、「リソース」ノードを展開します。

  2. 「リソース」ノードで、「JNDI」ノードを展開します。

  3. 「JNDI」ノードで、「カスタム・リソース」を選択します。

    「カスタム・リソース」ページが開きます。

  4. 「カスタム・リソース」ページで、編集するリソースの名前をクリックします。

    「カスタム・リソースを編集」ページが開きます。

    注意:

    JNDI名は読取り専用フィールドです。JNDI名を指定できるのは、新規カスタム・リソースの作成時のみです。

  5. 「リソース・タイプ」を変更するには、次のいずれかの操作を行います。

    • ドロップダウン・リストから、タイプ定義を選択します。

    • 2番目のラジオ・ボタンを選択し、完全修飾タイプ定義をxxx.xxxの形式で入力します。

  6. 「ファクトリ・クラス」フィールドに、カスタム・リソースのファクトリ・クラス名を入力します。

    「ファクトリ・クラス」はファクトリ・クラスのユーザー指定の名前です。このクラスはjavax.naming.spi.ObjectFactoryインタフェースを実装する必要があります。

  7. 「説明」フィールドに、リソースの簡単な説明を入力します。

    この説明は文字列値で、最大250文字を入力することができます。

  8. 「ステータス」の「有効」チェック・ボックスを選択し、リソースを有効にします。

    デフォルトでは、リソースは有効で、作成後すぐに使用できます。

  9. 「追加プロパティ」セクションで、カスタム・リソースの追加プロパティを指定します。

    GlassFish Serverはカスタム・リソースの追加プロパティを定義しません。

    • プロパティを追加するには、「プロパティを追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティ名を入力し、「値」フィールドにプロパティ値を入力します。

    • プロパティを変更するには、そのプロパティの「値」フィールドを編集します。

    • プロパティを削除するには、削除するプロパティの「名前」フィールドの左側にあるチェック・ボックスを選択し、「プロパティの削除」ボタンをクリックします。

  10. 「保存」をクリックします。
