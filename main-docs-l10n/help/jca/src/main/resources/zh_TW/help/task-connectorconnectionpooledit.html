編輯連線器連線集區

  1. 在瀏覽樹狀結構中，展開「資源」節點。

  2. 展開「連線器」節點。

  3. 在「連線器」節點下，選取「連線器連線集區」節點。

    「連線器連線集區」頁面會隨即開啟。

  4. 在「連線器連線集區」頁面上，按一下要編輯的連線器連線集區名稱。

    「編輯連線器連線集區」頁面會隨即開啟。

  5. 從「資源轉接器」下拉式清單中，選取不同的資源轉接器。

  6. 從「連線定義」下拉式清單中，選取連線定義的類別名稱。

    清單中提供的選項會視您所選取的資源轉接器而定。連線定義會識別資源轉接器在 ra.xml 檔案中的 connection-definition 元素。

  7. 選取「啟用 Ping」核取方塊，可在連線集區建立或重新組態期間 Ping 連線集區，以識別並警告任何錯誤的屬性值。

    此選項預設為停用。

  8. 在「建置順序」欄位中，鍵入數字來指定連線器連線集區的建置順序。

    系統會在伺服器啟動時先載入數字較小的資源。預設值為 100。

  9. 在「描述」欄位中，鍵入連線器連線集區的描述。

  10. 在「集區的初始大小和最小大小」欄位中，指定連線器連線集區中的最小連線數目。

    預設值為 8。

  11. 在「集區的最大大小」欄位中，指定連線器連線集區中的最大連線數目。

    預設值為 32。

  12. 在「集區設定大小數量」欄位中，指定集區的閒置逾時過期時要移除的連線數目。

    預設值為 2。

  13. 在「閒置逾時」欄位中，指定連線可以在集區中閒置的最長時間 (以秒為單位)。

    預設值為 300。

  14. 在「最大等待時間」欄位中，指定請求連線的應用程式在連線逾時前等待的時間。

    預設值為 60000。

  15. 選取「需要連線驗證」核取方塊以啟用連線驗證。

    若選取了「需要連線驗證」核取方塊，指定連線給應用程式之前會先驗證。若資源的驗證失敗，則會銷毀該資源，然後建立並傳回新的資源。此選項預設為停用。

  16. 選取「一旦失敗」核取方塊，可讓 GlassFish Server 在任何一個連線失敗時，關閉集區中的所有連線並重新建立連線。

    如果未選取此核取方塊，則僅在使用個別連線時才會重新建立這些連線。

  17. 從「作業事件支援」下拉式清單中，為連線集區選取作業事件支援類型。

    選擇的作業事件支援以向下相容方式置換與此連線集區關聯的資源介面中的作業事件支援屬性。也就是說，它可以支援比資源轉接器中指定的作業事件層級低或與其相同的作業事件層級，但它不能指定更高的層級。

    下拉式清單中可用的選項如下：

    • NoTransaction：表示資源轉接器不支援資源管理員的本機或 JTA 作業事件，且不會實行 XAResource 或 LocalTransaction 介面。

    • LocalTransaction：表示資源轉接器會透過實行 LocalTransaction 介面來支援本機作業事件。

      本機作業事件的管理在資源管理員內部進行，不涉及任何外部作業事件管理員。

    • XATransaction：表示資源轉接器會透過實行 LocalTransaction 和 XAResource 介面來支援資源管理員的本機和 JTA 作業事件。

      XA 作業事件由作業事件管理員在資源管理員外部進行控制和協調。

  18. 按一下「清除」按鈕可從集區中清除所有連線。

    按一下「清除」按鈕可銷毀任何現有的連線、重新建立已針對連線集區建立的連線，並將集區復原為其初始與最小的集區大小。

  19. 按一下「Ping」按鈕可驗證連線集區是否可以使用。

  20. 按一下「儲存」。

    提示:

    若要回到預設設定，請按一下「載入預設值」按鈕，然後按一下「儲存」。
