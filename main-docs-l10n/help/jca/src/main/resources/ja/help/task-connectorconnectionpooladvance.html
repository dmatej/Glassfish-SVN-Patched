コネクタ接続プールの詳細属性を編集するには

  1. ナビゲーション・ツリーで、「リソース」ノードを展開します。

  2. 「コネクタ」ノードを展開します。

  3. 「コネクタ」ノードで、「コネクタ接続プール」ノードを選択します。

    「コネクタ接続プール」ページが開きます。

  4. 「コネクタ接続プール」ページで、編集するコネクタ接続プールの名前をクリックします。

    「コネクタ接続プールを編集」ページが開きます。

  5. 「コネクタ接続プールを編集」ページで、「詳細」タブをクリックします。

    「コネクタ接続プールの詳細属性を編集」ページが開きます。

  6. 「最大で1回検証」フィールドに、接続を最大で1回のみ検証するまでの時間を秒単位で入力します。

    この属性を適切な値に設定すると、接続ごとの確認リクエスト回数を最小にできます。デフォルト値は0です。この属性が無効であることを意味します。

  7. 「リーク・タイムアウト」フィールドに、接続プールの接続リークを監視する時間を秒単位で入力します。

    0を指定すると、接続リークの監視が無効になります。デフォルト値は0です。

  8. 「リーク再要求」チェック・ボックスを選択して、リーク接続のトレースが完了した後に、リークした接続をプールに復元します。

    このオプションは、デフォルトでは無効になっています。

  9. 「作成再試行回数」フィールドに、新規接続の作成時に障害が発生した場合の試行回数を入力します。

    0を指定すると、接続の作成は再試行されません。デフォルト値は0です。

  10. 「再試行間隔」フィールドに、接続作成の試行間隔を秒単位で入力します。

    この属性は、「作成再試行回数」が1以上の場合にのみ使用されます。デフォルト値は10秒です。

  11. 「プール」チェック・ボックスを選択して、プールの接続プールを有効にします。

    このオプションはデフォルトで有効になっています。

  12. 「使用時にのみ関連付け」チェック・ボックスを選択して、接続で操作が実行されるときにのみ、接続が関連付けられるようにします。

    トランザクションが完了してコンポーネント・メソッドが終了したときに関連付けが解除されるため、物理的な接続を再利用しやすくなります。このオプションは、デフォルトでは無効になっています。

  13. 「使用時にのみ接続を登録」チェック・ボックスを選択して、リソースが実際にメソッドで使用されるときのみ、トランザクションにリソースを登録します。

    このオプションは、デフォルトでは無効になっています。

  14. 「スレッドとの関連付け」チェック・ボックスを選択して、接続をスレッドに関連付けます。

    同じスレッドが接続を必要とする場合、スレッドはすでに関連付けられている接続を再利用できるため、プールから接続を取得するためのオーバーヘッドが発生しません。このオプションは、デフォルトでは無効になっています。

  15. 「接続の照合」チェック・ボックスを選択して、プールに対する接続の照合を有効にします。

    プール内の接続が常に同じ種類で、プールから選択した接続とリソース・アダプタを照合する必要がない場合は、このオプションを無効にします。このオプションはデフォルトで有効になっています。

  16. 「接続使用数の照合」フィールドに、接続をプールで再利用する回数を入力します。

    指定された回数だけ再利用されると、その接続はクローズされます。0を指定すると、この属性は無効になります。デフォルト値は0です。

  17. 「保存」をクリックします。

    ヒント:

    デフォルト設定に返すには、「デフォルトのロード」ボタンをクリックし、「保存」をクリックします。
