Neue Admin-Objektressource

Auf der Seite "Neue Admin-Objektressource" können Sie eine neue verwaltete Objektressource erstellen.

Die Seite "Neue Admin-Objektressource" enthält die folgenden Optionen.

JNDI-Name

Ein eindeutiger Name, der die verwaltete Objektressource identifiziert.

Ressourcenadapter

Der Ressourcenadapter für die verwaltete Objektressource.

Ressourcentyp

Der Java-Typ für die verwaltete Objektressource (z.B. ↓javax.jms.Topic).

Klassenname

Der mit dem Ressourcentyp verknüpfte Implementierungsklassenname.

Beschreibung

Eine Beschreibung der verwalteten Objektressource.

Status

Bei aktivierter Option ist die verwaltete Objektressource aktiviert. Diese Option ist standardmäßig aktiviert.

Weitere Eigenschaften

Weitere Eigenschaften für die verwaltete Objektressource. Die verfügbaren Eigenschaften richten sich nach dem Ressourcenadapter, der von dem Connector Connection-Pool verwendet wird. Mit den in dieser Tabelle angegebenen Name/Werte-Paaren können die vom Ressourcenadapterprovider definierten Standardwerte für die Eigenschaften überschrieben werden.

Ziele

Cluster und Standalone-Instanzen für die verwaltete Objektressource. Mithilfe der Schaltflächen "Hinzufügen" und "Alle hinzufügen" verschieben Sie die benötigten Ziele in die Spalte "Ausgewählte Ziele". Verschieben Sie die nicht benötigten Ziele in die Spalte "Verfügbare Ziele" mithilfe der Schaltflächen "Entfernen" und "Alle entfernen".

Diese Option wird nur dann angezeigt, wenn in der Domain Cluster oder Standalone-Instanzen erstellt wurden.

Verwandte Aufgaben

  • So wird eine verwaltete Objektressource erstellt

  • So wird eine verwaltete Objektressource bearbeitet

  • So wird eine verwaltete Objektressource gelöscht
