Connector Connection-Pool bearbeiten - Erweiterte Attribute

Auf der Seite "Connector Connection-Pool bearbeiten Erweiterte Attribute" können Sie die erweiterten Einstellungen für einen Connector Connection-Pool ändern.

Die Seite "Connector Connection-Pool bearbeiten - Erweiterte Attribute" umfasst die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zur Wiederherstellung der Standardwerte von Einstellungen, die Standardwerte haben. Einstellungen, die keine Standardwerte haben, werden nicht geändert.

Höchstens einmal validieren

Gibt das Zeitintervall (in Sekunden) zwischen aufeinander folgenden Anforderungen an, eine Verbindung maximal ein Mal zu validieren. Der Standardwert ist 0, d.h. das Attribut ist nicht aktiviert.

Lecktimeout

Die Zeitspanne in Sekunden zur Verfolgung von Verbindungslecks in einem Connection-Pool. Der Wert 0 bedeutet, dass die Verfolgung von Verbindungslecks deaktiviert ist. Der Standardwert ist 0.

Leck zurückfordern

Wenn diese Option aktiviert ist, werden Verbindungen mit Leck im Pool wiederhergestellt, nachdem die Verfolgung von Verbindungslecks abgeschlossen ist. Diese Option ist standardmäßig deaktiviert.

Erstellungswiederholungsversuche

Die Anzahl der Neuversuche, wenn bei der Erstellung einer neuen Verbindung ein Fehler aufgetreten ist. Der Wert 0 bedeutet, dass keine Neuversuche zur Verbindungserstellung unternommen werden. Der Standardwert ist 0.

Wiederholungsintervall

Das Intervall (in Sekunden) zwischen zwei Versuchen zur Erstellung einer Verbindung. Dieses Attribut wird nur verwendet, wenn der Wert von "Erstellungswiederholungsversuche" größer als 0 ist. Der Standardwert ist 10 Sekunden.

Pooling

Aktiviert das Connection Pooling für den Pool. Diese Option ist standardmäßig aktiviert.

Verzögerte Verknüpfung

Bei aktivierter Option werden Verbindungen verzögert verknüpft, wenn ein Vorgang für sie durchgeführt wird. Die Verknüpfung wird wieder aufgehoben, wenn die Transaktion abgeschlossen ist und eine Komponentenmethode beendet wird. Auf diese Weise wird die Wiederverwendung der physischen Verbindungen vereinfacht. Diese Option ist standardmäßig deaktiviert.

Eintragung verzögerter Verbindungen

Bei aktivierter Option wird eine Ressource nur dann für die Transaktion eingetragen, wenn sie tatsächlich in einer Methode verwendet wird. Diese Option ist standardmäßig deaktiviert.

Mit Thread verknüpfen

Bei aktivierter Option wird eine Verbindung mit einem Thread verknüpft. Wenn derselbe Thread eine Verbindung benötigt, kann er die bereits mit diesem Thread verknüpfte Verbindung wiederverwenden, wodurch die zusätzliche Belastung durch das Abrufen einer Verbindung aus dem Pool vermieden wird. Diese Option ist standardmäßig deaktiviert.

Verbindungen abgleichen

Bei aktivierter Option ist der Verbindungsabgleich für den Pool aktiviert. Diese Option kann deaktiviert werden, wenn Sie wissen, dass die Verbindungen im Pool immer homogen sind und eine aus dem Pool abgerufene Verbindung daher nicht vom Ressourcenadapter abgeglichen werden muss. Diese Option ist standardmäßig aktiviert.

Max. Verbindungsnutzung

Die Anzahl der Wiederverwendungen einer Verbindung durch den Pool. Nachdem eine Verbindung so häufig wie angegeben wiederverwendet wurde, wird sie geschlossen. Der Wert 0 bedeutet, dass dieses Attribut deaktiviert ist. Der Standardwert ist 0.
