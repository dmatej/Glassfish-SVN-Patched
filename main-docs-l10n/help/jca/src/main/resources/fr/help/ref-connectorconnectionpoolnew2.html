Nouveau pool de connexions de connecteur (étape 2 sur 2)

La page Nouveau pool de connexions de connecteur (étape 2 sur 2) permet de créer un pool de connexions de connecteur.

Elle contient les options ci-après.

Ping

Lorsque cette option est activée, la commande ping est envoyée au pool de connexions lors de sa création ou de sa reconfiguration, afin d'identifier et de signaler toute erreur de valeur d'attribut. Cette option est désactivée par défaut.

Description

Description du pool de connexions de connecteur.

Taille initiale et minimale du pool

Nombre minimal de connexions dans le pool de connexions de connecteur. La valeur par défaut est 8.

Taille maximale du pool

Nombre maximal de connexions dans le pool de connexions de connecteur. La valeur par défaut est 32.

Valeur de redimensionnement du pool

Nombre de connexions à enlever lorsque le délai d'inactivité du pool est dépassé. La valeur par défaut est 2.

Délai d'inactivité

Durée maximale, en secondes, pendant laquelle une connexion peut rester inactive dans le pool. La valeur par défaut est 300.

Attente max.

Délai d'attente d'une application demandant une connexion au-delà duquel la connexion est temporisée. La valeur par défaut est 60 000.

Validation de la connexion

Si la case Requis est cochée, les connexions sont validées avant d'être affectées à l'application. Si la validation d'une ressource échoue, elle est détruite et une autre ressource est créée puis renvoyée. Cette option est désactivée par défaut.

Echec sur tout

Si la case Fermer toutes les connexions est cochée, GlassFish Server ferme toutes les connexions du pool et les rétablit en cas d'échec de l'une d'entre elles. Cette option est désactivée par défaut.

Prise en charge des transactions

La prise en charge des transactions choisie remplace l'attribut de prise en charge des transactions dans l'adaptateur de ressources associé au pool de connexions de façon à garantir la compatibilité descendante. En d'autres termes, la prise en charge d'un niveau de transaction inférieur ou égal à celui spécifié dans l'adaptateur de ressources est assurée, mais pas la prise en charge d'un niveau supérieur.

Les valeurs possibles sont les suivantes :

XATransaction

L'adaptateur de ressources prend en charge les transactions JTA et les transactions locales du gestionnaire de ressources en implémentant les interfaces LocalTransaction et XAResource. Les transactions XA sont contrôlées et déterminées par un gestionnaire de transactions externe à un gestionnaire de ressources.

LocalTransaction

L'adaptateur de ressources prend en charge les transactions locales en implémentant l'interface LocalTransaction. Les transactions locales sont gérées dans un gestionnaire de ressources et n'impliquent aucun gestionnaire de transactions externe.

NoTransaction

L'adaptateur de ressources ne prend pas en charge les transactions JTA ou les transactions locales du gestionnaire de ressources et n'implémente pas les interfaces XAResource et LocalTransaction.

Autres propriétés

Propriétés supplémentaires du pool de connexions de connecteur. Les propriétés disponibles dépendent de l'adaptateur de ressources du pool de connexions de connecteur. Les paires nom-valeur spécifiées dans cette table peuvent être utilisées pour remplacer les valeurs par défaut des propriétés définies par le fournisseur de l'adaptateur de ressources.

Si vous indiquez un nom mais pas de valeur pour une propriété, celle-ci est enlevée de la table à l'issue de la création du pool de connexions. Si vous ne souhaitez pas définir immédiatement la valeur d'une propriété mais conserver cette propriété dans la table afin de définir sa valeur ultérieurement, entrez des parenthèses vides dans le champ Valeur :
