Editar Atributos Avançados do Pool de Conexões do Conector

  1. Na árvore de navegação, expanda o nó Recursos.

  2. Expanda o nó Conectores.

  3. Sob o nó Conectores, selecione o nó Pools de Conexões do Conector.

    A página Pools de Conexões do Conector será aberta.

  4. Na página Pools de Conexões do Conector, clique no nome do pool de conexões do conector a ser editado.

    A página Editar Pool de Conexões do Conector será aberta.

  5. Na página Editar Pool de Conexões do Conector, clique na guia Avançado.

    A página Editar Atributos Avançados do Pool de Conexões do Conector será exibida.

  6. No campo Validar no Máximo uma Vez, digite o período, em segundos, após o qual uma conexão será validada no máximo uma vez.

    A definição desse atributo para um valor apropriado minimiza o número de solicitações de validação feitos por uma conexão. O valor default é 0, o que significa que o atributo não está ativado.

  7. No campo Time-out para Perda, digite o tempo, em segundos, para rastrear perdas de conexão em um pool de conexões.

    Um valor de 0 indica que o rastreamento da perda de conexão está desativado. O valor default é 0.

  8. Selecione a opção Reclamação de Perda, para permitir que as conexões perdidas sejam restauradas no pool após a conclusão do rastreamento de perda da conexão.

    Por default, essa opção está desativada.

  9. No campo Novas Tentativas de Criação, digite o número de tentativas a serem feitas se houver uma falha ao criar uma nova conexão.

    O valor de 0 indica que nenhuma tentativa será feita para criar a conexão novamente. O valor default é 0.

  10. No campo Intervalo para Nova Tentativa, digite o intervalo, em segundos, entre duas tentativas de criar uma conexão.

    Esse atributo só será usado se o valor Novas Tentativas de Criação for maior que 0. O valor default é 10 segundos.

  11. Marque a caixa de seleção Agrupamento para ativar o agrupamento de conexões para o pool.

    Por default, a opção está ativada.

  12. Marque a caixa de seleção Associação Lenta para permitir que as conexões sejam lentamente associadas quando uma operação for executada na mesmas.

    As conexões serão desassociadas quando a transação for concluída e um método de componente for finalizado, o que ajuda a reutilizar as conexões físicas. Por default, essa opção está desativada.

  13. Marque a caixa de seleção Alocação para Conexão Lenta, para inscrever um recurso para a transação somente quando ele for efetivamente utilizado em um método.

    Por default, essa opção está desativada.

  14. Marque a caixa de seleção Associate with Thread, para associar a conexão a um thread.

    Quando o mesmo thread for necessário a uma conexão, a conexão já associada ao thread poderá ser reutilizada, não incorrendo, portanto, na sobrecarga para obtenção de outra conexão do pool. Por default, essa opção está desativada.

  15. Marque a caixa de seleção Correspondência de Conexões, para ativar a correspondência de conexões para o pool.

    Desative esta opção se você souber que as conexões no pool serão sempre homogêneas e, portanto, uma conexão selecionada no pool não precisará coincidir com o adaptador de recurso. Por default, a opção está ativada.

  16. No campo Uso da Correspondência de Conexões, digite o número de vezes que uma conexão deve ser reutilizada pelo pool.

    Depois que uma conexão é reutilizada por um número especificado de vezes, ela será fechada. O valor de 0 indica que o atributo está desativado. O valor default é 0.

  17. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.
