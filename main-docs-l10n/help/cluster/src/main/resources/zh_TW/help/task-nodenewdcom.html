完成 DCOM 節點的建立

此作業是建立節點的 DCOM 節點的接續。

  1. 若要將 GlassFish Server 軟體從 DAS 主機複製至節點主機，請啟用「安裝 GlassFish Server」選項。

  2. 若要強制在網域管理伺服器(DAS) 的組態中建立節點 (即使驗證節點參數失敗)，請啟用「強制」選項。

  3. 如果您需要 GlassFish Server 先測試透過 DCOM 的節點主機連線，再建立節點，請執行下列步驟：

    1. 啟用「驗證 DCOM」選項。

    2. 在「遠端測試目錄」欄位中，鍵入遠端主機上 GlassFish Server 應該建立檔案以測試連線的資料夾名稱。

  4. 在「Windows 網域」欄位中，鍵入 Windows 網域名稱，內含「Windows 使用者名稱」選項指定的使用者。

  5. 在「Windows 使用者名稱」欄位中，鍵入要在此節點主機上執行透過 DCOM 連線至主機之處理作業的使用者名稱。

  6. 指定在登入此節點的主機時如何認證 Windows 使用者。

    備註:

    GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_WINDOWSPASSWORD 項目進行密碼驗證。密碼檔案指定於 asadmin(1) 公用程式的 --paswordfile 選項中。

    1. 若要指定透過儲存在 DAS 組態中的密碼來進行認證，請執行以下步驟：

      注意:

      此密碼在 DAS 組態中是以純文字儲存。如需增強安全性，請使用密碼別名

      1. 從「Windows 使用者驗證」下拉式清單中，選取「密碼」。

      2. 在「Windows 使用者密碼」欄位中，鍵入 Windows 使用者在登入此節點主機時將使用的密碼。

    2. 若要指定透過密碼別名來進行認證，請執行以下步驟：

      1. 從「Windows 使用者認證」下拉式清單中，選取「密碼別名」。

      2. 從「密碼別名」下拉式清單中選取現有的別名。

  7. 按一下「儲存」。
