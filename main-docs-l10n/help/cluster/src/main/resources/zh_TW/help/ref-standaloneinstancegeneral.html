一般資訊

「一般資訊」頁面包含下列選項。

啟動

啟動執行處理的按鈕。如果執行處理正在執行中，便會關閉此按鈕。

停止

停止執行處理的按鈕。如果執行處理已停止，便會關閉此按鈕。

重新啟動

重新啟動執行處理的按鈕。如果執行處理已停止，便會關閉此按鈕。

檢視日誌檔

檢視執行處理日誌檔的按鈕。

循環日誌

為執行處理循環日誌檔的按鈕。循環日誌檔時，會使用時間戳記名稱 (格式為 server.log_date-and-time) 來重新命名檔案，並且為新的日誌訊息建立空白日誌檔。變更會以動態方式套用。不需要重新啟動伺服器。

復原作業事件

復原執行處理的作業事件的按鈕。

名稱

建立執行處理時指定給執行處理的名稱。本欄位是唯讀的。

JVM

如果按一下「JVM 報告」，會開啟另外一個視窗並顯示 Java 虛擬機器的相關報告，包括摘要報告、記憶體管理和資源回收報告、類別載入報告以及目前繫線傾印。

狀態

指示執行處理是否正在執行中。本欄位是唯讀的。

節點

執行處理所在之節點的名稱。按一下該名稱，隨即會開啟節點的「編輯節點」頁面。

組態

執行處理參照的組態。按一下組態，隨即會開啟執行處理的 「組態」 頁面。

除錯

指出是否啟用執行處理的 Java 平台除錯程式架構 (Java Platform Debugger Architecture，JPDA) (http://java.sun.com/javase/technologies/core/toolsapis/jpda/) 除錯功能。本欄位是唯讀的。

如果是從 管理主控台 啟動執行處理，便不會啟用該執行處理的 JPDA 除錯功能。若要啟動已啟用 JPDA 除錯功能的執行處理，請使用 start-instance(1) 子命令。

HTTP 連接埠

執行處理監聽下列請求類型所用連接埠的連接埠號碼清單 (以逗號分隔)：

  • 管理請求

  • HTTP 請求

  • HTTPS 請求

本欄位是唯讀的。

IIOP 連接埠

執行處理針對下列連線類型所用連接埠的連接埠號碼清單 (以逗號分隔)：

  • 安全 IIOP 連線

  • 使用從屬端認證的安全 IIOP 連線

  • IIOP 連線

本欄位是唯讀的。
