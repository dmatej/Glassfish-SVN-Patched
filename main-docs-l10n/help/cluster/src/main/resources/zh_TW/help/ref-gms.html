編輯群組管理服務

使用「群組管理服務」頁面可變更 GMS 設定。

群組管理服務 (GMS) 是一種進程服務，可提供叢集監視、叢集成員身份和群組通訊服務。GMS 建立在群組成員身份探索和運作狀態監視協定的可組態堆疊之上。GMS 會執行下列功能：

  • 從叢集增加或移除一或多個成員執行處理，或者一或多個成員執行處理疑似故障或確認故障時，會通知已在 GlassFish Server 執行處理中註冊的模組。

  • 在叢集成員之間傳送和接收訊息。

「群組管理服務」頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原為設定值的按鈕。沒有預設值的設定則保持不變。

組態名稱

套用此頁面上之設定的組態名稱。本欄位是唯讀的。

遺失的活動訊號數上限

成員的遺失活動訊號數，超過此數目即表示可能發生伺服器執行處理故障。預設值為 3。

活動訊號頻率

成員遺失活動訊號數的時間間隔。預設值為 2000 毫秒。

群組探索逾時

GMS 等候探索其他叢集成員的時間量。預設值為 5000 毫秒。

故障驗證等待時間

驗證疑似故障的成員報告為故障之前所等候的時間量。預設值為 1500 毫秒。

其他特性

群組管理服務的其他特性表格。

GlassFish Server 未定義任何其他 GMS (群組管理服務) 特性。

對於每個特性，都會顯示以下資訊：

名稱

特性的名稱。

值

為所選取執行處理設定的特性值。

描述

提供有關特性之更多資訊的文字描述。

其他特性表也包含下列選項。

增加特性

增加特性的按鈕。按一下此按鈕即可在其他特性表中增加一列。

刪除特性

刪除一或多個所選取特性的按鈕。刪除的任何特性都會復原成其預設值或未定義 (如果未設定任何預設值)。
