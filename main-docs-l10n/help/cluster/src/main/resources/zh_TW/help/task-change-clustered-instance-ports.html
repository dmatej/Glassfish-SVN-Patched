修改叢集 GlassFish Server 執行處理的連接埠號碼

叢集 GlassFish Server 執行處理的連接埠號碼，最初是在執行處理繼承自其父系叢集的組態中設定。例如，如果出現兩個叢集執行處理位於相同主機的情況，則必須修改這些連接埠號碼。在此情況下，必須修改其中一個執行處理的連接埠號碼，因為兩個執行處理會共用相同的組態。

執行處理的連接埠號碼會儲存為 Java 系統特性。GlassFish Server 啟動時，會透過 Java 應用程式啟動程式的 -D 選項，將這些特性傳送至 Java 應用程式啟動程式。

  1. 在瀏覽樹狀結構中，展開「叢集」節點。

  2. 在展開的「叢集」節點下，選取該叢集的節點。

    此時將開啟叢集的「一般資訊」頁面。

  3. 在「一般資訊」頁面上，按一下「執行處理」標籤。

    此時將開啟該叢集的「叢集伺服器執行處理」頁面。

  4. 選取要修改其連接埠號碼的執行處理。

    此時將開啟執行處理的「一般資訊」頁面。

  5. 在「一般資訊」頁面上，按一下「特性」標籤。

    此時將開啟「執行處理系統特性」頁面。

    此頁面會顯示執行處理之下列系統特性目前的值：

    • 執行處理組態中預先定義的連接埠號碼

    • 為執行處理定義的自訂特性 (如果有)

  6. 對於您要修改的每個現有特性，請在該特性的「置換值」欄位中鍵入新值。

  7. 為所需的任何其他特性設定值。

    1. 在「其他特性」表格中，按一下「增加特性」。

      隨即會在其他特性表中增加一列。

    2. 在「執行處理變數名稱」欄位中，鍵入特性名稱。

    3. 在「置換值」欄位中，鍵入值。

  8. 按一下「儲存」。
