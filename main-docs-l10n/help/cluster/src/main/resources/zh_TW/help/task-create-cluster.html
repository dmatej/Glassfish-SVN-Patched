建立叢集

叢集是共用相同應用程式、資源和組態資訊的已命名 GlassFish Server 執行處理集合。

叢集可透過延展性、負載平衡和故障保護來提供高可用性。叢集中的執行處理或主機故障時，GlassFish Server 會偵測到該故障，並將請求從故障執行處理重新導向至叢集中的其他執行處理，然後復原使用者階段作業狀態。因為相同應用程式和資源均位於叢集中的所有執行處理上，因此執行處理可以容錯移轉至叢集中的任何其他執行處理。

無論執行處理是位於同一主機還是不同主機，GlassFish Server 都能讓您從單一主機以單一單元的方式管理叢集中的所有執行處理。

開始前

如果您打算將 GlassFish Server 執行處理增加至您要建立的叢集，請確認執行處理要位於的節點已經存在。

  1. 在瀏覽樹狀結構中，選取「叢集」節點。

    此時將開啟「叢集」頁面。

  2. 在「叢集」頁面上，按一下「新增」。

    此時將開啟「新建叢集」頁面。

  3. 在「叢集名稱」欄位中，鍵入叢集的名稱。

    名稱必須符合下列需求：

    • 名稱只能包含 ASCII 字元。

    • 名稱必須以字母、數字或底線為開頭。

    • 名稱只能包含下列字元：

      • 小寫字母

      • 大寫字母

      • 數字

      • 連字符

      • 小數點號

      • 底線

    • 名稱在網域中必須是唯一的，且不得為另一個叢集、已命名組態、GlassFish Server 執行處理或節點的名稱。

    • 名稱不得為 domain、server 或 GlassFish Server 所保留的任何其他關鍵字。

  4. 從「組態」下拉式清單中，選取叢集將使用的已命名組態。

    備註:

    如果您選取 default-config，就必須在下個步驟中將組態指定為不共用。default-config 組態無法共用。

  5. 指定是否要共用叢集將使用的組態。

    • 如果不共用組態，請保留選取「複製選取的組態」選項。

      建立叢集時，會複製選取的組態。為組態副本指定的名稱是 cluster-name-config，其中 cluster-name 是您要建立的叢集名稱。

    • 如果共用組態，請選取「參照選取的組態」選項。

      叢集將使用指定的現有已命名組態。

  6. 將 GlassFish Server 執行處理新增至叢集。

    備註:

    若要在類型為 CONFIG 且代表遠端主機的節點上建立執行處理，您必須使用 create-local-instance(1) 子命令。您無法使用 管理主控台 來達成此目的。

    您也可以在建立叢集後增加執行處理 (如將 GlassFish Server 執行處理新增至叢集所述)。

    增加各個執行處理，如下所示：

    1. 在「要建立的伺服器執行處理」區域，按一下「增加」。

      隨即會在叢集內的執行處理表中增加一列。

    2. 在「執行處理名稱」欄位中，鍵入執行處理的名稱。

    3. 在「加權」欄位中，鍵入表示執行處理負載平衡加權的整數。

      負載平衡加權可決定送至叢集供執行處理處理之所有請求的比例。

      例如，在有兩個執行處理的叢集中，您可能需要讓其中一個執行處理處理每四個請求中的一個請求，而讓另一個執行處理處理每四個請求中的三個請求。在這種情況下，可將處理每四個請求中之一個請求的執行處理加權設定為 1，將另一個執行處理加權設定為 3。如果您要使用百分比，請將這兩個執行處理的加權分別設定為 25 與 75。

      預設加權為 100。

    4. 從「節點」下拉式清單中，選取執行處理要位於的節點。

  7. 按一下「確定」。

    您會回到「叢集」頁面。叢集會增加至該頁面的叢集表。
