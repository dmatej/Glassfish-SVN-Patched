新建節點

使用「新建節點」頁面可建立 GlassFish Server 節點。節點代表其上安裝 GlassFish Server 軟體的主機。GlassFish Server 執行處理所在的每部主機都必須存在對應的節點。

「新節點」頁面包含的選項，取決於節點類型。

所有節點類型的選項

以下是所有節點類型可用的選項：

名稱

節點的名稱。

名稱必須符合下列需求：

  • 名稱只能包含 ASCII 字元。

  • 名稱必須以字母、數字或底線為開頭。

  • 名稱只能包含下列字元：

    • 小寫字母

    • 大寫字母

    • 數字

    • 連字符

    • 小數點號

    • 底線

  • 名稱在網域中必須是唯一的，且不得為另一個節點、叢集、已命名組態或 GlassFish Server 執行處理的名稱。

  • 名稱不得為 domain、server 或 GlassFish Server 所保留的任何其他關鍵字。

類型

下拉式清單，可指定下列其中一種節點類型：

組態

節點不支援遠端通訊。如果已選取 CONFIG，則不會顯示 SSH 組態或 DCOM 組態的任何選項，並會清除「安裝目錄」欄位。

DCOM

此節點支援透過分散式元件物件模型 (DCOM) 遠端協定進行通訊。DCOM 協定僅在 Windows 系統上提供。

SSH

節點支援透過安全 shell (SSH) 進行通訊。

節點主機

節點代表的主機名稱。此欄位是否為必要欄位，取決於節點類型：

  • 如果節點類型為 CONFIG，可能會省略主機名稱。可在建立位於節點上的執行處理時決定主機名稱。

  • 如果節點類型為 DCOM 或 SSH，則必須指定主機名稱。否則，將發生錯誤。

節點目錄

要包含在節點上所建立的 GlassFish Server 執行處理之目錄的路徑。如果指定相對路徑，則該路徑會相對於as-install 目錄。

預設值取決於節點類型：

  • 如果節點類型為 CONFIG，則不會為網域管理伺服器(DAS) 組態中的節點指定任何執行處理目錄。可在建立位於節點上的執行處理時決定節點目錄。

  • 如果節點類型為 DCOM，則預設值為 as-install\nodes，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

  • 如果節點類型為 SSH，則預設值為 as-install/nodes，其中 as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

安裝目錄

主機上 GlassFish Server 軟體基底安裝目錄之父系目錄的完整路徑 (例如 /export/glassfish3)。

預設值取決於節點類型：

  • 如果節點類型為 CONFIG，則不會為 DAS 組態中的節點指定任何安裝目錄。可在建立位於節點上的執行處理時決定安裝目錄。

  • 如果節點類型為 DCOM 或 SSH，則對於 DAS，預設值為 GlassFish Server 軟體預設基底安裝目錄的父系目錄。只有當 GlassFish Server 安裝在所有主機上的相同位置中，此預設值才有用。

DCOM 節點的其他選項

只有在「類型」欄位設為 DCOM 時，才可使用下列選項：

安裝 GlassFish Server

如果選取「已啟用」選項，DAS 會將 GlassFish Server 軟體從 DAS 主機複製到節點主機。若要將 GlassFish Server 軟體複製到主機，DAS 必須能夠透過 DCOM 連絡主機。

強制

如果選取「已啟用」選項，則即使驗證節點參數失敗，也會在 DAS 組態中建立節點。若要驗證節點的參數，DAS 必須能夠透過 DCOM 連絡節點的主機。

驗證 DCOM

如果選取「已啟用」選項，在建立節點之前，GlassFish Server 會測試透過 DCOM 與節點主機的連線。測試時，GlassFish Server 會將名稱為 delete_me.bat 的命令檔複製至主機。

如果您選取此選項，則會啟用「遠端測試目錄」欄位以讓您提供 GlassFish Server 複製命令檔的目標目錄。

遠端測試目錄

遠端主機上 GlassFish Server 應該在其中建立 delete_me.bat 檔案的資料夾。「Windows 使用者名稱」選項指定的使用者，必須具有此資料夾的寫入存取權。否則，將發生錯誤。預設值為 C:\。

Windows 網域

包含「Windows 使用者名稱」選項指定之使用者的 Windows 網域名稱。預設值為執行 管理主控台 的主機名稱。

Windows 使用者名稱

此節點之主機上要執行透過 DCOM 連線至主機處理作業的使用者。預設值為執行 DAS 處理作業的使用者。如果「節點主機」選項設定為 localhost-domain，便會忽略「Windows 使用者名稱」選項。

Windows 使用者驗證

可指定在登入此節點的主機時如何認證 Windows 使用者的下拉式清單。可能的認證方法如下：

密碼

Windows 使用者是透過儲存在 DAS 組態中的密碼來認證。

注意:

此密碼在 DAS 組態中是以純文字儲存。如需增強安全性，請使用密碼別名。

如果您選取此選項，即會啟用「Windows 使用者密碼」欄位，讓您可以提供密碼。

密碼別名

Windows 使用者是透過密碼別名來認證。如果選取此選項，請從「密碼別名」下拉式清單中選取密碼別名。

Windows 使用者密碼

Windows 使用者登入此節點主機時將使用的密碼。

密碼別名

為此網域定義之密碼別名的下拉式清單。

SSH 節點的其他選項

只有在「類型」欄位設為 SSH 時，才可使用下列選項：

安裝 GlassFish Server

如果選取「已啟用」選項，DAS 會將 GlassFish Server 軟體從 DAS 主機複製到節點主機。若要將 GlassFish Server 軟體複製到主機，DAS 必須能夠透過 SSH 連絡主機。

強制

如果選取「已啟用」選項，則即使驗證節點參數失敗，也會在 DAS 組態中建立節點。若要驗證節點的參數，DAS 必須能夠透過 SSH 連絡節點的主機。

SSH 連接埠

用於與此節點主機連線之 SSH 的連接埠。預設值為 22。如果「節點主機」選項設定為 localhost-domain，便會忽略「SSH 連接埠」選項。

SSH 使用者名稱

在此節點之主機上的使用者，會執行透過 SSH 連線至主機的處理作業。預設值為執行 DAS 處理作業的使用者。若要確保 DAS 可以讀取此使用者的 SSH 私密金鑰檔案，請指定執行 DAS 處理作業的使用者。如果「節點主機」選項設定為 localhost-domain，便會忽略「使用者名稱」選項。

SSH 使用者認證

可指定在登入此節點的主機時如何認證 SSH 使用者的下拉式清單。可能的認證方法如下：

金鑰檔案

SSH 使用者是透過 SSH 私密金鑰檔案來認證。如果選取此選項，請在「金鑰檔案」欄位中指定該檔案的絕對路徑。

密碼

SSH 使用者是透過儲存在 DAS 組態中的密碼來認證。

注意:

此密碼在 DAS 組態中是以純文字儲存。如需增強安全性，請使用金鑰檔案或密碼別名。

如果您選取此選項，即會啟用「SSH 使用者密碼」欄位，讓您可以提供密碼。

密碼別名

SSH 使用者是透過密碼別名來認證。如果選取此選項，請從「密碼別名」下拉式清單中選取密碼別名。

設定 SSH

如果選取「已啟用」選項，GlassFish Server 會在節點主機上設定 SSH 金鑰。GlassFish Server 設定 SSH 金鑰的方式是複製 DAS 主機中的現有金鑰檔案，或視需要於複製檔案前先產生檔案。只有在「SSH 使用者認證」選項設為「金鑰檔案」時，才可使用此選項。

產生金鑰

如果選取「已啟用」選項，則 GlassFish Server 會先產生 SSH 金鑰檔案，再將檔案複製至節點主機 (若檔案不存在)。若 SSH 金鑰檔案已存在，則即使選取「已啟用」選項，GlassFish Server 也不會產生檔案。

備註:

如果 SSH 金鑰檔案不存在，必須選取「已啟用」選項。否則，在您嘗試儲存變更時會發生錯誤。
SSH 使用者密碼

SSH 使用者將在登入此節點之主機時使用的密碼。

密碼別名

為此網域定義之密碼別名的下拉式清單。

公開金鑰檔案

SSH 公開金鑰檔案的絕對路徑，該檔案屬於「SSH 使用者名稱」選項指定的使用者。公開金鑰檔案的內容會附加至使用者在節點主機上的 .ssh/authorized_keys 檔案。如果主機上沒有 .ssh/authorized_keys 檔案，則 GlassFish Server 會建立此檔案。

金鑰檔案

SSH 私密金鑰檔案的絕對路徑，該檔案屬於「SSH 使用者名稱」選項指定的使用者。這個檔案用於向節點主機上的 sshd 常駐程式認證。

備註:

GlassFish Server 也支援透過密碼檔案中的 AS_ADMIN_SSHPASSWORD 項目進行密碼認證。密碼檔案指定於 asadmin(1) 公用程式的 --paswordfile 選項中。

如果 SSH 私密金鑰檔案受到密碼詞組保護，則密碼檔案必須包含 AS_ADMIN_SSHKEYPASSPHRASE 項目。

DAS 必須可以存取金鑰檔案的路徑，且 DAS 必須可以讀取金鑰檔案。

預設值為使用者之 .ssh 目錄中的金鑰檔案。若有多個金鑰檔案，GlassFish Server 會使用下列偏好設定順序：

  1. id_rsa

  2. id_dsa

  3. identity
