編輯節點

編輯節點會更新 GlassFish Server 節點的組態資料。編輯節點也可讓您將節點類型從 CONFIG 變更為 DCOM 或 SSH，以啟用節點的遠端通訊功能。

如果您將節點類型從 CONFIG 變更為 DCOM，則下列任一欄位保留空白時，便會套用預設值：

  • Windows 網域

  • Windows 使用者

如果您將節點類型從 CONFIG 變更為 SSH，則下列任一欄位保留空白時，便會套用預設值：

  • SSH 連接埠

  • SSH 使用者名稱

  • 金鑰檔案

  1. 在瀏覽樹狀結構中，展開「節點」節點。

  2. 從節點清單選取您要編輯的節點。

    此時將開啟「編輯節點」頁面。

  3. 從「類型」下拉式清單中，選取更新的節點類型。

    • 若要讓節點支援透過分散式元件物件模型 (DCOM) 遠端協定的通訊，請選取「DCOM」。

    • 若要讓節點支援透過安全 shell (SSH) 進行通訊，請選取「SSH」。

    • 若要移除節點中的遠端組態支援，請選取「CONFIG」。

      如果您選取「CONFIG」，則不會顯示 SSH 組態或 DCOM 組態的任何選項。如果目前的節點類型是 DCOM 或 SSH，而您選取「CONFIG」，則儲存變更時，會從節點組態中移除 DCOM 組態資料或 SSH 組態資料。

  4. 在「節點主機」欄位中，鍵入更新的節點要代表的主機名稱。

    此欄位是否為必要欄位，取決於節點類型：

    • 如果節點類型為 CONFIG，可能會省略主機名稱。可在建立位於節點上的執行處理時決定主機名稱。

    • 如果節點類型為 DCOM 或 SSH，則必須指定主機名稱。否則，將發生錯誤。

  5. 在「節點目錄」欄位中，鍵入要包含在節點上所建立的 GlassFish Server 執行處理之目錄的路徑。

    如果您鍵入相對路徑，則該路徑會相對於as-install 目錄，其中as-install 是主機上 GlassFish Server 軟體的基底安裝目錄。

  6. 在「安裝目錄」欄位中，鍵入主機上 GlassFish Server 軟體的基底安裝目錄之父系目錄的完整路徑 (例如 /export/glassfish3)。

  7. 請完成節點類型所需要的節點更新。

    1. 如果節點類型是 CONFIG，請按一下「儲存」。

    2. 如果節點類型是 DCOM，請依照完成 DCOM 節點的更新中的說明進行。

    3. 如果節點類型是 SSH，請依照完成 SSH 節點的更新中的說明進行。
