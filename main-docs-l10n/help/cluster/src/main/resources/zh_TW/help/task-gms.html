組態群組管理服務

群組管理服務 (GMS) 是一種進程服務，可提供叢集監視、叢集成員身份和群組通訊服務。GMS 會執行下列功能：

  • 從叢集增加或移除一或多個成員執行處理，或者一或多個成員執行處理疑似故障或確認故障時，會通知已在 GlassFish Server 執行處理中註冊的模組。

  • 在叢集成員之間傳送和接收訊息。

  1. 在瀏覽樹狀結構中，展開「組態」節點。

  2. 選取您要變更之叢集的組態。

  3. 在組態的節點下，選取「群組管理服務」節點。

    此時將開啟「群組管理服務」頁面。

  4. 在「遺失的活動訊號數上限」欄位中，輸入 GMS 確認疑似故障之前成員遺失的活動訊號數上限。

    預設值為 3。

  5. 在「活動訊號頻率」欄位中，輸入群組成員活動訊號連續廣播的時間間隔。

    預設值為 2000 毫秒。

  6. 在「群組探索逾時」欄位中，輸入 GMS 等候探索其他叢集成員的時間量。

    預設值為 5000 毫秒。

  7. 在「故障驗證等待時間」欄位中，輸入驗證疑似故障的成員報告為故障之前所等候的時間量。

    預設值為 1500 毫秒。

  8. 若要復原預設值，請按一下「載入預設值」按鈕。

  9. 在「其他特性」區段中，指定其他特性。

    • 若要增加特性，請按一下「增加特性」按鈕。在出現的空白列中，於「名稱」欄位中鍵入特性名稱，然後在「值」欄位中鍵入特性值。

    • 若要修改特性，請編輯特性的「值」欄位。

    • 若要刪除特性，請選取您要刪除之特性「名稱」欄位左側的核取方塊，然後按一下「刪除特性」按鈕。

  10. 按一下「儲存」。
