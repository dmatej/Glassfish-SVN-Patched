叢集伺服器執行處理

使用「叢集伺服器執行處理」頁面可建立和管理目前叢集中的 GlassFish Server 執行處理。

「叢集名稱」欄位是唯讀欄位，會顯示目前叢集的名稱。

「叢集伺服器執行處理」頁面會顯示隸屬於目前叢集的執行處理清單。對於每個執行處理，均提供以下資訊。

名稱

建立執行處理時指定給執行處理的名稱。按一下該名稱，隨即會開啟該執行處理的「一般資訊」頁面。

負載平衡加權

一個整數，代表執行處理的負載平衡加權。

負載平衡加權可決定送至叢集供執行處理處理之所有請求的比例。

例如，在有兩個執行處理的叢集中，您可能需要讓其中一個執行處理處理每四個請求中的一個請求，而讓另一個執行處理處理每四個請求中的三個請求。在這種情況下，可將處理每四個請求中之一個請求的執行處理加權設定為 1，將另一個執行處理加權設定為 3。如果您要使用百分比，請將這兩個執行處理的加權分別設定為 25 與 75。

預設加權為 100。

組態

執行處理參照的組態。按一下組態，隨即會開啟執行處理的 「組態」 頁面。

節點

執行處理所在之節點的名稱。按一下該名稱，隨即會開啟節點的「編輯節點」頁面。

狀態

指示執行處理是在執行中 (需要重新啟動才能更新組態變更) 還是不在執行中。本欄位是唯讀的。

伺服器執行處理表也包含下列選項。

新建

將執行處理增加至叢集的按鈕。

刪除

從叢集中移除一或多個所選取執行處理的按鈕。

注意:

如果透過主代理程式使用 Java Message Service (JMS) 叢集，請勿刪除與主代理程式相關聯的執行處理。如果必須刪除此執行處理，請使用 change-master-broker(1) 子命令，將主代理程式指定給不同的執行處理。
啟動

啟動叢集中一或多個所選取執行處理的按鈕。

停止

停止叢集中一或多個所選取執行處理的按鈕。
