Activation du service de gestion de groupes

Le service de gestion de groupes (GMS) est un service inclus dans un processus offrant des services de surveillance de clusters, d'appartenance au cluster et de communication de groupe. Il exécute les fonctions suivantes :

  • Il signale les modules inscrits dans une instance GlassFish Server lorsque des instances membres sont ajoutées à un cluster et enlevées d'un cluster, ou lorsque leur échec est suspecté ou a été confirmé.

  • Il permet l'envoi et la réception de messages entre les membres du cluster.

  1. Dans l'arborescence de navigation, développez le noeud Clusters.

  2. Sélectionnez le cluster pour lequel vous souhaitez activer le service de gestion de groupes (GMS).

    La page Informations générales du cluster apparaît.

  3. Sélectionnez la case Activé du champ GMS pour activer le service GMS pour le cluster.

    Cette option est sélectionnée par défaut. Pour désactiver le service GMS pour le cluster, désélectionnez la case Activé du champ GMS.

    Si cette option est activée, le service de gestion de groupes est démarré dans chaque instance de serveur du cluster et dans le serveur d'administration de domaine (DAS). Le serveur DAS se trouve dans chaque cluster dans lequel le service GMS est activé.

  4. Cliquez sur Enregistrer.
