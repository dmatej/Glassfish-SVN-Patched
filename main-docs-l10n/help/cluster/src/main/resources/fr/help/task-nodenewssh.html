Achèvement de la création d'un noeud SSH

Cette tâche constitue la suite de la procédure Création d'un noeud pour un noeud SSH.

  1. Pour copier le logiciel GlassFish Server à partir de l'hôte de serveur DAS vers l'hôte du noeud, activez l'option Installer GlassFish Server.

  2. Pour forcer la création du noeud dans la configuration du serveur d'administration de domaine (DAS) même si la validation des paramètres du noeud échoue, activez l'option Forcer.

  3. Dans le champ Port SSH, entrez le port à utiliser pour les connexions SSH à l'hôte de ce noeud.

  4. Dans le champ Nom utilisateur SSH, entrez le nom de l'utilisateur sur l'hôte de ce noeud qui exécutera le processus de connexion à l'hôte via SSH.

  5. Spécifiez le mode d'authentification de l'utilisateur SSH lors de sa connexion à l'hôte de ce noeud.

    • Pour spécifier l'authentification via un fichier de clés privées SSH, procédez comme suit :

      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Fichier de clés.

      2. Si vous voulez que GlassFish Server définisse une clé SSH sur l'hôte du noeud, activez l'option Configurer SSH.

        GlassFish Server configure la clé SSH en copiant les fichiers de clés existants à partir de l'hôte de serveur DAS ou, si nécessaire, en générant les fichiers avant de les copier.

        Les champs Générer une clé et Fichier de clés publiques sont ajoutés à la fenêtre.

      3. Si vous voulez que GlassFish Server génère les fichiers de clés SSH avant de les copier vers l'hôte du noeud, activez l'option Générer une clé.

        Si les fichiers de clés SSH existent, GlassFish Server ne génère pas les fichiers, même si l'option Activé est sélectionnée.

        Remarque:

        Si les fichiers de clés SSH n'existent pas, vous devez sélectionner l'option Activé. Sinon, une erreur se produit lorsque vous tentez d'enregistrer les modifications.

      4. Dans le champ Mot de passe utilisateur SSH, entrez le mot de passe que l'utilisateur SSH utilisera lors de la connexion à l'hôte de ce noeud.

      5. Dans le champ Fichier de clés publiques, entrez le chemin absolu du fichier de clés publiques SSH pour l'utilisateur spécifié par l'option Nom utilisateur SSH.

      6. Dans le champ Fichier de clés, entrez le chemin absolu du fichier de clés privées SSH pour l'utilisateur spécifié par l'option Nom utilisateur SSH.

        Remarque:

        GlassFish Server prend également en charge l'authentification par mot de passe via l'entrée AS_ADMIN_SSHPASSWORD dans le fichier de mots de passe. Le fichier de mots de passe est spécifié dans l'option -paswordfile de l'utilitaire asadmin(1).

    • Pour spécifier l'authentification via un mot de passe stocké dans la configuration du serveur DAS, procédez comme suit :

      Attention:

      Ce mot de passe est stocké sous forme de texte en clair dans la configuration du serveur DAS. Pour une sécurité accrue, utilisez un fichier de clés ou un alias de mot de passe.

      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Mot de passe.

      2. Dans le champ Mot de passe utilisateur SSH, entrez le mot de passe que l'utilisateur SSH utilisera lors de la connexion à l'hôte de ce noeud.

    • Pour spécifier l'authentification via un alias de mot de passe, procédez comme suit :

      1. Dans la liste déroulante Authentification de l'utilisateur SSH, sélectionnez Alias de mot de passe.

      2. Dans la liste déroulante Alias de mot de passe, sélectionnez un alias existant.

  6. Cliquez sur Enregistrer.
