Instances de serveur clusterisées

La page Instances de serveur clusterisées permet de créer et de gérer les instances GlassFish Server dans le cluster en cours.

Le champ Nom du cluster est un champ en lecture seule qui affiche le nom du cluster en cours.

La page Instances de serveur clusterisées affiche la liste des instances appartenant au cluster en cours. Pour chaque instance, les informations suivantes sont fournies.

Nom

Nom affecté à l'instance lors de sa création. Lorsque vous cliquez sur le nom, la page Informations générales de l'instance s'ouvre.

Poids EC

Entier qui représente la pondération de l'équilibrage de charge de l'instance.

La pondération de l'équilibrage de charge détermine la proportion de toutes les demandes destinées au cluster que l'instance doit traiter.

Par exemple, dans un cluster comportant deux instances, vous pouvez avoir besoin d'une instance pour le traitement de l'une des quatre demandes et d'une autre instance pour le traitement des trois demandes restantes. Dans ce cas, définissez la pondération de l'instance qui doit traiter une demande sur quatre sur 1 et celle de l'autre instance sur 3. Si vous préférez utiliser des pourcentages, définissez la pondération des instances sur 25 et 75, respectivement.

La pondération par défaut est de 100.

Configuration

Configuration que l'instance référence. Le fait de cliquer sur la configuration ouvre la page Configurations de l'instance.

Noeud

Nom du noeud sur lequel se trouve l'instance. Lorsque vous cliquez sur le nom, la page Modifier le noeud du noeud s'ouvre.

Statut

Mention indiquant si l'instance est en cours d'exécution, doit être redémarrée pour être mise à jour avec les modifications de la configuration ou n'est pas en cours d'exécution. Ce champ est en lecture seule.

La table Instances de serveur contient également les options ci-après.

Nouveau

Bouton permettant d'ajouter une instance au cluster.

Supprimer

Bouton permettant d'enlever les instances sélectionnées du cluster.

Attention:

Si vous utilisez un cluster JMS (Java Message Service) avec un broker maître, ne supprimez pas l'instance associée au broker maître. Si cette instance doit être supprimée, utilisez la sous-commande change-master-broker(1) pour affecter le broker maître à une autre instance.
Démarrer

Bouton permettant de démarrer les instances sélectionnées dans le cluster.

Arrêter

Bouton permettant d'arrêter les instances sélectionnées dans le cluster.
