Migration d'horloges EJB

Si une instance GlassFish Server est arrêtée, les horloges EJB (Enterprise JavaBeans) qui y sont installées s'arrêtent également. Si l'instance a été arrêtée de manière anormale ou inattendue, pour permettre l'exécution de ces horloges alors que l'instance est arrêtée, vous devez migrer les horloges vers une instance de serveur en cours d'exécution dans le cluster.

  1. Dans l'arborescence de navigation, développez le noeud Clusters.

  2. Dans l'arborescence de navigation, sous le noeud Clusters, sélectionnez le cluster.

    La page Informations générales du cluster apparaît.

  3. Sur la page Informations générales, cliquez sur le bouton Migrer les horloges EJB.

    La page Migrer les horloges EJB apparaît.

    Remarque:

    Si toutes les instances du cluster sont en cours d'exécution, le bouton Migrer les horloges EJB est désactivé.

    Pour vous aider à migrer les horloges, la page Migrer les horloges EJB présente une table d'horloges EJB comportant les informations ci-après.

    Nom du serveur

    Liste des instances GlassFish Server dans le cluster.

    Nombre d'horloges

    Nombre d'horloges persistantes sur chaque instance.

  4. Sur la page Migrer les horloges EJB, sélectionnez l'instance à partir de laquelle et vers laquelle vous souhaitez migrer les horloges.

    1. Dans la liste déroulante Source, sélectionnez l'instance arrêtée à partir de laquelle vous souhaitez migrer les horloges.

    2. Dans la liste déroulante Destination, sélectionnez l'instance en cours d'exécution vers laquelle vous souhaitez migrer les horloges.

    3. Cliquez sur OK.

Si l'instance source est en cours d'exécution ou si l'instance de destination n'est pas en cours d'exécution, la Console d'administration affiche un message d'erreur.
