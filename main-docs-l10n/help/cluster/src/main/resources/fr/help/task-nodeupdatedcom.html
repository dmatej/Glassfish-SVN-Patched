Achèvement de la mise à jour d'un noeud DCOM

Cette tâche constitue la suite de la procédure Modification d'un noeud pour un noeud DCOM.

  1. Pour forcer la création du noeud dans la configuration du serveur d'administration de domaine (DAS) même si la validation des paramètres du noeud échoue, activez l'option Forcer.

  2. Si vous voulez que GlassFish Server teste la connexion via DCOM à l'hôte du noeud avant de mettre à jour ce dernier, procédez comme suit :

    1. Activez l'option Valider DCOM.

    2. Dans le champ Répertoire de test distant, saisissez le nom du dossier sur l'hôte distant dans lequel GlassFish Server doit créer un fichier pour tester la connexion.

  3. Dans le champ Domaine Windows, saisissez le nom du domaine Windows contenant l'utilisateur spécifié par l'option Nom utilisateur Windows.

    Si vous changez le type du noeud en DCOM et que vous laissez ce champ vide, l'option Domaine Windows est définie sur le nom de l'hôte sur lequel la Console d'administration est exécutée.

  4. Dans le champ Nom utilisateur Windows, entrez le nom de l'utilisateur sur l'hôte de ce noeud qui exécutera le processus de connexion à l'hôte via DCOM.

    Si vous changez le type de noeud CONFIG en DCOM et que vous laissez ce champ vide, l'option Nom utilisateur Windows est définie sur l'utilisateur qui exécute le processus de serveur DAS.

  5. Spécifiez le mode d'authentification de l'utilisateur Windows lors de sa connexion à l'hôte de ce noeud.

    Remarque:

    GlassFish Server prend également en charge l'authentification par mot de passe via l'entrée AS_ADMIN_WINDOWSPASSWORD dans le fichier de mots de passe. Le fichier de mots de passe est spécifié dans l'option -paswordfile de l'utilitaire asadmin(1).

    1. Pour spécifier l'authentification via un mot de passe stocké dans la configuration du serveur DAS, procédez comme suit :

      Attention:

      Ce mot de passe est stocké sous forme de texte en clair dans la configuration du serveur DAS. Pour une sécurité accrue, utilisez un alias de mot de passe.

      1. Dans la liste déroulante Authentification de l'utilisateur Windows, sélectionnez Mot de passe.

      2. Dans le champ Mot de passe utilisateur Windows, entrez le mot de passe que l'utilisateur Windows utilisera lors de la connexion à l'hôte de ce noeud.

    2. Pour spécifier l'authentification via un alias de mot de passe, procédez comme suit :

      1. Dans la liste déroulante Authentification de l'utilisateur Windows, sélectionnez Alias de mot de passe.

      2. Dans la liste déroulante Alias de mot de passe, sélectionnez un alias existant.

  6. Cliquez sur Enregistrer.
