Informations générales

La page Informations générales contient les options ci-après.

Démarrer le cluster

Bouton permettant de démarrer le cluster. Si toutes les instances du cluster sont en cours d'exécution, ce bouton est désactivé.

Arrêter le cluster

Bouton permettant d'arrêter le cluster. Si toutes les instances du cluster sont arrêtées, ce bouton est désactivé.

Migrer les horloges EJB

Bouton permettant de migrer les horloges EJB (Enterprise Java Beans) d'une instance GlassFish Server arrêtée vers une instance en cours d'exécution dans le cluster. Un clic sur ce bouton ouvre la page Migrer les horloges EJB. Si toutes les instances du cluster sont en cours d'exécution, ce bouton est désactivé.

Changer de journal

Bouton permettant de changer le fichier journal du cluster. Lorsque le fichier journal est déplacé, il est renommé avec un horodatage, au format server.log_date-et-heure, et un fichier journal vide est créé pour les nouveaux messages de journalisation. Les modifications sont appliquées dynamiquement. Il n'est pas nécessaire de redémarrer le serveur.

Nom du cluster

Nom affecté au cluster lors de sa création. Ce champ est en lecture seule.

JVM

Si vous cliquez sur Rapport JVM, une fenêtre distincte s'ouvre et affiche des rapports sur la Java Virtual Machine, notamment un rapport récapitulatif, un rapport sur la gestion et le nettoyage de la mémoire, un rapport de chargement de classes et le vidage des threads en cours.

Configuration

Configuration sélectionnée pour le cluster lors de sa création. Le fait de cliquer sur la configuration ouvre la page Configurations du cluster. Ce champ est en lecture seule.

GMS

Si cette option est activée, le service de gestion de groupes (GMS) est démarré dans chaque instance GlassFish Server du cluster et dans le serveur d'administration de domaine (DAS). Le serveur DAS fonctionne dans chaque cluster pour lequel le service GMS est activé. Le service GMS fournit des fonctions de surveillance de cluster, d'appartenance au cluster et des services de communication de groupe. Cette option est activée par défaut.

Port de multidiffusion

Numéro du port de communication sur lequel le service GMS écoute les événements de groupe. Cette option doit spécifier un numéro de port valide compris entre 2048 et 49151. La valeur par défaut est générée automatiquement dans cette plage.

Adresse de multidiffusion

Adresse sur laquelle le service GMS écoute les événements de groupe. Cette option doit indiquer une adresse de multidiffusion dans la plage 224.0.0.0 à 239.255.255.255. La valeur par défaut est 228.9.XX.YY, XX et YY étant des valeurs indépendantes générées automatiquement, comprises entre 0 et 255.

Adresse d'interface de liaison

Adresse IP (Internet Protocol) de l'interface réseau à laquelle le service GMS est lié. Cette option doit spécifier l'adresse IP d'une interface réseau locale. La valeur par défaut correspond à toutes les adresses de l'interface réseau publique.

Sur un ordinateur à connexions multiples, cette option permet de configurer l'interface réseau utilisée pour le service GMS. Un ordinateur à connexions multiples possède au moins deux interfaces réseau.

Afin de spécifier une adresse valide pour toutes les instances GlassFish Server du cluster, utilisez une propriété système pour définir l'adresse de chaque instance séparément.

Par exemple, utilisez la page Propriétés système de cluster pour créer la propriété système GMS-BIND-INTERFACE-ADDRESS-cluster-name. Définissez ensuite l'option Adresse de liaison de cette page sur ${GMS-BIND-INTERFACE-ADDRESS-cluster-name} pour spécifier la propriété système. Enfin, pour chaque instance du cluster, définissez la propriété système GMS-BIND-INTERFACE-ADDRESS-cluster-name sur l'adresse d'interface réseau requise sur l'ordinateur de l'instance.

Statut

Indication du nombre d'instances en cours d'exécution ou non dans le cluster. Ce champ est en lecture seule.
