Instances de serveur autonomes

La page Instances de serveur autonomes permet de créer et de gérer des instances GlassFish Server autonomes.

Elle affiche la liste de toutes les instances autonomes du domaine. Pour chaque instance, les informations suivantes sont fournies.

Nom

Nom affecté à l'instance lors de sa création. Lorsque vous cliquez sur le nom, la page Informations générales de l'instance s'ouvre.

Poids EC

Entier qui représente la pondération de l'équilibrage de charge de l'instance.

La pondération de l'équilibrage de charge détermine la proportion de toutes les demandes destinées au cluster que l'instance doit traiter.

Par exemple, dans un cluster comportant deux instances, vous pouvez avoir besoin d'une instance pour le traitement de l'une des quatre demandes et d'une autre instance pour le traitement des trois demandes restantes. Dans ce cas, définissez la pondération de l'instance qui doit traiter une demande sur quatre sur 1 et celle de l'autre instance sur 3. Si vous préférez utiliser des pourcentages, définissez la pondération des instances sur 25 et 75, respectivement.

La pondération par défaut est de 100.

Configuration

Configuration sélectionnée pour l'instance lors de sa création. Le fait de cliquer sur la configuration ouvre la page Configurations de l'instance.

Noeud

Nom du noeud sur lequel se trouve l'instance. Lorsque vous cliquez sur le nom, la page Modifier le noeud du noeud s'ouvre.

Statut

Mention indiquant si l'instance est en cours d'exécution.

La table Instances de serveur contient également les options ci-après.

Nouveau

Bouton permettant de créer une instance autonome.

Supprimer

Bouton permettant de supprimer les instances autonomes sélectionnées.

Démarrer

Bouton permettant de démarrer les instances autonomes sélectionnées.

Arrêter

Bouton permettant d'arrêter les instances autonomes sélectionnées.
