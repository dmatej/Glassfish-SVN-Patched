SSHノードの作成を終了するには

このタスクは、ノードを作成するにはのSSHノードに対する続きです。

  1. GlassFish ServerソフトウェアをDASソフトからノード・ホストにコピーするには、GlassFish Serverのインストール・オプションを有効にします。

  2. ノードのパラメータの検証に失敗した場合にドメイン管理サーバー(DAS)の構成でノードが強制的に作成されるようにするには、「強制」オプションを有効にします。

  3. 「SSHポート」フィールドに、このノードのホストに対するSSH接続に使用するポートを入力します。

  4. 「SSHユーザー名」フィールドに、ホストにSSH経由で接続するためのプロセスを実行する、このノードのホストのユーザーの名前を入力します。

  5. このノードのホストへのログイン時にSSHユーザーを認証する方法を指定します。

    • SSH秘密鍵ファイルによる認証を指定するには、次のステップを実行します。

      1. 「SSHユーザー認証」ドロップダウン・リストから「鍵ファイル」を選択します。

      2. GlassFish Serverでノード・ホストにSSH鍵を設定する必要がある場合は、「SSHの設定」オプションを有効にします。

        GlassFish Serverでは、既存の鍵ファイルをDASホストからコピーするか、または必要な場合はファイルのコピー前にファイルを生成することによって、SSH鍵が設定されます。

        「鍵の生成」フィールドおよび「公開鍵ファイル」フィールドがウィンドウに追加されます。

      3. ファイルをノード・ホストにコピーする前に、GlassFish ServerでSSH鍵ファイルを生成する必要がある場合は、「鍵の生成」オプションを有効にします。

        SSH鍵ファイルが存在する場合は、「有効」オプションを選択してもGlassFish Serverによってファイルが生成されることはありません。

        注意:

        SSH鍵ファイルが存在しない場合は、「有効」オプションを選択する必要があります。そうしないと、変更を保存するときにエラーが発生します。

      4. 「SSHユーザー・パスワード」フィールドに、このノードのホストへのログイン時にSSHユーザーが使用するパスワードを入力します。

      5. 「公開鍵ファイル」フィールドに、「SSHユーザー名」オプションで指定するユーザーのSSH公開鍵ファイルの絶対パスを入力します。

      6. 「鍵ファイル」フィールドに、「SSHユーザー名」オプションで指定するユーザーのSSH秘密鍵ファイルの絶対パスを入力します。

        注意:

        GlassFish Serverでは、パスワード・ファイルのAS_ADMIN_SSHPASSWORDエントリを使用したパスワード認証もサポートされます。パスワード・ファイルは、asadmin(1)ユーティリティの--paswordfileオプションで指定します。

    • DAS構成に格納されているパスワードによる認証を指定するには、次のステップを実行します。

      注意:

      このパスワードは、DAS構成にクリア・テキストで格納されています。セキュリティ強化のため、鍵ファイルまたはパスワード・エイリアスを使用してください。

      1. 「SSHユーザー認証」ドロップダウン・リストから「パスワード」を選択します。

      2. 「SSHユーザー・パスワード」フィールドに、このノードのホストへのログイン時にSSHユーザーが使用するパスワードを入力します。

    • パスワード・エイリアスによる認証を指定するには、次のステップを実行します。

      1. 「SSHユーザー認証」ドロップダウン・リストから「パスワード・エイリアス」を選択します。

      2. 「パスワード・エイリアス」ドロップダウン・リストから既存のエイリアスを選択します。

  6. 「保存」をクリックします。
