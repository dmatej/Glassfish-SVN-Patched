So wird das Update eines DCOM-Knotens beendet

Diese Task ist die Fortsetzung für einen DCOM-Knoten von So wird ein Knoten bearbeitet.

  1. Damit der Knoten in der Konfiguration des Domainadministrationsservers (DAS, Domain Administration Server) erstellt wird, auch wenn die Validierung der Knotenparameter nicht erfolgreich verläuft, aktivieren Sie die Option "Erzwingen".

  2. Wenn GlassFish Server die Verbindung zum Knotenhost über DCOM testen soll, bevor der Knoten upgedatet wird, führen Sie die folgenden Schritte aus:

    1. Aktivieren Sie die Option "DCOM validieren".

    2. Geben Sie im Feld "Remote-Testverzeichnis" den Namen des Ordners auf dem Remote-Host ein, in dem GlassFish Server eine Datei zum Testen der Verbindung erstellen soll.

  3. Geben Sie im Feld "Windows-Domain" den Namen der Windows-Domain ein, die den Benutzer enthält, der mit der Option "Windows-Benutzername" angegeben wird.

    Wenn Sie den Knotentyp in DCOM ändern und dieses Feld leer lassen, wird die Option "Windows-Domain" auf den Namen des Hosts festgelegt, auf dem die &AdminConsole ausgeführt wird.

  4. Geben Sie im Feld "Windows-Benutzername" den Namen des Benutzers auf dem Host dieses Knotens an, der die Verbindung mit dem Host über DCOM herstellen soll.

    Wenn Sie den Knotentyp von CONFIG in DCOM ändern und dieses Feld leer lassen, wird für die Option "Windows-Benutzername" der Benutzer verwendet, der den DAS-Prozess ausführt.

  5. Geben Sie an, wie der Windows-Benutzer authentifiziert wird, wenn er sich beim Host dieses Knotens anmeldet.

    Hinweis:

    GlassFish Server unterstützt auch die Kennwortauthentifizierung über den Eintrag AS_ADMIN_WINDOWSPASSWORD in der Kennwortdatei. Die Kennwortdatei wird in der Option -paswordfile des Utilitys asadmin1 angegeben.

    1. Um die Authentifizierung durch ein Kennwort anzugeben, das in der DAS-Konfiguration gespeichert wird, gehen Sie wie folgt vor:

      Achtung:

      Dieses Kennwort wird als Klartext in der DAS-Konfiguration gespeichert. Zur erweiterten Sicherheit verwenden Sie einen Kennwortalias.

      1. Wählen Sie aus der Dropdown-Liste "Windows-Benutzerauthentifizierung" die Option "Kennwort".

      2. Geben Sie im Feld "Windows-Benutzerkennwort" das Kennwort ein, das der Windows-Benutzer verwendet, wenn er sich beim Host dieses Knotens anmeldet.

    2. Um die Authentifizierung durch einen Kennwortalias anzugeben, gehen Sie wie folgt vor:

      1. Wählen Sie aus der Dropdown-Liste "Windows-Benutzerauthentifizierung" die Option "Kennwortalias".

      2. Wählen Sie aus der Dropdown-Liste "Kennwortalias" einen vorhandenen Aliasnamen.

  6. Klicken Sie auf "Speichern".
