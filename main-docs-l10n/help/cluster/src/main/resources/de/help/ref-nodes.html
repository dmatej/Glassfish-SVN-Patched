Knoten

Auf der Seite "Knoten" werden die Knoten in der Domain aufgelistet. Für jeden Knoten werden folgende Informationen angezeigt:

Name

Der Name, der dem Knoten bei der Erstellung zugewiesen wurde. Wenn Sie auf den Namen klicken, wird die Seite Knoten bearbeiten für den Knoten geöffnet.

Knotenhost

Der Name des Hosts, den der Knoten darstellt.

Typ

Der Knotentyp ist einer der folgenden Typen:

CONFIG

Der Knoten unterstützt keine Remote-Kommunikation.

DCOM

Der Knoten unterstützt die Kommunikation über das Distributed Component Object Model-(DCOM-)Remote-Protokoll. Das DCOM-Protokoll ist nur auf Windows-Systemen verfügbar.

SSH

Der Knoten unterstützt die Kommunikation über Secure Shell (SSH).

Instanzen

Eine Liste der Instanzen auf dem Knoten und der Angabe, ob die Instanz ausgeführt wird:

  • Wenn die Instanz ausgeführt wird, wird ein Häkchen (✓) angezeigt.

  • Wenn die Instanz angehalten ist, wird ein Ausrufezeichen (!) angezeigt.

Wenn Sie auf den Namen einer Instanz klicken, wird die Seite "Allgemeine Informationen" für die Instanz geöffnet.

Aktion

Nur Knoten vom Typ DCOM und SSH: Ein Link zu der Ping-Aktion für den Knoten. Durch Klicken auf den Ping-Link wird getestet, ob der Knoten über das Kommunikationsprotokoll zugänglich ist, das der Knoten unterstützt.

Für CONFIG-Knoten sind keine Aktionen möglich.

Die Tabelle "Knoten" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen eines Knotens.

Löschen

Schaltfläche zum Löschen von einem oder mehreren ausgewählten Knoten.

Löschen und deinstallieren

Schaltfläche zum Löschen von mindestens einem gewählten Knoten und zur Deinstallation der GlassFish Server-Software von dem Host, den jeder gelöschte Knoten darstellt.

Hinweis:

Die GlassFish Server-Software wird nicht von einem Host deinstalliert, den ein Knoten des Typs CONFIG darstellt. Bei Knoten des Typs CONFIG führt diese Schaltfläche dieselbe Aktion aus wie die Schaltfläche "Löschen".
Verwandte Aufgaben

  • So wird ein Knoten erstellt

  • So werden Knoten gelöscht

  • So werden alle Knoten in der aktuellen Domain aufgeführt

  • So wird ein Knoten bearbeitet

  • So wird getestet, ob ein Knoten zugänglich ist
