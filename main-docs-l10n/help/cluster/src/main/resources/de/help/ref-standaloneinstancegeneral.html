Allgemeine Informationen

Die Seite "Allgemeine Informationen" enthält die folgenden Optionen.

Starten

Schaltfläche zum Starten der Instanz. Wenn die Instanz ausgeführt wird, ist diese Schaltfläche deaktiviert.

Anhalten

Schaltfläche zum Anhalten der Instanz. Wenn die Instanz angehalten ist, ist diese Schaltfläche deaktiviert.

Neustart

Schaltfläche zum erneuten Starten der Instanz. Wenn die Instanz angehalten ist, ist diese Schaltfläche deaktiviert.

Logdateien anzeigen

Schaltfläche zum Anzeigen von Logdateien für die Instanz.

Log rotieren

Schaltfläche zum Rotieren der Logdatei für die Instanz. Beim Rotieren der Logdatei wird die Datei mit einem Zeitstempelnamen im Format server.log; date-and-time umbenannt und eine leere Logdatei wird für neue Logmeldungen erstellt. Die Änderungen werden dynamisch angewendet. Ein Neustarten des Servers ist nicht erforderlich.

Transaktionen wiederherstellen

Schaltfläche zum Wiederherstellen von Transaktionen für die Instanz.

Name

Der Name, der der Instanz bei der Erstellung zugewiesen wurde. Dieses Feld ist schreibgeschützt.

JVM

Wenn Sie auf einen JVM-Bericht klicken, wird ein separates Fenster geöffnet, das Berichte über die Java Virtual Machine anzeigt, einschließlich eines zusammenfassenden Berichts, Speichermanagement- und Garbage Collection-Berichts, Class Loading-Berichts und einem aktuellen Threaddump.

Status

Gibt an, ob die Instanz ausgeführt wird. Dieses Feld ist schreibgeschützt.

Knoten

Der Name des Knotens, auf dem sich die Instanz befindet. Wenn Sie auf den Namen klicken, wird die Seite Knoten bearbeiten für den Knoten geöffnet.

Konfiguration

Die Konfiguration, auf die die Instanz verweist. Durch Klicken auf die Konfiguration wird die Seite Konfigurationen für die Instanz geöffnet.

Debug

Gibt an, ob das Java Platform Debugger Architecture-(JPDA-)Debugging (http://java.sun.com/javase/technologies/core/toolsapis/jpda/) für die Instanz aktiviert ist. Dieses Feld ist schreibgeschützt.

Wenn eine Instanz über die Administrationskonsole gestartet wird, wird das JPDA-Debugging für die Instanz nicht aktiviert. Um eine Instanz mit aktiviertem JPDA-Debugging zu starten, verwenden Sie den Unterbefehl start-instance(1).

HTTP-Ports

Eine durch Komma getrennte Liste der Nummern von Ports, auf denen die Instanz auf die folgenden Anforderungstypen horcht:

  • Administrationsanforderungen

  • HTTP-Anforderungen

  • HTTPS-Anforderungen

Dieses Feld ist schreibgeschützt.

IIOP-Ports

Eine durch Komma getrennte Liste der Nummern von Ports, die die Instanz für die folgenden Verbindungstypen verwendet:

  • Sichere IIOP-Verbindung

  • Sichere IIOP-Verbindungen mit Clientauthentifizierung

  • IIOP-Verbindungen

Dieses Feld ist schreibgeschützt.

Verwandte Aufgaben

  • So werden Standalone-GlassFish Server-Instanzen gestartet

  • So werden Standalone-GlassFish Server-Instanzen angehalten

  • So zeigen Sie Logdateien für eine Standalone-GlassFish Server-Instanz an

  • So werden Transaktionen für eine Standalone-GlassFish Server-Instanz wiederhergestellt
