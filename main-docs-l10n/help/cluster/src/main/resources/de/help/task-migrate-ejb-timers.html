So werden EJB-Timer migriert

Wenn eine GlassFish Server-Instanz angehalten wird, werden ebenfalls die auf dieser Instanz installierten EJB-Timer (Enterprise JavaBeans) angehalten. Wenn die Instanz anormal oder unerwartet angehalten wird, können Sie bei angehaltener Instanz diese Timer aktivieren, indem Sie die Timer auf eine ausgeführte Serverinstanz im Cluster migrieren.

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Cluster" ein.

  2. Wählen Sie in der Navigationsbaumstruktur im Knoten "Cluster" das Cluster.

    Die Seite "Allgemeine Informationen" für das Cluster wird geöffnet.

  3. Klicken Sie auf der Seite "Allgemeine Informationen" auf "EJB-Timer migrieren".

    Die Seite "EJB-Timer migrieren" wird geöffnet.

    Hinweis:

    Wenn alle Instanzen im Cluster ausgeführt werden, ist die Schaltfläche "EJB-Timer migrieren" deaktiviert.

    Die Seite "EJB-Timer migrieren" enthält eine Tabelle der EJB-Timer mit den folgenden Informationen, um Sie bei der Migration der Timer zu unterstützen..

    Server Name (Servername)

    Liste der GlassFish Server-Instanzen im Cluster.

    Timerwert

    Anzahl persistenter Timer auf jeder Instanz.

  4. Wählen Sie auf der Seite "EJB-Timer migrieren" die Instanz, von der und auf die die Timer migriert werden sollen.

    1. Wählen Sie in der Dropdown-Liste "Quelle" die angehaltene Instanz, von der aus Sie die Timer migrieren möchten.

    2. Wählen Sie in der Dropdown-Liste "Ziel" die ausgeführte Instanz, auf die Sie die Timer migrieren möchten.

    3. Klicken Sie auf "OK".

Wenn die Quellinstanz ausgeführt oder die Zielinstanz nicht ausgeführt wird, zeigt die Administrationskonsole eine Fehlermeldung an.
