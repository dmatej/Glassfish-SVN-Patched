DCOM 노드 생성을 완료하는 방법

이 작업은 DCOM 노드의 노드를 생성하는 방법에 이어서 수행되는 작업입니다.

  1. GlassFish Server 소프트웨어를 DAS 호스트에서 노드 호스트로 복사하려면 [GlassFish Server 설치] 옵션을 사용으로 설정합니다.

  2. 노드 매개변수 검증이 실패하더라도 DAS(도메인 관리 서버) 구성에 노드가 생성되도록 하려면 [강제 적용] 옵션을 사용으로 설정합니다.

  3. 노드를 생성하기 전에 GlassFish Server에서 DCOM을 통한 노드 호스트 접속을 테스트하도록 하려면 다음 단계를 수행합니다.

    1. [DCOM 검증] 옵션을 사용으로 설정합니다.

    2. [원격 테스트 디렉토리] 필드에 접속 테스트를 위해 GlassFish Server에서 파일을 생성해야 하는 원격 호스트 폴더의 이름을 입력합니다.

  4. [Windows 도메인] 필드에 [Windows 사용자 이름] 옵션에 지정된 사용자를 포함하는 Windows 도메인의 이름을 입력합니다.

  5. [Windows 사용자 이름] 필드에 DCOM을 통해 호스트에 접속하기 위한 프로세스를 실행할 이 노드 호스트의 사용자 이름을 입력합니다.

  6. 이 노드 호스트에 로그인할 때 Windows 사용자 인증 방식을 지정합니다.

    주:

    GlassFish Server에서는 비밀번호 파일의 AS_ADMIN_WINDOWSPASSWORD 항목을 통한 비밀번호 인증도 지원합니다. 비밀번호 파일은 asadmin(1) 유틸리티의 --paswordfile 옵션에 지정되어 있습니다.

    1. DAS 구성에 저장된 비밀번호를 통해 인증을 지정하려면 다음 단계를 수행합니다.

      주의:

      이 비밀번호는 DAS 구성에 일반 텍스트로 저장됩니다. 보안 향상을 위해 비밀번호 별칭을 사용하십시오.

      1. [Windows 사용자 인증] 드롭다운 목록에서 비밀번호를 선택합니다.

      2. [Windows 사용자 비밀번호] 필드에 Windows 사용자가 이 노드 호스트에 로그인할 때 사용하는 비밀번호를 입력합니다.

    2. 비밀번호 별칭을 통해 인증을 지정하려면 다음 단계를 수행합니다.

      1. [Windows 사용자 인증] 드롭다운 목록에서 비밀번호 별칭을 선택합니다.

      2. [비밀번호 별칭] 드롭다운 목록에서 기존 별칭을 선택합니다.

  7. [저장]을 누릅니다.
