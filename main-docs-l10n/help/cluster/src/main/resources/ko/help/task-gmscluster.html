클러스터에 대한 그룹 관리 서비스 설정을 구성하는 방법

GMS(그룹 관리 서비스)는 클러스터 모니터링, 클러스터 멤버쉽 및 그룹 통신 서비스를 제공하는 in-process 서비스입니다. GMS는 다음 기능을 수행합니다.

  • 하나 이상의 멤버 인스턴스가 클러스터에 추가되거나 클러스터에서 제거되거나 장애가 발생한 것으로 의심되거나 확인되면 GlassFish Server 인스턴스의 등록된 모듈에 알립니다.

  • 클러스터 멤버 간에 메시지를 주고 받을 수 있는 기능을 제공합니다.

  1. 탐색 트리에서 [클러스터] 노드를 확장합니다.

  2. GMS(그룹 관리 서비스) 설정을 구성할 클러스터를 선택합니다.

    클러스터에 대한 [일반 정보] 페이지가 열립니다.

  3. [GMS 사용] 체크 박스를 선택하여 클러스터에 대해 GMS를 사용으로 설정합니다.

    이 옵션은 기본적으로 선택되어 있습니다. 클러스터에 대해 GMS를 사용 안함으로 설정하려면 [GMS 사용] 체크 박스에서 선택 표시를 제거합니다.

    사용으로 설정된 경우 그룹 관리 서비스가 클러스터 및 DAS(도메인 관리 서버)의 각 서버 인스턴스에서 시작됩니다. GMS가 사용으로 설정된 각 클러스터에 DAS가 참여합니다.

  4. [멀티캐스트 포트] 필드에 그룹 관리 서비스가 그룹 이벤트를 수신하는 데 사용할 포트를 입력합니다.

    2048-49151 범위의 표준 UDP 포트 번호를 지정합니다. 기본값은 이 범위에서 자동으로 생성된 값입니다.

  5. [멀티캐스트 주소] 필드에 그룹 관리 서비스가 그룹 이벤트를 수신하는 데 사용할 주소를 입력합니다.

    클래스 D IP 멀티캐스트 주소를 지정합니다. 클래스 D IP 주소는 224.0.0.0 이상 239.255.255.255 이하 범위에 있습니다. 주소 224.0.0.0는 예약되어 있어 사용하지 않습니다. 기본값은 228.9.XX.YY입니다. 여기서 XXYY는 0에서 255 사이의 자동으로 생성된 독립 값입니다.

  6. 둘 이상의 네트워크 인터페이스가 있는 경우 [바인드 인터페이스 주소] 필드에 그룹 관리 서비스에 대한 네트워크 인터페이스 주소를 입력합니다.

    클러스터의 서버 인스턴스를 호스팅하는 시스템에 둘 이상의 네트워크 인터페이스가 있는 경우 이 설정을 통해 GMS에 사용할 네트워크 인터페이스를 구성할 수 있습니다. 이 값은 로컬 네트워크 인터페이스 IP 주소여야 합니다. 기본적으로 모든 공용 네트워크 인터페이스 주소가 사용됩니다.

    클러스터의 모든 GlassFish Server 인스턴스에 대해 적합한 주소를 지정하려면 각 인스턴스에 대한 주소를 개별적으로 설정하는 시스템 속성을 사용합니다.

    예를 들어 DAS에 시스템 속성 GMS-BIND-INTERFACE-ADDRESS-cluster-name을 생성합니다. 그런 다음 이 바인드 인터페이스 주소 옵션을 ${GMS-BIND-INTERFACE-ADDRESS-cluster-name}으로 설정하여 시스템 속성을 지정합니다. 마지막으로, 클러스터의 각 인스턴스에 대해 GMS-BIND-INTERFACE-ADDRESS-cluster-name 시스템 속성을 인스턴스의 시스템에 필요한 네트워크 인터페이스 주소로 설정합니다.

  7. 추가 속성을 지정하려면 [속성] 탭을 선택한 다음 [클러스터 속성] 하위 탭을 선택합니다.

    • 속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

    • 속성을 수정하려면 속성의 [값] 필드를 편집합니다.

    • 속성을 삭제하려면 삭제할 속성의 [이름] 필드 왼쪽에 있는 체크 박스를 선택한 다음 [속성 삭제] 단추를 누릅니다.

  8. [저장]을 누릅니다.
