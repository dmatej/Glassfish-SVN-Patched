클러스터를 시작하는 방법

클러스터를 시작하면 아직 실행되고 있지 않은 클러스터의 모든 GlassFish Server 인스턴스가 시작됩니다.

시작하기 전에

다음 필수 조건이 충족되는지 확인합니다.

  • 클러스터의 인스턴스가 있는 모든 노드가 원격 통신에 대해 사용으로 설정되어 있습니다.

  • DAS(도메인 관리 서버) 사용자가 DCOM(Distributed Component Object Model) 원격 프로토콜 또는 SSH(보안 셸)를 사용하여 클러스터의 인스턴스가 있는 노드의 호스트에 로그인할 수 있습니다.

이러한 필수 조건 중 일부가 충족되지 않은 경우 start-local-instance(1) 하위 명령으로 명령행에서 개별적으로 각 인스턴스를 시작하여 클러스터를 시작합니다.

  1. 탐색 트리에서 [클러스터] 노드를 선택합니다.

    [클러스터] 페이지가 열립니다.

  2. [클러스터] 페이지에서 클러스터 이름 옆에 있는 체크 박스를 선택합니다.

  3. [클러스터 시작]을 누릅니다.

    클러스터를 시작할 것인지 확인하는 메시지가 표시됩니다.

  4. 클러스터를 시작할 것임을 확인합니다.

    클러스터가 시작됩니다. 인스턴스가 시작되면 [클러스터] 페이지에서 클러스터에 있는 각 인스턴스의 상태가 업데이트됩니다.
