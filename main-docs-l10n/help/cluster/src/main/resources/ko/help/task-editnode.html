노드를 편집하는 방법

노드를 편집하면 GlassFish Server 노드의 구성 데이터가 업데이트됩니다. 노드를 편집하면 노드 유형을 CONFIG에서 DCOM 또는 SSH로 변경하여 노드에 대한 원격 통신이 가능하도록 할 수도 있습니다.

노드 유형을 CONFIG에서 DCOM으로 변경하는 경우 다음 필드를 비워 두면 기본값이 적용됩니다.

  • Windows 도메인

  • Windows 사용자

노드 유형을 CONFIG에서 SSH로 변경하는 경우 다음 필드를 비워 두면 기본값이 적용됩니다.

  • SSH 포트

  • SSH 사용자 이름

  • 키 파일

  1. 탐색 트리에서 [노드] 노드를 확장합니다.

  2. 노드 목록에서 편집할 노드를 선택합니다.

    [노드 편집] 페이지가 열립니다.

  3. [유형] 드롭다운 목록에서 업데이트된 노드 유형을 선택합니다.

    • 노드가 DCOM(Distributed Component Object Model) 원격 프로토콜을 통한 통신을 지원하도록 하려면 DCOM을 선택합니다.

    • 노드가 SSH(보안 셸)를 통한 통신을 지원하도록 하려면 SSH를 선택합니다.

    • 노드에서 원격 구성에 대한 지원을 제거하려면 CONFIG를 선택합니다.

      CONFIG를 선택하면 SSH 구성 또는 DCOM 구성에 대한 옵션이 표시되지 않습니다. 현재 노드 유형이 DCOM 또는 SSH인데 CONFIG를 선택하는 경우 변경 사항을 저장하면 노드 구성에서 SSH 구성 데이터 또는 DCOM 구성 데이터가 제거됩니다.

  4. [노드 호스트] 필드에 업데이트된 노드가 나타낼 호스트의 이름을 입력합니다.

    이 필드가 필수인지 여부는 노드 유형에 따라 결정됩니다.

    • 노드 유형이 CONFIG이면 호스트 이름을 생략할 수 있습니다. 호스트 이름은 노드에 있는 인스턴스를 생성할 때 지정할 수 있습니다.

    • 노드 유형이 DCOM 또는 SSH인 경우 호스트 이름을 지정해야 합니다. 그렇지 않으면 오류가 발생합니다.

  5. [노드 디렉토리] 필드에 노드에 생성된 GlassFish Server 인스턴스를 포함할 디렉토리의 경로를 입력합니다.

    상대 경로를 입력하는 경우 경로는 as-install 디렉토리에 대한 상대 경로입니다. 여기서 as-install은 호스트에 있는 GlassFish Server 소프트웨어의 기본 설치 디렉토리입니다.

  6. [설치 디렉토리] 필드에 호스트에 있는 GlassFish Server 소프트웨어 기본 설치 디렉토리의 상위 디렉토리에 대한 전체 경로를 입력합니다(예: /export/glassfish3).

  7. 노드 유형에 따라 노드 업데이트를 완료합니다.

    1. 노드 유형이 CONFIG인 경우 [저장]을 누릅니다.

    2. 노드 유형이 DCOM인 경우 DCOM 노드 업데이트를 완료하는 방법의 지침을 따릅니다.

    3. 노드 유형이 SSH인 경우 SSH 노드 업데이트를 완료하는 방법의 지침을 따릅니다.

관련 항목

  • DCOM 노드 업데이트를 완료하는 방법

  • SSH 노드 업데이트를 완료하는 방법

  • 노드를 생성하는 방법

  • 노드를 삭제하는 방법

  • 현재 도메인의 모든 노드를 나열하는 방법

  • 연결 가능한 노드인지 테스트하는 방법
