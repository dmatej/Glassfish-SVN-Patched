클러스터 속성

[클러스터 속성] 페이지에는 선택된 클러스터에 대해 설정된 속성 목록이 표시됩니다. 이러한 속성은 클러스터에 대한 선택적 구성 정보를 추가합니다.

[클러스터 이름] 필드는 선택한 클러스터의 이름을 표시하는 읽기 전용 필드입니다.

각 속성에 대해 표시되는 정보는 다음과 같습니다.

이름

속성의 이름입니다.

값

선택된 인스턴스에 대해 설정된 속성의 값입니다.

설명

속성에 대한 자세한 정보를 제공하는 텍스트 설명입니다.

추가 속성 테이블에는 다음 옵션도 포함되어 있습니다.

속성 추가

속성을 추가하는 단추입니다. 이 단추를 누르면 추가 속성 테이블에 행이 추가됩니다.

속성 삭제

선택한 속성을 하나 이상 삭제하는 단추입니다. 삭제된 속성은 해당 기본값으로 되돌려지거나 기본값이 설정되어 있지 않은 경우 정의되지 않습니다.

GlassFish Server에 정의된 클러스터 속성은 다음과 같습니다.

GMS_DISCOVERY_URI_LIST

클러스터 검색에 사용할 클러스터 내 GlassFish Server 인스턴스의 위치입니다. 이 속성은 GMS(그룹 관리 서비스)가 메시지 브로드캐스팅에 멀티캐스트를 사용하지 않는 경우에만 필요합니다.

이 속성의 적합한 값은 다음과 같습니다.

  • 쉼표로 구분된 URI(Uniform Resource Identifier)의 목록. 각 URI는 GlassFish Server 인스턴스 또는 DAS를 가리켜야 합니다. 이 형식은 여러 GlassFish Server 인스턴스가 동일한 호스트에서 실행 중인 경우 필요합니다.

    목록에 있는 각 URI의 형식은 다음과 같습니다.

    scheme://host-name-or -IP-address:port

    • scheme은 URI 체계(tcp)입니다.

    • host-name-or -IP-address는 인스턴스가 실행 중인 호스트의 호스트 이름 또는 IP 주소입니다.

    • port는 인스턴스가 GMS의 메시지를 수신하는 포트의 포트 번호입니다. 시스템 속성 GMS_LISTENER_PORT-clustername을 인스턴스에 설정해야 합니다. 이 시스템 속성을 인스턴스에 설정하는 방법은 Oracle GlassFish Server High Availability Administration Guide의 "Discovering a Cluster When Multicast Transport Is Unavailable"을 참조하십시오.

  • DAS 또는 인스턴스가 실행 중인 쉼표로 구분된 IP 주소 또는 호스트 이름의 목록입니다. IP 주소와 호스트 이름의 혼합 목록을 포함할 수 있습니다. 이 형식은 하나의 클러스터된 인스턴스가 각 호스트에서 실행 중인 경우에만 사용할 수 있습니다. GMS_LISTENER_PORT 속성값은 도메인의 각 클러스터에 대해 고유해야 합니다.

  • 키워드 generate. 이 형식은 클러스터의 한 인스턴스가 각 호스트에서 실행 중이고 DAS가 개별 호스트에서 실행 중인 경우에만 사용할 수 있습니다. 동일한 호스트의 복수 인스턴스가 동일한 클러스터의 멤버일 수 없습니다. GMS_LISTENER_PORT 속성값은 도메인의 각 클러스터에 대해 고유해야 합니다.

GMS_LISTENER_PORT

클러스터가 GMS의 메시지를 수신하는 포트의 포트 번호입니다.

기본값은 GMS_LISTENER_PORT-cluster-name 시스템 속성에 대한 참조입니다. 기본적으로 이 시스템 속성은 설정되어 있지 않습니다. 이 경우 GMS가 GMS_TCPSTARTPORT 및 GMS_TCPENDPORT 속성으로 정의된 범위에서 사용 가능한 포트를 선택합니다. 기본적으로 이 범위는 9090-9200입니다. 대부분의 경우 기본 동작으로도 충분합니다.

그러나 GMS가 메시지 브로드캐스팅에 멀티캐스트를 사용하지 않는 경우 이 속성이 클러스터의 모든 GlassFish Server 인스턴스에 적합한 포트 번호를 지정해야 합니다. 기본값으로 이 요구 사항을 충족하려면 각 인스턴스에 대한 포트 번호를 개별적으로 설정하는 시스템 속성을 사용합니다.

예를 들어 클러스터 시스템 속성 페이지에서 DAS에 대한 시스템 속성 GMS_LISTENER_PORT-cluster-name을 생성할 수 있습니다. 그런 다음, 클러스터의 각 인스턴스에 대해 GMS_LISTENER_PORT-cluster-name 시스템 속성을 인스턴스가 GMS의 메시지를 수신하는 포트 번호로 설정합니다. 클러스터에서 GMS_LISTENER_PORT 속성의 기본값이 이 시스템 속성을 참조합니다.

GMS_LOOPBACK

인스턴스가 해당 인스턴스가 클러스터에 브로드캐스트하는 자체 응용 프로그램 레벨 메시지를 수신할 수 있는지 여부를 지정합니다.

가능한 값은 다음과 같습니다.

false

인스턴스가 자체 메시지를 수신할 수 없습니다(기본값).

true

인스턴스가 자체 메시지를 수신할 수 있습니다. 인스턴스가 클러스터의 유일한 인스턴스일 경우 인스턴스 테스트에 이 설정을 사용합니다.

GMS_MULTICAST_TIME_TO_LIVE

메시지가 무시되기 전에 다음 이벤트 유형에 대한 멀티캐스트 메시지에 발생할 수 있는 최대 반복 또는 전송 횟수입니다.

  • 그룹 검색

  • 멤버 하트비트

  • 멤버쉽 변경

DAS 및 클러스터된 인스턴스가 배치된 네트워크의 구성을 일치시키려면 이 값을 가능한 한 낮게 설정합니다. 시스템의 사용 가능한 최저값을 확인하려면 validate-multicast(1) 하위 명령을 사용합니다.

0 값은 멀티캐스트 메시지가 해당 멀티캐스트 메시지가 브로드캐스트되는 호스트를 절대 떠나지 않도록 합니다.

1 값은 스위치 또는 라우터로 접속된 동일한 서브넷에 있는 호스트 간에 메시지가 브로드캐스트되지 않도록 막을 수 있습니다.

기본값 4는 호스트가 스위치 또는 라우터로 접속된 네트워크에 있는 모든 클러스터 멤버에 메시지가 성공적으로 브로드캐스트되도록 합니다.

GMS_TCPENDPORT

GMS_LISTENER_PORT-cluster-name 시스템 속성이 설정되지 않은 경우 GMS가 사용 가능한 포트를 선택하는 범위에서 가장 높은 포트 번호입니다. 기본값은 9200입니다.

GMS_TCPSTARTPORT

GMS_LISTENER_PORT-cluster-name 시스템 속성이 설정되지 않은 경우 GMS가 사용 가능한 포트를 선택하는 범위에서 가장 낮은 포트 번호입니다. 기본값은 9090입니다.
