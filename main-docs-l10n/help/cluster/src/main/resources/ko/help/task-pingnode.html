연결 가능한 노드인지 테스트하는 방법

원격 호스트를 나타내는 노드에서 GlassFish Server 인스턴스를 생성하거나 시작하려면 DAS가 해당 노드에 연결할 수 있어야 합니다. 해당 노드에서 인스턴스를 생성하거나 시작하기 전에 DAS가 해당 노드에 연결할 수 있는지 테스트합니다.

테스트할 노드의 유형은 DCOM 또는 SSH여야 합니다. CONFIG 유형의 노드는 원격 통신에 사용할 수 없으므로 테스트할 수 없습니다.

시작하기 전에

노드가 나타내는 호스트에 노드 유형에 적합한 프로토콜이 구성되었는지 확인합니다. 이 프로토콜을 통해 노드를 원격 통신에 사용할 수 있습니다.

  • 노드 유형이 DCOM인 경우 DCOM(Distributed Component Object Model) 프로토콜을 구성해야 합니다.

  • 노드 유형이 SSH인 경우 SSH(보안 셸) 프로토콜을 구성해야 합니다.

    1. 탐색 트리에서 [노드] 노드를 선택합니다.

      노드 페이지가 열립니다. 이 페이지에는 현재 도메인의 모든 노드가 나열됩니다.

    2. 노드 테이블에서 테스트하려는 노드의 [핑] 링크를 누릅니다.

      DAS가 노드에 연결할 수 있는지 여부를 나타내는 메시지가 [노드] 페이지에 추가됩니다.

관련 항목

  • 노드를 생성하는 방법

  • 노드를 삭제하는 방법

  • 현재 도메인의 모든 노드를 나열하는 방법

  • 노드를 편집하는 방법
