클러스터된 서버 인스턴스

[클러스터된 서버 인스턴스] 페이지에서는 현재 클러스터의 GlassFish Server 인스턴스를 생성하고 관리할 수 있습니다.

[클러스터 이름] 필드는 현재 클러스터의 이름을 표시하는 읽기 전용 필드입니다.

[클러스터된 서버 인스턴스] 페이지에는 현재 클러스터의 멤버인 인스턴스의 목록이 표시됩니다. 각 인스턴스에 대해 제공되는 정보는 다음과 같습니다.

이름

인스턴스가 생성될 때 인스턴스에 지정된 이름입니다. 이름을 누르면 해당 인스턴스에 대한 일반 정보 페이지가 열립니다.

LB 가중치

인스턴스의 로드 균형 조정 가중치를 나타내는 정수입니다.

로드 균형 조정 가중치는 인스턴스가 처리해야 하는 클러스터에 대한 모든 요청의 비율을 결정합니다.

예를 들어 두 개의 인스턴스로 구성된 클러스터에서 한 인스턴스가 네 개의 요청 중에서 하나를 처리하고 다른 인스턴스가 네 개의 요청 중에서 세 개를 처리해야 할 수 있습니다. 이 경우 네 개의 요청 중에서 하나를 처리할 인스턴스의 가중치를 1로 설정하고 다른 인스턴스의 가중치를 3으로 설정합니다. 백분율을 사용하려는 경우 인스턴스의 가중치를 각각 25와 75로 설정합니다.

기본 가중치는 100입니다.

구성

인스턴스가 참조하는 구성입니다. 구성을 누르면 인스턴스에 대한 구성 페이지가 열립니다.

노드

인스턴스가 있는 노드의 이름입니다. 이름을 누르면 해당 노드에 대한 노드 편집 페이지가 열립니다.

상태

인스턴스가 실행 중인지, 구성 변경 사항으로 업데이트하기 위해 재시작해야 하는지 또는 실행 중이지 않은지 여부를 나타냅니다. 이 필드는 읽기 전용입니다.

서버 인스턴스 테이블에는 다음 옵션도 포함되어 있습니다.

새로 만들기

클러스터에 인스턴스를 추가하는 단추입니다.

삭제

클러스터에서 선택한 하나 이상의 인스턴스를 제거하는 단추입니다.

주의:

마스터 중개자에서 JMS(Java Message Service) 클러스터를 사용하는 경우에는 이 마스터 중개자와 연관된 인스턴스를 삭제하지 마십시오. 이 인스턴스를 삭제해야 하는 경우에는 change-master-broker(1) 하위 명령을 사용하여 마스터 중개자를 다른 인스턴스에 지정하십시오.
시작

클러스터의 선택한 하나 이상의 인스턴스를 시작하는 단추입니다.

정지

클러스터의 선택한 하나 이상의 인스턴스를 정지하는 단추입니다.

관련 작업

  • 클러스터에 GlassFish Server 인스턴스를 추가하는 방법

  • 클러스터된 GlassFish Server 인스턴스를 삭제하는 방법

  • 클러스터된 GlassFish Server 인스턴스를 시작하는 방법

  • 클러스터된 GlassFish Server 인스턴스를 정지하는 방법

동일한 asadmin 하위 명령

  • create-instance(1)

  • delete-instance(1)

  • list-instances(1)

  • start-instance(1)

  • stop-instance(1)
