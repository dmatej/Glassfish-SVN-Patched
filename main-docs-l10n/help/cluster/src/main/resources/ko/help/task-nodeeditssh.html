SSH 노드 업데이트를 완료하는 방법

이 작업은 SSH 노드의 노드를 편집하는 방법에 이어서 수행되는 작업입니다.

  1. 노드 매개변수 검증이 실패하더라도 DAS(도메인 관리 서버) 구성에서 노드가 업데이트되도록 하려면 [강제 적용] 옵션을 사용으로 설정합니다.

  2. [SSH 포트] 필드에 이 노드의 호스트에 대한 SSH 접속에 사용할 포트를 입력합니다.

    노드 유형을 CONFIG에서 SSH로 변경하고 이 필드를 비워 두면 SSH 포트 옵션이 22로 설정됩니다.

  3. [SSH 사용자 이름] 필드에 SSH를 통해 호스트에 접속하기 위한 프로세스를 실행할 이 노드의 호스트 사용자 이름을 입력합니다.

    노드 유형을 CONFIG에서 SSH로 변경하고 이 필드를 비워 두면 SSH 사용자 이름 옵션이 DAS 프로세스를 실행하고 있는 사용자로 설정됩니다.

  4. 이 노드의 호스트에 로그인할 때 SSH 사용자 인증 방식을 지정합니다.

    • SSH 전용 키 파일을 통해 인증을 지정하려면 다음 단계를 수행합니다.

      1. [SSH 사용자 인증] 드롭다운 목록에서 [키 파일]을 선택합니다.

      2. GlassFish Server에서 노드 호스트에 SSH 키를 생성하도록 하려면 [SSH 설정] 옵션을 사용으로 설정합니다.

        GlassFish Server에서는 기존 키 파일을 DAS 호스트에서 복사하거나, 필요한 경우 파일을 복사하기 전에 파일을 생성하여 SSH 키를 생성합니다.

        [키 생성] 필드 및 [공용 키 파일] 필드가 창에 추가됩니다.

      3. 파일을 노드 호스트로 복사하기 전에 GlassFish Server에서 SSH 키 파일을 생성하도록 하려면 [키 생성] 옵션을 사용으로 설정합니다.

        SSH 키 파일이 있는 경우에는 [사용] 옵션을 선택해도 GlassFish Server에서 이 파일을 생성하지 않습니다.

        주:

        SSH 키 파일이 없는 경우 [사용] 옵션을 선택해야 합니다. 그렇지 않을 경우 변경 사항을 저장하려고 하면 오류가 발생합니다.

      4. [SSH 사용자 비밀번호] 필드에 SSH 사용자가 이 노드 호스트에 로그인할 때 사용하는 비밀번호를 입력합니다.

      5. [공용 키 파일] 필드에 [SSH 사용자 이름] 옵션에 지정된 사용자의 SSH 공용 키 파일에 대한 절대 경로를 입력합니다.

      6. [키 파일] 필드에 [SSH 사용자 이름] 옵션에 지정된 사용자의 SSH 전용 키 파일에 대한 절대 경로를 입력합니다.

        주:

        GlassFish Server에서는 비밀번호 파일의 AS_ADMIN_SSHPASSWORD 항목을 통한 비밀번호 인증도 지원합니다. 비밀번호 파일은 asadmin(1) 유틸리티의 --paswordfile 옵션에 지정되어 있습니다.

        노드 유형을 SSH로 변경하고 이 필드를 비워 두면 키 파일 옵션이 사용자의 .ssh 디렉토리에 있는 키 파일로 설정됩니다. 키 파일이 여러 개 있을 경우 GlassFish Server에서는 다음 우선순위로 사용합니다.

        id_rsa
id_dsa
identity

    • DAS 구성에 저장된 비밀번호를 통해 인증을 지정하려면 다음 단계를 수행합니다.

      주의:

      이 비밀번호는 DAS 구성에 일반 텍스트로 저장됩니다. 보안 향상을 위해 키 파일이나 비밀번호 별칭을 사용하십시오.

      1. [SSH 사용자 인증] 드롭다운 목록에서 비밀번호를 선택합니다.

      2. [SSH 사용자 비밀번호] 필드에 SSH 사용자가 이 노드 호스트에 로그인할 때 사용하는 비밀번호를 입력합니다.

    • 비밀번호 별칭을 통해 인증을 지정하려면 다음 단계를 수행합니다.

      1. [SSH 사용자 인증] 드롭다운 목록에서 비밀번호 별칭을 선택합니다.

      2. [비밀번호 별칭] 드롭다운 목록에서 기존 별칭을 선택합니다.

  5. [저장]을 누릅니다.
