일반 정보

[일반 정보] 페이지에는 다음 옵션이 포함되어 있습니다.

클러스터 시작

클러스터를 시작하는 단추입니다. 클러스터에 있는 모든 인스턴스가 실행 중인 경우 이 단추가 비활성화됩니다.

클러스터 정지

클러스터를 정지하는 단추입니다. 클러스터에 있는 모든 인스턴스가 정지된 경우 이 단추가 비활성화됩니다.

EJB 타이머 마이그레이션

정지된 GlassFish Server 인스턴스에서 클러스터의 실행 중인 인스턴스로 EJB(Enterprise JavaBeans) 기술 타이머를 마이그레이션하는 단추입니다. 이 단추를 누르면 EJB 타이머 마이그레이션 페이지가 열립니다. 클러스터에 있는 모든 인스턴스가 실행 중인 경우 이 단추가 비활성화됩니다.

로그 교체

클러스터에 대한 로그 파일을 교체하는 단추입니다. 로그 파일이 교체되면 파일이 server.log_date-and-time 형식의 시간 기록 이름이 포함된 이름으로 바뀌고 새 로그 메시지에 대한 빈 로그 파일이 생성됩니다. 변경 사항이 동적으로 적용됩니다. 서버를 재시작할 필요는 없습니다.

클러스터 이름

클러스터가 생성될 때 클러스터에 지정된 이름입니다. 이 필드는 읽기 전용입니다.

JVM

JVM 보고서를 누르면 별도의 창이 열리고 요약 보고서, 메모리 관리 및 불필요한 정보 수집 보고서, 클래스 로딩 보고서 및 현재 스레드 덤프를 포함한 Java Virtual Machine에 대한 보고서가 표시됩니다.

구성

클러스터가 생성될 때 클러스터에 대해 선택된 구성입니다. 구성을 누르면 클러스터에 대한 구성 페이지가 열립니다. 이 필드는 읽기 전용입니다.

GMS

이 옵션을 사용으로 설정하면 GMS(그룹 관리 서비스)가 클러스터 및 DAS(도메인 관리 서버)에 있는 각 GlassFish Server 인스턴스에서 시작됩니다. GMS가 사용으로 설정된 각 클러스터에 DAS가 참여합니다. GMS는 클러스터 모니터링, 클러스터 멤버쉽 및 그룹 통신 서비스를 제공합니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다.

멀티캐스트 포트

GMS가 그룹 이벤트를 수신하는 통신 포트의 포트 번호입니다. 이 옵션은 2048-49151 범위의 적합한 포트 번호를 지정해야 합니다. 기본값은 이 범위에서 자동으로 생성된 값입니다.

멀티캐스트 주소

GMS가 그룹 이벤트를 수신하는 주소입니다. 이 옵션은 224.0.0.0 - 239.255.255.255 사이의 범위에서 멀티캐스트 주소를 지정해야 합니다. 기본값은 228.9.XX.YY입니다. 여기서 XXYY는 0에서 255 사이의 자동으로 생성된 독립 값입니다.

바인드 인터페이스 주소

GMS가 바인딩되는 네트워크 인터페이스의 IP(인터넷 프로토콜) 주소입니다. 이 옵션은 로컬 네트워크 인터페이스의 IP 주소를 지정해야 합니다. 기본값은 모든 공용 네트워크 인터페이스 주소입니다.

복수 홈 시스템에서 이 옵션은 GMS에 사용되는 네트워크 인터페이스를 구성합니다. 복수 홈 시스템은 둘 이상의 네트워크 인터페이스를 가집니다.

클러스터의 모든 GlassFish Server 인스턴스에 대해 적합한 주소를 지정하려면 각 인스턴스에 대한 주소를 개별적으로 설정하는 시스템 속성을 사용합니다.

예를 들어 시스템 속성 클러스터 시스템 속성GMS-BIND-INTERFACE-ADDRESS-cluster-name을 생성하려면 페이지를 사용합니다. 그런 다음 이 페이지의 바인드 주소 옵션을 ${GMS-BIND-INTERFACE-ADDRESS-cluster-name}으로 설정하여 시스템 속성을 지정합니다. 마지막으로, 클러스터의 각 인스턴스에 대해 GMS-BIND-INTERFACE-ADDRESS-cluster-name 시스템 속성을 인스턴스의 시스템에 필요한 네트워크 인터페이스 주소로 설정합니다.

상태

실행 중인 있는 클러스터의 인스턴스 수와 실행 중이지 않은 클러스터의 인스턴스 수를 나타냅니다. 이 필드는 읽기 전용입니다.
