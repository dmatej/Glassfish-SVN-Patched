클러스터에 GlassFish Server 인스턴스를 추가하는 방법

GlassFish Server 인스턴스는 응용 프로그램 배치의 기본을 구성합니다. 클러스터된 인스턴스는 클러스터의 멤버입니다. 각 인스턴스는 단일 도메인에 속하며 자체 디렉토리 구조, 구성 및 배치된 응용 프로그램을 가집니다. 또한 각 인스턴스에는 Java EE(Java Platform, Enterprise Edition) 웹 컨테이너 및 EJB(Enterprise JavaBeans) 기술 컨테이너도 포함됩니다. 모든 새 인스턴스에는 인스턴스가 있게 될 호스트를 정의하는 노드에 대한 참조가 포함되어야 합니다.

주:

원격 호스트를 나타내는 CONFIG 유형의 노드에 인스턴스를 생성하려면 create-local-instance(1) 하위 명령을 사용해야 합니다. 이 용도로는 관리 콘솔을 사용할 수 없습니다.
시작하기 전에

다음 항목이 있는지 확인합니다.

  • 인스턴스를 추가할 클러스터. 지침은 클러스터를 생성하는 방법을 참조하십시오.

  • 인스턴스가 상주할 노드. 지침은 노드를 생성하는 방법을 참조하십시오.

  1. 탐색 트리에서 [클러스터] 노드를 확장합니다.

  2. 확장된 [클러스터] 노드에서 클러스터에 대한 노드를 선택합니다.

    클러스터에 대한 [일반 정보] 페이지가 열립니다.

  3. [일반 정보] 페이지에서 [인스턴스] 탭을 누릅니다.

    클러스터에 대한 [클러스터된 서버 인스턴스] 페이지가 열립니다.

  4. [클러스터된 서버 인스턴스] 페이지에서 [새로 만들기]를 누릅니다.

    [클러스터된 새 서버 인스턴스] 페이지가 열립니다.

  5. [이름] 필드에 인스턴스의 이름을 입력합니다.

    이름은 다음 요구 사항을 충족해야 합니다.

    • 이름에는 ASCII 문자만 포함될 수 있습니다.

    • 이름은 문자, 숫자 또는 밑줄로 시작해야 합니다.

    • 이름에는 다음 문자만 포함될 수 있습니다.

      • 소문자

      • 대문자

      • 숫자

      • 하이픈

      • 마침표

      • 밑줄

    • 이름은 도메인에서 고유해야 하며 다른 GlassFish Server 인스턴스, 클러스터, 명명된 구성 또는 노드의 이름이어서는 안됩니다.

    • 이름은 domain, server 또는 GlassFish Server에서 예약된 다른 키워드여서는 안됩니다.

  6. [노드] 드롭다운 목록에서 인스턴스가 상주할 노드를 선택합니다.

  7. [확인]을 누릅니다.

관련 항목

  • 클러스터된 GlassFish Server 인스턴스의 포트 번호를 수정하는 방법
