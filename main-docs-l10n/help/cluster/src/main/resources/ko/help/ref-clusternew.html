새 클러스터

[새 클러스터] 페이지에서는 클러스터를 생성할 수 있습니다.

[새 클러스터] 페이지에는 다음 옵션이 포함되어 있습니다.

클러스터 이름

클러스터의 이름입니다.

이름은 다음 요구 사항을 충족해야 합니다.

  • 이름에는 ASCII 문자만 포함될 수 있습니다.

  • 이름은 문자, 숫자 또는 밑줄로 시작해야 합니다.

  • 이름에는 다음 문자만 포함될 수 있습니다.

    • 소문자

    • 대문자

    • 숫자

    • 하이픈

    • 마침표

    • 밑줄

  • 이름은 도메인에서 고유해야 하며 다른 클러스터, 명명된 구성, GlassFish Server 인스턴스 또는 노드의 이름이어서는 안됩니다.

  • 이름은 domain, server 또는 GlassFish Server에서 예약된 다른 키워드여서는 안됩니다.

구성

클러스터에서 사용할 수 있는 기존에 명명된 구성의 드롭다운 목록입니다. 클러스터에는 클러스터에 추가된 모든 인스턴스의 구성을 정의하는 명명된 구성이 필요합니다. 목록에서 하나의 구성을 선택해야 합니다. 클러스터는 클러스터를 생성할 때 선택한 구성을 사용합니다.

선택한 구성의 복사본 만들기

이 옵션을 선택한 경우 클러스터를 생성할 때 선택한 구성이 복사됩니다. 구성 복사본에는 cluster-name-config라는 이름이 지정됩니다. 여기서 cluster-name은 [클러스터 이름] 필드에 입력한 클러스터의 이름입니다.

구성 드롭다운 목록에서 default-config를 선택한 경우 이 옵션을 선택해야 합니다. default-config 구성은 복사만 할 수 있고 참조할 수는 없습니다.

선택한 구성 참조

이 옵션을 선택한 경우 클러스터가 지정된 기존 명명된 구성을 사용합니다.

구성 드롭다운 목록에서 default-config를 선택한 경우 이 옵션을 선택하면 안됩니다. default-config 구성은 참조할 수는 없고 복사만 할 수 있습니다.

메시지 대기열 클러스터 구성 유형

이 옵션을 사용하여 새 GlassFish Server 클러스터에 사용할 Message Queue 중개자 클러스터의 유형을 지정할 수 있습니다. 기본적으로 GlassFish Server에서는 기본 Message Queue 중개자 클러스터를 GlassFish Server 클러스터의 각 인스턴스에 포함된 중개자인 마스터 중개자와 함께 사용합니다.

이 옵션을 [사용자 정의]로 설정할 경우 GlassFish Server 클러스터에 사용할 Message Queue 중개자 클러스터를 구성할 수 있는 몇 개의 추가 옵션이 표시됩니다. 사용자 정의 Message Queue 중개자 클러스터에 대한 옵션을 참조하십시오.

생성할 서버 인스턴스

클러스터를 생성할 때 생성할 GlassFish Server 인스턴스 목록입니다. 각 인스턴스에 대해 제공되는 정보는 다음과 같습니다.

인스턴스 이름

인스턴스의 이름입니다.

가중치

인스턴스의 로드 균형 조정 가중치를 나타내는 정수입니다.

로드 균형 조정 가중치는 인스턴스가 처리해야 하는 클러스터에 대한 모든 요청의 비율을 결정합니다.

예를 들어 두 개의 인스턴스로 구성된 클러스터에서 한 인스턴스가 네 개의 요청 중에서 하나를 처리하고 다른 인스턴스가 네 개의 요청 중에서 세 개를 처리해야 할 수 있습니다. 이 경우 네 개의 요청 중에서 하나를 처리할 인스턴스의 가중치를 1로 설정하고 다른 인스턴스의 가중치를 3으로 설정합니다. 백분율을 사용하려는 경우 인스턴스의 가중치를 각각 25와 75로 설정합니다.

기본 가중치는 100입니다.

노드

인스턴스가 있을 수 있는 기존 노드의 드롭다운 목록입니다. 목록에서 하나의 노드를 선택해야 합니다. 클러스터를 생성할 때 인스턴스가 선택한 노드에 있게 됩니다.

생성할 서버 인스턴스 테이블에는 다음 옵션도 포함되어 있습니다.

새로 만들기

인스턴스를 생성하는 단추입니다. 이 단추를 누르면 [생성할 서버 인스턴스] 테이블에 행이 추가됩니다.

삭제

인스턴스를 삭제하는 단추입니다.

사용자 정의 Message Queue 중개자 클러스터에 대한 옵션

[메시지 대기열 클러스터 구성 유형]을 [사용자 정의]로 설정할 경우 GlassFish Server 클러스터에 사용할 Message Queue 중개자 클러스터를 구성할 수 있는 다음 옵션이 표시됩니다.

JMS 서비스 유형

중개자 클러스터에 사용할 중개자(GlassFish Server의 JMS 호스트)의 유형입니다. 선택한 유형에 따라 사용 가능한 중개자 클러스터의 유형이 결정됩니다. 중개자 유형은 다음과 같습니다.

Embedded

중개자 구성 및 수명 주기는 GlassFish Server에서 관리합니다. Each GlassFish 클러스터된 인스턴스는 인스턴스와 동일한 JVM에서 실행되는 중개자에서 제공합니다.

[JMS 서비스 유형] 옵션이 [Embedded]로 설정된 경우 기본 Message Queue 중개자 클러스터만 지원됩니다. 기본 클러스터를 구성하는 옵션이 표시됩니다.

로컬

중개자 구성 및 수명 주기는 GlassFish Server에서 관리합니다. Each GlassFish 클러스터된 인스턴스는 인스턴스와 동일한 호스트에 있는 별도의 JVM에서 실행되는 중개자에서 제공합니다.

[JMS 서비스 유형] 옵션이 [로컬]로 설정된 경우 기본 및 Enhanced(HA) Message Queue 중개자 클러스터가 모두 지원됩니다. 클러스터 유형을 선택하고 클러스터를 구성하는 옵션이 표시됩니다.

원격

중개자/중개자 클러스터 구성 및 수명 주기는 Message Queue 관리 툴을 사용하여 관리됩니다.

[JMS 서비스 유형] 옵션이 [원격]으로 설정된 경우 다른 옵션이 표시되지 않습니다.

JMS 클러스터 유형

GlassFish Server 클러스터에 사용할 Message Queue 중개자 클러스터의 유형입니다.

[JMS 클러스터 유형] 옵션이 [기본]으로 설정된 경우 다음 옵션이 표시됩니다.

JMS 구성 저장소 유형

기본 클러스터 구성 데이터에 사용할 데이터 저장소의 유형입니다. 이 옵션이 [마스터 중개자]로 설정된 경우 클러스터 내의 한 중개자가 마스터 중개자로 지정되고 마스터 중개자에 의해 구성 데이터가 저장됩니다. 이 옵션이 [공유 DB]로 설정된 경우 모든 중개자에서 액세스할 수 있는 JDBC 데이터 저장소에 구성 데이터가 저장됩니다. 이 경우 데이터베이스 관련 옵션이 표시됩니다.

JMS 메시지 저장소 유형

각 중개자의 메시지 데이터를 저장하는 데 사용되는 데이터 저장소의 유형입니다. 이 옵션이 [파일]로 설정된 경우 각 중개자의 메시지 데이터가 파일 기반 데이터 저장소에 저장됩니다. 이 옵션이 [JDBC]로 설정된 경우 각 중개자의 메시지 데이터가 JDBC 데이터 저장소에 저장됩니다.

[JMS 클러스터 유형] 옵션이 [Enhanced(HA)]로 설정된 경우 데이터베이스 관련 옵션이 표시됩니다.

데이터베이스 공급업체
데이터베이스 URL
데이터베이스 사용자

다음 경우에 사용할 JDBC 데이터베이스의 데이터베이스 공급업체, 액세스 URL 및 JDBC 데이터베이스의 사용자입니다.

  • 기본 클러스터의 구성 저장소 유형이 공유 DB로 설정된 경우

  • 기본 클러스터의 메시지 저장소 유형이 JDBC로 설정된 경우

  • 클러스터 유형이 Enhanced(HA)로 설정된 경우

지원되는 공급업체 및 각 공급업체의 액세스 URL 형식에 대한 자세한 내용은 Oracle GlassFish Server Message Queue Administration Guide의 JDBC-Based Persistence를 참조하십시오.

데이터베이스 인증
데이터베이스 비밀번호
비밀번호 별칭

데이터베이스 사용자에 지정된 JDBC 데이터베이스 사용자의 비밀번호 정보입니다.

[데이터베이스 인증]이 [비밀번호]로 설정된 경우 [데이터베이스 비밀번호] 옵션이 표시됩니다. [데이터베이스 인증]이 [비밀번호 별칭]으로 설정된 경우 [비밀번호 별칭] 옵션이 표시됩니다.

JMS 클러스터 속성

중개자를 구성하는 하나 이상의 Message Queue 중개자 속성 목록입니다. 이 목록은 콜론(:)으로 구분되며 다음과 같은 형식으로 되어 있습니다.

prop1Name=prop1Value:prop2Name=prop2Value:...

중개자 속성 이름에 점이 포함된 경우, 점 앞에 두 개의 백슬래시(\\)를 추가하십시오. 예를 들어 imq.system.max_count 속성을 포함시키려면 목록에 imq\\.system\\.max_count를 지정하십시오.
