Per avviare le istanze di GlassFish Server in cluster

Prima di iniziare

Assicurarsi che siano soddisfatti i seguenti prerequisiti:

  • tutti i nodi in cui risiedono le istanze sono abilitati per la comunicazione remota;

  • l'utente del server di amministrazione del dominio (DAS) può utilizzare il protocollo remoto DCOM (Distributed Component Object Model) o SSH (Secure Shell) per collegarsi all'host per qualsiasi nodo in cui risiedono le istanze.

Se uno qualsiasi di questi prerequisiti non è soddisfatto, avviare singolarmente ogni istanza dalla riga di comando utilizzando il comando secondario start-local-instance(1).

  1. Espandere il nodo Cluster nell'albero di navigazione.

  2. Selezionare il nodo per il cluster sotto il nodo Cluster espanso.

    Viene visualizzata la pagina Informazioni generali per il cluster.

  3. Nella pagina Informazioni generali, fare clic sulla scheda Istanze.

    Viene visualizzata la pagina Istanze server in cluster per il cluster.

  4. Per ogni istanza da avviare, selezionare la casella di controllo corrispondente nella tabella Istanze server.

  5. Fare clic su Avvia.

    Viene richiesto di confermare l'avvio delle istanze.

  6. Confermare che si desidera avviare le istanze.

    Lo stato di ogni istanza viene aggiornato nella tabella Istanze server quando l'istanza viene avviata.
