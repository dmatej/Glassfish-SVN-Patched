修改集群 GlassFish Server 实例的端口号

集群 GlassFish Server 实例的端口号最初是在实例从其父集群继承的配置中设置的。例如，如果两个集群实例驻留在同一主机上，则必须修改这些端口号。在这种情况下，必须修改一个实例的端口号，因为这两个实例共享同一配置。

实例的端口号存储为 Java 系统属性。启动 GlassFish Server 时，这些属性会通过 Java 应用程序启动器的 -D 选项传递给 Java 应用程序启动器。

  1. 在导航树中，展开“集群”节点。

  2. 在展开的“集群”节点下，选择某个集群的节点。

    此时将打开集群的“一般信息”页。

  3. 在“一般信息”页中，单击“实例”选项卡。

    此时将打开集群的“集群服务器实例”页。

  4. 选择要修改端口号的实例。

    此时将打开实例的“一般信息”页。

  5. 在“一般信息”页中，单击“属性”选项卡。

    此时将打开“实例系统属性”页。

    此页会显示实例的以下系统属性的当前值：

    • 实例的配置中预定义的端口号

    • 为实例定义的定制属性 (如果有)

  6. 对于要修改的每个现有属性，在该属性的“覆盖值”字段中键入新的值。

  7. 对于其他属性，可根据需要设置相应的值。

    1. 在“其他属性”表中，单击“添加属性”。

      会向“其他属性”表中添加一行。

    2. 在“实例变量名称”字段中，键入属性名称。

    3. 在“覆盖值”字段中，键入值。

  8. 单击“保存”。
