完成更新 SSH 节点

此任务是 编辑节点 的 SSH 节点的延续。

  1. 要强制在域管理服务器 (DAS) 的配置中更新节点 (即使节点的参数验证失败)，请启用“强制”选项。

  2. 在“SSH 端口”字段中，键入该端口，以用于与该节点主机的 SSH 连接。

    如果将节点类型从“配置”更改为“SSH”并将此字段留空，“SSH 端口”选项将设置为 22。

  3. 在“SSH 用户名”字段中，键入该节点主机上要运行通过 SSH 连接到主机的进程的用户名称。

    如果将节点类型从“配置”更改为“SSH”并将此字段留空，“SSH 用户名”选项将设置为运行 DAS 进程的用户。

  4. 指定 SSH 用户登录到该节点主机时如何对其进行验证。

    • 要指定通过 SSH 私有密钥文件进行验证，请执行以下步骤：

      1. 从“SSH 用户验证”下拉列表中，选择“密钥文件”。

      2. 如果需要 GlassFish Server 在节点主机上设置 SSH 密钥，请启用“设置 SSH”选项。

        GlassFish Server 通过从 DAS 主机复制现有密钥文件，或通过在复制文件之前生成这些文件 (如果需要) 来设置 SSH 密钥。

        “生成密钥”字段和“公共密钥文件”字段将添加到窗口中。

      3. 如果需要 GlassFish Server 在将 SSH 密钥文件复制到节点主机之前生成这些文件，请启用“生成密钥”选项。

        如果 SSH 密钥文件存在，则 GlassFish Server 不会生成这些文件，即使选中“已启用”选项也是如此。

        注:

        如果 SSH 密钥文件不存在，则必须选中“已启用”选项。否则，在尝试保存更改时会出现错误。

      4. 在“SSH 用户口令”字段中，键入 SSH 用户登录到该节点主机时要使用的口令。

      5. 在“公共密钥文件”字段中，为“SSH 用户名”选项指定的用户键入 SSH 公共密钥文件的绝对路径。

      6. 在“密钥文件”字段中，为“SSH 用户名”选项指定的用户键入 SSH 私有密钥文件的绝对路径。

        注:

        GlassFish Server 还支持通过口令文件中的 AS_ADMIN_SSHPASSWORD 条目进行口令验证。口令文件在 asadmin(1) 实用程序的 --paswordfile 选项中指定。

        如果将节点类型更改为 SSH 并将此字段留空，则会将“密钥文件”选项设置为用户的 .ssh 目录中的密钥文件。如果找到多个密钥文件，GlassFish Server 遵循以下先后顺序：

        id_rsa
id_dsa
identity

    • 要指定通过存储在 DAS 配置中的口令进行验证，请执行以下步骤：

      注意:

      此口令以明文形式存储在 DAS 配置中。为了增强安全性，请使用密钥文件或口令别名。

      1. 从“SSH 用户验证”下拉列表中，选择“口令”。

      2. 在“SSH 用户口令”字段中，键入 SSH 用户登录到该节点主机时要使用的口令。

    • 要指定通过口令别名进行验证，请执行以下步骤：

      1. 从“SSH 用户验证”下拉列表中，选择“口令别名”。

      2. 从“口令别名”下拉列表中，选择一个现有别名。

  5. 单击“保存”。
