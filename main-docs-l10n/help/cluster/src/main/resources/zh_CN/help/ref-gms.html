编辑组管理服务

使用“组管理服务”页可更改 GMS 设置。

组管理服务 (GMS) 是一种进程内服务，可提供集群监视、集群成员和组通信服务。GMS 构建于可配置的组成员搜索和健康状况监视协议堆栈之上。GMS 执行以下功能：

  • 在集群中添加或删除一个或多个成员实例，或者怀疑或确认一个或多个成员实例已失败时，通知 GlassFish Server 实例中的已注册模块。

  • 提供在集群成员之间发送和接收消息的功能。

“组管理服务”页包含以下选项。

加载默认值

用于将具有默认值的设置还原为其默认值的按钮。没有默认值的设置不发生更改。

配置名称

此页上的设置应用到的配置的名称。此字段为只读。

最大未命中脉动数

在怀疑发生服务器实例故障前某个成员出现的未命中脉动次数。默认值为 3。

脉动频率

某个成员出现未命中脉动之间的时间段。默认值为 2000 毫秒。

组搜索超时

GMS 等待搜索其他集群成员的时间。默认值为 5000 毫秒。

故障确认等待时间

确认应将怀疑出现故障的成员报告为失败前等待的时间长度。默认值为 1500 毫秒。

其他属性

“组管理服务”的其他属性表。

GlassFish Server 不定义任何其他 GMS 属性。

对于每个属性，都将显示以下信息：

名称

属性名称。

值

为所选的实例设置的属性的值。

说明

提供有关属性的详细信息的文本说明。

“其他属性”表还包含以下选项。

添加属性

用于添加属性的按钮。单击此按钮可向“其他属性”表中添加一行。

删除属性

用于删除一个或多个所选属性的按钮。删除的任何属性会恢复其默认值，或者如果没有设置默认值，则未定义。
