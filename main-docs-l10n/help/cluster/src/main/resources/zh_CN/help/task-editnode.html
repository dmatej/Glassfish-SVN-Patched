编辑节点

编辑节点会更新 GlassFish Server 节点的配置数据。通过编辑节点，还可以将节点类型从“配置”更改为“DCOM”或“SSH”，以便为节点启用远程通信。

如果将节点类型从“配置”更改为“DCOM”，则在以下任意字段留空时将应用默认值：

  • Windows 域

  • Windows 用户

如果将节点类型从“配置”更改为“SSH”，则在以下任意字段留空时将应用默认值：

  • SSH 端口

  • SSH 用户名

  • 密钥文件

  1. 在导航树中，展开“节点”节点。

  2. 从节点列表中，选择要编辑的节点。

    此时将打开“编辑节点”页。

  3. 从“类型”下拉列表中，选择已更新节点的类型。

    • 要允许节点支持通过分布式组件对象模型 (DCOM) 远程协议进行通信，请选择“DCOM”。

    • 要允许节点支持通过安全 Shell (SSH) 进行通信，请选择“SSH”。

    • 要从节点中删除对远程配置的支持，请选择“配置”。

      如果选择“配置”，则不会显示“SSH”配置或“DCOM”配置的选项。如果在节点的当前类型是“DCOM”或“SSH”时选择“配置”，则在保存更改时，将从节点的配置中删除 DCOM 配置数据或 SSH 配置数据。

  4. 在“节点主机”字段中，键入已更新节点要代表的主机的名称。

    该字段是否是必需字段取决于节点类型：

    • 如果节点类型为“配置”，可以省略主机名。可在创建驻留在节点上的实例时确定该主机名。

    • 如果节点类型为 DCOM 或 SSH，必须指定主机名。否则，会出现错误。

  5. 在“节点目录”字段中，键入要包含节点上创建的 GlassFish Server 实例的目录路径。

    如果键入一个相对路径，则该路径相对于as-install 目录，其中as-install 是主机上 GlassFish Server 软件的基安装目录。

  6. 在“安装目录”字段中，键入主机上 GlassFish Server 软件基安装目录的父目录完整路径，例如 /export/glassfish3。

  7. 按节点类型的要求完成更新节点。

    1. 如果节点的类型为“配置”，请单击“保存”。

    2. 如果节点的类型为 DCOM，请按照完成更新 DCOM 节点中的说明操作。

    3. 如果节点的类型为 SSH，请按照完成更新 SSH 节点中的说明操作。
