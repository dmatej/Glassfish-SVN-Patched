Para Testar se um Nó é Acessível

Para criar ou iniciar as instâncias do GlassFish Server em um nó que representa um host remoto, o DAS deve ser capaz de contatar o nó. Antes de tentar criar ou iniciar as instâncias em tal nó, teste se o DAS pode contatar o nó.

O tipo do nó que você está testando deve ser DCOM ou SSH. Você não pode testar um nó do tipo CONFIG porque um nó desse tipo não é ativado para comunicação remota.

Antes de Começar

Certifique-se de que o protocolo apropriado para o tipo do nó esteja configurado no host representado pelo nó. O nó está ativado para comunicação remota por meio deste protocolo.

  • Se o tipo do nó for DCOM, o protocolo DCOM (Distributed Component Object Model) deverá ser configurado.

  • Se o tipo do nó for SSH, o protocolo secure shell (SSH) deverá ser configurado.

    1. Na árvore de navegação, selecione o nó Nós.

      A página Nós será aberta. Esta página lista todos os nós do domínio atual.

    2. Na tabela de nós, siga o link Ping relativo ao nó que você estiver testando.

      Uma mensagem que indica se o DAS pode contatar o nó é adicionada à página Nós.
