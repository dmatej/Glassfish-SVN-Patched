Para Migrar Timers EJB

Se uma instância do GlassFish Server for interrompida, os timers da tecnologia Enterprise JavaBeans (EJB) instalados em tal instância também interrompem a execução. Se a instância foi interrompida de forma anormal ou inesperada, para permitir que estes timers funcionem enquanto a instância estiver interrompida, é necessário migrar os timers para uma instância do servidor em execução no cluster.

  1. Na árvore de navegação, expanda o nó Clusters.

  2. No nó Clusters da árvore de navegação, selecione o cluster.

    A página Informações Gerais do cluster será aberta.

  3. Na página Informações Gerais, clique em Migrar Timers EJB.

    A página Migrar Timers EJB será aberta.

    Observação:

    Se todas as instâncias do cluster estiverem em execução, o botão Migrar Timers EJB ficará desativado.

    Para ajudá-lo na migração de timers, a página Migrar Timers de EJB contém uma tabela de timers de EJB com as informações a seguir.

    Nome do Servidor

    Lista de instâncias de GlassFish Server no cluster.

    Contagem do Timer

    Número de timers persistências em cada instância.

  4. Na página Migrar Timers EJB, selecione a instância da qual e para a qual migrar os Timers.

    1. Na lista drop-down Origem, selecione a instância interrompida da qual migrar os timers.

    2. Na lista drop-down Destino, selecione a instância em execução para a qual migrar os timers.

    3. Clique em OK.

Se a instância de origem estiver em execução ou se a instância de destino não estiver em execução, o Console de Administração exibe uma mensagem de erro.
