Para Iniciar um Cluster

Iniciar um cluster inicial todas as instâncias de GlassFish Server no cluster que ainda não foram executadas.

Antes de Começar

Certifique-se de que os seguintes pré-requisitos foram atendidos:

  • Todos os nós nos quais se encontram as instâncias do cluster estão ativados para comunicação remota.

  • O usuário do servidor de administração de domínios (DAS) pode usar o protocolo remoto DCOM (Distributed Component Object Model) ou secure shell (SSH) para efetuar log-in no host para qualquer nó em que residam as instâncias no cluster.

Se algum desses pré-requisitos não for atendido, inicie o cluster, iniciando cada instância individualmente pela linha de comandos, usando o subcomando start-local-instance(1).

  1. Na árvore de navegação, selecione o nó Clusters.

    A página Clusters será aberta.

  2. Na página Clusters, marque a caixa de seleção ao lado do nome do cluster.

  3. Clique em Iniciar Cluster.

    Será solicitado que você confirme se deseja iniciar o cluster.

  4. Confirme se deseja iniciar o cluster.

    O cluster será iniciado. O status de cada instância do cluster será atualizado na página Clusters quando a instância for iniciada.
