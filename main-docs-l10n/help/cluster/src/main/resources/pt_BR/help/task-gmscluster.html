Para Configurar as Definições do Serviço de Gerenciamento de Grupos de um Cluster

O Serviço de Gerenciamento de Grupos (GMS) é um serviço em andamento que oferece monitoramento de cluster, associação de cluster e serviços de comunicação de grupos. O GMS realiza as seguintes funções:

  • Notifica os módulos registrados em uma instância do GlassFish Server quando uma ou mais instâncias do membro são adicionadas ou removidas de um cluster, ou quando tais instâncias são suspeitas de ter falhas ou realmente tenham falhas.

  • Fornece a capacidade de enviar e receber mensagens entre membros do cluster.

  1. Na árvore de navegação, expanda o nó Clusters.

  2. Selecione o cluster para o qual você está configurando as definições do Serviço de Gerenciamento de Grupos (GMS).

    A página Informações Gerais do cluster será aberta.

  3. Marque a caixa de seleção GMS Ativado para ativar o GMS do cluster.

    Por default, essa opção está selecionada. Para desativar o GMS do cluster, desmarque a caixa de seleção GMS Ativado.

    Se ativado, o Serviço de Gerenciamento de Grupos (GMS) é iniciado em cada instância do servidor do cluster e do Servidor de Administração de Domínios (DAS). O DAS participa de cada cluster no qual o GMS estiver ativado.

  4. No campo Porta Multicast, digite a porta que o Serviço de Gerenciamento de Grupos utiliza para fazer listening dos eventos dos grupos.

    Especifique um número de porta UDP padrão na faixa de 2048 a 49151. O default é um valor gerado automaticamente dentro dessa faixa.

  5. No campo Endereço Multicast, digite o endereço que o Serviço de Gerenciamento de Grupos utiliza para fazer listening dos eventos dos grupos.

    Especifique um endereço IP multicast de classe D. Os endereços IP de classe D estão na faixa de 224.0.0.0 a 239.255.255.255. O endereço 224.0.0.0 está reservado e não deve ser utilizado. O default é 228.9.XX. YY, em que XX e YY são valores standalone gerados automaticamente entre 0 e 255.

  6. No campo Endereço da Interface de Bind, digite o endereço da interface de rede do Serviço de Gerenciamento de Grupos se houver mais de uma interface de rede.

    Esta definição permite configurar qual interface de rede será utilizada no GMS se houver mais de uma em qualquer uma das máquinas que hospedam as instâncias do servidor no cluster. Este valor deve ser um endereço IP de interface de rede local. Por default, são utilizados todos os endereços de interface de rede.

    Para especificar um endereço que seja válido para todas as instâncias do GlassFish Server do cluster, utilize uma propriedade do sistema para definir o endereço de forma individual de cada instância.

    Por exemplo, crie a propriedade de sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name no DAS. Em seguida, defina essa opção Endereço de Interface de Bind como ${GMS-BIND-INTERFACE-ADDRESS-cluster-name} para especificar a propriedade de sistema. Por fim, para cada instância no cluster, defina a propriedade do sistema GMS-BIND-INTERFACE-ADDRESS-cluster-name para o endereço da interface de rede solicitada na máquina da instância.

  7. Para especificar as propriedades adicionais, selecione a guia Propriedades e a subguia Propriedades do Cluster.

    • Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que for exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    • Para modificar uma propriedade, edite o campo Valor da propriedade.

    • Para deletar uma propriedade, marque a caixa de seleção à esquerda do campo Nome da Propriedade a ser deletada. Clique no botão Deletar Propriedades.

  8. Clique em Salvar.
