Para Finalizar a Atualização de um Nó DCOM

Esta tarefa é a continuação de um nó DCOM de Para Editar um Nó.

  1. Para forçar a criação do nó na configuração do servidor de administração de domínios (DAS) mesmo se a validação dos parâmetros do nó falhar, ative a opção Forçar.

  2. Se você quiser que o GlassFish Server teste a conexão por meio do DCOM com o host do nó antes de atualizar o nó, execute estas etapas:

    1. Ative a opção Validar DCOM.

    2. No campo Diretório de Teste Remoto, digite o nome da pasta no host remoto na qual o GlassFish Server deverá criar um arquivo para testar a conexão.

  3. No campo Domínio do Windows, digite o nome do domínio Windows que contém o usuário especificado pela opção Nome do Usuário do Windows.

    Se você alterar o tipo do nó para DCOM e deixar este campo em branco, a opção Domínio do Windows será definida para o nome do host no qual a Console de Administração estiver em execução.

  4. No campo Nome do Usuário do Windows, digite o nome do usuário no host deste nó que deverá executar o processo de conexão com o host por meio do DCOM.

    Se você alterar o tipo do nó de CONFIG para DCOM e deixar este campo em branco, a opção Nome do Usuário do Windows será definida para o usuário que estiver executando o processo DAS.

  5. Especifique como o usuário do Windows será autenticado ao efetuar log-in no host deste nó.

    Observação:

    O GlassFish Server também suporta autenticação de senha por meio da entrada AS_ADMIN_WINDOWSPASSWORD no arquivo de senha. O arquivo de senhas é especificado na opção -paswordfiledo utilitário asadmin(1).

    1. Para especificar a autenticação através de uma senha armazenada na configuração do DAS, execute estas etapas:

      Cuidado:

      Esta senha é armazenada em forma de texto criptografado na configuração do DAS. Para aumentar a segurança, use um alias de senha.

      1. Na lista drop-down Autenticação do Usuário Windows, selecione Senha.

      2. No campo Senha do Usuário do Windows, digite a senha que o usuário do Windows usará ao efetuar log-in no host deste nó.

    2. Para especificar a autenticação através de um alias de senha, execute estas etapas:

      1. Na lista drop-down Autenticação do Usuário do Windows, selecione Alias de Senha.

      2. Na lista drop-down Alias de Senha, selecione o alias existente.

  6. Clique em Salvar.
