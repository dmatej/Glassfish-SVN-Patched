Instâncias Clusterizadas do Servidor

Utilize a página Instâncias Clusterizadas do Servidor para criar e gerenciar as instâncias do GlassFish Server no cluster atual.

O campo Nome do Cluster é um campo somente para leitura que exibe o nome do cluster atual.

A página Instâncias Clusterizadas do Servidor exibe uma lista de instâncias que são membros do cluster atual. Para cada instância, são fornecidas as seguintes informações:

Nome

Nome designado à instância quando a instância foi criada. Clicar no nome abre a página Informações Gerais para a instância.

Peso do Balanceador de Cargas

Um inteiro que representa o peso do balanceamento de carga da instância.

O peso do balanceamento de carga determina a proporção de todas as solicitações do cluster que a instância deve processar.

Por exemplo, em um cluster com duas instâncias, é possível processar uma das quatro solicitações em uma instância e processar três das quatro solicitações na outra instância. Nesta situação, defina como 1 o peso da instância que processa uma das quatro solicitações e como 3 o peso da outra instância. Se preferir utilizar porcentagem, defina os pesos das instâncias como 25 e 75, respectivamente.

O valor default é 100.

Configuração

A configuração à qual a instância faz referência. Ao clicar na configuração, a página Configurações da instância se abre.

Nó

O nome do nó no qual a instância reside. Clicar no nome abre a página Editar Nó para o nó.

Status

Indica se a instância está em execução, se requer reinicialização para que seja atualizada com as alterações na configuração ou se não está em execução. Este campo é somente para leitura.

A tabela Instâncias do Servidor também contém as seguintes opções.

Novo

Botão para adicionar uma instância ao cluster.

Deletar

Botão para remover uma ou mais instâncias selecionadas do cluster.

Cuidado:

Se estiver utilizando um cluster Java Message Service (JMS) com um broker mestre, não delete a instância que está associada ao broker mestre. Se esta instância precisar ser deletada, use o subcomando change-master-broker(1) para designar o broker mestre a outra instância.
Iniciar

Botão para iniciar uma ou mais instâncias selecionadas no cluster.

Interromper

Botão para interromper uma ou mais instâncias selecionadas no cluster.
