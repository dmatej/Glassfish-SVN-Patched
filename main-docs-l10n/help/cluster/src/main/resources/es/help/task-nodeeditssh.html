Terminar la Actualización de un Nodo SSH

Esta tarea es la continuación para un nodo SSH de Editar un nodo.

  1. Para forzar la actualización del nodo en la configuración del Servidor de Administración de Dominio (DAS) aunque falle la validación de los parámetros del nodo, active la opción Forzar.

  2. En el campo Puerto SSH, escriba el puerto que se utilizará para las conexiones SSH con el host de este nodo.

    Si cambia el tipo de nodo de CONFIG a SSH y deja este campo vacío, la opción Puerto SSH se define en 22.

  3. En el campo Nombre de Usuario SSH, escriba el nombre del usuario del host de este nodo que ejecutará el proceso para conectar al host mediante SSH.

    Si cambia el tipo de nodo de CONFIG a SSH y deja este campo en blanco, la opción Nombre de Usuario SSH se configura con el usuario que ejecuta el proceso DAS.

  4. Especifique cómo se autenticará el usuario de SSH al iniciar sesión en el host de este nodo.

    • Para especificar la autenticación a través de un archivo de claves SSH privadas, efectúe los pasos siguientes:

      1. Seleccione Archivo de Claves en la lista desplegable Autenticación de Usuario de SSH.

      2. Si necesita que GlassFish Server configure una clave SSH en el host del nodo, active la opción Configurar SSH.

        GlassFish Server configura la clave SSH copiando archivos de claves existentes desde el host de DAS o, en caso necesario, generando los archivos antes de copiarlos.

        Se agregan a la ventana el campo Generar Clave y el campo Archivo de Claves Públicas.

      3. Si necesita que GlassFish Server genere los archivos de claves SSH antes de copiar los archivos en el host del nodo, active la opción Generar Clave.

        Si existen archivos de claves SSH, GlassFish Server no generará los archivos aunque seleccione la opción Activada.

        Nota:

        Si no existen archivos de claves SSH, deberá seleccionar la opción Activada. En caso contrario, se producirá un error al intentar guardar los cambios.

      4. En el campo Contraseña de Usuario de SSH, escriba la contraseña que el usuario de SSH utilizará al conectarse al host de este nodo.

      5. En el campo Archivo de Claves Públicas, escriba la ruta de acceso absoluta al archivo de claves públicas de SSH para el usuario especificado en la opción Nombre de Usuario SSH.

      6. En el campo Archivo de Claves, escriba la ruta de acceso absoluta al archivo de claves SSH privadas para el usuario especificado en la opción Nombre de Usuario SSH.

        Nota:

        GlassFish Server también soporta la autenticación con contraseña a través de la entrada AS_ADMIN_SSHPASSWORD en el archivo de contraseñas. El archivo de contraseñas se especifica en la opción -paswordfile de la utilidad asadmin(1).

        Si cambia el tipo de nodo a SSH y deja este campo en blanco, la opción Archivo de Claves se configura con un archivo de claves situado en el directorio .ssh del usuario. Si se encuentran varios archivos de claves, GlassFish Server utiliza el siguiente orden de preferencia:

        id_rsa
id_dsa
identity

    • Para especificar la autenticación mediante una contraseña guardada en la configuración de DAS, efectúe los pasos siguientes:

      Atención:

      Esta contraseña se almacena en forma de texto no cifrado en la configuración de DAS. Para mayor seguridad, utilice un archivo de claves o un alias de contraseña.

      1. Seleccione Contraseña en la lista desplegable Autenticación de Usuario de SSH.

      2. En el campo Contraseña de Usuario de SSH, escriba la contraseña que el usuario de SSH utilizará al conectarse al host de este nodo.

    • Para especificar la autenticación mediante un alias de contraseña, efectúe los pasos siguientes:

      1. Seleccione Alias de Contraseña en la lista desplegable Autenticación de Usuario de SSH.

      2. En la lista desplegable Alias de Contraseña, seleccione un alias existente.

  5. Haga clic en Guardar.
