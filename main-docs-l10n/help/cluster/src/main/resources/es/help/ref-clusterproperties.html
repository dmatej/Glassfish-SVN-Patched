Propiedades del cluster

La página Propiedades del cluster muestra una lista de las propiedades que se definen para los clusters seleccionados. Estas propiedades agregan información de configuración opcional sobre el cluster.

El campo Nombre de cluster es un campo de sólo lectura que muestra el nombre del cluster seleccionado.

Para cada propiedad se muestra la información siguiente:

Nombre

Nombre de la propiedad.

Valor

Valor de la propiedad que se ha definido para la instancia seleccionada.

Descripción

Descripción textual que ofrece más información sobre la propiedad.

La tabla Propiedades adicionales también contiene las opciones siguientes.

Agregar propiedad

Botón para agregar una propiedad. Al hacer clic en este botón se agrega una fila a la tabla Propiedades adicionales.

Suprimir propiedades

Botón para suprimir una o varias propiedades seleccionadas. Cualquier propiedad que se suprima vuelve a su valor por defecto o, si no se ha definido ningún valor por defecto, se queda sin definir.

GlassFish Server define las siguientes propiedades del cluster:

gms_discovery_uri_list

Ubicaciones de las instancias de GlassFish Server del cluster que se van a usar para detectar dicho cluster. Esta propiedad sólo es necesaria si el servicio de gestión de grupos (GMS) no utiliza multidifusión para enviar mensajes.

Los valores válidos de esta propiedad son los siguientes:

  • Una lista de identificadores de recursos uniformes (URI) separados por comas. Cada URI debe localizar una instancia de GlassFish Server o el DAS. Este formato es necesario si hay varias instancias de GlassFish Server en ejecución en el mismo host.

    El formato de cada URI en la lista es el siguiente:

    scheme://host-name-or -IP-address:port

    • scheme es el esquema de URI, que es tcp.

    • host-name-or -IP-address es el nombre de host o la dirección IP del host en el que se ejecuta la instancia.

    • port es el número del puerto en el que la instancia recibe mensajes de GMS. Debe definirse la propiedad del sistema GMS_LISTENER_PORT-clustername para la instancia. Para obtener información sobre cómo definir esta propiedad del sistema para una instancia, consulte la sección sobre la detección de un cluster cuando el transporte de multidifusión no está disponible en Oracle GlassFish Server High Availability Administration Guide.

  • Lista separada por comas de direcciones IP o nombres de host en los que se estén ejecutando el DAS o las instancias. Esta lista puede contener una mezcla de direcciones IP y nombres de host. Este formato puede utilizarse sólo si se ejecuta una instancia en cluster en cada host. El valor de la propiedad GMS_LISTENER_PORT debe ser único para cada cluster de un dominio.

  • La palabra clave generate. Este formato sólo se puede usar si se está ejecutando una instancia de cluster en cada host y si el DAS se está ejecutando en un host diferente. No puede haber varias instancias en el mismo host que pertenezcan al mismo cluster. El valor de la propiedad GMS_LISTENER_PORT debe ser único para cada cluster de un dominio.

GMS_LISTENER_PORT

Número del puerto en el que el cluster recibe mensajes de GMS.

El valor por defecto es una referencia a la propiedad del sistema GMS_LISTENER_PORT-cluster-name. Por defecto, esta propiedad de sistema no está definida. En esta situación, GMS seleccionará un puerto libre de los que se incluyen en el rango definido por las propiedades GMS_TCPSTARTPORT y GMS_TCPENDPORT. Por defecto, este rango es 9090-9200. En la mayoría de los casos, este comportamiento por defecto debe ser suficiente.

Sin embargo, si GMS no utiliza la multidifusión para difundir mensajes, esta propiedad debe indicar un número de puerto válido para todas las instancias de GlassFish Server del cluster. Para utilizar el valor por defecto para cubrir este requisito, utilice una propiedad del sistema para definir el número de puerto para cada una de las instancias.

Por ejemplo, utilice la página Propiedades del sistema de cluster para crear la propiedad del sistema GMS_LISTENER_PORT-cluster-name para el DAS. A continuación, para cada instancia del cluster, defina la propiedad del sistema GMS_LISTENER_PORT-cluster-name en el número de puerto en el que la instancia recibe mensajes de GMS. El valor por defecto de la propiedad GMS_LISTENER_PORT para el cluster hace referencia a esta propiedad del sistema.

GMS_LOOPBACK

Especifica si una instancia puede recibir de ella misma mensajes de nivel de aplicación que difunda al cluster.

Los valores posibles son los siguientes:

false

La instancia no puede recibir mensajes de ella misma (por defecto).

true

La instancia puede recibir mensajes de ella misma. Utilice esta configuración para probar una instancia cuando sea la única de un cluster.

GMS_MULTICAST_TIME_TO_LIVE

Número máximo de iteraciones o transmisiones que puede experimentar un mensaje de multidifusión para los siguientes tipos de eventos antes de descartar el mensaje:

  • Detección de grupos

  • Latidos de miembros

  • Cambios de miembros

Para coincidir con la configuración de la red en la que se despliegan el DAS y las instancias agrupadas en cluster, configure esta opción con el valor más bajo posible. Para determinar el valor más bajo posible para el sistema, utilice el subcomando validate-multicast(1).

Un valor de 0 garantiza que los mensajes de multidifusión nunca dejarán el host desde el cual se difunden.

Un valor de 1 puede impedir la difusión de mensajes entre hosts de la misma subred que están conectados mediante un conmutador o un enrutador.

El valor por defecto es 4, que garantiza que los mensajes se difundan correctamente a todos los miembros del cluster de las redes en las que se han conectado los hosts mediante conmutadores o enrutadores.

GMS_TCPENDPORT

Número de puerto más alto dentro del rango en el que GMS selecciona un puerto libre si la propiedad del sistema GMS_LISTENER_PORT-cluster-name no se ha definido. El valor por defecto es 9200.

GMS_TCPSTARTPORT

Número de puerto más bajo dentro del rango en el que GMS selecciona un puerto libre si la propiedad del sistema GMS_LISTENER_PORT-cluster-name no se ha definido. El valor por defecto es 9090.
