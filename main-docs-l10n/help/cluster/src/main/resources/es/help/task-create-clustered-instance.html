Para agregar una instancia de GlassFish Server a un Cluster

Las instancias de GlassFish Server forman la base del despliegue de aplicaciones. Una instancia en cluster es un miembro de un cluster. Cada instancia pertenece a un solo dominio y tiene su propia estructura de directorios, configuración y aplicaciones desplegadas. Cada instancia incluye también el contenedor web de la plataforma Java, Enterprise Edition (Java EE) y el contenedor de la tecnología Enterprise JavaBeans (tecnología EJB). Cada instancia nueva debe contener una referencia a un nodo que defina el host donde residirá la instancia.

Nota:

Para crear una instancia en un nodo de tipo CONFIG que representa un host remoto, debe utilizar el subcomando create-local-instance(1). No se puede utilizar Consola de administración para este fin.
Antes de Empezar

Asegúrese de que existan los elementos siguientes:

  • Cluster al que se agregará la instancia. Consulte Para crear un cluster para obtener instrucciones.

  • Nodo en el que residirá la instancia. Consulte Crear un nodo para obtener instrucciones.

  1. En el árbol de navegación, amplíe el nodo Clusters.

  2. En el nodo Clusters ampliado, seleccione el nodo para el cluster.

    Se abre la página Información General para el cluster.

  3. En la página Información General, haga clic en el separador Instancias.

    Se abrirá la página Instancias del Servidor en Cluster para el cluster.

  4. En la página Instancias de Servidor en Cluster, haga clic en Nuevo.

    Se abrirá página Nueva Instancia del Servidor en Cluster.

  5. En el campo Nombre, escriba un nombre para la instancia.

    El nombre debe cumplir los siguientes requisitos:

    • El nombre sólo puede contener caracteres ASCII.

    • El nombre debe empezar por una letra, un número o un carácter de subrayado.

    • El nombre sólo puede contener los siguientes caracteres:

      • Letras en minúscula

      • Letras en mayúscula

      • Números

      • Guión

      • Punto

      • Subrayado

    • El nombre debe ser único en el dominio y no debe ser el nombre de otro nodo, cluster, configuración con nombre ni instancia de GlassFish Server.

    • El nombre no debe ser domain, server ni otra palabra clave reservada por GlassFish Server.

  6. En la lista desplegable Nodo, seleccione el nodo donde residirá la instancia.

  7. Haga clic en Aceptar.
