組態 ORB

  1. 在樹形元件中，展開「組態」節點。

  2. 選取要組態的執行處理：

    • 若要組態特定的執行處理，請選取該執行處理的組態節點。例如，對於預設執行處理 server，請選取 server-config 節點。

    • 若要組態使用 default-config 副本之未來執行處理的預設設定，請選取 default-config 節點。

  3. 選取「ORB」節點。

    「ORB」頁面會隨即開啟。

  4. 從「繫線集區 ID」下拉式清單中選擇 ORB 使用的繫線集區。

    ORB 使用繫線集區回應來自使用 RMI-IIOP 進行通訊的企業 Bean 的遠端從屬端和其他從屬端的請求。ORB 的預設繫線集區為 thread-pool-1，它會使用適合 RMI/IIOP 請求的預設值。

    請參閱 Thread Pools 以取得更多資訊。

  5. 從「最大訊息片段大小」下拉式清單中，選取 IIOP 訊息的最大片段大小。

    大於該大小的訊息將被分段。預設值為 1024 位元組。

  6. 在「連線總數」欄位中，設定所有 IIOP 監聽器的最大內送連線數目。

    預設值為 1024。

  7. 選取「需要 IIOP 從屬端認證」核取方塊可指定需要 IIOP 從屬端認證。

    若啟用了 IIOP 從屬端認證，伺服器會拒絕未經認證的請求，並在傳送給從屬端的可互通物件參照 (IOR) 中插入需要認證的位元。此選項預設為停用。

  8. 在「其他特性」區域中，指定其他特性。

    GlassFish Server 未定義 ORB 的其他特性。

    • 若要增加特性，請按一下「增加特性」按鈕。在出現的空白列中，於「名稱」欄位中鍵入特性名稱，然後在「值」欄位中鍵入特性值。

    • 若要修改特性，請編輯特性的「值」欄位。

    • 若要刪除特性，請選取您要刪除之特性「名稱」欄位左側的核取方塊，然後按一下「刪除特性」按鈕。

  9. 按一下「儲存」。

    提示:

    若要回到預設設定，請按一下「載入預設值」按鈕，然後按一下「儲存」。

  10. 重新啟動 GlassFish Server。
