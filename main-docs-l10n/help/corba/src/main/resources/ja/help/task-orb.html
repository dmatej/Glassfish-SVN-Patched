ORBを構成するには

  1. ツリー・コンポーネントで、「構成」ノードを展開します。

  2. 構成するインスタンスを選択します:

    • 特定のインスタンスを構成するには、そのインスタンスの構成ノードを選択します。たとえば、デフォルト・インスタンスserverの場合は、server-configノードを選択します。

    • default-configのコピーを使用する将来のインスタンスのためにデフォルトの設定値を構成するには、default-configノードを選択します。

  3. 「ORB」ノードを選択します。

    「ORB」ページが開きます。

  4. 「スレッド・プールID」ドロップダウン・リストから、ORBが使用するスレッド・プールを選択します。

    ORBはスレッド・プールを使用して、エンタープライズBeanのリモート・クライアントおよびRMI-IIOPを使用して通信する他のクライアントからのリクエストに応答します。ORBのデフォルトのスレッド・プールはthread-pool-1です。このスレッド・プールは、RMI/IIOPリクエストに適切なデフォルト設定を使用します。

    詳細は、「スレッド・プール」を参照してください。

  5. 「最大メッセージ・フラグメント・サイズ」ドロップダウン・リストから、IIOPメッセージの最大フラグメント・サイズを選択します。

    このサイズより大きいメッセージは分割されます。デフォルト値は1024バイトです。

  6. 「合計接続数」フィールドに、すべてのIIOPリスナーの受信接続の最大数を設定します。

    デフォルト値は1024です。

  7. 「IIOPクライアント認証」の「必須」チェック・ボックスを選択して、IIOPクライアント認証が必要であることを指定します。

    IIOPクライアント認証が有効な場合、サーバーは認証されていないリクエストを拒否し、クライアントに送信されたInteroperable Object Reference (IOR)に認証必須のビットを挿入します。このオプションは、デフォルトでは無効になっています。

  8. 「追加プロパティ」領域に、追加プロパティを指定します。

    GlassFish ServerはORBの追加プロパティを定義しません。

    • プロパティを追加するには、「プロパティの追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティ名を入力し、「値」フィールドにプロパティ値を入力します。

    • プロパティを変更するには、そのプロパティの「値」フィールドを編集します。

    • プロパティを削除するには、削除するプロパティの「名前」フィールドの左側にあるチェック・ボックスを選択し、「プロパティの削除」ボタンをクリックします。

  9. 「保存」をクリックします。

    ヒント:

    デフォルト設定に返すには、「デフォルトのロード」ボタンをクリックし、「保存」をクリックします。

  10. GlassFish Serverを再起動します。
