ロガーの設定

「ロガーの設定」ページを使用し、選択したGlassFish Serverインスタンスまたはクラスタのロギングを構成します。

「ロガーの設定」ページには、次のオプションが含まれます。

デフォルトのロード

デフォルト値のある設定をデフォルト値に復元するボタン。デフォルト値のない設定は変更されません。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

システム・ログに書込み

有効な場合、ロギング出力がサーバー・ログに加え、syslog機能にも送信されます。この機能はSolarisおよびLinuxシステムのみで使用できます。このオプションは、デフォルトでは無効になっています。

コンソールにログを記録

有効な場合、ロギング出力がシステム・コンソールに送信されます。このオプションは、デフォルトでは無効になっています。

日付変更のローテーション

有効な場合、GlassFish Serverは、ログ・ファイルを毎日午前0時にローテーションします。このオプションは、デフォルトでは無効になっています。

複数行モード

有効化されていると、ログ・メッセージの本文が、メッセージ・ヘッダーの後の新しい行から開始されます。このオプションはデフォルトで有効になっています。

コンソールのロギング・フォーマット

コンソールへのロギングに使用されるフォーマット。使用可能な選択肢は、ULF (UniformLogFormatter)またはODL (Oracle Diagnostics Logging)です。デフォルトのフォーマットはULFです。

ログ・ファイルのロギング・フォーマット

サーバー・ログ・ファイルへのロギングに使用されるフォーマット。使用可能な選択肢はULFまたはODLです。デフォルトのフォーマットはODLです。

フォーマット除外フィールド

ログ・レコードで除外するフィールド。デフォルトで、すべてのフィールドが含まれています。

ファイル・ローテーション制限

ローテーションされるまでにサーバー・ログ・ファイルが到達するサイズ(バイト単位)。サーバー・ログが指定したサイズに達すると、GlassFish Serverはserver.logという名前の新しい空のファイルを作成し、古いファイルの名前をserver.log_dateに変更します(ここで、dateはファイルがローテーションされた日時です)。デフォルト値は2MBです。制限の最小値は500KBです。それより小さな値を指定すると、ファイルは500KBに達した時点でローテーションされます。ログ・ファイルのローテーションを無効にするには、この値を0に設定します。

ファイル・ローテーションの制限時間

サーバー・ログがローテーションされるまでの時間(分単位)。デフォルト値は0です。これは、「ファイル・ローテーション制限」フィールドで指定したサイズに達したときに、ファイルがローテーションされることを意味しています。1分以上の値を指定すると、「ファイル・ローテーション制限」フィールドで指定したサイズ制限よりも、時間による制限が優先されます。

フラッシュの頻度

キューからサーバー・ログに一度に書き込むメッセージの最大数。デフォルト値は1です。

最大履歴ファイル数

最も古いファイルを削除する前にGlassFish Serverが保持するログ・ファイルの最大数。この値をゼロ(0)に設定すると、GlassFish Serverは古いログ・ファイルを削除しません。

ログ・ファイル

サーバー・ログ・ファイルの別名または場所。デフォルトの場所は、domain-dir/logs/server.logです。

ログ・ハンドラ

カスタム・ログ・ハンドラの絶対クラス名。カスタム・ログ・ハンドラを使用すると、server.logまたはsyslog以外の宛先にログを送信することができます。カスタム・ハンドラを使用する場合は、クラスjava.util.logging.Handler(JSR 047準拠のAPI)を拡張する必要があります。ハンドラ・クラスをGlassFish Serverのクラスパスに置き、サーバーの起動時にハンドラがインストールされるようにします。
