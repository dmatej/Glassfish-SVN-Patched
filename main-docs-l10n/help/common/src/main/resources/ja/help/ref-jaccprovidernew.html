新規JACCプロバイダ

「新規JACCプロバイダ」ページを使用して、GlassFish Serverを実行しているアプリケーションのサードパーティ製認証モジュールで使用できる、JSR-115/196準拠の新しいJava Authorization Contract for Containers (JACC)プロバイダを作成します。JACCプロバイダは、ドメインのdomain.xmlファイルのsecurity-service要素内にjacc-provider要素として作成されます。

デフォルトのGlassFish Serverインストールには、defaultおよびsimpleの2つのJACCプロバイダが含まれます。ユーザーが作成する新規JACCプロバイダはすべて、これら2つのデフォルト・プロバイダに対する追加となります。デフォルトのGlassFish Server JACCプロバイダは、JACC仕様に準拠した簡単なファイルベース認証エンジンを実装します。

GlassFish Server JACCプロバイダはすべて、JAAS(Java Authentication and Authorization Service)APIを使用する必要があります。JAASによって、サービスが認証およびユーザーに対するアクセス制御を行うことが可能になります。JAASは、標準PAM(Pluggable Authentication Module)フレームワークのJavaテクノロジ・バージョンを実装しています。

JSR 196により、別のレイヤーでプラグインを開発できます。AuthConfigProviderやAuthConfigFactoryなどの、新規認証メカニズムを構成する方法を変更するプラグインを定義できます。また、ServerAuthModuleやClientAuthModuleなどの、新規認証メカニズムを定義することもできます。

「新規JACCプロバイダ」ページには、次のオプションが含まれます。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

名前

JACCプロバイダの名前。この名前を使用して、domain.xmlのjacc-provider要素を参照します。

ポリシーの構成

プロバイダ固有のjavax.security.jacc.PolicyConfigurationFactoryを実装するjavax.security.jacc.PolicyConfigurationFactory.providerの完全修飾クラス名。

ポリシー・プロバイダ

java.security.Policyを実装するjavax.security.jacc.policy.providerの完全修飾クラス名。

追加プロパティ

JACCプロバイダの追加プロパティ。JACCプロバイダで使用可能な追加プロパティについては、JACCプロバイダに固有のプロパティを参照してください。
