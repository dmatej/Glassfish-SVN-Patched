JACCプロバイダ

「JACCプロバイダ」ページを使用して、選択したGlassFish ServerインスタンスまたはクラスタにJava Authorization Contract for Containers (JACC)プロバイダを構成します。

JACCプロバイダは、GlassFish Serverで実行しているアプリケーションのサードパーティ製認証モジュールで使用できます。デフォルトのGlassFish Serverインストールには、defaultおよびsimpleの2つのJACCプロバイダが含まれます。これらのデフォルトのプロバイダは、JACC仕様に準拠した簡単なファイルベース認証エンジンを実装します。独自のカスタムJACCプロバイダを作成することも可能です。JACCプロバイダを作成するにはを参照してください。

JACCプロバイダの実装クラスの書込み方法については、http://jcp.org/en/jsr/detail?id=115のJACC仕様を参照してください。

JACCプロバイダごとに、次の情報が表示されます。

名前

JACCプロバイダの名前。この名前を使用して、domain.xmlのjacc-provider要素を参照します。

ポリシーの構成

プロバイダ固有のjavax.security.jacc.PolicyConfigurationFactoryを実装するjavax.security.jacc.PolicyConfigurationFactory.providerの完全修飾クラス名。

ポリシー・プロバイダ

java.security.Policyを実装するjavax.security.jacc.policy.providerの完全修飾クラス名。

デフォルトのJACCプロバイダは、次のプロパティもサポートします。

リポジトリ

JACCポリシー・ファイルが含まれるディレクトリ。default GlassFish Server JACCプロバイダの場合、デフォルト・ディレクトリは${com.sun.aas.instanceRoot}/generated/policyです。このプロパティは、デフォルトではsimple GlassFish Server JACCプロバイダに定義されません。

「JACCプロバイダ」の表には、次のオプションも表示されます。

新規

新規JACCプロバイダを作成するボタン。

削除

選択した1つまたは複数のJACCプロバイダを削除するボタン。
