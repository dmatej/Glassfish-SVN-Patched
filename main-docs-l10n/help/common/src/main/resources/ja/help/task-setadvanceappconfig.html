アプリケーション構成を設定するには

「アプリケーション構成」ページを使用して、デプロイされたアプリケーションに対する変更の検出と変更されたクラスの再ロードを確実に実行できるプロパティを設定します。

  1. ナビゲーション・ツリーで、「ドメイン」ノードを選択します。

    「ドメイン属性」ページが開きます。

  2. 「アプリケーション構成」タブを選択します。

    「アプリケーション構成」ページが開きます。

  3. 「再ロード」の「有効」チェック・ボックスを選択して、アプリケーションの動的再ロードを有効にします。

    動的再ロードが有効な場合、アプリケーションやモジュールのコードやデプロイメント記述子を変更したときは、そのアプリケーションやモジュールを再デプロイする必要はありません。必要な操作は、変更されたJavaServer Pages(JSP)またはクラス・ファイルをアプリケーションまたはモジュールのデプロイメント・ディレクトリにコピーすることです。サーバーは、デプロイメント・ディレクトリ内の変更を定期的に確認し、変更に伴いアプリケーションを自動的および動的に再デプロイします。この機能は、変更したコードをすぐにテストできるため、開発環境で役に立ちます。しかし、本番環境では、動的再ロードはパフォーマンスを低下させる可能性があります。また、再ロードが行われているときは、その転送時のセッションが無効になります。クライアントのセッションを再起動する必要があります。

    このオプションはデフォルトで有効になっています。

  4. 「再ロードのポーリング間隔」フィールドに、コード変更についてアプリケーションとモジュールを確認し、動的に再ロードする間隔(秒単位)を入力します。

    デフォルト値は2です。

  5. 「管理セッション・タイムアウト」フィールドに、アクティブでない状態が続き、管理コンソールがタイムアウトするまでの分数を入力します。

    デフォルト値は60です。

  6. 「自動デプロイ」の「有効」チェック・ボックスを選択して、アプリケーションの自動デプロイメントを有効にします。

    アプリケーションまたはモジュール・ファイル(JAR、WAR、RAR、またはEAR)を専用のディレクトリにコピーして自動デプロイメントを有効にすると、GlassFish Serverによってアプリケーションまたはモジュールが自動的にデプロイされます。このオプションはデフォルトで有効になっています。

  7. 「自動デプロイのポーリング間隔」フィールドに、コード変更についてアプリケーションとモジュールを確認し、動的に再ロードする間隔(秒単位)を入力します。

    デフォルト値は2です。ポーリング間隔を変更しても、アプリケーションやモジュールのデプロイにかかる時間には影響ありません。

  8. 「自動デプロイの再試行タイムアウト」フィールドに、エラーが発生するまでに部分的にコピーされたファイルのサイズが変更されずに残る秒数を入力します。

    デフォルト値は4です。

  9. 「自動デプロイ・ディレクトリ」フィールドに、アプリケーションの自動デプロイメントの対象としてモニターするディレクトリのフルパスを入力します。

    デフォルト値はdomain-dir/autodeployです。

  10. 「XML検査」ドロップダウン・リストから、標準およびGlassFish Serverのデプロイメント記述子で実行するXML検査のタイプを選択します。

    次のいずれかのタイプを指定できます。

    完全

    XML検査が失敗すると、デプロイメントも失敗します。この値はデフォルトです。

    解析中

    XML検査のエラーがレポートされますが、デプロイメントは実行されます。

    なし

    XML検査は実行されません。

  11. デプロイメント記述子ファイルを確認するには、「ベリファイア」の「有効」チェック・ボックスを選択します。

    ベリファイアは自動デプロイメントの前にファイルの構造とコンテンツを調べます。確認が失敗すると、デプロイメントは実行されません。大きなアプリケーションの確認は時間がかかる可能性があります。このオプションは、デフォルトでは無効になっています。

  12. JSPファイルをプリコンパイルするには、プリコンパイルを有効チェック・ボックスを選択します。

    このオプションが無効な場合、JSPファイルは初めてアクセスされた実行時にコンパイルされます。このオプションは、デフォルトでは無効になっています。

  13. 「追加プロパティ」セクションで、追加プロパティを指定します。

    GlassFish Serverはアプリケーションの追加プロパティを定義しません。

    • プロパティを追加するには、「プロパティの追加」ボタンをクリックします。表示される空白行で、「名前」フィールドにプロパティ名を入力し、「値」フィールドにプロパティ値を入力します。

    • プロパティを変更するには、そのプロパティの「値」フィールドを編集します。

    • プロパティを削除するには、削除するプロパティの「名前」フィールドの左側にあるチェック・ボックスを選択し、「プロパティの削除」ボタンをクリックします。

  14. 「保存」をクリックします。

    ヒント:

    デフォルト設定に返すには、「デフォルトのロード」ボタンをクリックし、「保存」をクリックします。

  15. サーバーを再起動します。
