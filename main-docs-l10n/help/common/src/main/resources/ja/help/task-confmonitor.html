モニター・サービスを構成するには

  1. 「モニター・サービス」ページを開くには、次のいずれかの操作を実行します。

    • 「モニター・データ」ノードを選択し、モニターを構成するサーバー・インスタンスの「モニターを構成」リンクをクリックします。クラスタのモニターを構成するには、クラスタの任意のインスタンスを選択します。

    • 「構成」ノードの下で、モニターを構成するサーバー・インスタンスまたはクラスタ構成を展開し、「モニター」ノードを展開します。

    「モニター・サービス」ページが開きます。

  2. 「モニター・サービス」の「有効」チェック・ボックスを選択してGlassFish Serverのモニターを有効にするか、チェック・ボックスを選択解除してモニターを無効にします。

    このオプションはデフォルトで有効になっています。

  3. 「モニターMBean」の「有効」チェック・ボックスを選択してモニターに必要なすべてのMBeanをデプロイするか、チェック・ボックスを選択解除してこれらのMBeanをアンデプロイします。

    このオプションはデフォルトで有効になっています。

    チェック・ボックスが選択されていない場合は、「モニター・サービス」チェック・ボックスが選択されている場合でも、管理コンソールでモニター・データを表示することはできません。この場合は、コマンド・ラインを使用してモニター・データを表示する必要があります。チェック・ボックスの選択を解除すると、GlassFish Serverのメモリー・フットプリントが減少します。

  4. DTraceの「有効」チェック・ボックスを選択し、SolarisのDTrace機能の使用を有効にします。

    SolarisのDTrace機能を使用するには、Oracle Enterprise Manager DTraceモニタリングも必要です。このオプションは、デフォルトでは無効になっています。

  5. 表で次のいずれかのアクションを実行します。

    • 「コンポーネント」列の1つまたは複数のコンポーネントに対して、「モニター・レベル」列のドロップダウン・リストから「LOW」または「HIGH」を選択し、コンポーネントのモニターを有効にします。

    • 1つまたは複数のコンポーネントの横にあるチェック・ボックスを選択した後、表の上部にある「レベル」ドロップダウン・リストから「LOW」または「HIGH」を選択し、「レベルの変更」ボタンをクリックして、選択したコンポーネントのモニター・レベルを変更します。

    現在のリリースでは、「LOW」オプションと「HIGH」オプションによるモニター・レベルに差はありません。1つまたは複数のコンポーネントのモニターを無効にするには、「OFF」を選択します。デフォルト値は「OFF」です。

    モニターを構成する前に、モニター対象のアプリケーション、コンポーネント、またはサービスをデプロイする必要があります。

  6. 「保存」をクリックします。
