モニター・サービス

「モニター・サービス」ページを使用して、個々のサーバー・コンポーネントまたはサービスのモニター・オプションを構成します。

モニターを構成する前に、モニター対象のアプリケーション、コンポーネント、またはサービスをデプロイする必要があります。

「モニター・サービス」ページには、次のオプションが含まれます。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

モニター・サービス

「モニター・サービス」の「有効」チェック・ボックスを選択すると、GlassFish Serverでモニターが有効になります。このオプションはデフォルトで有効になっています。

モニターMBean

「モニターMBean」の「有効」チェック・ボックスを選択すると、モニターに必要なすべてのMBeanがGlassFish Serverにデプロイされます。このオプションはデフォルトで有効になっています。

チェック・ボックスが選択されていない場合は、「モニター・サービス」チェック・ボックスが選択されている場合でも、管理コンソールでモニター・データを表示することはできません。この場合は、コマンド・ラインを使用してモニター・データを表示する必要があります。チェック・ボックスの選択を解除すると、GlassFish Serverのメモリー・フットプリントが減少します。

DTrace

Oracle Enterprise Manager DTrace Monitoringを使用している場合は、「DTrace」の「有効」チェック・ボックスを選択すると、SolarisのDTrace機能を使用できます。このオプションは、デフォルトでは無効になっています。

使用可能なコンポーネントごとに、次の情報が表示されます。

コンポーネント

GlassFish Serverコンポーネントの名前。

モニター・レベル

コンポーネントに設定されたモニターのレベル。コンポーネントごとに、「LOW」、「HIGH」、および「OFF」のレベルを使用できます。デフォルト値は「OFF」です。現在のリリースでは、「LOW」オプションと「HIGH」オプションによるモニター・レベルに差はありません。

「コンポーネント・レベルの設定」の表には、次のオプションも表示されます。

レベルの変更

選択した1つまたは複数のコンポーネントのモニター・レベルを変更するボタン。

レベル

選択したすべてのコンポーネントに適用するモニター・レベルを選択するドロップダウン・リスト。
