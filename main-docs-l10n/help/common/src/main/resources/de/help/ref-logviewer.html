Log-Viewer

Verwenden Sie die Seite "Log-Viewer", um die Logdateien für ausgewählte GlassFish Server-Instanzen und -Cluster anzuzeigen und die Loganzeige anzupassen.

Die Seite "Log-Viewer" ist in vier Bereiche unterteilt:

  • Suchkriterien

  • Erweiterte Suche

  • Suche ändern

  • Suchergebnisse

Der Bereich "Suchkriterien" enthält die folgenden Optionen.

Erweiterte Suche

Wenn Sie auf diesen Link klicken, wird ein Bereich geöffnet, in dem Sie weitere Feineinstellungen im Log-Viewer vornehmen können.

Zeitstempel

Im Optionsfeld "Neu" werden die neuen Nachrichten angezeigt. Über das Optionsfeld "Bestimmter Bereich" lassen sich die Felder "Von" und "Bis" öffnen. Auf diese Weise können Sie Nachrichten anzeigen, die einem bestimmten Zeitraum entstammen.

Für den Wert Zeit muss die Syntax folgende Form annehmen (SSS steht für Millisekunden):

hh:mm:ss.SSS

Beispiel:

17:10:00.000

Wenn der Wert im Feld "Von" zeitlich hinter dem Wert in "Bis" liegt, wird eine Fehlermeldung angezeigt.

Logebene

Die für den Filtervorgang ausgewählte Logebene. Standardmäßig werden alle im Serverlog enthaltenen Meldungen angezeigt, die der gewählten oder einer höheren Logebene zugeordnet sind. Aktivieren Sie das Kontrollkästchen "Keine weiteren als 'schwerwiegend' eingestuften Meldungen aufnehmen", um nur Meldungen der gewählten Ebene anzuzeigen.

Um sicherzustellen, dass die Meldungen, die Sie anzeigen möchten, im Serverlog angezeigt werden, legen Sie zunächst die geeigneten Logebenen auf der Seite "Modul-Logebenen" fest. Siehe So werden Logebenen konfiguriert

Wenn Sie sich für eine Filterung von Logmeldungen basierend auf der Logebene entschieden haben, werden nur Meldungen angezeigt, die den angegebenen Filterkriterien entsprechen. Diese Filterung wirkt sich jedoch nicht darauf aus, welche Meldungen im Serverlog aufgezeichnet werden.

Der Bereich "Erweiterte Suche" enthält die folgenden Optionen.

Logger

Logger Namespaces zum Filtern nach Modul. Klicken Sie bei gedrückter Umschalt- oder Strg+Taste, um mehrere Namespaces auszuwählen.

Die Auswahl eines Namespaces einer höheren Ebene führt zur Auswahl aller darunter angeordneten Namespaces. Beispiel: Wenn Sie javax.enterprise.system auswählen, werden auch die Logger aller Module unterhalb dieses Namespaces ausgewählt: javax.enterprise.system.core, javax.enterprise.system.tools.admin usw.

Benutzerdefinierter Logger

Logger, die zu einer bestimmten Anwendung gehören, einer pro Zeile.

Wenn die Anwendung über mehrere Module verfügt, können Sie einige oder alle dieser Module anzeigen. Angenommen, die Anwendung weist Logger mit den folgenden Namen auf:

com.mycompany.myapp.module1
com.mycompany.myapp.module2
com.mycompany.myapp.module3

Um Meldungen aller Module in der Anwendung anzuzeigen, geben Sie com.mycompany.myapp ein. Um nur Meldungen von module2 anzuzeigen, geben Sie com.mycompany.myapp.module2 ein.

Wenn Sie mindestens einen benutzerdefinierten Logger angeben, werden Meldungen von GlassFish Server-Modulen nur dann angezeigt, wenn Sie sie im Bereich "Logger" explizit aufführen.

Anzeige

Die Anzahl der gleichzeitig angezeigten Nachrichten. Als Standardanzeige für Meldungen wird 40 angezeigt. Darüber hinaus stehen 100, 250 und 1000 zur Auswahl.

Um Stacktraces anzuzeigen, deaktivieren Sie das Kontrollkästchen "Sehr lange Meldungen begrenzen". Standardmäßig werden in der Anzeige keine Stacktraces angezeigt. Um diese einzublenden, klicken Sie auf den Link (Details) für eine Meldung.

Mithilfe des Bereichs "Suche modifizieren" können Sie die Serverinstanz oder das Cluster angeben, sowie den Namen einer zugehörigen Logdatei, die angezeigt werden soll.

Instanz

Die Serverinstanz oder das Cluster, für die/das Datensätze angezeigt werden. Der Standardwert ist "server". Andere Optionen hängen von den Clustern und Instanzen ab, die für die Domain definiert und aktiviert werden können.

Logdatei

Der Name der Logdatei, die Sie anzeigen möchten. Der Standardname lautet server.log. Andere Optionen hängen von den Logeinstellungen für das gewählte Ziel ab.

Die Tabelle "Log-Viewer-Ergebnisse" zeigt die Meldungen des Serverlogs an. Verwenden Sie die Schaltflächen "Einträge vor" und "Einträge nach", die sich am oberen Rand der Tabelle befinden, um den Bereich der Datensätze auszuwählen, die Sie anzeigen möchten.

Für jede Nachricht enthält die Tabelle "Log-Viewer-Ergebnisse" die folgenden Informationen.

Datensatznummer

Die Nummer der Meldung im Serverlog.

Logebene

Die Logebene der Meldung.

Meldung

Der kurze Text der Logmeldung. Klicken Sie auf den Link (Details), um eine formatierte Version einer Meldung anzuzeigen.

Logger

Das Logger-Modul, das die Logmeldung generiert hat.

Zeitstempel

Datum und Uhrzeit der Erstellung der Logmeldung.

Name/Wertpaare

Der Name und die ID des Threads der Logmeldung.

Verwandte Aufgaben

  • So werden allgemeine Logging-Einstellungen konfiguriert

  • So werden Logebenen konfiguriert

  • So werden Logdateien für DAS angezeigt

Zugehörige asadmin-Befehle

  • set-log-level
