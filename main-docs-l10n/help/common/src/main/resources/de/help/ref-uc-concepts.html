Updatetool-Schlüsselkonzepte

Das Updatetool verwaltet die Installationen der GlassFish Server-Softwarekomponenten, die in einem einzelnen Dateisystemverzeichnis auf Ihrem System enthalten sind.

Die Administrationskonsole-Version von Updatetool bietet die folgenden Möglichkeiten:

  • Add-on-Komponenten installieren

  • Aktualisierungen für installierte Komponenten anzeigen

  • Installierte Komponenten anzeigen

Um installierte Komponenten aktualisieren oder entfernen zu können, müssen Sie die GlassFish Server-Domain anhalten und dann einen der folgenden Befehle verwenden:

  • as-install-parent/bin/updatetool, das das grafische Updatetoolutility startet

  • as-install-parent/bin/pkg, eine Befehlszeilenversion des Updatetools

Die Schlüsselkonzepte für das Updatetool werden in den folgenden Abschnitten beschrieben.

Komponente

Eine Komponente ist die installierbare Einheit, die von Updatetool verwaltet wird. Eine Komponente besteht aus Dateien, die als Einheit auf Ihrem System innerhalb eines Installationsimages installiert sind.

Installationsimage

Ein Installationsimage ist das Verzeichnis, das die Softwarekomponenten enthält, einschließlich des Verzeichniscontents. Das Installationsimage wird dem Updatetool als Ergebnis des Herunterladens und Installierens der GlassFish Server-Software hinzugefügt. Um das Hinzufügen und Aktualisieren von GlassFish Server-Softwarekomponenten zu ermöglichen, enthält jedes Installationsimage Informationen zur Installation von Komponenten.

Netzwerk-Repository

Ein Netzwerk-Repository enthält folgende Elemente:

  • Komponenten, die einem Installationsimage hinzugefügt werden können

  • Updates von vorhandenen Komponenten eines Installationsimages

Ein Installationsimage ist mit mindestens einem Netzwerk-Repository verbunden. Jedes Netzwerk-Repository wird durch einen Namen und eine entsprechende URL dargestellt. Ein Repository-Name ist eindeutig für jedes Installationsimage und steht sowohl für die Organisation oder das Projekt, die das Repository verwalten, als auch für einen klar definierten Contentbereich. Weitere Informationen zu den für GlassFish Server vorkonfigurierten Netzwerk-Repositorys finden Sie unter Preconfigured Repositories for GlassFish Server im Oracle GlassFish Server Administration Guide.

Add-on-Komponente

Eine Add-on-Komponente ist eine Komponente des Installationsimages, die noch nicht installiert ist.
