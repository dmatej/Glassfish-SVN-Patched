Sichere Administration

Verwenden Sie die Seite "Sichere Administration", um die sichere Administration zu aktivieren oder zu deaktivieren.

Wenn die sichere Administration aktiviert ist:

  • verwenden der DAS und die Instanzen in der Domain SSL-Zertifikate zur Verschlüsselung der Nachrichten, die sie untereinander versenden.

  • akzeptiert der DAS Administrationsnachrichten von Remote-Admin-Clients wie dem asadmin-Utility und IDEs.

Die Seite "Sichere Administration" enthält die folgenden Informationen.

Sichere Administration aktivieren

Schaltfläche zur Aktivierung der sicheren Administration. Diese Schaltfläche wird nur angezeigt, wenn die sichere Administration nicht aktiviert ist.

Hinweis:

Wenn das Kennwort eines Benutzers mit Administratorrechten in der Domain leer ist, verlaufen Versuche zur Aktivierung der sicheren Administration nicht erfolgreich.
Sichere Administration deaktivieren

Schaltfläche zur Deaktivierung der sicheren Administration. Diese Schaltfläche wird nur angezeigt, wenn die sichere Administration aktiviert ist.

Administrationsalias

Der Alias, der auf das SSL-/TLS-Zertifikat auf dem DAS verweist. Mit diesem Alias identifiziert sich der DAS bei den Instanzen. Der Standardwert ist s1as. Diese Option wird nur angezeigt, wenn die sichere Administration nicht aktiviert ist.

Instanzalias

Der Alias, der auf das SSL-/TLS-Zertifikat auf den Instanzen verweist. Mit diesem Alias identifizieren sich die Instanzen beim DAS. Der Standardwert ist glassfish-instance. Diese Option wird nur angezeigt, wenn die sichere Administration nicht aktiviert ist.

Verwandte Aufgaben

  • So aktivieren Sie die sichere Administration

  • So deaktivieren Sie die sichere Administration

Zugehörige asadmin-Befehle

  • enable-secure-admin(1)

  • disable-secure-admin(1)
