Editar Aplicação

Use a página Editar Aplicação para modificar uma aplicação existente.

A página Editar Aplicação contém algumas ou todas as seguintes opções, dependendo do tipo de aplicação.

Nome

Campo somente para leitura que exibe o nome da aplicação em edição.

O nome pode incluir um identificador de versão opcional, que segue o nome e é separado do nome por dois-pontos (:). Para obter mais informações sobre versões modulares ou da aplicação, consulte Module and Application Versions em Oracle GlassFish Server Application Deployment Guide.

Status

Se a caixa de seleção Ativado estiver marcada, a aplicação está ativada. Por default, essa opção está selecionada.

Raiz de Contexto

Para aplicações Web, especifica o caminho da aplicação. No URL da aplicação web, a raiz do contexto segue imediatamente o número da porta (http://host:porta/context-root/). A raiz do contexto deve começar com uma barra à direita, por exemplo, /hello.

Disponibilidade

Se a caixa de seleção Ativado estiver selecionada, a alta disponibilidade será ativada para sessões Web e para checkpoint e, potencialmente, a passivação do bean da sessão stateful (SFSB). Se esta opção estiver definida como falsa (default), todo os salvamentos de sessão Web e checkpoints SFSB serão desativados para a aplicação, a aplicação Web ou o módulo EJB especificado. Se esta opção estiver definida como verdadeira, a aplicação ou o módulo especificado será ativado para alta disponibilidade. Defina essa opção como verdadeira apenas se a alta disponibilidade estiver configurada e ativada em níveis mais altos, como os níveis do servidor e contêiner.

Esta opção aparece se clusters ou instâncias de servidores standalone, além da instância default do servidor (server), existirem.

Servidores Virtuais

Os servidores virtuais associados a esta aplicação.

A opção Servidores Virtuais aparece somente se a instância default do servidor, server, existir. Se os clusters ou outras instâncias de servidores standalone existirem, é possível selecionar os servidores virtuais depois da implantação. Vá à página Editar Aplicação, selecione a guia Alvo e selecione Gerenciar Servidores Virtuais para o alvo desejado.

Java Web Start

(Para alguns tipos de aplicações) Se a caixa de seleção Ativado estiver selecionada, a aplicação usará o software Java Web Start. O Java Web Start fornece uma maneira independente de browser de implantar aplicações Java a serem executadas em uma Máquina Virtual Java dedicada.

Descrição

Uma breve descrição da aplicação.

Local

O local da aplicação implantada. Este é um campo somente para leitura.

Ordem de Implantação

A ordem de implantação da aplicação.

Aplicações com um número mais baixo são carregadas primeiro na inicialização do servidor. Um aplicação com uma ordem de implantação de 102 é carregada antes de uma de 110. Se uma ordem de implantação não for especificada no tempo uma aplicação será implantada, a ordem de implantação default de 100 será designada. Se duas aplicações tiverem a mesma ordem de implantação, a aplicação que foi implantada primeiro será carregada primeiro. Especificar uma ordem de implantação é útil se a aplicação tiver dependências e deve ser carregada em uma determinada ordem.

Bibliotecas

Os arquivos JAR da biblioteca exigidos pela aplicação. Este é um campo somente para leitura.

Módulos e Componentes

Tabela que exibe os nomes dos módulos da aplicação e seus respectivos mecanismos. Em um ponto final de Web service, é possível selecionar Exibir Ponto Final para exibir a página Informações do Ponto Final de Web Service. Para um cliente da aplicação, você pode selecionar Acionar para exibir a página Acionar Cliente da Aplicação ou para Fazer Download de Stubs do Cliente para fazer download de stubs do cliente.
