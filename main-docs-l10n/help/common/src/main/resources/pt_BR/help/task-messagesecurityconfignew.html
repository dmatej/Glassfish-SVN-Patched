Para Criar uma Configuração de Segurança de Mensagem

Normalmente, você não precisa criar uma configuração de segurança de mensagem. O GlassFish Server inclui uma configuração de segurança de mensagens para a camada de autenticação SOAP. Os provedores desta configuração não são ativados por default, mas você pode ativá-los editando a configuração de segurança de mensagem SOAP. Você pode, contudo, criar uma configuração para a camada de autenticação HttpServlet, e especificar uma ou mais configurações de provedor para ela.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Expanda o nó Segurança.

  3. Selecione o nó Segurança de Mensagem.

    A página Configurações de Segurança de Mensagem será aberta.

  4. Na página Configurações de Segurança de Mensagem, clique em Novo.

    A página Configuração de Segurança de Nova Mensagem será aberta.

    Observação:

    É possível criar ou deletar os provedores de segurança de mensagem da configuração server-config default. Os únicos provedores de segurança de mensagem que são suportados no server-config default são SOAP e HttpServlet. Como estes dois provedores de mensagem já são criados por default quando o GlassFish Server é instalado, o botão Novo fica desativado na página Configurações de Segurança de Mensagem no server-config default.

  5. Na página Configuração de Segurança de Nova Mensagem, na lista drop-down Camada de Autenticação, selecione a camada de mensagens para a quar você está criando uma configuração.

    Como uma configuração SOAP está disponível por default, a única opção é HttpServlet.

  6. No campo ID do Provedor, digite o identificador exclusivo da configuração do provedor para essa configuração de segurança de mensagem.

  7. Marque a caixa de seleção Provedor Default para especificar que essa configuração de provedor é o provedor default.

  8. Na lista drop-down Tipo de Provedor, selecione o tipo deste provedor.

    As opções disponíveis são:

    client

    Especifica que o provedor é o provedor de autenticação do cliente.

    server

    Especifica que o provedor é o provedor de autenticação do servidor.

    client-server

    Especifica que o provedor é um provedor de autenticação do cliente e do servidor.

  9. No campo Nome da Classe, digite o nome da classe de Implementação Java do provedor.

    Provedores de autenticação do cliente devem implementar a interface com.sun.enterprise.security.jauth.ClientAuthModule. Provedores do servidor devem implementar a interface com.sun.enterprise.security.jauth.ServerAuthModule. Os provedores de cliente-servidor devem implementar as duas interfaces.

  10. Na seção Propriedades Adicionais, especifique as propriedades adicionais.

    Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que for exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    O GlassFish Server não define nenhuma propriedade adicional para configurações de segurança de mensagem.
