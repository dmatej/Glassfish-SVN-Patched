更改滚动日志文件的数量限制

当 GlassFish Server 滚动日志文件时，它会创建名为 server.log 的新的空文件并将旧文件重命名为 server.log_date，其中 date 是文件滚动的日期和时间。

默认情况下，GlassFish Server 将滚动日志文件的数量限制为 10。当到达此限制时，GlassFish Server 接下来滚动日志文件时会删除最旧的日志文件。

如果需要，您可以更改滚动日志文件的数量限制。您设置的任何新限制都会同时应用于访问日志记录和服务器日志的滚动日志文件。

  1. 将 com.sun.enterprise.server.logging.max_history_files 系统属性设置为要保留的滚动日志文件的最大数量。

    有关如何设置系统属性的详细信息，请参阅管理系统属性。

    com.sun.enterprise.server.logging.max_history_files 系统属性的行为如下：

    • 如果未设置此属性，则 GlassFish Server 保留最大值，即 10 个滚动日志文件。

    • 如果此属性设置为无效数字或为空，则 GlassFish Server 保留最大值，即 10 个滚动日志文件。

    • 如果此属性设置为 0，则 GlassFish Server 不保留滚动日志文件。
