新建生命周期模块

使用“新建生命周期模块”页可以定义新的生命周期模块。

“新建生命周期模块”页包含以下选项。

名称

生命周期模块名称。

类名

生命周期模块类文件的全限定名称。此类必须实现 com.sun.appserv.server.LifecycleListener 接口。

类路径

生命周期模块的类路径。类路径指定生命周期模块的位置。默认值为 domain-dir/applications。如果模块已经在此目录中 (即，在服务器类路径中)，则此字段可以留为空白。

加载顺序

启动时加载此生命周期模块的顺序。加载顺序值为较小整数的模块会较早加载。值的范围在 101 和操作系统的 MAXINT 之间。1 和 100 之间的值为保留值。

说明

生命周期模块说明。

状态

如果选中此选项，则将启用生命周期模块。默认情况下启用此选项。

加载失败时

如果选中此选项，则在生命周期模块无法加载时将关闭服务器。默认情况下禁用此选项。

目标

生命周期模块的集群和独立实例。使用“添加”和“全部添加”按钮将所需目标移动到“所选目标”列。使用“删除”和“全部删除”按钮将任何不需要的目标移动到“可用目标”列。

仅当在域中已创建集群或独立实例时才会显示此选项。
