授予对管理工具的访问权限

只有 asadmin 组中的用户才能访问管理控制台和 asadmin 命令行实用程序。要授予用户对这些管理工具的访问权限，请将用户添加到 admin-realm 的 asadmin 组中。

  1. 在“配置”节点下，展开您要为其配置管理访问权限的服务器实例或集群配置。

  2. 展开“安全性”节点。

  3. 展开“领域”节点。

  4. 选择 admin-realm 节点。

    将打开“编辑领域”页。

  5. 在“编辑领域”页上，单击“管理用户”按钮。

    此时将打开“管理员用户”页。

    初次完成安装之后，安装过程中输入的管理员用户名和口令将列在名为 admin-keyfile 的文件中。默认情况下，此用户属于 asadmin 组，该组可以授予修改 GlassFish Server 的权限。仅当您要为这些用户授予所选 GlassFish Server 实例或集群的管理员权限时，才应将其指定到该组。

    如果您将用户添加到 admin-realm 领域中，但将该用户指定到 asadmin 组之外的组中，则用户信息仍将写入名为 admin-keyfile 的文件，但该用户没有访问管理工具或文件领域中的应用程序的权限。

  6. 在“管理员用户”页上，单击“新建”以向 admin-realm 领域添加新用户。

    将打开“新建文件领域用户”页。

  7. 在“用户 ID”字段中，键入新用户名。

  8. 在“组列表”字段中，键入用户将属于的组名称。

    要授予用户修改 GlassFish Server 的权限，请将 asadmin 组包括在“组列表”中。

  9. 在“新口令”和“确认新口令”字段中，键入新用户的口令。

  10. 单击“确定”将此用户添加到 admin-realm 领域中，或者单击“取消”退出而不保存更改。
