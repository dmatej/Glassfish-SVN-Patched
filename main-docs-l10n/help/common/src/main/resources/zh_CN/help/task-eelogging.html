配置常规日志记录设置

  1. 在导航树中，展开“配置”节点。

  2. 在“配置”节点下，单击您要为其配置日志记录程序设置的服务器实例或集群配置。

    此时将打开“配置”页。

  3. 在“配置”页上，单击“日志记录程序设置”。

    此时将打开所选配置目标的“日志记录程序设置”页。

  4. 选中“启用写入系统日志”复选框，以便除了服务器日志以外，还将日志记录输出发送到 syslog 工具。

    此功能仅在 Solaris 和 Linux 系统上可用。默认情况下禁用此选项。

  5. 选中“启用记录到控制台”复选框可将日志记录输出发送到系统控制台。

  6. 选择“启用日期更改时滚动”复选框可在每天午夜滚动服务器日志文件。

  7. 选择“启用多行模式”复选框可设置日志消息格式以便于阅读。日志消息正文将缩进并在新行上开始。

  8. 从“控制台日志记录格式”下拉列表中，选择记录到控制台的格式。默认选项为 ODL (Oracle 诊断日志记录)。其他选项为 ULF (UniformLogFormatter)。

  9. 从“日志文件日志记录格式”下拉列表中，选择记录到服务器日志文件的格式。默认选项为 ODL。

  10. 在“文件滚动限制”字段中，键入在滚动之前，服务器日志文件必须达到的大小 (字节)。

    服务器日志达到指定大小时，GlassFish Server 将创建一个新的名为 server.log 的空文件，并将旧文件重命名为 server.log_date，其中 date 是文件滚动的日期和时间。默认值为 2 兆字节。该限制的最小值为 500 千字节；如果指定更低的值，则要在达到 500 千字节时该文件才滚动。要关闭日志文件滚动，请将该值设置为 0。

  11. 在“文件滚动时间限制”字段中，键入服务器日志必须滚动之前的分钟数。

    默认值为 0，表示文件达到“文件滚动限制”字段中指定的大小时进行滚动。

    注:

    如果指定一分钟或若干分钟，则时间限制将优先于“文件滚动限制”字段指定的大小限制。

  12. 在“刷新频率”字段中，键入一次从队列写入到服务器日志的最大消息数。

    默认值为 1。

  13. 在“最大历史记录文件数”字段中，键入 GlassFish Server 在删除最旧文件之前应保留的最大日志文件数。如果将此值设置为零 (0)，GlassFish Server 不会删除任何旧日志文件。

  14. 在“日志文件”字段中，键入服务器日志文件的替代名称或位置。

    默认位置为 ${com.sun.aas.instanceRoot}/logs/server.log。

  15. 在“日志处理程序”字段中，键入定制日志处理程序的绝对类名。

    使用定制日志处理程序可将日志发送到除了 server.log 或 syslog 以外的其他目标。定制处理程序必须扩展 java.util.logging.ConsoleHandler 类 (符合 JSR 047 的 API)。将处理程序类置于 GlassFish Server 类路径中，以便在服务器启动过程中安装该处理程序。

  16. 单击“保存”。

    注:

    可能需要重新启动服务器，具体取决于启用的日志记录配置。

    如果日志文件名称从默认设置改为其他值，可能也需要重新启动服务器。如果需要重新启动服务器，则将在管理控制台的右上角显示必须重新启动消息。
