Création d'un module de cycle de vie

  1. Dans l'arborescence de navigation, sélectionnez le noeud Modules de cycle de vie.

    La page Module de cycle de vie apparaît.

  2. Sur la page Modules de cycle de vie, cliquez sur Nouveau.

    La page Nouveau module de cycle de vie apparaît.

  3. Dans le champ Nom, saisissez le nom du module de cycle de vie.

  4. Dans le champ Nom de classe, saisissez le nom qualifié complet du fichier de classe du module de cycle de vie.

    La classe doit implémenter l'interface com.sun.appserv.server.LifecycleListener.

  5. Dans le champ Classpath, saisissez le classpath du module de cycle de vie.

    Le classpath indique l'emplacement du module de cycle de vie. La valeur par défaut est domain-dir/applications. Si le module est déjà dans ce répertoire (c'est-à-dire dans le classpath du serveur), ce champ peut être laissé vide.

  6. Dans le champ Ordre de chargement, saisissez une valeur indiquant l'ordre dans lequel ce module de cycle de vie doit être chargé au démarrage.

    Les modules aux valeurs entières les plus faibles dans l'ordre de chargement sont chargés en premier. Ce chiffre peut aller de 101 à la valeur MAXINT du système d'exploitation. Les valeurs de 1 à 100 sont réservées.

  7. Dans le champ Description, saisissez une description du module de cycle de vie.

  8. Cochez la case Statut activé pour activer le module de cycle de vie.

    Cette option est activée par défaut.

  9. Cochez la case Empêcher le démarrage de l'instance en cas d'échec du chargement pour indiquer que le serveur doit être arrêté en cas d'échec du chargement du module de cycle de vie.

    Cette option est désactivée par défaut.

  10. Pour ajouter des clusters et des instances de serveur cible sur lesquels activer le module de cycle de vie, sélectionnez-les dans la liste Cibles disponibles et utilisez le bouton Ajouter pour les déplacer vers la liste Cibles sélectionnées.

    L'option Cible apparaît si des clusters ou des instances de serveur autonomes existent en plus de l'instance de serveur par défaut (server). Par défaut, le module de cycle de vie est activé sur l'instance de serveur par défaut, nommée server. Utilisez le bouton Tout ajouter pour le/la déployer vers toutes les cibles. Utilisez les boutons Enlever et Tout enlever pour enlever des cibles de la liste Cibles sélectionnées.

  11. Cliquez sur OK.
