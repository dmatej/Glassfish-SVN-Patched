Configurazione applicazioni

Utilizzare la pagina Configurazione applicazioni per impostare proprietà che garantiscono l'individuazione delle modifiche apportate alle applicazioni distribuite e il ricaricamento delle classi modificate.

La pagina Configurazione applicazioni contiene le opzioni riportate di seguito.

Carica impostazioni predefinite

Pulsante per ripristinare i valori predefiniti delle impostazioni che dispongono di valori predefiniti. Le impostazioni che non dispongono di valori predefiniti non vengono modificate.

Ricarica

Se questa opzione è abilitata, le classi modificate vengono ricaricate in modo dinamico. Il server controlla periodicamente la presenza di eventuali modifiche nella directory di distribuzione e ridistribuisce l'applicazione, in modo automatico e dinamico, con le modifiche. Ciò risulta utile in un ambiente di sviluppo, in quanto consente di verificare rapidamente le modifiche apportate al codice. In un ambiente di produzione, tuttavia, il ricaricamento dinamico può avere un impatto negativo sulle prestazioni. Inoltre, ogni volta che si esegue un ricaricamento, le sessioni eseguite al momento del transito diventano non valide. Il client deve riavviare la sessione. Questa opzione è abilitata per impostazione predefinita.

Intervallo di polling ricaricamento

Indica la frequenza con cui si desidera che il server controlli l'eventuale presenza di modifiche al codice nelle applicazioni e nei moduli distribuiti. Il valore predefinito è 2.

Timeout sessione di amministrazione

Indica il numero di minuti di inattività prima che si verifichi il timeout di Console di amministrazione e sia necessario eseguire nuovamente il login. Il valore predefinito è 60.

Distribuzione automatica

Se questa opzione è abilitata, le applicazioni nella directory di distribuzione automatica vengono distribuite in modo automatico. Questa opzione è abilitata per impostazione predefinita.

Intervallo di polling distribuzione automatica

Specifica la frequenza con cui si desidera che il server controlli l'eventuale presenza di modifiche a un'applicazione o a un modulo nella directory di distribuzione automatica. La modifica dell'intervallo di polling non influisce sul tempo necessario per la distribuzione di un'applicazione o di un modulo. Il valore predefinito è 2.

Timeout tentativi distribuzione automatica

Specifica il numero di secondi durante i quali la dimensione di un file copiato parzialmente può rimanere invariata prima che si verifichi un errore. Il valore predefinito è 4.

Se viene copiato lentamente nella directory di distribuzione automatica, un file può essere visualizzato prima di essere stato copiato per intero. Di conseguenza, il tentativo di distribuire l'applicazione in modo automatico non riesce. Se un tentativo di distribuzione automatica di un'applicazione non riesce per questo motivo, GlassFish Server tenta di ridistribuire l'applicazione. Se la dimensione del file copiato parzialmente rimane invariata per il periodo di tempo specificato, si verifica un errore.

Directory di distribuzione automatica

Indica la directory per monitorare la distribuzione automatica delle applicazioni. Il valore predefinito è domain-dir/autodeploy.

Convalida XML

Specifica il tipo di convalida XML da eseguire su descrittori di distribuzione standard e di GlassFish Server. Di seguito sono indicati i tipi disponibili.

Completo

Se la convalida XML non riesce, non riesce neanche la distribuzione. Questo è il valore predefinito.

Analisi

Vengono segnalati errori di convalida XML, ma la distribuzione viene eseguita.

Nessuno

La convalida XML non viene eseguita.

Verificatore

Se questa opzione è abilitata, il verificatore viene eseguito prima della distribuzione automatica. Se la verifica non riesce, la distribuzione non viene eseguita. Il verificatore esamina la struttura e il contenuto del file. La verifica di applicazioni di grandi dimensioni richiede spesso molto tempo. Questa opzione è disabilitata per impostazione predefinita.

Precompila

Se questa opzione è abilitata, i file JSP (JavaServer Pages) vengono precompilati. Se questa opzione è disabilitata, i file JSP vengono compilati in fase di esecuzione al primo accesso. Questa opzione è disabilitata per impostazione predefinita.

Proprietà aggiuntive

Proprietà aggiuntive per le applicazioni. GlassFish Server non definisce alcuna proprietà aggiuntiva per le applicazioni.
