Provider JACC

Utilizzare la pagina Provider JACC per configurare i provider JACC (Java Authorization Contract for Containers) per l'istanza o il cluster GlassFish Server selezionato.

I provider JACC possono essere usati dai moduli di autorizzazione di terze parti per applicazioni in esecuzione su GlassFish Server. L'installazione predefinita di GlassFish Server comprende due provider JACC, denominati default e simple. Questi provider predefiniti implementano un motore di autorizzazione semplice, basato su file e conforme alla specifica JACC. È inoltre possibile creare provider JACC personalizzati. Vedere Per creare un provider JACC.

Per informazioni sulle modalità di scrittura delle classi di implementazione per un provider JACC, vedere la specifica JACC all'indirizzo http://jcp.org/en/jsr/detail?id=115.

Per ogni provider JACC vengono fornite le informazioni riportate di seguito.

Nome

Il nome del provider JACC. Questo nome viene utilizzato per fare riferimento all'elemento jacc-provider nel file domain.xml.

Configurazione criteri

Il nome di classe completamente qualificato per javax.security.jacc.PolicyConfigurationFactory.provider che implementa la classe javax.security.jacc.PolicyConfigurationFactory specifica del provider.

Provider criteri

Il nome di classe completamente qualificato per javax.security.jacc.policy.provider che implementa java.security.Policy.

I provider JACC predefiniti supportano inoltre la proprietà riportata di seguito.

repository

La directory che contiene il file dei criteri JACC. Per il provider JACC default di GlassFish Server, la directory predefinita è ${com.sun.aas.instanceRoot}/generated/policy. Questa proprietà non è definita per impostazione predefinita per il provider JACC simple di GlassFish Server.

La tabella Provider JACC contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare un nuovo provider JACC.

Elimina

Pulsante che consente di eliminare uno o più provider JACC selezionati.
