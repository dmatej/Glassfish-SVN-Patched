Crear un módulo de ciclo de vida

  1. En el árbol de navegación, seleccione el nodo Módulos de Ciclo de Vida.

    Se abrirá la página Módulos de Ciclo de Vida.

  2. En la página Módulos de Ciclo de Vida, haga clic en Nuevo.

    Se abrirá la página Nuevo Módulo de Ciclo de Vida.

  3. En el campo Nombre, introduzca un nombre para el módulo de ciclo de vida.

  4. En el campo Nombre de Clase, escriba el nombre completo del archivo de clase del módulo de ciclo de vida.

    La clase debe implantar la interfaz com.sun.appserv.server.LifecycleListener.

  5. En el campo Classpath, introduzca la classpath para el módulo de ciclo de vida.

    La classpath especifica la ubicación del módulo de ciclo de vida. El valor por defecto es domain-dir/applications. Si el módulo ya se encuentra en este directorio (es decir, en la classpath del servidor), este campo puede dejarse en blanco.

  6. En el campo Orden de Carga, introduzca un valor que especifique el orden en que este módulo de ciclo de vida se cargará al inicio.

    Los módulos con valores de orden de carga de enteros inferiores se cargarán antes. El valor puede estar entre 101 y el MAXINT del sistema operativo. Los valores de 1 a 100 están reservados.

  7. En el campo Descripción, introduzca una descripción del módulo de ciclo de vida.

  8. Active la casilla de control Estado Activado para activar el módulo de ciclo de vida.

    Esta opción se encuentra activada por defecto.

  9. Active la casilla de control Ante Fallo de Carga: Evitar Inicio de Instancia para especificar que el servidor debe apagarse si se produce un fallo en la carga del módulo de ciclo de vida.

    Esta opción se activada por defecto.

  10. Para agregar instancias de servidor y clusters de destino en los que activar el módulo de ciclo de vida, selecciónelos en la lista Destinos Disponibles y utilice el botón Agregar para moverlos a la lista Destinos Seleccionados.

    La opción Destino aparece si existen clusters o instancias de servidor independientes aparte de la instancia de servidor por defecto (server). Por defecto, el módulo de ciclo de vida está activado en la instancia de servidor por defecto, denominada server. Utilice el botón Agregar Todos para desplegar en todos los destinos. Utilice los botones Eliminar y Eliminar Todos para eliminar destinos de la lista Destinos Seleccionados.

  11. Haga clic en Aceptar.
