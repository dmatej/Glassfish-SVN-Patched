Definir la configuración de aplicaciones

Utilice la página Configuración de Aplicaciones para definir las propiedades que ayudan a garantizar que se detecten los cambios en aplicaciones desplegadas y que se vuelvan a cargar las clases modificadas.

  1. En el árbol de navegación, seleccione el nodo Dominio.

    Se abrirá la página Atributos de Dominio.

  2. Seleccione el separador Configuración de Aplicaciones.

    Se abrirá la página Configuración de Aplicaciones.

  3. Active la casilla de control Recarga Activada para activar la recarga dinámica de las aplicaciones.

    Si se activa la recarga dinámica, no tiene que volver a desplegar una aplicación o módulo cuando cambia su código o descriptores de despliegue. Lo único que tiene que hacer es copiar los archivos JavaServer Pages (JSP) o de clase modificados en el directorio de despliegue de la aplicación o módulo. El servidor comprueba periódicamente los cambios realizados en el directorio de despliegue y vuelve a desplegar la aplicación de forma automática y dinámica con los cambios. Esto es útil en un entorno de desarrollo ya que permite poner a prueba los cambios realizados en el código de forma rápida. En cambio, en un entorno de producción, la nueva carga dinámica puede hacer que disminuya el rendimiento. Además, cuando ha finalizado la nueva carga, las sesiones del período de tránsito pierden su validez. El cliente debe reiniciar la sesión.

    Esta opción se encuentra activada por defecto.

  4. En el campo Intervalo de Sondeo de Nueva Carga, introduzca la longitud, en segundos, del intervalo en el que se comprobarán los cambios de código de las aplicaciones y módulos y si se han recargado de forma dinámica.

    El valor por defecto es 2.

  5. En el campo Timeout de Sesión de Administrador, introduzca el número de minutos de inactividad tras los que la sesión de Consola de administración debe sufrir un timeout.

    El valor por defecto es 60.

  6. Active la casilla de control Despliegue Automático Activado para activar el despliegue automático de aplicaciones.

    Si copia un archivo de aplicación o módulo (JAR, WAR, RAR o EAR) en un directorio especial y, a continuación, activa el despliegue automático, GlassFish Server desplegará automáticamente la aplicación o el módulo. Esta opción se encuentra activada por defecto.

  7. En el campo Intervalo de Sondeo de Despliegue Automático, introduzca la longitud, en segundos, del intervalo en el que se comprobarán los cambios de código de las aplicaciones y módulos y si se han recargado de forma dinámica.

    El valor por defecto es 2. Si cambia el intervalo de sondeo no se verá afectado el tiempo que se tarda en desplegar una aplicación o un módulo.

  8. En el campo Timeout para el Reintento de Despliegue Automático, introduzca el número de segundos que un archivo copiado parcialmente puede permanecer sin cambios en el tamaño antes de que se produzca un error.

    El valor por defecto es 4.

  9. En el campo Directorio de Despliegue Automático, introduzca la ruta de acceso completa del directorio para supervisar el despliegue automático de aplicaciones.

    El valor por defecto es domain-dir/autodeploy.

  10. En la lista desplegable Validación de XML, seleccione el tipo de validación de XML que se llevará a cabo en los descriptores de despliegue estándar y de GlassFish Server.

    El tipo puede ser cualquiera de los siguientes:

    full

    Si se produce un fallo en la validación de XML, no se realizará el despliegue. Éste es el valor por defecto.

    parsing

    Se informa acerca de errores en la validación de XML, pero se realiza el despliegue.

    none

    No se realiza ninguna validación de XML.

  11. Active la casilla de control Verificador Activado para verificar los archivos de descriptor de despliegue.

    El verificador examinará la estructura y contenido del archivo antes del despliegue automático. Si se produce un fallo en la verificación, no se realiza el despliegue. La verificación de aplicaciones de gran tamaño tiende a llevar mucho tiempo. Esta opción se activada por defecto.

  12. Active la casilla de control Precompilación Activada para precompilar los archivos JSP.

    Si está desactivada esta opción, los archivos JSP se compilarán en tiempo de ejecución cuando se acceda a los mismos por primera vez. Esta opción se activada por defecto.

  13. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    GlassFish Server no define ninguna propiedad adicional para las aplicaciones.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, introduzca el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite el campo Valor.

    • Para suprimir una propiedad, active la casilla de control a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  14. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.

  15. Reinicie el servidor.
