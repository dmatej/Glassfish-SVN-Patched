Seguridad

Utilice la página Seguridad para definir las propiedades de seguridad para la instancia o cluster de GlassFish Server seleccionado.

La página Seguridad contiene las siguientes opciones.

Nombre de Configuración

Nombre de la configuración a la que se aplican los valores de esta página. Este campo es de sólo lectura.

Gestor de seguridad

Si se selecciona esta opción, se activará el gestor de seguridad para el dominio. Esta opción se activada por defecto.

Si se activa esta opción, se agregará una opción de JVM, -Djava.security.manager, a la configuración de JVM de GlassFish Server. Consulte Configurar opciones de JVM. Debe reiniciar el servidor para activar este cambio.

Asegúrese de que ha otorgado los permisos correctos para todas las aplicaciones. Puede desactivar el gestor de seguridad para mejorar el rendimiento.

Registro de Auditoría

Si esta opción está activada, el servidor cargará y ejecutará todos los módulos de auditoría especificados en la configuración Módulos de Auditoría. Si la opción está desactivada, el servidor no accederá a los módulos de auditoría. Esta opción se activada por defecto.

Dominio por Defecto

Dominio activo (por defecto) que el servidor utiliza para la autenticación. Las aplicaciones utilizan este dominio a menos que su descriptor de despliegue especifique un dominio distinto. Todos los dominios configurados aparecen en la lista. El valor por defecto es file.

Principal por Defecto

Especifica el nombre de usuario por defecto. El servidor utiliza este nombre de usuario cuando no se proporciona ningún otro principal. Si introduce un valor en este campo, tiene que introducir la contraseña correspondiente en el campo Contraseña Principal por Defecto.

No se requiere un principal por defecto para el funcionamiento normal de un servidor.

Contraseña Principal por Defecto

Contraseña del principal por defecto especificada en el campo Principal por Defecto. No se requiere un principal por defecto para el funcionamiento normal de un servidor.

JACC

Nombre de clase de un proveedor JACC configurado. El valor por defecto es default.

Módulos de Auditoría

Módulos del proveedor de auditoría que utilizará el subsistema de auditorías si se activa el registro de auditorías. Por defecto el servidor utiliza un módulo de auditoría llamado default. Para obtener información sobre la creación de nuevos módulos de auditoría, consulte Crear un módulo de auditoría.

Asignación de principal por defecto a rol activada

Si esta opción está seleccionada, la asignación de principal a rol por defecto se aplicará a las aplicaciones que no tengan una asignación específica de las aplicaciones.

Clase Principal Asignada

Especifica una implantación personalizada de la clase java.security.Principal utilizada en la asignación de principal a rol por defecto, en caso de que la asignación de principal a rol por defecto esté activada.

Propiedades Adicionales

Propiedades de seguridad adicionales del servidor.

Las propiedades válidas dependen del tipo de dominio seleccionado en el campo Dominio por Defecto y suelen especificarse cuando edita un dominio. Para obtener más información, consulte los siguientes temas:

  • Propiedades específicas de la clase FileRealm

  • Propiedades específicas de la clase CertificateRealm

  • Propiedades específicas de la clase JDBCRealm

  • Propiedades específicas de la clase LDAPRealm

  • Propiedades específicas de la clase SolarisRealm

  • Propiedades específicas de la clase PamRealm
