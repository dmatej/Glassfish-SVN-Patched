기본 GlassFish Server 설치에는 default와 simple이라는 두 개의 JACC 제공자가 포함됩니다. 이러한 두 개의 기본 제공자 외에도 새로 생성한 모든 JACC 제공자가 포함됩니다. 기본 GlassFish Server JACC 제공자는 JACC 사양을 준수하는 간단한 파일 기반 권한 부여 엔진을 구현합니다.

JACC 제공자에 대한 구현 클래스를 작성하는 방법은 JACC 사양(http://jcp.org/en/jsr/detail?id=115)을 참조하십시오.

  1. 탐색 트리의 [구성] 노드에서 JACC 제공자를 생성할 GlassFish Server 인스턴스나 클러스터를 확장합니다.

    모든 인스턴스와 클러스터에서 사용할 수 있는 제공자를 생성하려면 default-config 노드를 선택합니다.

  2. 확장된 인스턴스나 클러스터 구성 노드에서 [보안] 노드를 확장합니다.

  3. [보안] 노드에서 [JACC 제공자] 노드를 선택합니다.

    [JACC 제공자] 페이지가 열립니다.

  4. [JACC 제공자] 페이지에서 [새로 만들기]를 누릅니다.

    [새 JACC 제공자] 페이지가 열립니다.

  5. [새 JACC 제공자] 페이지의 [이름] 필드에 이 제공자를 식별하는 이름을 입력합니다.

    이 이름은 domain.xml의 jacc-provider 요소를 참조하는 데 사용됩니다.

  6. [정책 구성] 필드에 정책 구성 팩토리를 구현하는 클래스 이름을 입력합니다.

    default 제공자는 com.sun.enterprise.security.provider.PolicyConfigurationFactoryImpl을 사용합니다. simple 제공자는 com.sun.enterprise.security.jacc.provider.SimplePolicyConfigurationFactory 를 사용합니다.

  7. [정책 제공자] 필드에 정책 팩토리를 구현하는 클래스 이름을 입력합니다.

    default 제공자는 com.sun.enterprise.security.provider.PolicyWrapper를 사용합니다. simple 제공자는 com.sun.enterprise.security.jacc.provider.SimplePolicyProvider 를 사용합니다.

  8. [추가 속성] 섹션에서 추가 속성을 지정합니다.

    속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

    JACC 제공자에 사용 가능한 추가 속성에 대한 설명은 JACC 제공자 관련 속성을 참조하십시오.

  9. [확인]을 누릅니다.
