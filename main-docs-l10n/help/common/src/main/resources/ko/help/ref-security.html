보안

[보안] 페이지에서는 선택한 GlassFish Server 인스턴스나 클러스터에 대한 보안 속성을 설정할 수 있습니다.

[보안] 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

보안 관리자

이 옵션을 선택하면 도메인의 보안 관리자가 사용으로 설정됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

이 옵션을 사용으로 설정하면 JVM 옵션 -Djava.security.manager가 GlassFish Server의 JVM 설정에 추가됩니다. JVM 옵션을 구성하는 방법을 참조하십시오. 이 변경 사항을 사용으로 설정하려면 서버를 재시작해야 합니다.

모든 응용 프로그램에 올바른 권한 설정을 부여했는지 확인하십시오. 성능 향상을 위해 보안 관리자를 사용 안함으로 설정할 수 있습니다.

감사 로깅

이 옵션을 사용으로 설정하면 감사 모듈 목록에서 지정한 모든 감사 모듈을 서버에서 로드하여 실행합니다. 이 옵션이 사용 안함으로 설정된 경우 서버가 감사 모듈에 액세스하지 않습니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

기본 영역

서버가 인증에 사용하는 활성(기본) 영역입니다. 배치 기술자가 다른 영역을 지정하지 않는 한 응용 프로그램은 이 영역을 사용합니다. 구성된 모든 영역이 목록에 표시됩니다. 기본값은 file입니다.

기본 주체

기본 사용자 이름을 지정합니다. 다른 주체를 제공하지 않을 경우 서버는 이 사용자 이름을 사용합니다. 이 필드에 값을 입력하는 경우에는 [기본 주체 비밀번호] 필드에 해당 비밀번호를 입력합니다.

정상적인 서버 작업에는 기본 주체가 필요하지 않습니다.

기본 주체 비밀번호

[기본 주체] 필드에 지정한 기본 주체의 비밀번호입니다. 정상적인 서버 작업에는 기본 주체가 필요하지 않습니다.

JACC

구성된 JACC 제공자의 클래스 이름입니다. 기본값은 default입니다.

감사 모듈

감사 로깅이 사용으로 설정된 경우 감사 하위 시스템이 사용할 감사 제공자 모듈입니다. 기본적으로 서버는 이름이 default인 감사 모듈을 사용합니다. 새 감사 모듈 생성에 대한 자세한 내용은 감사 모듈을 생성하는 방법을 참조하십시오.

롤 매핑에 대한 기본 주체 사용 가능

이 옵션을 선택하면 롤 매핑에 대한 기본 주체가 응용 프로그램 관련 매핑이 없는 응용 프로그램에 적용됩니다.

매핑된 주체 클래스

주체와 롤 사이의 주체 매핑이 사용으로 설정된 경우 주체와 원칙 사이의 매핑에 사용되는 java.security.Principal 클래스의 사용자 정의 구현을 지정합니다.

추가 속성

서버의 추가 보안 속성입니다.

적합한 속성은 [기본 영역] 필드에서 선택한 영역의 유형에 따라 달라지며 일반적으로 영역을 편집할 때 지정됩니다. 자세한 내용은 다음 항목을 참조하십시오.

  • FileRealm 클래스 관련 속성

  • CertificateRealm 클래스 관련 속성

  • JDBCRealm 클래스 관련 속성

  • LDAPRealm 클래스 관련 속성

  • SolarisRealm 클래스 관련 속성

  • PamRealm 클래스 관련 속성

  • 감사 모듈을 생성하는 방법
