새 메시지 보안 구성

[새 메시지 보안 구성] 페이지에서는 메시지 보안에 대한 HttpServlet 구성을 생성할 수 있습니다.

[새 메시지 보안 구성] 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

인증 계층

인증이 수행되는 메시지 계층을 지정합니다. 값은 SOAP 또는 HttpServlet이어야 합니다. SOAP 구성은 기본적으로 사용 가능하기 때문에 선택 항목은 HttpServlet뿐입니다.

제공자 ID

이 메시지 보안 구성에 대한 제공자 구성의 고유 식별자입니다.

기본 제공자

이 옵션을 선택하면 이 제공자가 기본 제공자가 됩니다. 기본 제공자는 특정 제공자를 식별하지 않는 응용 프로그램에 호출됩니다. 제공자가 기본 클라이언트 제공자나 기본 서버 제공자인지 또는 둘 다인지는 제공자 유형으로 결정됩니다.

제공자 유형

제공자의 유형을 지정합니다. 사용 가능한 선택 항목은 다음과 같습니다.

client

제공자가 클라이언트 인증 제공자임을 지정합니다.

server

제공자가 서버 인증 제공자임을 지정합니다.

client-server

제공자가 클라이언트 및 서버 인증 제공자임을 지정합니다.

클래스 이름

제공자의 Java 구현 클래스의 이름입니다. 클라이언트 인증 제공자는 com.sun.enterprise.security.jauth.ClientAuthModule 인터페이스를 구현해야 합니다. 서버측 제공자는 com.sun.enterprise.security.jauth.ServerAuthModule 인터페이스를 구현해야 합니다. 클라이언트-서버 제공자는 두 인터페이스를 모두 구현해야 합니다.

추가 속성

메시지 보안 구성에 대한 추가 속성입니다.

관련 작업

  • 메시지 보안 제공자가 사용하도록 GlassFish Server 기능을 구성하는 방법

  • 메시지 보안 구성을 생성하는 방법

  • 메시지 보안 구성을 편집하는 방법

  • 메시지 보안 구성을 삭제하는 방법

  • 메시지 보안 제공자 구성을 생성하는 방법

  • 메시지 보안 제공자 구성을 편집하는 방법

  • 메시지 보안 제공자 구성을 삭제하는 방법

  • 응용 프로그램 클라이언트에 대해 메시지 보안을 사용으로 설정하는 방법
