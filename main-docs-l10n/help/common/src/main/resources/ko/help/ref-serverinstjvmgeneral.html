JVM 일반 설정

JVM(Java Virtual Machine)을 통해 Java 바이트 코드를 물리적 시스템에서 실행할 수 있습니다. JVM 설정을 조정하여 성능을 향상시키고 메모리 할당 오류를 방지할 수 있습니다.

Java Virtual Machine은 GlassFish Server에 필요한 Java Platform, Standard Edition(Java SE 플랫폼)에 포함되어 있습니다. JVM 설정이 올바르지 않으면 서버가 실행되지 않으므로 이 설정을 변경할 때 주의해야 합니다.

주:

"Java Virtual Machine"과 "JVM"은 Java 플랫폼용 가상 시스템을 의미합니다.

[JVM 일반 설정] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값을 사용하는 설정을 기본값으로 복원하는 단추입니다. 기본값을 사용하지 않는 설정은 변경되지 않습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

Java 홈

GlassFish Server를 실행하는 데 필요한 Java SE 소프트웨어의 설치 디렉토리 이름입니다. 지정한 Java SE 소프트웨어 버전이 이 릴리스에서 지원되는지 확인하려면 Oracle GlassFish Server 릴리스 노트를 참조하십시오. 존재하지 않는 디렉토리 이름을 입력하거나 지원되지 않는 Java SE 소프트웨어 버전의 디렉토리를 입력하면 GlassFish Server가 시작되지 않습니다.

Javac 옵션

Java 프로그래밍 언어 컴파일러에 대한 명령행 옵션입니다. EJB(Enterprise JavaBeans) 구성 요소가 배치되면 GlassFish Server에서 컴파일러가 실행됩니다.

디버그

JPDA(Java Platform Debugger Architecture)로 디버깅할 수 있도록 설정합니다.

JPDA는 응용 프로그램 개발자가 사용합니다. 자세한 내용은 Oracle GlassFish Server Application Development Guide를 참조하십시오.

디버그 옵션

디버깅이 사용으로 설정된 경우 JVM에 전달되는 JPDA 옵션을 지정합니다.

RMI 컴파일 옵션

rmic 컴파일러에 대한 명령행 옵션입니다. EJB 구성 요소가 배치되면 GlassFish Server가 rmic 컴파일러를 실행합니다.

바이트 코드 선행 프로세서

바이트 코드 선행 처리를 위한 클래스의 이름입니다. 클래스 이름을 구분하려면 콤마를 사용합니다. 클래스마다 com.sun.appserv.BytecodePreprocessor 인터페이스를 구현해야 합니다. 지정한 순서대로 클래스가 호출됩니다.

추가 속성

JVM에 대한 추가 속성입니다. GlassFish Server에서는 JVM에 대한 추가 속성을 정의하지 않습니다.
