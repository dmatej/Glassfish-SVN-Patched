메시지 보안 구성을 편집하는 방법

GlassFish Server에 배치된 웹 서비스 끝점에 대한 메시지 보안을 사용으로 설정하려면 서버측에서 기본적으로 사용되는 제공자를 지정해야 합니다. 메시지 보안을 위한 기본 제공자를 사용으로 설정한 경우 GlassFish Server에 배치된 웹 서비스의 클라이언트에서 사용할 제공자도 사용으로 설정해야 합니다. 클라이언트에서 사용하는 제공자를 사용으로 설정하는 방법은 응용 프로그램 클라이언트에 대해 메시지 보안을 사용으로 설정하는 방법을 참조하십시오.

배치된 끝점에서 발생한 웹 서비스 호출에 대해 메시지 보안을 사용으로 설정하려면 기본 클라이언트 제공자를 지정해야 합니다. GlassFish Server에 대한 기본 클라이언트 제공자를 사용으로 설정한 경우 GlassFish Server에 배치된 끝점에서 호출한 모든 서비스가 메시지 계층 보안과 호환 가능하게 구성되도록 해야 합니다.

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. [보안] 노드를 확장합니다.

  3. [메시지 보안] 노드를 선택합니다.

    [메시지 보안 구성] 페이지가 열립니다.

  4. [메시지 보안 구성] 페이지에서 편집 중인 구성의 인증 계층 이름을 누릅니다.

    기본적으로 구성되어 있는 유일한 메시지 보안 구성은 SOAP입니다.

    [메시지 보안 구성 편집] 페이지가 열립니다.

  5. [메시지 보안 구성 편집] 페이지의 [기본 제공자] 드롭다운 목록에서 서버측 제공자를 선택합니다.

    기본 제공자라고도 하는 기본 서버 제공자는 특정 서버 제공자를 식별하지 않는 응용 프로그램에 대해 호출된 서버 제공자를 식별합니다. ServerProvider, XWS_ServerProvider 또는 없음을 선택할 수 있습니다. 기본적으로 제공자가 선택되어 있지 않으며, GlassFish Server에서는 서버측의 메시지 보안 제공자를 호출하지 않습니다. 일반적으로 ServerProvider를 기본 제공자로 선택합니다.

  6. [기본 클라이언트 제공자] 드롭다운 목록에서 클라이언트 제공자를 선택합니다.

    기본 클라이언트 제공자는 특정 클라이언트 제공자를 식별하지 않는 응용 프로그램에 호출된 클라이언트 제공자를 식별합니다. ClientProvider, XWS_ClientProvider 또는 없음을 선택할 수 있습니다. 기본적으로 제공자가 선택되어 있지 않으며, GlassFish Server에서는 클라이언트측의 메시지 보안 제공자를 호출하지 않습니다. 일반적으로 기본 클라이언트 제공자에 대해 [없음]을 선택합니다. ClientProvider를 선택하면 기본 제공자 및 메시지 보호 정책을 사용으로 설정하여 GlassFish Server에 배치된 웹 서비스 끝점에서 발생한 웹 서비스 호출에 적용할 수 있습니다.

  7. [저장]을 누릅니다.

  8. 사용으로 설정된 클라이언트 또는 서버 제공자의 메시지 보호 정책을 수정하는 방법에 대한 지침은 메시지 보안 제공자 구성을 편집하는 방법을 참조하십시오.
