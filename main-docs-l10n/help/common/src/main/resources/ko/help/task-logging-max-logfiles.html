교체된 로그 파일 수 제한을 변경하는 방법

GlassFish Server에서 로그 파일을 교체할 때 server.log라는 빈 파일을 새로 생성한 다음 이전 파일의 이름을 server.log_ date로 바꿉니다. 여기서 date는 파일이 교체된 날짜 및 시간입니다.

기본적으로 GlassFish Server에서는 교체된 로그 파일 수를 10으로 제한합니다. 이 제한에 도달하면 GlassFish Server에서 다음에 로그 파일을 교체할 때 가장 오래된 로그 파일이 삭제됩니다.

필요한 경우 교체된 로그 파일의 수 제한을 변경할 수 있습니다. 설정하는 새 제한은 액세스 로깅 및 서버 로그에 대한 교체된 로그 파일에 적용됩니다.

  1. com.sun.enterprise.server.logging.max_history_files 시스템 속성을 보존할 최대 교체 액세스 로그 파일 수로 설정합니다.

    시스템 속성을 설정하는 방법은 시스템 속성을 관리하는 방법을 참조하십시오.

    com.sun.enterprise.server.logging.max_history_files 시스템 속성의 동작은 다음과 같습니다.

    • 속성이 설정되지 않은 경우 GlassFish Server에 최대 10개의 교체된 로그 파일이 보존됩니다.

    • 속성이 부적합한 수 또는 널로 설정된 경우 GlassFish Server에 최대 10개의 교체된 로그 파일이 보존됩니다.

    • 속성이 0으로 설정된 경우 GlassFish Server에 교체된 로그 파일이 보존되지 않습니다.
