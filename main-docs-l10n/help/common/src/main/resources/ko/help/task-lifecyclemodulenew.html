수명 주기 모듈을 생성하는 방법

  1. 탐색 트리에서 [수명 주기 모듈] 노드를 선택합니다.

    [수명 주기 모듈] 페이지가 열립니다.

  2. [수명 주기 모듈] 페이지에서 [새로 만들기]를 누릅니다.

    [새 수명 주기 모듈] 페이지가 열립니다.

  3. [이름] 필드에 수명 주기 모듈의 이름을 입력합니다.

  4. [클래스 이름] 필드에 수명 주기 모듈의 클래스 파일에 대한 전체 이름을 입력합니다.

    클래스는 com.sun.appserv.server.LifecycleListener 인터페이스를 구현해야 합니다.

  5. [클래스 경로] 필드에 수명 주기 모듈의 클래스 경로를 입력합니다.

    수명 주기 모듈이 있는 위치를 지정하는 클래스 경로입니다. 기본값은 domain-dir/applications입니다. 모듈이 이미 이 디렉토리(즉, 서버 클래스 경로)에 있으면 이 필드를 비워둘 수 있습니다.

  6. [로드 순서] 필드에 시작할 때 이 수명 주기 모듈을 로드할 순서를 지정하는 값을 입력합니다.

    정수 로드 순서 값이 작은 모듈부터 로드됩니다. 값의 범위는 101부터 운영 체제의 MAXINT까지일 수 있습니다. 1부터 100까지의 값은 예약되어 있습니다.

  7. [설명] 필드에 수명 주기 모듈의 설명을 입력합니다.

  8. 수명 주기 모듈을 사용하려면 [상태 사용] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용으로 설정되어 있습니다.

  9. 수명 주기 모듈이 로드에 실패할 경우 서버를 종료하도록 지정하려면 [로드 실패 시 인스턴스 시작 방지] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

  10. 수명 주기 모듈을 사용으로 설정할 대상 서버 인스턴스와 클러스터를 추가하려면 사용 가능한 대상 목록에서 해당 서버 인스턴스와 클러스터를 선택한 다음 [추가] 단추를 사용하여 [선택한 대상] 목록으로 이동합니다.

    [대상] 옵션은 기본 서버 인스턴스(server) 외에 클러스터 또는 독립형 서버 인스턴스가 있는 경우에 표시됩니다. 기본적으로 수명 주기 모듈은 server라는 기본 서버 인스턴스에서 사용으로 설정되어 있습니다. 모든 대상에 배치하려면 [모두 추가] 단추를 사용합니다. [선택한 대상] 목록에서 대상을 제거하려면 [제거] 및 [모두 제거] 단추를 사용합니다.

  11. [확인]을 누릅니다.
