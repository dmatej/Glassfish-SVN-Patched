모니터링 서비스를 구성하는 방법

  1. [모니터링 서비스] 페이지를 열려면 다음 중 하나를 수행합니다.

    • [모니터링 데이터] 노드를 선택한 다음 모니터링을 구성할 서버 인스턴스에 대한 [모니터링 구성] 링크를 누릅니다. 클러스터에 대한 모니터링을 구성하려면 클러스터에 있는 인스턴스를 선택합니다.

    • [구성] 노드에서 모니터링을 구성할 서버 인스턴스나 클러스터 구성을 확장한 다음 [모니터링] 노드를 확장합니다.

    [모니터링 서비스] 페이지가 열립니다.

  2. [모니터링 서비스 사용] 체크 박스를 선택하여 GlassFish Server에 대한 모니터링을 사용으로 설정하거나, 이 체크 박스를 선택 해제하여 모니터링을 사용 안함으로 설정합니다.

    이 옵션은 기본적으로 사용으로 설정되어 있습니다.

  3. [MBean 모니터링 사용] 체크 박스를 선택하여 모니터링에 필요한 모든 MBean을 배치하거나, 이 체크 박스를 선택 해제하여 이 MBean의 배치를 취소합니다.

    이 옵션은 기본적으로 사용으로 설정되어 있습니다.

    이 체크 박스가 선택되어 있지 않으면 [모니터링 서비스] 체크 박스가 선택되어 있어도 관리 콘솔에서 모니터링 데이터를 볼 수 없습니다. 모니터링 데이터를 보려면 명령행을 사용해야 합니다. 체크 박스 선택을 해제하면 GlassFish Server의 메모리 범위가 줄어듭니다.

  4. Solaris DTrace 기능을 사용하도록 설정하려면 [DTrace 사용] 체크 박스를 선택합니다.

    Solaris DTrace 기능을 사용하려면 Oracle Enterprise Manager DTrace Monitoring도 있어야 합니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

  5. 테이블에서 다음 작업 중 하나를 수행합니다.

    • [모니터링 레벨] 열에서 구성 요소 열의 하나 이상의 구성 요소에 대한 드롭다운 목록에서 LOW 또는 HIGH를 선택하여 해당 구성 요소에 대한 모니터링을 사용으로 설정합니다.

    • 하나 이상의 구성 요소 옆에 있는 체크 박스를 선택한 다음 테이블 맨 위의 레벨 드롭다운 목록에서 LOW 또는 HIGH를 선택하고 [레벨 변경] 단추를 눌러 해당 구성 요소 모두에 대해 모니터링 레벨을 이 값으로 설정합니다.

    이 릴리스에서는 LOW 및 HIGH 옵션에서 제공되는 모니터링 레벨 간에 차이가 없습니다. 하나 이상의 구성 요소에 대한 모니터링을 사용 안함으로 설정하려면 OFF를 선택합니다. 기본값은 OFF입니다.

    응용 프로그램, 구성 요소 또는 서비스에 대한 모니터링을 구성하려면 먼저 배치해야 합니다.

  6. [저장]을 누릅니다.
