메시지 보안 구성을 생성하는 방법

일반적으로 메시지 보안 구성은 생성할 필요가 없습니다. GlassFish Server에는 SOAP 인증 계층에 대한 메시지 보안 구성이 포함됩니다. 이 구성에 대한 제공자는 기본적으로 활성화되지 않지만 SOAP 메시지 보안 구성을 편집하여 사용으로 설정할 수 있습니다. 하지만 HttpServlet 인증 계층에 대한 구성을 생성하고 이에 대한 하나 이상의 제공자 구성을 지정할 수 있습니다.

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. [보안] 노드를 확장합니다.

  3. [메시지 보안] 노드를 선택합니다.

    [메시지 보안 구성] 페이지가 열립니다.

  4. [메시지 보안 구성] 페이지에서 [새로 만들기]를 누릅니다.

    [새 메시지 보안 구성] 페이지가 열립니다.

    주:

    기본 server-config 구성에 대한 메시지 보안 제공자는 생성하거나 삭제할 수 없습니다. 기본 server-config에 대해 지원되는 메시지 보안 제공자는 SOAP와 HttpServlet뿐입니다. 이 두 메시지 제공자는 GlassFish Server 설치 시 이미 기본적으로 생성되었기 때문에 [새로 만들기] 단추가 기본 server-config의 [메시지 보안 구성] 페이지에서 회색으로 표시됩니다.

  5. [새 메시지 보안 구성] 페이지의 [인증 계층] 드롭다운 목록에서 구성을 생성하고 있는 메시지 계층을 선택합니다.

    SOAP 구성은 기본적으로 사용 가능하기 때문에 선택 항목은 HttpServlet뿐입니다.

  6. [제공자 ID] 필드에 이 메시지 보안 구성에 대한 제공자 구성의 고유 식별자를 입력합니다.

  7. [기본 제공자] 체크 박스를 선택하여 이 제공자 구성을 기본 제공자로 지정합니다.

  8. [제공자 유형] 드롭다운 목록에서 이 제공자의 유형을 선택합니다.

    사용 가능한 선택 항목은 다음과 같습니다.

    client

    제공자가 클라이언트 인증 제공자임을 지정합니다.

    server

    제공자가 서버 인증 제공자임을 지정합니다.

    client-server

    제공자가 클라이언트 및 서버 인증 제공자임을 지정합니다.

  9. [클래스 이름] 필드에 제공자의 Java 구현 클래스의 이름을 입력합니다.

    클라이언트 인증 제공자는 com.sun.enterprise.security.jauth.ClientAuthModule 인터페이스를 구현해야 합니다. 서버측 제공자는 com.sun.enterprise.security.jauth.ServerAuthModule 인터페이스를 구현해야 합니다. 클라이언트-서버 제공자는 두 인터페이스를 모두 구현해야 합니다.

  10. [추가 속성] 섹션에서 추가 속성을 지정합니다.

    속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

    GlassFish Server에서는 메시지 보안 구성에 대한 추가 속성을 정의하지 않습니다.

관련 항목

  • 메시지 보안 제공자가 사용하도록 GlassFish Server 기능을 구성하는 방법

  • 메시지 보안 구성을 편집하는 방법

  • 메시지 보안 구성을 삭제하는 방법

  • 메시지 보안 제공자 구성을 생성하는 방법

  • 메시지 보안 제공자 구성을 편집하는 방법

  • 메시지 보안 제공자 구성을 삭제하는 방법

  • 응용 프로그램 클라이언트에 대해 메시지 보안을 사용으로 설정하는 방법
