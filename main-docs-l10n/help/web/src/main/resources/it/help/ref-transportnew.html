Nuovo trasporto

Utilizzare la pagina Nuovo trasporto per creare un nuovo trasporto.

In genere non è necessario utilizzare altri trasporti in aggiunta al trasporto tcp predefinito.

La pagina Nuovo trasporto contiene le opzioni riportate di seguito.

Nome configurazione

Il nome della configurazione a cui si applicano le impostazioni presenti in questa pagina. Questo campo è di sola lettura.

Nome trasporto

Il nome univoco del trasporto.

Nome classe

Il nome classe dell'implementazione del trasporto. Il valore predefinito è com.sun.grizzly.TCPSelectorHandler.

Handler chiavi di selezione

Il nome classe dell'handler delle chiavi di selezione. Per handler delle chiavi di selezione si intende una classe astratta che gestisce il ciclo di vita delle chiavi di selezione. Se non si specifica un valore, verrà utilizzata un'implementazione predefinita.

Tipo di buffer di byte

Il tipo di ByteBuffer da utilizzare. Le scelte disponibili sono HEAP e DIRECT. Il valore predefinito è HEAP.

Thread processo di accettazione

Il numero di processori nel computer. Il valore -1 indica che Grizzly calcolerà direttamente il numero di thread del processo di accettazione, basandosi sul numero di processori del computer. Il valore predefinito è 1.

Per impostare il numero dei thread di elaborazione delle richieste, impostare il valore Dimensione massima pool di thread del pool di thread utilizzato da un listener di rete che utilizza questo trasporto.

Numero massimo connessioni

Il numero massimo di connessioni in sospeso in un listener di rete che utilizza questo trasporto. Il valore predefinito è 4096.

Dimensione buffer

La dimensione, in byte, del buffer da fornire per i flussi di input creati da un listener di rete che utilizza questo trasporto. Il valore predefinito è 8192.

Timeout chiave inattiva

Il numero di secondi allo scadere dei quali una chiave inattiva viene annullata e il canale viene chiuso. Il valore predefinito è 30.

Timeout di lettura

Il numero di millisecondi di attesa di GlassFish Server durante la fase di analisi dell'intestazione e del corpo di un'operazione di lettura. Il valore predefinito è 30.000.

Timeout di polling del selettore

Il numero di millisecondi di blocco di un selettore NIO in attesa di eventi (richieste di utenti). Il valore predefinito è 1000.

Timeout di scrittura

Il numero di millisecondi di attesa di GlassFish Server prima che venga considerata terminata la connessione del client remoto durante la scrittura della risposta. Il valore predefinito è 30.000.

Visualizza configurazione

Se questa opzione è selezionata, viene eseguito il flush della configurazione interna di Grizzly nei log del server. Questa opzione può fornire informazioni utili per il debug. Questa opzione è disabilitata per impostazione predefinita.

Snooping

Se questa opzione è selezionata, viene eseguito il dump delle informazioni di richiesta/risposta nel log del server. Questa opzione può fornire informazioni utili per il debug, ma causa un peggioramento significativo delle prestazioni. Questa opzione è disabilitata per impostazione predefinita.

TCP senza ritardo

Se questa opzione è selezionata, viene abilitato TCP_NODELAY (noto anche come algoritmo di Nagle). Questa opzione è abilitata per impostazione predefinita.
