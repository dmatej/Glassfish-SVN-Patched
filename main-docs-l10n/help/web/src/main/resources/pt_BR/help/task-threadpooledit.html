Para Editar um Pool de Threads

  1. Na árvore de navegação, expanda o nó Configuração.

  2. No nó Configuração, selecione o nó Pool de Threads.

    A página Pools de Thread será aberta.

  3. Na página Pool de Threads, clique no nome do pool de threads a ser editado.

    A página Editar Pool de Threads será aberta.

  4. Na página Editar Pool de Threads, no campo Nome da Classe, digite o nome totalmente qualificado da classe de implementação do pool de threads.

    O valor default é com.sun.grizzly.http.StatsThreadPool.

  5. No campo Tamanho Máximo da Fila, digite o número máximo de mensagens que podem ser colocadas na fila até que os threads estejam disponíveis para processá-las.

    Um valor de -1 indica que não existe limite para o tamanho da linha. O valor default é 4096.

  6. No campo Tamanho Máximo do Pool de Threads, digite a quantidade máxima de threads que pode haver no pool de threads.

    O valor default é 5. Esse é o valor recomendado para pool de threads de listener de rede.

  7. No campo Tamanho Mínimo do Pool de Threads, digite a quantidade mínima de threads que pode haver no pool de threads.

    Esses pools são criados quando o pool de threads é instanciado. O valor default é 2.

  8. No campo Time-out para Ociosidade do Thread, digite o tempo máximo, em segundos, durante o qual um thread pode permanecer ocioso no pool.

    Após esse período, o thread será removido do pool. O valor default é 900.

  9. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.
