Editar Servidor Virtual

Use a página Editar Servidor Virtual para modificar as definições de um servidor virtual.

A página Editar Servidor Virtual contém as seguintes opções.

Carregar Defaults

Botão para restaurar definições para seus valores default. Definições que não tiveram seus valores default alterados.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Id

Identificador visível internamente do servidor virtual. Não é mostrado aos clientes HTTP. Os nomes de host apresentados aos clientes HTTP devem ser especificados no campo Hosts.

Hosts

O nome ou os nomes de host do computador em que o servidor está em execução. Use nomes de host reais ou virtuais que são registrados com o servidor DNS para a sua rede (e, em um sistema UNIX, no seu arquivo/etc/hosts). O valor default está no valor da propriedade do sistema ${com.sun.aas.hostName}.

Estado

O estado desejado do servidor. O valor pode ser um dos seguintes:

Ligado

O servidor virtual está ativo. Esse é o valor default.

Desligado

O servidor virtual está inativo. As tentativas de acessar o servidor retornam o código de erro 404 (recurso indisponível).

Desativado

O servidor virtual está desativado. As tentativas de acessar o servidor retornam o código de erro 403 (execução da solicitação recusada).

SSO

Especifique se o comportamento de sign-on único é herdado do serviço HTTP e se está ativado ou desativado. Por default, é herdado do Serviço HTTP. Se esta opção estiver ativada, o sign-on único será ativado para aplicações Web neste servidor virtual que estiverem configuradas para o mesmo realm. Se esta opção não estiver ativada, o sign-on único será desativado para este servidor virtual, e os usuários precisarão ser autenticados separadamente em cada aplicação no servidor virtual.

Listeners de Rede

O listener ou listeners de rede associados a esse servidor.

Módulo Web Default

O módulo da Web implantado (se existente), designado para responder todas as solicitações que não puderem ser mapeadas por outros módulos Web implantados no servidor virtual. Se um Módulo da Web Default não for especificado, o módulo da Web que tiver uma raiz de contexto vazia será utilizado. Se não houver nenhum módulo da Web com raiz de contexto vazia, um módulo da Web default do sistema será criado e utilizado.

Arquivo de Log

O nome do caminho do arquivo em que as mensagens de log do servidor virtual serão exibidas. Por default, as mensagens de log aparecerão em domain-dir/logs/server.log.

Docroot

O caminho absoluto do diretório de documento raiz do servidor. O valor default é domain-dir/docroot.

Log de Acesso

Especifica se o log de acesso é herdado do serviço HTTP e se está ativado ou desativado. Por default, é herdado do Serviço HTTP.

Diretório

O caminho absoluto de diretório para os logs de acesso do servidor. O default é domain-dir/logs/access.

Propriedades Adicionais

Propriedades adicionais do servidor virtual.

Para obter informações sobre propriedades disponíveis, consulte Propriedades Específicas para Servidores Virtuais.
