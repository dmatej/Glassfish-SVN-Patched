Para Editar um Listener de Rede

  1. Na árvore de navegação, expanda o nó Configuração.

  2. No nó Configuração, expanda o nó Configuração da Rede.

  3. No nó Config da Rede, selecione o nó Listeners de Rede.

    A página Listeners de Rede será aberta.

  4. Na página Listeners de Rede, clique no nome do listener de rede que está editando.

    A página Editar Listener de Rede será aberta.

  5. Selecione o Protocolo usado pelo listener de rede para alterar suas definições. Por exemplo, você pode ativar segurança. Não é possível escolher um protocolo diferente.

  6. Marque ou desmarque a caixa de seleção Status Ativado para alterar o status do listener de rede.

    O listener pode ser ativado ou desativado. Por default, a opção está ativada. Se o listener estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

  7. A caixa de seleção somente para leitura Segurança Ativada indica se a segurança está ativada para o protocolo usado pelo listener de rede.

    Se você estiver configurando o admin-listener para o servidor de administração de domínios (DAS), que usa a configuração server-config, esta opção será somente para leitura. Para ativar a segurança deste listener, é necessário ativar a administração segura. Consulte Administração Segura.

  8. Marque a caixa de seleção Listener JK ativado para especificar que o listener suporta o protocolo mod-jk do Apache.

    O protocolo mod-jk é um protocolo proprietário de comunicação/rede entre o httpd do Apache (servidor da web) e uma instância de back-end do GlassFish Server. Se o listener oferecer suporte a esse protocolo, o httpd deverá tratar todo o conteúdo estático, enquanto as solicitações por recursos dinâmicos (ou seja, Servlets e JSPs) serão roteadas para o listener de rede ativado por mod-jk do GlassFish Server, utilizando o protocolo mod-jk. Por default, essa opção está desativada.

  9. No campo Porta, digite o número da porta do listener de rede.

    Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

  10. No campo Endereço, digite o endereço IP do listener de rede.

    O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

  11. Na lista drop-down Pool de Threads, selecione o pool de threads a ser associado ao listener de rede.

  12. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.
Próximas Etapas

Você pode usar as guias SSL, HTTP e Cache de Arquivos para editar as definições para o protocolo associado ao listener de rede. A guia SSL somente é significativa se você tiver ativado a segurança para o protocolo.
