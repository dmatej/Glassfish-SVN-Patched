Novo Listener HTTP

Use a página Novo Listener HTTP para criar um novo listener HTTP.

Ao criar um listener HTTP, a guia SSL é disponibilizada para você também editar.

A página Novo Listener HTTP contém as seguintes opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Nome

Nome de um listener exclusivo. O nome de um listener HTTP não pode começar com um número.

Porta

O número da porta na qual o listener fará listening. Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

Status

O status do listener HTTP. O listener pode ser ativado ou desativado. Por default, a opção está ativada. Se o listener estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

Nas versões do GlassFish Server anteriores a 9.1, um listener cujo atributo ativado estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter esse comportamento na versão atual do GlassFish Server, defina o status do listener como ativado e cada estado de servidor virtual associado como OFF. Um servidor virtual lista suas listeners associadas na sua lista drop-down Listeners de Rede.

Segurança

Se esta opção estiver selecionada, a segurança será ativada para o listener HTTP. Por default, essa opção está desativada.

Se você estiver configurando o admin-listener para o servidor de administração de domínios (DAS), que usa a configuração server-config, esta opção será somente para leitura. Para ativar a segurança deste listener, é necessário ativar a administração segura. Consulte Administração Segura.

Listener JK

Se esta opção estiver selecionada, o listener suportará o protocolo mod-jk do Apache. O protocolo mod-jk é um protocolo proprietário de comunicação/rede entre o httpd do Apache (servidor da web) e uma instância de back-end do GlassFish Server. Se o listener suportar este protocolo, httpd deverá suportar todo o conteúdo estático, enquanto qualquer solicitação de recurso dinâmico (ou seja, Servlets e JSPs) são roteados para o listener HTTP ativado do mod-jk do &Productname;, usando o protocolo mod-jk. Por default, essa opção está desativada.

Endereço

O endereço IP no qual o listener de rede fará listening. O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

Servidor Virtual Default

O servidor virtual a ser associado a este listener HTTP. Use a página Servidores Virtuais para definir servidores virtuais.

Pool de Threads

O pool de threads associado ao listener HTTP. Normalmente, você seleciona um dos dois pool de threads que estão configurados quando o GlassFish Server é instalado.

Nome do Servidor

O nome do host a ser utilizado nas URLs que o servidor envia ao cliente. Esse nome será o nome alias se o servidor usar um alias. Se o servidor não usar alias, deixe o campo em branco.

Esse valor somente afeta os URLs gerados automaticamente pelo servidor; ele não afeta os URLs para diretórios e arquivos armazenados no servidor. Caso seu servidor use um alias, o nome do servidor deve ser o nome do alias. Se uma vírgula e um número de porta forem acrescentados, essa porta será usada nos URLs que o servidor envia para o cliente.
