Para Editar um Listener HTTP

  1. Na árvore de navegação, expanda o nó Configuração.

  2. No nó Configuração, expanda o nó Serviço HTTP.

  3. No nó Configuração de Rede, selecione o nó Listeners HTTP.

    A página Listeners HTTP será aberta.

  4. Na página Listeners HTTP, clique no nome do listener HTTP que você estiver editando.

    A página Editar Listener HTTP será aberta.

  5. No campo Porta, digite o número da porta na qual o listener HTTP fará o listening.

    Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

  6. Marque ou desmarque a caixa de seleção Status Ativado para alterar o status do listener HTTP.

    O listener pode ser ativado ou desativado. Por default, a opção está ativada. Se o listener estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

  7. Marque a caixa de seleção Segurança Ativada para ativar a segurança para o listener HTTP.

    Por default, essa opção está desativada.

    Se você estiver configurando o admin-listener para o servidor de administração de domínios (DAS), que usa a configuração server-config, esta opção será somente para leitura. Para ativar a segurança deste listener, é necessário ativar a administração segura. Consulte Administração Segura.

  8. Marque a caixa de seleção Listener JK ativado para especificar que o listener suporta o protocolo mod-jk do Apache.

    O protocolo mod-jk é um protocolo proprietário de comunicação/rede entre o httpd do Apache (servidor da web) e uma instância de back-end do GlassFish Server. Se o listener suportar este protocolo, httpd deverá suportar todo o conteúdo estático, enquanto qualquer solicitação de recurso dinâmico (ou seja, Servlets e JSPs) são roteados para o listener HTTP ativado do mod-jk do &Productname;, usando o protocolo mod-jk. Por default, essa opção está desativada.

  9. No campo Endereço, digite o endereço IP no qual o listener HTTP fará o listening.

    O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

  10. Na lista drop-down Servidor Virtual Default, selecione o servidor virtual a ser associado a este listener HTTP.

    Use a página Servidores Virtuais para definir servidores virtuais.

  11. Na lista drop-down Pool de Threads, selecione o pool de threads a ser associado ao listener HTTP.

    Normalmente, você seleciona um dos dois pool de threads que estão configurados quando o GlassFish Server é instalado.

  12. No campo Nome do Servidor, digite o nome do host a ser usado nas URLs que o servidor envia para o cliente.

    Esse nome será o nome alias se o servidor usar um alias. Se o servidor não usar alias, deixe o campo em branco.

    Esse valor somente afeta os URLs gerados automaticamente pelo servidor; ele não afeta os URLs para diretórios e arquivos armazenados no servidor. Caso seu servidor use um alias, o nome do servidor deve ser o nome do alias. Se uma vírgula e um número de porta forem acrescentados, essa porta será usada nos URLs que o servidor envia para o cliente.

  13. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.
Próximas Etapas
