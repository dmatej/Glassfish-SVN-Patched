Novo Listener de Rede

Utilize a página Novo Listener de Rede para criar um novo listener.

Quando você cria um listener de rede, pode criar simultaneamente um protocolo que será utilizado por ele ou um protocolo criado primeiro. Normalmente, há um relacionamento um-a-um entre o listener de rede e seu protocolo associado, mas isso não é necessário.

A página Novo Listener de Rede contém estas opções.

Nome da Configuração

O nome da configuração à qual as definições nesta página se aplicam. Este campo é somente para leitura.

Nome

Nome de um listener exclusivo. Um nome do listener de rede não pode começar com um número.

Protocolo

O protocolo associado ao listener de rede. As opções a seguir, especificam se deve ser criado um protocolo ou se um existente deve ser utilizado.

Criar um Novo Protocolo

Um novo protocolo com definições default é criado para o listener de rede. O nome default do protocolo é o nome do listener de rede com -protocol acrescentado, mas é possível editar esse valor.

Servidor Virtual

Para um novo protocolo, o servidor virtual a ser associado ao protocolo.

Utilizar um Protocolo Existente

O protocolo selecionado é utilizado. O protocolo determina se a segurança está ativada para o listener de rede.

Status

O status do listener de rede. O listener pode ser ativado ou desativado. Por default, a opção está ativada. Se o listener estiver desativado, qualquer tentativa de conectar ao listener resulta em uma exceção de soquete (java.net.ConnectException).

Nas versões do GlassFish Server anteriores a 9.1, um listener cujo atributo ativado estava definido como falso retornava um código de resposta 404, para quaisquer solicitações enviadas ao mesmo. Para obter esse comportamento na versão atual do GlassFish Server, defina o status do listener como ativado e cada estado de servidor virtual associado como OFF. Um servidor virtual lista suas listeners associadas na sua lista drop-down Listeners de Rede.

Segurança

Se essa opção estiver selecionada, a segurança será ativada para o protocolo utilizado pelo listener de rede. O campo Segurança é somente para leitura.

Se você estiver configurando o admin-listener para o servidor de administração de domínios (DAS), que usa a configuração server-config, esta opção será somente para leitura. Para ativar a segurança deste listener, é necessário ativar a administração segura. Consulte Administração Segura.

Listener JK

Se esta opção estiver selecionada, o listener suportará o protocolo mod-jk do Apache. O protocolo mod-jk é um protocolo proprietário de comunicação/rede entre o httpd do Apache (servidor da web) e uma instância de back-end do GlassFish Server. Se o listener oferecer suporte a esse protocolo, o httpd deverá tratar todo o conteúdo estático, enquanto as solicitações por recursos dinâmicos (ou seja, Servlets e JSPs) serão roteadas para o listener de rede ativado por mod-jk do GlassFish Server, utilizando o protocolo mod-jk. Por default, essa opção está desativada.

Porta

O número da porta na qual o listener fará listening. Os valores válidos são 1 a 65.535. Em sistemas UNIX, criar soquetes que fazem listening nas portas 1 - 1024 exige privilégios de superusuário.

Endereço

O endereço IP no qual o listener de rede fará listening. O endereço pode estar na notação de par de pontos ou IPv6. Pode ser qualquer (para INADDR_ANY) para fazer listening em todos os endereços IP. Ele pode ser um nome de host.

Pool de Threads

O pool de threads associado ao listener de rede. Normalmente, você seleciona um dos dois pool de threads que estão configurados quando o GlassFish Server é instalado.

Transporte

O tipo de transporte do listener de rede. Por default, a escolha é tcp.
