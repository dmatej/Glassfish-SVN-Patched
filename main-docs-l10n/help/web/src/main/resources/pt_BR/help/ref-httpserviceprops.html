Propriedades Específicas para o Serviço HTTP

As propriedades adicionais a seguir estão disponíveis para o serviço HTTP. Se você especificar essas propriedades, elas se aplicarão a todos Listeners de Rede.

ssl-cache-entriese

O número se sessões SSL a serem armazenadas no cache. O valor default é 10.000.

ssl3-session-timeout

O intervalo no qual as sessões SSL3 são armazenadas no cache. O valor default é 86.400.

ssl-session-timeout

O intervalo no qual as sessões SSL2 são armazenadas no cache. O valor default é 100.

proxyHandler

O nome da classe totalmente qualificada de uma implementação personalizada da classe abstrata com.sun.appserv.ProxyHandler que permite uma instância back-end do GlassFish Server, para recuperar informações sobre a solicitação original do cliente que foi interceptada por um servidor proxy em que a conexão SSL proveniente do cliente termina no servidor proxy (por exemplo, um balanceador de carga). Uma implementação desta classe abstrata inspeciona um determinado cabeçalho de solicitação do cliente, através da qual o proxy do servidor comunica as informações sobre a solicitação original do cliente para a instância do Enterprise Server, e retorna aquela informação para seu chamador.

A implementação default lê o endereço IP do cliente de um cabeçalho de solicitação HTTP denominado Proxy-ip, o keysize do SSL de um cabeçalho de solicitação denominado Proxy-keysize, e o certificado do cliente SSL de um cabeçalho de solicitação HTTP denominado Proxy-auth-cert. O valor Proxy-auth-cert deve conter a cadeia de certificados do cliente codificados por BASE-64 sem os limites BEGIN CERTIFICATE e END CERTIFICATE e com \n substituído por % d% a.

Essa definição só será utilizada se a caixa de seleção Passagem de Autorização para a página HTTP do protocolo do listener de rede está definida como verdadeira.

connectionTimeout

Especifica o número de segundos aguardados pelos listeners de rede, após aceitar a conexão, pela apresentação de linha URI de solicitação. O valor default é 30.
