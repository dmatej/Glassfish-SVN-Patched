Nouveau pool de threads

La page Nouveau pool de threads permet de créer un pool de threads.

Elle contient les options ci-après.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

Nom

Nom du pool de threads.

Nom de classe

Nom qualifié complet de la classe qui implémente le pool de threads. La valeur par défaut est com.sun.grizzly.http.StatsThreadPool.

Taille max. de la file d'attente

Nombre maximal de messages pouvant être mis en file d'attente jusqu'à ce que des threads soient disponibles pour les traiter. La valeur -1 signifie que la taille de la file d'attente est illimitée. La valeur par défaut est 4 096.

Taille max. du pool de threads

Nombre maximal de threads dans le pool de threads. La valeur par défaut est 5. Il s'agit de la valeur recommandée pour les pools de threads de processus d'écoute réseau.

Taille min. du pool de threads

Nombre minimal de threads dans le pool de threads. La valeur par défaut est 2.

Expiration de thread inactif

Temps maximal (en secondes) durant lequel un thread peut rester inactif dans le pool. Une fois ce délai expiré, le thread est enlevé du pool. La valeur par défaut est 900.
