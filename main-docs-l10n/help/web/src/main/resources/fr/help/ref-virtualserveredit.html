Modifier un serveur virtuel

La page Modifier un serveur virtuel permet de modifier les paramètres d'un serveur virtuel.

La page Modifier un serveur virtuel contient les options ci-après.

Charger les valeurs par défaut

Bouton permettant de restaurer les valeurs par défaut des paramètres qui en ont. Les paramètres qui n'ont pas de valeurs par défaut ne sont pas modifiés.

Nom de configuration

Nom de la configuration à laquelle s'appliquent les paramètres de cette page. Ce champ est en lecture seule.

ID

Identificateur de serveur virtuel visible en interne. Il n'est pas visible par les clients HTTP. Les noms d'hôte visibles par les clients HTTP doivent être spécifiés dans le champ Hôtes.

Hôtes

Noms d'hôte de l'ordinateur sur lequel le serveur est exécuté. Utilisez des noms d'hôte réels ou virtuels inscrits sur le serveur DNS pour votre réseau (et, sous UNIX, dans le fichier /etc/hosts). La valeur par défaut correspond à la valeur de propriété système ${com.sun.aas.hostName}.

Etat

Etat souhaité du serveur. Les valeurs possibles sont les suivantes :

Activé

Le serveur virtuel est actif. Il s'agit de la valeur par défaut.

Désactivé

Le serveur virtuel est inactif. Les tentatives d'accès au serveur renvoient le code d'erreur 404 (ressource non disponible).

Désactivé

Le serveur virtuel est désactivé. Les tentatives d'accès au serveur renvoient le code d'erreur 403 (refus d'exécuter la demande).

SSO

Spécifie si le comportement de connexion unique est hérité du service HTTP, activé ou désactivé. Par défaut, il est hérité du service HTTP. Si cette option est activée, la connexion unique est activée pour les applications Web sur ce serveur virtuel qui sont configurées pour le même domaine. Si cette option est désactivée, la connexion unique est désactivée pour ce serveur virtuel et les utilisateurs doivent être authentifiés séparément pour chaque application sur le serveur virtuel.

Processus d'écoute réseau

Processus d'écoute réseau associés à ce serveur.

Module Web par défaut

Module Web déployé (le cas échéant) qui doit répondre à toutes les demandes ne pouvant pas être mappées avec d'autres modules Web déployés sur le serveur virtuel. Si aucun module Web par défaut n'est spécifié, le module Web qui possède une racine de contexte vide est utilisé. S'il n'existe aucun module Web avec une racine de contexte vide, un module Web système par défaut est créé et utilisé.

Fichier journal

Nom de chemin du fichier dans lequel les messages de journalisation provenant de ce serveur virtuel apparaîtront. Par défaut, les messages de journalisation apparaissent dans domain-dir/logs/server.log.

Racine doc.

Chemin absolu du répertoire de documents racine pour le serveur. La valeur par défaut est domain-dir/docroot.

Journalisation des accès

Spécifie si la journalisation des accès est héritée du service HTTP, activée ou désactivée. Par défaut, il est hérité du service HTTP.

Répertoire

Chemin de répertoire absolu des journaux des accès du serveur. La valeur par défaut est domain-dir/logs/access.

Autres propriétés

Autres propriétés du serveur virtuel.

Pour plus d'informations sur les propriétés disponibles, reportez-vous à Propriétés propres aux serveurs virtuels.
