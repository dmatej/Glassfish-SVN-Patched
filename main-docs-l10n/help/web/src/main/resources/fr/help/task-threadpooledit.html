Modification d'un pool de threads

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sous le noeud Configuration, sélectionnez le noeud Pools de threads.

    La page Pools de threads apparaît.

  3. Sur la page Pools de threads, cliquez sur le nom du pool de threads à modifier.

    La page Modifier un pool de threads apparaît.

  4. Sur la page Modifier un pool de threads, dans le champ Nom de classe, saisissez le nom qualifié complet de la classe qui implémente le pool de threads.

    La valeur par défaut est com.sun.grizzly.http.StatsThreadPool.

  5. Dans le champ Taille max. de la file d'attente, saisissez le nombre maximal de messages pouvant être mis en file d'attente jusqu'à ce que des threads soient disponibles pour les traiter.

    La valeur -1 signifie que la taille de la file d'attente est illimitée. La valeur par défaut est 4 096.

  6. Dans le champ Taille max. du pool de threads, saisissez le nombre maximal de threads pouvant figurer dans le pool de threads.

    La valeur par défaut est 5. Il s'agit de la valeur recommandée pour les pools de threads de processus d'écoute réseau.

  7. Dans le champ Taille min. du pool de threads, saisissez le nombre minimal de threads pouvant figurer dans le pool de threads.

    Ces threads sont créés lors de l'instanciation du pool de threads. La valeur par défaut est 2.

  8. Dans le champ Expiration de thread inactif, saisissez la durée maximale, en secondes, pendant laquelle un thread peut demeurer inactif dans le pool.

    Une fois ce délai expiré, le thread est enlevé du pool. La valeur par défaut est 900.

  9. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
