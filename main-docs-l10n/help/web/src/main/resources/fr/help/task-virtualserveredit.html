Modification d'un serveur virtuel

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez la configuration à modifier.

  3. Sous le noeud Configuration, sélectionnez le noeud Serveurs virtuels.

    La page Serveurs virtuels apparaît.

  4. Sur la page Serveurs virtuels, cliquez sur le nom du serveur virtuel que vous modifiez.

    La page Modifier le serveur virtuel apparaît.

  5. Dans le champ Hôtes, entrez les noms d'hôte ou les adresses IP de l'ordinateur sur lequel le serveur est exécuté.

    Utilisez des noms d'hôte réels ou virtuels inscrits sur le serveur DNS pour votre réseau (et, sous UNIX, dans le fichier /etc/hosts). La valeur par défaut correspond à la valeur de propriété système ${com.sun.aas.hostName}.

  6. Sélectionnez l'option Etat pour le serveur virtuel.

    Les valeurs possibles sont les suivantes :

    Activé

    Le serveur virtuel est actif. Il s'agit de la valeur par défaut.

    Désactivé

    Le serveur virtuel est inactif. Les tentatives d'accès au serveur renvoient le code d'erreur 404 (ressource non disponible).

    Désactivé

    Le serveur virtuel est désactivé. Les tentatives d'accès au serveur renvoient le code d'erreur 403 (refus d'exécuter la demande).

  7. Sélectionnez une option SSO pour le nouveau serveur virtuel.

    Par défaut, un serveur virtuel hérite du paramètre de connexion unique (SSO) du service HTTP. Sélectionnez Activé ou Désactivé pour annuler le paramètre du service HTTP en activant ou en désactivant la connexion unique pour ce serveur virtuel.

    La connexion unique sur l'ensemble des applications de GlassFish Server est prise en charge par les servlets et les fichiers JSP. Cette fonctionnalité permet à plusieurs applications exigeant les mêmes informations de connexion utilisateur de partager ces dernières au lieu de contraindre l'utilisateur à s'authentifier séparément pour chaque application.

  8. Dans la liste Processus d'écoute réseau, sélectionnez au moins un processus d'écoute réseau ou laissez le champ vide.

  9. Dans la liste déroulante Module Web par défaut, sélectionnez le module devant répondre à toutes les demandes qui ne peuvent pas être mappées avec les autres modules Web déployés sur le serveur virtuel.

    Si aucun module Web par défaut n'est spécifié, le module Web qui possède une racine de contexte vide est utilisé. S'il n'existe aucun module Web avec une racine de contexte vide, un module Web système par défaut est créé et utilisé.

  10. Dans le champ Fichier journal, saisissez le nom de chemin du fichier dans lequel les messages de journalisation provenant de ce serveur virtuel apparaîtront.

    Par défaut, les messages de journalisation sont envoyés au journal de serveur par défaut, domain-dir/logs/server.log.

  11. Dans le champ Racine doc., entrez le chemin absolu du répertoire racine de documents sur le serveur.

    La valeur par défaut est domain-dir/docroot.

  12. Sélectionnez l'option Journalisation des accès pour activer ou désactiver la journalisation des accès.

    Par défaut, un serveur virtuel hérite du paramètre de journalisation des accès du service HTTP. Sélectionnez Activé ou Désactivé pour remplacer le paramètre du service HTTP en activant ou en désactivant la journalisation des accès pour ce serveur virtuel.

  13. Dans le champ Répertoire, entrez le chemin absolu des journaux des accès du serveur.

    La valeur par défaut est domain-dir/logs/access.

  14. Dans la section Autres propriétés, spécifiez les autres propriétés du serveur virtuel.

    Pour plus d'informations sur les propriétés disponibles, reportez-vous à Propriétés propres aux serveurs virtuels.

    • Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

    • Pour modifier une propriété, modifiez le champ Valeur de cette propriété.

    • Pour supprimer une propriété, sélectionnez la case située à gauche du champ Nom de la propriété à supprimer, puis cliquez sur le bouton Supprimer des propriétés.

  15. Cliquez sur Enregistrer.

    Conseil:

    Pour revenir aux paramètres par défaut, cliquez sur le bouton Charger les valeurs par défaut, puis sur Enregistrer.
