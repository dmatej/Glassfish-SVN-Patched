Création d'un serveur virtuel

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez la configuration à modifier.

  3. Sous le noeud Configuration, sélectionnez le noeud Serveurs virtuels.

    La page Serveurs virtuels apparaît.

  4. Sur la page Serveurs virtuels, cliquez sur Nouveau.

    La page Nouveau serveur virtuel apparaît.

  5. Dans le champ ID, entrez une valeur identifiant le serveur virtuel en interne.

    Cette valeur n'est pas visible par les clients HTTP. Les noms d'hôte visibles par les clients HTTP doivent être spécifiés dans le champ Hôtes.

  6. Dans le champ Hôtes, entrez les noms d'hôte ou les adresses IP de l'ordinateur sur lequel le serveur est exécuté.

    Utilisez des noms d'hôte réels ou virtuels inscrits sur le serveur DNS pour votre réseau (et, sous UNIX, dans le fichier /etc/hosts).

    La valeur par défaut correspond à la valeur de propriété système ${com.sun.aas.hostName}.

  7. Sélectionnez l'option Etat pour le nouveau serveur virtuel.

    Le serveur virtuel peut être défini sur Actif, Inactif ou Désactivé. La valeur par défaut est Actif.

  8. Sélectionnez une option SSO pour le nouveau serveur virtuel.

    Par défaut, un serveur virtuel hérite du paramètre de connexion unique (SSO) du service HTTP. Sélectionnez Activé ou Désactivé pour annuler le paramètre du service HTTP en activant ou en désactivant la connexion unique pour ce serveur virtuel.

    La connexion unique sur l'ensemble des applications de GlassFish Server est prise en charge par les servlets et les fichiers JSP. Cette fonctionnalité permet à plusieurs applications exigeant les mêmes informations de connexion utilisateur de partager ces dernières au lieu de contraindre l'utilisateur à s'authentifier séparément pour chaque application.

  9. Dans la liste Processus d'écoute réseau, sélectionnez au moins un processus d'écoute réseau ou laissez le champ vide.

  10. Dans la liste déroulante Module Web par défaut, sélectionnez le module devant répondre à toutes les demandes qui ne peuvent pas être mappées avec les autres modules Web déployés sur le serveur virtuel.

    Si aucun module Web par défaut n'est spécifié, le module Web qui possède une racine de contexte vide est utilisé. S'il n'existe aucun module Web avec une racine de contexte vide, un module Web système par défaut est créé et utilisé.

  11. Dans le champ Fichier journal, saisissez le nom de chemin du fichier dans lequel les messages de journalisation provenant de ce serveur virtuel apparaîtront.

    Par défaut, les messages de journalisation sont envoyés au journal de serveur par défaut, domain-dir/logs/server.log.

  12. Dans le champ Racine doc., entrez le chemin absolu du répertoire racine de documents sur le serveur.

    Si ce champ est vide, domain-dir/docroot est utilisé.

  13. Sélectionnez l'option Journalisation des accès pour activer ou désactiver la journalisation des accès.

    Vous pouvez également indiquer si la journalisation des accès est héritée du service HTTP. Par défaut, il est hérité du service HTTP.

  14. Dans le champ Répertoire, entrez le chemin absolu des journaux des accès du serveur.

    La valeur par défaut est domain-dir/logs/access.

  15. Dans la section Autres propriétés, spécifiez les autres propriétés du serveur virtuel.

    Pour plus d'informations sur les propriétés disponibles, reportez-vous à Propriétés propres aux serveurs virtuels.

    Pour ajouter une propriété, cliquez sur le bouton Ajouter une propriété. Sur la ligne vide qui apparaît, saisissez le nom de la propriété dans le champ Nom et la valeur de la propriété dans le champ Valeur.

  16. Cliquez sur OK.
