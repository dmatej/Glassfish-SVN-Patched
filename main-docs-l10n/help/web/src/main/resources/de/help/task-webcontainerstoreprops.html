So werden Speichereigenschaften für den Webcontainer konfiguriert

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Wählen Sie die zu ändernde Konfiguration.

  3. Wählen Sie unter dem Konfigurationsknoten den Webcontainerknoten.

    Die Seite "Allgemeine Eigenschaften" wird geöffnet.

  4. Klicken Sie auf der Seite "Allgemeine Eigenschaften" auf die Registerkarte "Speichereigenschaften".

    Die Seite "Speichereigenschaften" wird geöffnet.

  5. Geben Sie im Feld "Verzeichnis" den absoluten oder relativen Pfadnamen des Verzeichnisses an, in das einzelne Sessiondateien geschrieben werden.

    Der Standardwert ist domain-dir/generated/jsp/j2ee-apps/appname/appname_war.

    Ein relativer Pfad wird relativ zum temporären Arbeitsverzeichnis für diese Webanwendung angegeben.

    Diese Einstellung findet nur dann Anwendung, wenn der Persistenztyp file lautet. In So werden Verfügbarkeitseinstellungen für den Webcontainer konfiguriert wird beschrieben, wie der Persistenztyp festgelegt wird.

  6. Geben Sie im Abschnitt "Weitere Eigenschaften" weitere Eigenschaften an.

    GlassFish Server definiert keine zusätzlichen Eigenschaften für den Speicher.

    • Um eine Eigenschaft hinzuzufügen, klicken Sie auf die Schaltfläche "Eigenschaft hinzufügen". Geben Sie in die daraufhin angezeigte leere Zeile den Eigenschaftsnamen in das Feld "Name" und den Eigenschaftswert in das Feld "Wert" ein.

    • Um eine Eigenschaft zu ändern, bearbeiten Sie das Feld "Wert" dieser Eigenschaft.

    • Um eine Eigenschaft zu löschen, aktivieren Sie das Kontrollkästchen links neben dem Feld "Name" der zu löschenden Eigenschaft und klicken anschließend auf die Schaltfläche "Eigenschaften löschen".

  7. Klicken Sie auf "Speichern".
