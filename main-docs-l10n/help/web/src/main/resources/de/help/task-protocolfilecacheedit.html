So werden Dateicacheeinstellungen für ein Protokoll bearbeitet

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Blenden Sie unter dem Konfigurationsknoten den Knoten "Netzwerkkonfiguration" ein.

  3. Wählen Sie unter dem Knoten "Netzwerkkonfiguration" den Knoten "Protokolle".

    Die Seite "Protokolle" wird geöffnet.

  4. Klicken Sie auf der Seite "Protokolle" auf den Namen des zu bearbeitenden Protokolls.

    Die Seite "Protokoll bearbeiten" wird geöffnet.

  5. Klicken Sie auf der Seite "Protokoll bearbeiten" auf die Registerkarte "Dateicache".

    Die Seite "Dateicache" wird geöffnet.

  6. Wählen Sie das Kontrollkästchen "Status aktiviert", um den Dateicache zu aktivieren.

    Diese Option ist standardmäßig aktiviert.

  7. Geben Sie im Feld "Maximales Alter" das maximale Alter (in Sekunden) für einen gültigen Cacheeintrag ein.

    Dieser Parameter kontrolliert, wie lange gecachte Informationen verwendet werden, nachdem eine Datei gecacht wurde. Ein Eintrag, der älter ist als das angegebene Maximalalter, wird durch einen neuen Eintrag für dieselbe Datei ersetzt. Wenn sich der verwendete Content nur selten ändert, erhöhen Sie diesen Wert, um die Performance zu verbessern.

    Das optimale maximale Alter hängt davon ab, ob die vorhandenen Dateien regelmäßig geändert werden. Wenn die Dateien beispielsweise vier Mal täglich in regelmäßigen Abständen geändert werden, können Sie das maximale Alter auf 21600 Sekunden (6 Stunden) festlegen. Andernfalls sollten Sie das maximale Alter auf den längsten Zeitraum einstellen, für den die frühere Version einer Inhaltsdatei auf dem Server verbleiben soll, nachdem die Datei geändert wurde.

    Der Standardwert ist 30.

  8. Geben Sie im Feld "Maximale Cachegröße" die Maximalgröße (in Bytes) des Dateicaches auf der Festplatte an.

    Der Standardwert ist 10485760.

  9. Geben Sie im Feld "Maximale Anzahl an Dateien" die maximale Anzahl der Dateien an, die im Dateicache gespeichert werden können.

    Wenn dieser Wert zu hoch gewählt ist, cacht der Server selten verwendete Dateien, wodurch Arbeitsspeicher vergeudet wird. Ist dieser Wert zu niedrig eingestellt, gehen die Vorteile der Cachings verloren. Probieren Sie verschiedene Werte für dieses Attribut aus, um die optimale Einstellung für spezifische Anwendungen zu finden.

    Der Standardwert ist 1024.

  10. Klicken Sie auf "Speichern".

    Tipp:

    Um zu den Standardeinstellungen zurückzukehren, klicken Sie auf die Schaltfläche "Standardwerte laden" und anschließend auf "Speichern".
