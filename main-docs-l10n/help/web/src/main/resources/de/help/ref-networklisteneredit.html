Netzwerk-Listener bearbeiten

Auf der Seite "Netzwerk-Listener bearbeiten" können Sie einen vorhandenen Netzwerk-Listener bearbeiten.

Beim Bearbeiten eines Netzwerk-Listeners stehen die Registerkarten "SSL", "HTTP" und "Dateicache" für das verbundene Protokoll ebenfalls zur Bearbeitung zur Verfügung.

Die Seite "Netzwerk-Listener bearbeiten" enthält die folgenden Optionen.

Standardeinstellungen laden

Schaltfläche zur Wiederherstellung der Standardwerte von Einstellungen, die Standardwerte haben. Einstellungen, die keine Standardwerte haben, werden nicht geändert.

Name der Konfiguration

Der Name der Konfiguration, für die die Einstellungen auf dieser Seite angewendet werden. Dieses Feld ist schreibgeschützt.

Name

Der eindeutige Listener-Name. Das Feld "Name" ist ein schreibgeschütztes Feld. Sie können den Namen nur festlegen, wenn Sie einen neuen Netzwerk-Listener erstellen.

Protokoll

Das mit dem Netzwerk-Listener verbundene Protokoll. Das Feld "Protokoll" ist ein schreibgeschütztes Feld. Sie können nur dann ein Protokoll erstellen oder festlegen, wenn Sie einen neuen Netzwerk-Listener erstellen.

Wählen Sie das vom Netzwerk-Listener verwendete Protokoll, um die Einstellungen zu ändern. Beispiel: Sie können die Sicherheit aktivieren

Status

Der Status des Netzwerk-Listeners. Der Listener kann aktiviert oder deaktiviert werden. Diese Option ist standardmäßig aktiviert. Wenn der Listener deaktiviert ist, führen alle Versuche, eine Verbindung zum Listener herzustellen, zu einer Socket-Ausnahme (java.net.ConnectException).

In Versionen von GlassFish Server vor Application Server 9.1 wurde bei einem Listener, dessen Attribut "Aktiviert" auf "False" gesetzt war, für alle an den Listener gesendeten Anforderungen der Antwortcode 404 zurückgegeben. Um dieses Behavior in der aktuellen Version von GlassFish Server zu erzielen, setzen Sie den Status des Listeners auf "Aktiviert" und den Status aller verbundenen virtuellen Server auf "Deaktiviert". Die mit einem virtuellen Server verbundenen Listener werden in der Dropdown-Liste "Netzwerk-Listener" des Servers aufgeführt.

Sicherheit

Bei aktivierter Option ist für das vom Netzwerk-Listener verwendete Protokoll Sicherheit aktiviert. Das Feld "Sicherheit" ist ein schreibgeschütztes Feld.

Wenn Sie den admin-listener für den Domain Administration Server (DAS) konfigurieren, der die Konfiguration server-config verwendet, ist diese Option schreibgeschützt. Um die Sicherheit für diesen Listener zu aktivieren, müssen Sie die sichere Administration aktivieren. Siehe Sichere Administration.

JK-Listener

Bei aktivierter Option unterstützt der Listener das Apache-Protokoll mod-jk. Das Protokoll mod-jk ist ein proprietäres Kommunikations-/Netzwerkprotokoll zwischen dem Apache-Webserver httpd und einer Backend-Instanz von GlassFish Server. Wenn der Listener dieses Protokoll unterstützt, wird davon ausgegangen, dass httpd den gesamten statischen Content verarbeitet, während alle Anforderungen von dynamischen Ressourcen (d.h. Servlets und JSPs) an den mod-jk-fähigen Listener von GlassFish Server weitergeleitet werden, wobei das Protokoll mod-jk verwendet wird. Diese Option ist standardmäßig deaktiviert.

Port

Die Nummer des Ports, auf dem der Netzwerk-Listener horcht. Gültige Werte sind 1 bis 65535. Bei einem UNIX-System sind zum Erstellen von Sockets, die auf den Ports 1 bis 1024 horchen, Superuser-Berechtigungen erforderlich.

Adresse

Die IP-Adresse, auf der der Netzwerk-Listener horcht. Die Adresse kann als Punktepaar oder in IPv6-Notation angegeben werden. Beim Wert any (für INADDR_ANY) wird auf allen IP-Adressen gehorcht. Auch die Angabe eines Hostnamens ist möglich.

Threadpool

Der mit dem Netzwerk-Listener verbundene Threadpool.

Transport

Der Transporttyp für den Netzwerk-Listener. Standardmäßig steht tcp als einziger Transporttyp zur Auswahl.

Verwandte Aufgaben

  • So wird ein Netzwerk-Listener bearbeitet

  • So wird ein Netzwerk-Listener gelöscht

  • So wird ein Protokoll erstellt

  • So wird ein Protokoll bearbeitet

  • So werden SSL-Einstellungen für ein Protokoll bearbeitet

  • So werden HTTP-Einstellungen für ein Protokoll bearbeitet

  • So werden Dateicacheeinstellungen für ein Protokoll bearbeitet

  • So wird ein Protokoll gelöscht
