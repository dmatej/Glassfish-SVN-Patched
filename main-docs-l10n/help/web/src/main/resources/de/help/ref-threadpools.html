Threadpools

Sie verwenden die Seite "Threadpools" zum Konfigurieren von Threadpools.

Verwenden Sie Threadpools, um einen Service auf eine bestimmte Anzahl gleichzeitiger Threads zu beschränken.

Wenn Sie GlassFish Server erstmalig installieren, stehen standardmäßig zwei Threadpools zur Verfügung. Der Threadpool http-thread-pool ist für die Verwendung durch Netzwerk-Listener konfiguriert, während thread-pool-1 für die Verwendung durch den ORB für RMI/IIOP-Anforderungen konfiguriert ist. Möglicherweise müssen Sie einen Threadpool mit unterschiedlichen Einstellungen für bestimmte Arten von Anwendungen und bestimmte Hardwaresysteme erstellen oder den standardmäßigen Threadpool an Ihre spezifischen Anforderungen anpassen.

Die Java Virtual Machine (JVM) kann gleichzeitig eine Vielzahl von Ausführungsthreads unterstützen. Für eine gute Performance verfügt GlassFish Server über mindestens einen Threadpool. Sie können bestimmten Services bestimmte Threadpools zuweisen.

Ein Threadpool kann für mehrere Services verwendet werden. Anforderungsthreads verarbeiten Benutzeranforderungen. Wenn der Server eine Anforderung empfängt, weist er die Anforderung einem freien Thread im Threadpool zu. Der Thread führt die Clientanforderung aus und gibt Ergebnisse zurück. Beispiel: Wenn die Anforderung eine Systemressource verwenden muss, die gegenwärtig belegt ist, wartet der Thread, bis die Ressource frei ist, bevor er die Verwendung dieser Ressource zulässt.

Geben Sie die Mindestanzahl und die maximale Anzahl von Threads an, die für Anforderungen reserviert sind. Der Threadpool wird gemäß diesen Werten dynamisch angepasst. Die angegebene Mindestgröße des Threadpools weist den Server an, mindestens diese Anzahl von Threads für Anforderungen zu reservieren. Diese Anzahl wird bis zur angegebenen maximalen Threadpoolgröße erhöht.

Durch das Erhöhen der Anzahl von verfügbaren Threads für einen Prozess kann dieser Prozess auf eine höhere Anzahl von Anforderungen gleichzeitig reagieren.

Für jeden Threadpool stehen die folgenden Informationen zur Verfügung.

Threadpool-ID

Der Name des Threadpools.

Max. Threadpoolgröße

Die maximale Anzahl an Threads im Threadpool.

Minimale Threadpoolgröße

Die minimale Anzahl an Threads im Threadpool. Diese Threads werden bei der Instanziierung des Threadpools erstellt.

Max. Queuegröße

Die maximale Anzahl von Meldungen, die in die Queue gestellt werden können, bis Threads für ihre Verarbeitung zur Verfügung stehen.

Threadleerlauftimeout

Die Anzahl Sekunden, nach der inaktive Threads aus dem Pool entfernt werden.

Die Tabelle "Threadpools" enthält darüber hinaus die folgenden Optionen.

Neu

Schaltfläche zum Erstellen eines neuen Threadpools.

Löschen

Schaltfläche zum Löschen von mindestens einem ausgewählten Threadpool.

  • So wird ein Threadpool bearbeitet

  • So wird ein Threadpool gelöscht
