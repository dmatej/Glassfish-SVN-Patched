ネットワーク・リスナーの編集

既存のネットワーク・リスナーを変更するには、「ネットワーク・リスナーの編集」ページを使用します。

ネットワーク・リスナーを編集するときに、関連付けられたプロトコルの「SSL」、「HTTP」、および「ファイル・キャッシュ」タブを使用して編集することもできます。

「ネットワーク・リスナーの編集」ページには、次のオプションが含まれます。

デフォルトのロード

デフォルト値のある設定をデフォルト値に復元するボタン。デフォルト値のない設定は変更されません。

構成名

このページの設定が適用される構成の名前。このフィールドは読取り専用です。

名前

リスナーの一意の名前。「名前」フィールドは読取り専用です。名前を指定できるのは、新規ネットワーク・リスナーの作成時のみです。

プロトコル

ネットワーク・リスナーに関連付けられているプロトコル。「プロトコル」フィールドは読取り専用です。プロトコルを作成または指定できるのは、新規ネットワーク・リスナーの作成時のみです。

ネットワーク・リスナーが使用するプロトコルを選択して、その設定を変更します。たとえば、セキュリティを有効にできます。

ステータス

ネットワーク・リスナーのステータス。リスナーは、有効または無効の状態になります。このオプションはデフォルトで有効になっています。リスナーが無効の場合は、リスナーに接続しようとするとソケット例外(java.net.ConnectException)が発生します。

Application Server 9.1より前のGlassFish Serverでは、有効化された属性がfalseに設定されたリスナーは、送信されたリクエストに対して404レスポンス・コードを返します。現在のバージョンのGlassFish Serverでこの動作を行うには、リスナーのステータスを有効に設定し、関連する各仮想サーバーの状態をオフに設定します。仮想サーバーは、関連付けられたリスナーを「ネットワーク・リスナー」ドロップダウン・リストに一覧表示します。

セキュリティ

このオプションを選択すると、ネットワーク・リスナーが使用するプロトコルのセキュリティが有効になります。「セキュリティ」フィールドは読取り専用です。

server-config構成を使用するドメイン管理サーバー(DAS)に対してadmin-listenerを構成している場合、このオプションは読取り専用です。このリスナーに対してセキュリティを有効にするには、セキュリティ保護された管理を有効にする必要があります。Secure Administrationを参照してください。

JKリスナー

このオプションを選択すると、リスナーはApacheのmod-jkプロトコルをサポートします。mod-jkプロトコルは、Apacheのhttpd(Webサーバー)とGlassFish Serverのバックエンド・インスタンス間で使用される、専用の通信/ネットワークのプロトコルです。リスナーがこのプロトコルをサポートする場合、httpdはすべての静的なコンテンツを処理すると想定され、動的なリソース(サーブレットおよびJSP)のリクエストは、mod-jkに対応するGlassFish Serverのネットワーク・リスナーにmod-jkプロトコルを使用してルーティングされます。このオプションは、デフォルトでは無効になっています。

ポート

リスナーがリスニングするポート番号。適切な値は1から65535です。UNIXシステムで、ポート1から1024でリスニングするソケットを作成するには、スーパーユーザー権限が必要です。

アドレス

ネットワーク・リスナーがリスニングするIPアドレス。アドレスはドット区切りかIPv6の表記法で指定します。すべてのIPアドレスでリスニングする場合は、any(INADDR_ANY)を指定できます。ホスト名を指定できます。

スレッド・プール

ネットワーク・リスナーに関連付けられているスレッド・プール。

トランスポート

ネットワーク・リスナーのトランスポートのタイプ。デフォルトでは、トランスポートはtcpのみです。
