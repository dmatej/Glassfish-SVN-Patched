HTTP 리스너를 편집하는 방법

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. [구성] 노드에서 [HTTP 서비스] 노드를 확장합니다.

  3. [네트워크 구성] 노드에서 [HTTP 리스너] 노드를 선택합니다.

    [HTTP 리스너] 페이지가 열립니다.

  4. [HTTP 리스너] 페이지에서 편집할 HTTP 리스너의 이름을 누릅니다.

    [HTTP 리스너 편집] 페이지가 열립니다.

  5. [포트] 필드에 HTTP 리스너에서 수신할 포트 번호를 입력합니다.

    유효한 값은 1 - 65535입니다. UNIX 시스템에서 1 - 1024 포트를 수신하는 소켓을 생성하려면 수퍼유저 권한이 필요합니다.

  6. HTTP 리스너의 상태를 변경하려면 [상태 사용] 체크 박스를 선택하거나 선택 해제합니다.

    리스너는 사용 또는 사용 안함으로 설정할 수 있습니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다. 리스너가 사용 안함으로 설정된 경우 리스너에 접속하려고 하면 소켓 예외 사항(java.net.ConnectException)이 발생합니다.

  7. HTTP 리스너에 대해 보안을 사용하려면 [보안 사용] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

    DAS(도메인 관리 서버)에 대해 admin-listener를 구성하여 server-config 구성을 사용하는 경우, 이 옵션은 읽기 전용입니다. 이 리스너에 대해 보안을 사용으로 설정하려면 보안 관리를 사용으로 설정해야 합니다. 보안 관리를 참조하십시오.

  8. [JK 리스너 사용] 체크 박스를 선택하여 리스너가 Apache의 mod-jk 프로토콜을 지원하도록 지정합니다.

    mod-jk 프로토콜은 Apache httpd(웹 서버)와 GlassFish Server 백엔드 인스턴스 간 전용 통신/네트워크 프로토콜입니다. 리스너에서 이 프로토콜을 지원하는 경우 httpd가 모든 정적 콘텐츠를 처리하지만, 동적 리소스(즉, 서블릿 및 JSP)에 대한 요청은 mod-jk 프로토콜을 통해 GlassFish Server의 mod-jk 지원 HTTP 리스너로 라우팅됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

  9. [주소] 필드에 네트워크 HTTP 리스너에서 수신할 IP 주소를 입력합니다.

    주소는 점으로 된 쌍이거나 IPv6 표기 형식일 수 있습니다. 모든 IP 주소를 수신하려면 any(INADDR_ANY의 경우)일 수 있습니다. 호스트 이름일 수 있습니다.

  10. [기본 가상 서버] 드롭다운 목록에서 이 HTTP 리스너와 연관될 가상 서버를 선택합니다.

    [가상 서버] 페이지에서는 가상 서버를 정의할 수 있습니다.

  11. [스레드 풀] 드롭다운 목록에서 HTTP 리스너와 연관될 스레드 풀을 선택합니다.

    일반적으로 GlassFish Server를 설치할 때 구성되는 두 개의 스레드 풀 중 하나를 선택합니다.

  12. 서버가 클라이언트에 보내는 URL에 사용할 호스트 이름을 [서버 이름] 필드에 입력합니다.

    서버가 별칭을 사용하는 경우 이 이름은 별칭 이름입니다. 서버가 별칭을 사용하지 않는 경우에는 이 필드를 비워 두십시오.

    이 값은 서버에서 자동으로 생성하는 URL에 영향을 주지만, 서버에 저장된 디렉토리 및 파일의 URL에는 영향을 주지 않습니다. 서버에서 별칭을 사용하는 경우 서버 이름이 별칭 이름이어야 합니다. 콜론과 포트 번호를 추가하면 서버에서 클라이언트에 전송하는 URL에 해당 포트가 사용됩니다.

  13. [저장]을 누릅니다.

    팁:

    기본 설정으로 돌아가려면 [기본값 로드] 단추를 누른 다음 [저장]을 누릅니다.
