가상 서버를 편집하는 방법

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. 변경 중인 구성을 선택합니다.

  3. [구성] 노드에서 [가상 서버] 노드를 선택합니다.

    [가상 서버] 페이지가 열립니다.

  4. [가상 서버] 페이지에서 편집할 가상 서버의 이름을 누릅니다.

    [가상 서버 편집] 페이지가 열립니다.

  5. [호스트] 필드에 서버가 실행되고 있는 시스템의 호스트 이름 또는 IP 주소를 입력합니다.

    네트워크의 DNS 서버(UNIX 시스템의 경우 /etc/hosts 파일)에 등록된 실제 또는 가상 호스트 이름을 사용합니다. 기본값은 시스템 속성값 ${com.sun.aas.hostName}입니다.

  6. 가상 서버에 대한 상태 옵션을 선택합니다.

    다음과 같은 값을 사용할 수 있습니다.

    켜기

    가상 서버가 활성화됩니다. 이 값은 기본값입니다.

    끄기

    가상 서버가 비활성화됩니다. 서버에 액세스하려 하면 오류 코드 404(사용 가능한 리소스 없음)가 반환됩니다.

    사용 안함

    가상 서버가 사용 안함으로 설정됩니다. 서버에 액세스하려 하면 오류 코드 403(요청 수행 거부)이 반환됩니다.

  7. 새 가상 서버에 대한 SSO 옵션을 선택합니다.

    기본적으로 가상 서버는 HTTP 서비스의 SSO(Single Sign-On) 설정을 상속합니다. [사용] 또는 [사용 안함]을 선택하여 이 가상 서버에 대한 Single Sign-On을 사용 또는 사용 안함으로 설정함으로써 HTTP 서비스 설정을 대체합니다.

    GlassFish Server의 응용 프로그램에 대한 Single Sign-On은 서블릿 및 JSP 파일에서 지원됩니다. 이 기능을 통해 사용자가 응용 프로그램마다 따로 사인온하는 것이 아니라 같은 사용자 사인온 정보가 필요한 여러 응용 프로그램이 이 정보를 공유할 수 있습니다.

  8. [네트워크 리스너] 드롭다운 목록에서 하나 이상의 네트워크 리스너를 선택하거나 필드를 비워 둡니다.

  9. [기본 웹 모듈] 드롭다운 목록에서 가상 서버에 배치된 다른 웹 모듈에 매핑할 수 없는 모든 요청에 응답할 모듈을 선택합니다.

    기본 웹 모듈이 지정되지 않은 경우 빈 컨텍스트 루트가 있는 웹 모듈이 사용됩니다. 빈 컨텍스트 루트가 있는 웹 모듈이 없을 때는 시스템 기본 웹 모듈이 생성되어 사용됩니다.

  10. [로그 파일] 필드에 이 가상 서버로부터의 로깅 메시지가 나타날 파일의 경로 이름을 입력합니다.

    기본적으로 로깅 메시지는 기본 서버 로그 domain-dir/logs/server.log로 전송됩니다.

  11. Docroot 필드에 서버의 루트 문서 디렉토리에 대한 절대 경로를 입력합니다.

    기본값은 domain-dir/docroot입니다.

  12. 액세스 로깅을 사용 또는 사용 안함으로 설정하려면 [액세스 로깅] 옵션을 선택합니다.

    기본적으로 가상 서버는 HTTP 서비스의 액세스 로깅 설정을 상속합니다. [사용] 또는 [사용 안함]을 선택하여 이 가상 서버에 대한 액세스 로깅을 사용 또는 사용 안함으로 설정함으로써 HTTP 서비스 설정을 대체합니다.

  13. [디렉토리] 필드에 서버 액세스 로그의 절대 디렉토리 경로를 입력합니다.

    기본값은 domain-dir/logs/access입니다.

  14. [추가 속성] 섹션에 가상 서버에 대한 추가 속성을 지정합니다.

    사용 가능한 속성에 대한 자세한 내용은 가상 서버 관련 속성을 참조하십시오.

    • 속성을 추가하려면 [속성 추가] 단추를 누릅니다. 빈 행이 표시되면 [이름] 필드에 속성 이름을 입력하고 [값] 필드에 속성값을 입력합니다.

    • 속성을 수정하려면 속성의 [값] 필드를 편집합니다.

    • 속성을 삭제하려면 삭제할 속성의 [이름] 필드 왼쪽에 있는 체크 박스를 선택한 다음 [속성 삭제] 단추를 누릅니다.

  15. [저장]을 누릅니다.

    팁:

    기본 설정으로 돌아가려면 [기본값 로드] 단추를 누른 다음 [저장]을 누릅니다.
