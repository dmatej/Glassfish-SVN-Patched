새 네트워크 리스너

[새 네트워크 리스너] 페이지에서는 새 네트워크 리스너를 생성할 수 있습니다.

네트워크 리스너를 생성할 때 동시에 네트워크 리스너에서 사용할 프로토콜을 생성하거나 먼저 생성한 프로토콜을 사용할 수 있습니다. 일반적으로 네트워크 리스너와 관련 프로토콜 사이는 일대일 관계이지만 필수 사항은 아닙니다.

[새 네트워크 리스너] 페이지에는 다음 옵션이 포함되어 있습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

이름

고유한 리스너 이름입니다. 네트워크 리스너 이름은 숫자로 시작할 수 없습니다.

프로토콜

네트워크 리스너와 연관된 프로토콜입니다. 다음 옵션은 프로토콜을 생성할지 또는 기존 프로토콜을 사용할지 여부를 지정합니다.

새 프로토콜 생성

기본 설정을 사용하여 네트워크 리스너에 대한 새 프로토콜을 생성합니다. 프로토콜의 기본 이름은 네트워크 리스너 이름에 -protocol을 추가하여 지정됩니다. 이 값은 편집할 수 있습니다.

가상 서버

새 프로토콜의 경우 프로토콜과 연관될 가상 서버입니다.

기존 프로토콜 사용

선택한 프로토콜이 사용됩니다. 프로토콜에 따라 네트워크 리스너에 대한 보안을 사용으로 설정할지 여부가 결정됩니다.

상태

네트워크 리스너의 상태입니다. 리스너는 사용 또는 사용 안함으로 설정할 수 있습니다. 이 옵션은 기본적으로 사용으로 설정되어 있습니다. 리스너가 사용 안함으로 설정된 경우 리스너에 접속하려고 하면 소켓 예외 사항(java.net.ConnectException)이 발생합니다.

GlassFish Server 9.1 이전 버전에서는 사용으로 설정된 속성이 false로 설정된 리스너는 해당 리스너로 전송된 모든 요청에 대해 404 응답 코드를 반환했습니다. 현재 GlassFish Server 버전에서 이 동작을 수행하려면 리스너의 상태를 사용 가능으로 설정하고 연관된 모든 가상 서버의 상태를 OFF로 설정합니다. 가상 서버의 네트워크 리스너 드롭다운 목록에는 해당 가상 서버와 연관된 리스너가 나열됩니다.

보안

이 옵션을 선택하면 네트워크 리스너에서 사용하는 프로토콜에 대해 보안이 사용으로 설정됩니다. [보안] 필드는 읽기 전용 필드입니다.

DAS(도메인 관리 서버)에 대해 admin-listener를 구성하여 server-config 구성을 사용하는 경우, 이 옵션은 읽기 전용입니다. 이 리스너에 대해 보안을 사용으로 설정하려면 보안 관리를 사용으로 설정해야 합니다. 보안 관리를 참조하십시오.

JK 리스너

이 옵션을 선택하면 리스너에서 Apache의 mod-jk 프로토콜을 지원합니다. mod-jk 프로토콜은 Apache httpd(웹 서버)와 GlassFish Server 백엔드 인스턴스 간 전용 통신/네트워크 프로토콜입니다. 리스너에서 이 프로토콜을 지원하는 경우 httpd에서 모든 정적 내용을 처리하는 것으로 간주되지만, 동적 리소스(즉, 서블릿 및 JSP)에 대한 요청은 mod-jk 프로토콜을 통해 GlassFish Server의 mod-jk 사용 네트워크 리스너에 라우팅됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

포트

리스너에서 수신할 포트 번호입니다. 유효한 값은 1 - 65535입니다. UNIX 시스템에서 1 - 1024 포트를 수신하는 소켓을 생성하려면 수퍼유저 권한이 필요합니다.

주소

네트워크 리스너에서 수신할 IP 주소입니다. 주소는 점으로 된 쌍이거나 IPv6 표기 형식일 수 있습니다. 모든 IP 주소를 수신하려면 any(INADDR_ANY의 경우)일 수 있습니다. 호스트 이름일 수 있습니다.

스레드 풀

네트워크 리스너와 연관된 스레드 풀입니다. 일반적으로 GlassFish Server를 설치할 때 구성되는 두 개의 스레드 풀 중 하나를 선택합니다.

전송

네트워크 리스너의 전송 유형입니다. 기본적으로 유일한 선택 항목은 tcp입니다.
