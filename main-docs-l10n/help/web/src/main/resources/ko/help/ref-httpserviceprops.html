HTTP 서비스 관련 속성

HTTP 서비스에 사용할 수 있는 추가 속성은 다음과 같습니다. 이러한 속성을 지정하면 모든 네트워크 리스너에 적용됩니다.

ssl-cache-entriese

캐시할 SSL 세션의 수입니다. 기본값은 10,000입니다.

ssl3-session-timeout

SSL3 세션이 캐시되는 간격입니다. 기본값은 86,400입니다.

ssl-session-timeout

SSL2 세션이 캐시되는 간격입니다. 기본값은 100입니다.

proxyHandler

SSL 종료 프록시 서버(예: 로드 밸런서)가 가로챈 원래 클라이언트 요청에 대한 정보를 백엔드 GlassFish Server 인스턴스가 검색할 수 있도록 하는 com.sun.appserv.ProxyHandler 추상 클래스의 사용자 정의 구현에 대한 정규화된 클래스 이름입니다. 이 추상 클래스 구현은 지정된 요청에서 사용자 정의 요청 헤더를 검사합니다.사용자 정의 요청 헤더를 통해 프록시 서버는 원래 클라이언트 요청에 대한 정보를 Enterprise Server 인스턴스에 전달하고 해당 정보를 호출자에게 반환합니다.

기본 구현은 Proxy-ip라는 HTTP 요청 헤더에서 클라이언트 IP 주소를, Proxy-keysize라는 HTTP 요청 헤더에서 SSL keysize를, Proxy-auth-cert라는 HTTP 요청 헤더에서 SSL 클라이언트 인증서 체인을 읽습니다. Proxy-auth-cert 값은 BEGIN CERTIFICATE 및 END CERTIFICATE 경계가 없고 \n이 % d% a로 바뀌는 BASE-64 인코딩 클라이언트 인증서 체인을 포함해야 합니다.

이 설정은 네트워크 리스너 프로토콜의 HTTP 페이지에 대한 [인증 통과] 체크 박스가 true로 설정된 경우에만 사용됩니다.

connectionTimeout

네트워크 리스너가 접속을 승인한 후 요청 URI 행이 제출될 때까지 기다리는 시간(초)을 지정합니다. 기본값은 30입니다.
