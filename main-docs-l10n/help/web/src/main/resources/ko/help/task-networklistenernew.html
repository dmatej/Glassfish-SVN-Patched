네트워크 리스너를 생성하는 방법

시작하기 전에

네트워크 리스너를 생성할 때 동시에 네트워크 리스너에서 사용할 프로토콜을 생성하거나 먼저 생성한 프로토콜을 사용할 수 있습니다. 일반적으로 네트워크 리스너와 관련 프로토콜 사이는 일대일 관계이지만 필수 사항은 아닙니다.

  1. 탐색 트리에서 [구성] 노드를 확장합니다.

  2. [구성] 노드에서 [네트워크 구성] 노드를 확장합니다.

  3. [네트워크 구성] 노드에서 [네트워크 리스너] 노드를 선택합니다.

    [네트워크 리스너] 페이지가 열립니다.

  4. [네트워크 리스너] 페이지에서 [새로 만들기]를 누릅니다.

    [새 네트워크 리스너] 페이지가 열립니다.

  5. [이름] 필드에 네트워크 리스너의 이름을 입력합니다.

    네트워크 리스너 이름은 숫자로 시작할 수 없습니다.

  6. [프로토콜] 필드에서 네트워크 리스너와 연결할 프로토콜을 지정합니다.

    • [새 프로토콜 생성]을 선택하여 기본 설정으로 새 프로토콜을 생성할 수 있습니다. 또한 [가상 서버] 드롭다운 목록에서 프로토콜과 연관될 가상 서버를 선택합니다.

      프로토콜의 기본 이름은 네트워크 리스너 이름에 -protocol을 추가하여 지정됩니다. 이 값은 편집할 수 있습니다.

    • [기존 프로토콜 사용]을 선택한 다음 드롭다운 목록에서 프로토콜을 선택합니다. 네트워크 리스너에 대해 보안을 사용으로 설정하려면 보안이 사용으로 설정된 프로토콜을 선택합니다.

      이 옵션을 선택하는 경우 일반적으로 이 리스너에 대한 프로토콜을 이미 생성했습니다.

    네트워크 리스너를 생성할 때만 프로토콜을 지정할 수 있습니다. 연관된 프로토콜은 나중에 변경할 수 없습니다. 프로토콜에 따라 네트워크 리스너에 대한 보안을 사용으로 설정할지 여부가 결정됩니다.

  7. 네트워크 리스너를 사용으로 설정하려면 [상태 사용] 체크 박스를 선택합니다.

    이 옵션은 기본적으로 사용으로 설정되어 있습니다. 리스너가 사용 안함으로 설정된 경우 리스너에 접속하려고 하면 소켓 예외 사항(java.net.ConnectException)이 발생합니다.

    GlassFish Server 9.1 이전 버전에서는 사용으로 설정된 속성이 false로 설정된 리스너는 해당 리스너로 전송된 모든 요청에 대해 404 응답 코드를 반환했습니다. 현재 GlassFish Server 버전에서 이 동작을 수행하려면 리스너의 상태를 사용 가능으로 설정하고 연관된 모든 가상 서버의 상태를 OFF로 설정합니다. 가상 서버의 네트워크 리스너 드롭다운 목록에는 해당 가상 서버와 연관된 리스너가 나열됩니다.

  8. 읽기 전용 [보안 사용] 체크 박스는 네트워크 리스너에서 사용하는 프로토콜에 대해 보안이 사용으로 설정되었는지 여부를 나타냅니다.

    DAS(도메인 관리 서버)에 대해 admin-listener를 구성하여 server-config 구성을 사용하는 경우, 이 옵션은 읽기 전용입니다. 이 리스너에 대해 보안을 사용으로 설정하려면 보안 관리를 사용으로 설정해야 합니다. 보안 관리를 참조하십시오.

  9. [JK 리스너 사용] 체크 박스를 선택하여 리스너가 Apache의 mod-jk 프로토콜을 지원하도록 지정합니다.

    mod-jk 프로토콜은 Apache httpd(웹 서버)와 GlassFish Server 백엔드 인스턴스 간 전용 통신/네트워크 프로토콜입니다. 리스너에서 이 프로토콜을 지원하는 경우 httpd에서 모든 정적 내용을 처리하는 것으로 간주되지만, 동적 리소스(즉, 서블릿 및 JSP)에 대한 요청은 mod-jk 프로토콜을 통해 GlassFish Server의 mod-jk 사용 네트워크 리스너에 라우팅됩니다. 이 옵션은 기본적으로 사용 안함으로 설정되어 있습니다.

  10. [포트] 필드에 네트워크 리스너에서 수신할 포트 번호를 입력합니다.

    유효한 값은 1 - 65535입니다. UNIX 시스템에서 1 - 1024 포트를 수신하는 소켓을 생성하려면 수퍼유저 권한이 필요합니다.

  11. [주소] 필드에 네트워크 리스너에서 수신할 IP 주소를 입력합니다.

    주소는 점으로 된 쌍이거나 IPv6 표기 형식일 수 있습니다. 모든 IP 주소를 수신하려면 any(INADDR_ANY의 경우)일 수 있습니다. 호스트 이름일 수 있습니다.

  12. [스레드 풀] 드롭다운 목록에서 네트워크 리스너와 연관될 스레드 풀을 선택합니다.

    일반적으로 GlassFish Server를 설치할 때 구성되는 두 개의 스레드 풀 중 하나를 선택합니다.

  13. [전송] 드롭다운 목록에서 네트워크 리스너에 사용할 전송 유형을 선택합니다.

    기본적으로 유일한 선택 항목은 tcp입니다.

  14. [확인]을 누릅니다.

다음 단계
