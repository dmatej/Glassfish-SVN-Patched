프로토콜 편집

[프로토콜 편집] 페이지에서는 기존 네트워크 프로토콜의 설정을 수정할 수 있습니다.

[프로토콜 편집] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값을 사용하는 설정을 기본값으로 복원하는 단추입니다. 기본값을 사용하지 않는 설정은 변경되지 않습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

이름

프로토콜의 이름입니다. [이름] 필드는 읽기 전용 필드입니다. 새 프로토콜을 생성할 때만 이름을 지정할 수 있습니다.

보안

이 옵션을 선택하면 프로토콜에 대해 보안이 사용으로 설정됩니다. 보안이 사용으로 설정되면 SSL 탭을 사용하여 보안 설정을 수정할 수 있습니다.

DAS(도메인 관리 서버)에 대해 admin-listener를 구성하여 server-config 구성을 사용하는 경우, 이 옵션은 읽기 전용입니다. 이 프로토콜에 대해 보안을 사용으로 설정하려면 보안 관리를 사용으로 설정해야 합니다. 보안 관리를 참조하십시오.

관련 작업

  • 프로토콜을 생성하는 방법

  • 프로토콜을 편집하는 방법

  • 프로토콜의 SSL 설정을 편집하는 방법

  • 프로토콜의 HTTP 설정을 편집하는 방법

  • 프로토콜의 파일 캐시 설정을 편집하는 방법

  • 프로토콜을 삭제하는 방법

관련 asadmin 명령

  • create-protocol(1)

  • list-protocols(1)

  • delete-protocol(1)
