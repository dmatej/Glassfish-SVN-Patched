전송

[전송] 페이지에서는 전송을 구성할 수 있습니다. 각 네트워크 리스너는 전송과 연관됩니다.

GlassFish Server를 처음 설치하면 기본적으로 이름이 tcp인 TCP 전송이 하나 있습니다. TCP 또는 UDP 전송을 수정하거나 생성할 수 있으며, 사용자 정의 전송 구현을 제공할 수도 있습니다.

각 전송에 대해 제공되는 정보는 다음과 같습니다.

전송 이름

전송의 이름입니다.

클래스 이름

전송 구현의 클래스 이름입니다.

바이트 버퍼 유형

전송과 함께 사용할 ByteBuffer의 유형입니다.

전송 테이블에는 다음 옵션도 포함되어 있습니다.

새로 만들기

새 전송을 생성하는 단추입니다.

삭제

선택한 전송을 하나 이상 삭제하는 단추입니다.

관련 작업

  • 전송을 생성하는 방법

  • 전송을 편집하는 방법

  • 전송을 삭제하는 방법

관련 asadmin 명령

  • create-transport(1)

  • list-transports(1)

  • delete-transport(1)
