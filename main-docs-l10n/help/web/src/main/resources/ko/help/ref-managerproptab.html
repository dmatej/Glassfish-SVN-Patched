관리자 속성

세션 관리자는 세션을 생성하고 삭제하는 방법, 세션 상태 저장 위치 및 사용 가능한 최대 세션 수를 구성할 수 있는 방법을 제공합니다.

[관리자 속성] 페이지에서는 웹 컨테이너의 세션 관리자 속성을 추가하거나 수정할 수 있습니다.

[관리자 속성] 페이지에는 다음 옵션이 포함되어 있습니다.

기본값 로드

기본값을 사용하는 설정을 기본값으로 복원하는 단추입니다. 기본값을 사용하지 않는 설정은 변경되지 않습니다.

구성 이름

이 페이지의 설정이 적용되는 구성의 이름입니다. 이 필드는 읽기 전용입니다.

리프 간격

비활성 세션 데이터가 저장소에서 삭제되기까지의 시간(초)입니다. 기본값은 60입니다.

이 값을 세션 데이터가 변경되는 빈도보다 낮게 설정합니다. 예를 들어 자주 액세스하는 웹 사이트의 적중 횟수 서블릿에 대해서는 이 값을 가능한 한 낮게(1초) 설정해야 합니다.그렇지 않으면 서버를 재시작할 때마다 마지막 몇 개의 적중 항목이 손실될 수 있습니다.

최대 세션

캐시에 있을 수 있는 최대 세션 수입니다. 값이 –1이면 세션 수에 제한이 없습니다. 기본값은 -1입니다.

제한에 도달한 후에 새 세션을 생성하려고 시도하면 IllegalStateException이 발생합니다.

세션 파일 이름

상태를 보존할 수 있는 경우 응용 프로그램이 재시작되는 사이에 세션 상태가 보존되는 디렉토리의 절대 또는 상대 경로입니다. 이 경로를 설정하지 않으면 세션 상태가 보존되지 않습니다. 기본적으로 이 경로는 설정되지 않습니다.

상대 경로는 이 웹 응용 프로그램의 임시 디렉토리에 상대적입니다.

세션 ID 생성기 클래스 이름

고유한 세션 ID를 생성하기 위한 사용자 정의 클래스입니다. 서버 인스턴스당 하나의 세션 ID 생성기 클래스만 허용되며, 클러스터의 모든 인스턴스는 세션 키 충돌이 발생하지 않도록 동일한 세션 ID 생성기를 사용해야 합니다. 이 클래스는 애플리케이션 서버의 클래스 경로에 있어야 합니다. 기본적으로 사용자 정의 클래스는 설정되지 않습니다.

추가 속성

세션 관리자에 대한 추가 속성입니다. GlassFish Server에서는 세션 관리자에 대한 추가 속성을 정의하지 않습니다.

  • 웹 컨테이너의 관리자 속성을 구성하는 방법

  • 웹 컨테이너의 저장소 속성을 구성하는 방법

  • 웹 컨테이너 가용성 설정을 구성하는 방법
