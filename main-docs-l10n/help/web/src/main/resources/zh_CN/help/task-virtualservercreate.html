创建虚拟服务器

  1. 在导航树中，展开“配置”节点。

  2. 选择您要更改的配置。

  3. 在“配置”节点下，选择“虚拟服务器”节点。

    此时将打开“虚拟服务器”页。

  4. 在“虚拟服务器”页上，单击“新建”。

    此时将打开“新建虚拟服务器”页。

  5. 在“ID”字段中，键入将在内部标识虚拟服务器的值。

    此值不会向 HTTP 客户机公开。必须在“主机”字段中指定向 HTTP 客户机公开的主机名。

  6. 在“主机”字段中，键入运行服务器的计算机的主机名或 IP 地址。

    请使用已向网络的 DNS 服务器注册 (而且在 UNIX 系统上，位于 /etc/hosts 文件中) 的实际主机名或虚拟主机名。

    默认值为系统属性值 ${com.sun.aas.hostName}。

  7. 为新的虚拟服务器选择“状态”选项。

    虚拟服务器可以处于“打开”(即活动)、“关闭”或“已禁用”状态。默认值为“打开”。

  8. 为新的虚拟服务器选择 SSO 选项。

    默认情况下，虚拟服务器继承 HTTP 服务的单点登录 (SSO) 设置。选择“已启用”或“已禁用”，以启用或禁用此虚拟服务器的单点登录，从而覆盖 HTTP 服务设置。

    Servlet 和 JSP 文件支持在 GlassFish Server 上跨应用程序的单点登录。此功能允许需要同一用户登录信息的多个应用程序共享登录信息，用户不必在使用每个应用程序时都分别进行登录。

  9. 从“网络监听程序”下拉列表中，选择一个或多个网络监听程序，或将该字段留空。

  10. 从“默认 Web 模块”下拉列表中，选择一个模块，此模块将响应所有无法映射到已部署到虚拟服务器中的其他 Web 模块的请求。

    如果未指定默认 Web 模块，系统将使用具有空上下文根目录的 Web 模块。如果不存在包含空上下文根目录的 Web 模块，系统将创建并使用默认 Web 模块。

  11. 在“日志文件”字段中，键入来自此虚拟服务器的日志记录消息所在的文件的路径名。

    默认情况下，日志记录消息将发送至默认服务器日志 domain-dir/logs/server.log。

  12. 在“文档根目录”字段中，键入服务器的文档根目录的绝对路径。

    如果将此字段保留为空，将使用 domain-dir/docroot。

  13. 选择“访问日志记录”选项，以启用或禁用访问日志记录。

    此外，您可以指定是否从 HTTP 服务继承访问日志记录。默认情况下，会从 HTTP 服务继承。

  14. 在“目录”字段中，键入服务器访问日志的绝对目录路径。

    默认值为 domain-dir/logs/access。

  15. 在“其他属性”部分中，指定虚拟服务器的其他属性。

    有关可用属性的信息，请参阅特定于虚拟服务器的属性。

    要添加属性，请单击“添加属性”按钮。在显示的空白行中，在“名称”字段中键入属性名称，在“值”字段中键入属性值。

  16. 单击“确定”。
