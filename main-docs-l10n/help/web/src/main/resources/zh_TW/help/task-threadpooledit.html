編輯繫線集區

  1. 在瀏覽樹狀結構中，展開「組態」節點。

  2. 在「組態」節點下，選取「繫線集區」節點。

    「繫線集區」頁面會隨即開啟。

  3. 在「繫線集區」頁面上，按一下要編輯的繫線集區名稱。

    「編輯繫線集區」頁面會隨即開啟。

  4. 在「編輯繫線集區」頁面的「類別名稱」欄位中，鍵入實行繫線集區之類別的完全合格名稱。

    預設值為 com.sun.grizzly.http.StatsThreadPool。

  5. 在「佇列最大大小」欄位中，鍵入在有繫線可處理訊息之前，可排入佇列的最大訊息數目。

    -1 值表示佇列大小沒有限制。預設值為 4096。

  6. 在「最大繫線集區大小」欄位中，鍵入繫線集區中可存在的最大繫線數目。

    預設值為 5。此為網路監聽器繫線集區的建議值。

  7. 在「最小繫線集區大小」欄位中，鍵入繫線集區中可存在的最小繫線數目。

    這些繫線是在實例化繫線集區時建立的。預設值為 2。

  8. 在「閒置繫線逾時」欄位中，鍵入繫線可以在集區中閒置的最長時間 (以秒為單位)。

    一旦超過此時間，即從集區中移除該繫線。預設值為 900。

  9. 按一下「儲存」。

    提示:

    若要回到預設設定，請按一下「載入預設值」按鈕，然後按一下「儲存」。
