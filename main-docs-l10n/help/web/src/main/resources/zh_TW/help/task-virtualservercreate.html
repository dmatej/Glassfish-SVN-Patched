建立虛擬伺服器

  1. 在瀏覽樹狀結構中，展開「組態」節點。

  2. 選取您要變更的組態。

  3. 在「組態」節點下，選取「虛擬伺服器」節點。

    「虛擬伺服器」頁面會隨即開啟。

  4. 在「虛擬伺服器」頁面上，按一下「新建」。

    「新建虛擬伺服器」頁面會隨即開啟。

  5. 在「ID」欄位中，鍵入內部識別虛擬伺服器的值。

    HTTP 從屬端無法取得此值。提供給 HTTP 從屬端的主機名稱必須在「主機」欄位中指定。

  6. 在「主機」欄位中，鍵入執行伺服器的機器主機名稱或 IP 位址。

    使用已在網路之 DNS 伺服器註冊 (對於 UNIX 系統，則是已在 /etc/hosts 檔案中註冊) 的實際主機名稱或虛擬主機名稱。

    預設值為系統特性值 ${com.sun.aas.hostName}。

  7. 選取新虛擬伺服器的「狀態」選項。

    虛擬伺服器可以設為「開啟」(亦即使用中)、「關閉」或「停用」。預設值為「開啟」。

  8. 選取新虛擬伺服器的「SSO」選項。

    依預設，虛擬伺服器會繼承 HTTP 服務的 Single Sign-On (SSO) 設定。選取「已啟用」或「已停用」，透過啟用或停用此虛擬伺服器的 Single Sign-On 以置換 HTTP 服務設定。

    Servlet 和 JSP 檔案支援在 GlassFish Server 上跨應用程式的 Single Sign-On。此功能允許需要相同使用者登入資訊的多個應用程式共用該資訊，使用者不必在使用每個應用程式時分別進行登入。

  9. 從「網路監聽器」下拉式清單中選取一或多個網路監聽器，或保持欄位空白。

  10. 從「預設 Web 模組」下拉式清單中選取模組，該模組將會回應所有無法對應至其他 Web 模組 (已建置到虛擬伺服器) 的請求。

    如果未指定預設 Web 模組，則使用具有空環境根目錄的 Web 模組。如果沒有環境根目錄為空的 Web 模組，則會建立和使用系統預設的 Web 模組。

  11. 在「日誌檔」欄位中，鍵入將顯示該虛擬伺服器日誌訊息之檔案的路徑名稱。

    依預設，記錄日誌訊息將會傳送至預設伺服器日誌 domain-dir/logs/server.log。

  12. 在「文件根」欄位中，鍵入伺服器根文件目錄的絕對路徑。

    如果此欄位保留空白，將會使用 domain-dir/docroot。

  13. 選取「存取記錄日誌」選項，可啟用或停用存取記錄日誌。

    或者，您也可以指定是否從 HTTP 服務繼承存取記錄日誌功能。依預設，會繼承自 HTTP 服務。

  14. 在「目錄」欄位中，鍵入伺服器存取日誌的絕對目錄路徑。

    預設值為 domain-dir/logs/access。

  15. 在「其他特性」區段中，指定虛擬伺服器的其他特性。

    如需可用特性的相關資訊，請參閱虛擬伺服器特定的特性。

    若要增加特性，請按一下「增加特性」按鈕。在出現的空白列中，於「名稱」欄位中鍵入特性名稱，然後在「值」欄位中鍵入特性值。

  16. 按一下「確定」。
