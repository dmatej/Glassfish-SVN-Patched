建立繫線集區

繫線集區是供網路監聽器和 ORB 使用。一般而言，網路監聽器會使用您在安裝 GlassFish Server 時建立的兩個繫線集區之一，而 ORB 會使用另一個。您可以編輯上述任何預設的繫線集區，指定適合您的應用程式或硬體系統的繫線集區，也可以建立並使用具有這些設定的繫線集區。

  1. 在瀏覽樹狀結構中，展開「組態」節點。

  2. 在「組態」節點下，選取「繫線集區」節點。

    「繫線集區」頁面會隨即開啟。

  3. 在「繫線集區」頁面上，按一下「新建」。

    「新建繫線集區」頁面會隨即開啟。

  4. 在「名稱」欄位中，鍵入繫線集區的名稱。

  5. 在「類別名稱」欄位中，鍵入實行繫線集區之類別的完全合格名稱。

    預設值為 com.sun.grizzly.http.StatsThreadPool。

  6. 在「佇列最大大小」欄位中，鍵入在有繫線可處理訊息之前，可排入佇列的最大訊息數目。

    -1 值表示佇列大小沒有限制。預設值為 4096。

  7. 在「最大繫線集區大小」欄位中，鍵入繫線集區中可存在的最大繫線數目。

    預設值為 5。此為網路監聽器繫線集區的建議值。

  8. 在「最小繫線集區大小」欄位中，鍵入繫線集區中可存在的最小繫線數目。

    這些繫線是在實例化繫線集區時建立的。預設值為 2。

  9. 在「閒置繫線逾時」欄位中，鍵入繫線可以在集區中閒置的最長時間 (以秒為單位)。

    一旦超過此時間，即從集區中移除該繫線。預設值為 900。

  10. 按一下「確定」。
