建立網路監聽程式

開始前

當您建立網路監聽器時，您可以同步建立供網路監聽器使用的協定，或使用最初建立的協定。網路監聽器及其相關聯的協定間通常存在一對一的關係，但並不是必須的。

  1. 在瀏覽樹狀結構中，展開「組態」節點。

  2. 在「組態」節點下，展開「網路組態」節點。

  3. 在「網路組態」節點下，選取「網路監聽器」節點。

    「網路監聽器」頁面會隨即開啟。

  4. 在「網路監聽器」頁面上，按一下「新建」。

    「新建網路監聽器」頁面會隨即開啟。

  5. 在「名稱」欄位中，鍵入網路監聽器的名稱。

    網路監聽器名稱不能以數字為開頭。

  6. 在「協定」欄位中，指定要與網路監聽器關聯的協定。

    • 選取「建立新的協定」，以預設設定建立新的協定。此外，從「虛擬伺服器」下拉式清單中，選取要與此協定關聯的虛擬伺服器。

      協定的預設名稱為網路監聽器名稱加上 -protocol，但您可以編輯此值。

    • 選取「使用現有協定」，然後從下拉式清單中選取協定。若要啟用網路監聽器的安全性，請選擇已啟用安全性的協定。

      如果您選取了此選項，通常會預先為此監聽器建立協定。

    您僅能在建立網路監聽器時指定協定。之後您將無法變更關聯的協定。協定會決定是否為網路監聽器啟用安全性。

  7. 選取「狀態」的「已啟用」核取方塊，以啟用網路監聽器。

    此選項預設為啟用。如果停用監聽器，嘗試連線至監聽器時皆會發生通訊端異常 (java.net.ConnectException)。

    在 GlassFish Server 9.1 之前的版本中，若將監聽器已啟用的屬性設為 False，監聽器會針對任何傳送給它的請求傳回 404 回應碼。若要在目前的 GlassFish Server 版本中實現此運作方式，請將監聽器的狀態設為已啟用，並將每個關聯的虛擬伺服器狀態設為關閉。虛擬伺服器會在其「網路監聽器」下拉式清單中列出其關聯的監聽器。

  8. 唯讀的「啟用安全性」核取方塊會指示是否已為網路監聽器使用的協定啟用安全性。

    如果您正在組態使用 server-config 組態的網域管理伺服器 (DAS) admin-listener，此選項會是唯讀的。如果要為此監聽器啟用安全性，您必須啟用安全管理。請參閱 Secure Administration。

  9. 選取「JK 監聽器」的「已啟用」核取方塊，指定監聽器支援 Apache 的 mod-jk 協定。

    mod-jk 協定是 Apache 的 httpd (Web 伺服器) 和 GlassFish Server 後端執行處理之間的專用通訊/網路協定。如果監聽器支援此協定，httpd 應會處理所有靜態內容，而任何對動態資源 (亦即 Servlet 和 JSP) 的請求都會使用 mod-jk 協定路由至 GlassFish Server 已啟用 mod-jk 的網路監聽器。此選項預設為停用。

  10. 在「連接埠」欄位中，鍵入網路監聽器將監聽的連接埠號碼。

    合法值為 1 - 65535。在 UNIX 系統上，建立監聽連接埠 1 - 1024 的通訊端需要超級使用者權限。

  11. 在「位址」欄位中，鍵入網路監聽器將監聽的 IP 位址。

    此位址可以用「點對」或 IPv6 表示法。它可以表示為 any (用於 INADDR_ANY) 以監聽所有 IP 位址。它可以是主機名稱。

  12. 從「繫線集區」下拉式清單中，選取要與網路監聽器關聯的繫線集區。

    一般而言，您需要在安裝 GlassFish Server 時所組態的兩個繫線集區中選擇其一。

  13. 從「傳輸」下拉式清單中，選取網路監聽器所要使用的傳輸類型。

    依預設，唯一的選擇是 tcp。

  14. 按一下「確定」。

接下來的步驟

如果您針對網路監聽器使用的協定啟用了安全性，便可以使用「SSL」標籤來編輯協定的 SSL 設定。
