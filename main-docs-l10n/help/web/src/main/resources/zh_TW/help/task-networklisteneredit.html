編輯網路監聽程式

  1. 在瀏覽樹狀結構中，展開「組態」節點。

  2. 在「組態」節點下，展開「網路組態」節點。

  3. 在「網路組態」節點下，選取「網路監聽器」節點。

    「網路監聽器」頁面會隨即開啟。

  4. 在「網路監聽器」頁面上，按一下您要編輯的網路監聽器名稱。

    「編輯網路監聽器」頁面會隨即開啟。

  5. 選取網路監聽器使用的協定以變更其設定。例如，您可以啟用安全性。但您不能選擇其他協定。

  6. 選取或取消選取「狀態」的「已啟用」核取方塊，以變更網路監聽器的狀態。

    您可啟用或停用此監聽器。此選項預設為啟用。如果停用監聽器，嘗試連線至監聽器時皆會發生通訊端異常 (java.net.ConnectException)。

  7. 唯讀的「啟用安全性」核取方塊會指示是否已為網路監聽器使用的協定啟用安全性。

    如果您正在組態使用 server-config 組態的網域管理伺服器 (DAS) admin-listener，此選項會是唯讀的。如果要為此監聽器啟用安全性，您必須啟用安全管理。請參閱 Secure Administration。

  8. 選取「JK 監聽器」的「已啟用」核取方塊，指定監聽器支援 Apache 的 mod-jk 協定。

    mod-jk 協定是 Apache 的 httpd (Web 伺服器) 和 GlassFish Server 後端執行處理之間的專用通訊/網路協定。如果監聽器支援此協定，httpd 應會處理所有靜態內容，而任何對動態資源 (亦即 Servlet 和 JSP) 的請求都會使用 mod-jk 協定路由至 GlassFish Server 已啟用 mod-jk 的網路監聽器。此選項預設為停用。

  9. 在「連接埠」欄位中，鍵入網路監聽器將監聽的連接埠號碼。

    合法值為 1 - 65535。在 UNIX 系統上，建立監聽連接埠 1 - 1024 的通訊端需要超級使用者權限。

  10. 在「位址」欄位中，鍵入網路監聽器將監聽的 IP 位址。

    此位址可以用「點對」或 IPv6 表示法。它可以表示為 any (用於 INADDR_ANY) 以監聽所有 IP 位址。它可以是主機名稱。

  11. 從「繫線集區」下拉式清單中，選取要與網路監聽器關聯的繫線集區。

  12. 按一下「儲存」。

    提示:

    若要回到預設設定，請按一下「載入預設值」按鈕，然後按一下「儲存」。
接下來的步驟

您可以使用「SSL」、「HTTP」和「檔案快取」標籤編輯與網路監聽器關聯之協定的設定。「SSL」標籤只有在您已針對協定啟用安全性時才有意義。
