Para configurar el servicio HTTP

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Seleccione la configuración que desea cambiar.

  3. En el nodo Configuración, seleccione el nodo Servicio HTTP.

    Se abrirá la página Servicio HTTP.

  4. Active la casilla de control de activación de SSO para activar por defecto el single sign-on para todas las aplicaciones web en todos los servidores virtuales de este servidor que se hayan configurado para el mismo dominio.

    Puede sustituir esta configuración para un servidor virtual individual. Esta opción se activada por defecto.

  5. Active la casilla de control de activación de registro de acceso para activar el registro de acceso de todos los subelementos de servidores virtuales que no tengan especificada esta propiedad.

    Si desactiva la casilla de control de activación de registro de acceso, se desactivará el registro de acceso de todos los subelementos de servidores virtuales que no tengan especificada esta propiedad. Esta opción se activada por defecto.

  6. Active la casilla de control de activación de rotación para activar la rotación del archivo log.

    Esta opción se encuentra activada por defecto.

  7. En la lista desplegable Política de Rotación, seleccione una política de rotación.

    La única política de rotación disponible es time.

  8. En el campo Intervalo de Rotación, introduzca el número de minutos entre las rotaciones del log de acceso.

    Este campo sólo es válido si la política de rotación es time. El valor por defecto es 1.440 minutos (24 horas).

  9. En el campo Sufijo de Rotación, introduzca el valor de la cadena del sufijo que se va a agregar al final del nombre del archivo log tras la rotación.

    El valor por defecto es aaaa-MM-dd.

    Asegúrese de que el sufijo de rotación contenga valores suficientes para crear un sufijo único para cada rotación del log. Por ejemplo, si el log de acceso rota más de una vez al día, se incluye la hora del día en el sufijo de rotación.

    Es necesario un sufijo único para asegurar que el archivo log de acceso rota cada vez que transcurre el intervalo de rotación. Si el sufijo de rotación del nuevo archivo log de acceso es igual que el sufijo de rotación del archivo log de acceso existente, el archivo no se rota.

  10. En el campo Recuento Máximo de Archivos, escriba el número máximo de archivos log de acceso rotados que se deben guardar.

    Un valor negativo indica que no hay límite en el recuento de archivos. El valor por defecto es -1.

  11. En el campo Tamaño de Buffer, introduzca el tamaño en bytes del buffer del log de acceso.

    Un valor inferior o igual a cero desactiva el almacenamiento en el buffer del log de acceso. El valor por defecto es 32768.

  12. En el campo Intervalo de Escritura, introduzca el número de segundos que deben transcurrir para que el log se escriba en el disco.

    El log de acceso se escribe cuando el buffer está lleno o cuando caduca el intervalo. Si el valor es menor o igual a cero, el buffer siempre se escribe aunque no esté lleno. Esto significa que cada vez que se accede al servidor, el mensaje de log se almacena directamente en el archivo. El valor por defecto es 300.

  13. En el campo Formato, introduzca el valor de la cadena para especificar las propiedades siguientes del log de acceso:

    • Información que se capturará en el log de acceso

    • Orden en el que se capturará la información

    El valor por defecto es el siguiente:

    %client.name% %auth-user-name% %datetime% %request% %status% %response.length%

    Los valores siguientes están disponibles para su inclusión en la cadena de formato:

    %attribute.nombre%

    Registra el valor (mediante una llamada a su método toString) del atributo de la solicitud con el nombre indicado

    %auth-user-name%

    Nombre de usuario autorizado

    %client.name%

    Nombre de host del cliente

    %client.dns%

    DNS de cliente

    %cookie.nombre%

    El valor de la cookie de la solicitud con el nombre indicado

    %cookie.value%

    Valor de la primera cookie que se encuentra en la solicitud

    %datetime%

    Fecha del sistema

    %request%

    Línea de solicitud HTTP completa

    %status%

    Estado

    %response.length%

    Longitud del contenido de respuesta

    %header.referer%

    Cabecera del referencia

    %header.user-agent%

    Cabecera del agente del usuario

    %http-method%

    Método HTTP

    %http-uri%

    URI de HTTP

    %query-str%

    Cadena de consulta HTTP

    %http-version%

    Versión de protocolo HTTP

    %header.accept%

    Cabecera de aceptación

    %header.date%

    Cabecera de fecha

    %header.if-mod-since%

    Cabecera de fecha de modificación

    %header.auth%

    Cabecera de autorización

    %header.cualquiera%

    Cualquier valor de cabecera HTTP válido definido en RFC 2616 (cualquiera también es un valor de cabecera válido; se especifica como variable aquí)

    %session.nombre%

    El valor del atributo de la sesión con el nombre nombre o NULL-SESSION-ATTRIBUTE- si el atributo indicado no existe en la sesión, o NULL-SESSION si no existe ninguna sesión

    %time-taken%

    Tiempo (en milisegundos) que se tardó en prestar servicio a cada solicitud

    %vs.id%

    ID de servidor virtual

  14. En la sección Propiedades Adicionales, especifique las propiedades adicionales.

    Para obtener una descripción de las propiedades disponibles, consulte Propiedades específicas del servicio HTTP.

    • Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, introduzca el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

    • Para modificar una propiedad, edite el campo Valor.

    • Para suprimir una propiedad, active la casilla de control a la izquierda del campo Nombre de la propiedad que está suprimiendo y, a continuación, haga clic en el botón Suprimir Propiedades.

  15. Haga clic en Guardar.

    Sugerencia:

    Para volver a la configuración por defecto, haga clic en el botón Cargar Valores por Defecto y, a continuación, en Guardar.
