Para crear un servidor virtual

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Seleccione la configuración que desea cambiar.

  3. En el nodo Configuración, seleccione el nodo Servidores Virtuales.

    Se abrirá la página Servidores Virtuales.

  4. En la página Servidores Virtuales, haga clic en Nuevo.

    Se abrirá la página Nuevo Servidor Virtual.

  5. En el campo ID, introduzca un valor que identificará el servidor virtual de forma interna.

    Este valor no se mostrará a los clientes HTTP. Los nombres de host que se muestran a los clientes HTTP se deben especificar en el campo Hosts.

  6. En el campo Hosts, introduzca los nombres de host o las direcciones IP de la maquina en el que se está ejecutando el servidor.

    Utilice nombres de host reales o virtuales que estén registrados en el servidor DNS de la red (y, en un sistema UNIX, en el archivo /etc/hosts).

    El valor por defecto es el valor de la propiedad del sistema ${com.sun.aas.hostName}.

  7. Seleccione una opción de estado para el nuevo servidor virtual.

    El servidor virtual puede estar activado (activo), o desactivado. El valor por defecto es Activado.

  8. Seleccione una opción SSO para el nuevo servidor virtual.

    Por defecto, un servidor virtual hereda la configuración de single sign-on (SSO) del servicio HTTP. Seleccione Activado o Desactivado para sustituir la configuración del servicio HTTP activando o desactivando el single sign-on para este servidor virtual.

    Los servlets y los archivos JSP soportan el single sign-on en las aplicaciones de GlassFish Server. Esta función permite a varias aplicaciones que requieren la misma información de inicio de sesión compartir esta información de modo que el usuario no tenga que iniciar una sesión por separado para cada aplicación.

  9. Desde la lista desplegable Listeners de Red, seleccione uno o varios listeners de red o deje el campo en blanco.

  10. En la lista desplegable Módulo Web por Defecto, seleccione el módulo que va a responder a todas las solicitudes que no se pueden asignar a otros módulos web desplegados en el servidor virtual.

    Si no se especifica un módulo web por defecto, se utiliza un módulo web con una raíz de contexto vacía. Si no hay ningún módulo web con una raíz de contexto vacía, se crea y utiliza un módulo web por defecto del sistema.

  11. En el campo Archivo Log, introduzca el nombre de ruta de acceso del archivo en el que aparecerán los mensajes de registro de este servidor virtual.

    Por defecto, los mensajes de registro se enviarán al log del servidor por defecto, domain-dir/logs/server.log.

  12. En el campo Raíz de Documento, introduzca la ruta de acceso absoluta al directorio raíz de documentos del servidor.

    Si el campo se deja en blanco, se utilizará domain-dir/docroot.

  13. Seleccione la opción Registro de Acceso para activar o desactivar el registro de acceso.

    Como opción alternativa, puede especificar si el registro de acceso debe heredarse del servicio HTTP. Por defecto, se hereda del servicio HTTP.

  14. En el campo Directorio, introduzca la ruta de acceso de directorio absoluta en los logs de acceso del servidor.

    El valor por defecto es domain-dir/logs/access.

  15. En la sección Propiedades Adicionales, especifique propiedades adicionales del servidor virtual.

    Para obtener más información sobre las propiedades disponibles, consulte Propiedades específicas de servidores virtuales.

    Para agregar una propiedad, haga clic en el botón Agregar Propiedad. En la fila en blanco que aparece, introduzca el nombre de propiedad en el campo Nombre y escriba el valor de propiedad en el campo Valor.

  16. Haga clic en Aceptar.
