Définition de l'intervalle entre points clés

Les opérations de point clé permettent de compresser les fichiers journaux des transactions. L'intervalle entre points clés désigne le nombre de transactions effectuées entre les opérations de point clé dans le journal. Les opérations de point clé permettent de réduire la taille des fichiers journaux des transactions. Plus le nombre d'intervalles entre points clés est élevé, plus la taille des fichiers journaux est importante, ce qui peut altérer les performances. Un nombre plus faible d'intervalles entre points clés (par exemple, 256) génère des fichiers journaux plus petits, mais diminue légèrement les performances en raison du plus grand nombre d'opérations de point clé.

  1. Dans l'arborescence de navigation, développez le noeud Configuration.

  2. Sélectionnez l'instance à configurer :

    • Pour configurer une instance spécifique, sélectionnez le noeud de configuration de l'instance. Par exemple, pour l'instance par défaut, server, sélectionnez le noeud server-config.

    • Pour configurer les paramètres par défaut pour toutes les instances, sélectionnez le noeud default-config.

  3. Sélectionnez le noeud Service de transaction.

    La page Service de transaction apparaît.

  4. Dans le champ Intervalle de point clé, saisissez le nombre de transactions entre les opérations de point clé.

    La valeur par défaut est 65 536.

  5. Cliquez sur Enregistrer.
