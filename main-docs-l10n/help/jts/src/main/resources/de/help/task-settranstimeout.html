So wird ein Wert für das Transaktionstimeout festgelegt

Standardmäßig ist auf dem Server kein Timeout für Transaktionen festgelegt. Das heißt, der Server wartet eine unbegrenzte Zeit, bis eine Transaktion abgeschlossen ist. Wenn Sie einen Timeoutwert für Transaktionen festlegen, setzt GlassFish Server die Transaktion zurück, wenn sie nicht innerhalb des konfigurierten Zeitraums abgeschlossen werden kann.

  1. Blenden Sie in der Navigationsbaumstruktur den Knoten "Konfiguration" ein.

  2. Wählen Sie die zu konfigurierende Instanz:

    • Um eine bestimmte Instanz zu konfigurieren, wählen Sie den Konfigurationsknoten der Instanz. Beispiel: Für die Standardinstanz server wählen Sie den Knoten server-config.

    • Um die Standardeinstellungen für alle Instanzen zu konfigurieren, wählen Sie den Knoten default-config.

  3. Wählen Sie den Knoten "Transaktionsservice".

    Die Seite "Transaktionsservice" wird geöffnet.

  4. Geben Sie im Feld "Transaktionstimeout" den Zeitraum (in Sekunden) bis zum Timeout für die Transaktion an.

    Der Standardwert ist 0. Mit diesem Wert ist das Timeout bei Transaktionen deaktiviert.

  5. Klicken Sie auf "Speichern".

  6. Starten Sie GlassFish Server neu.
