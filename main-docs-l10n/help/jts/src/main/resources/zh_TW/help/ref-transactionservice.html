作業事件服務

使用「作業事件服務」頁面可組態作業事件復原、逾時與記錄日誌。

「作業事件服務」頁面包含下列選項。

載入預設值

用於將具有預設值的設定復原為設定值的按鈕。沒有預設值的設定則保持不變。

組態名稱

套用此頁面上之設定的組態名稱。本欄位是唯讀的。

在重新啟動時

若選取「已啟用」核取方塊，當伺服器重新啟動時，GlassFish Server 會嘗試復原未完成的作業事件。此選項預設為停用。

作業事件逾時

伺服器在倒回未完成之作業事件前要等待的秒數。預設值為 0，表示伺服器會一直等待作業事件完成。

重試逾時

GlassFish Server 嘗試連線至無法連線的伺服器的秒數。預設值為 600 (10 分鐘)。

作業事件日誌位置

保存伺服器日誌的目錄。作業事件日誌會保存在此欄位指定之目錄的 tx 子目錄中。預設值為「網域屬性」頁面上的「日誌根目錄」欄位所指定的目錄，頁面位於「GlassFish Server」節點的「進階」標籤之下。

啟發式決策

決定要確認還是倒回與無法連接伺服器有關的作業事件。預設值為「倒回」。確認不確認的作業事件會破壞應用程式的資料完整性。

關鍵點間隔

壓縮作業事件日誌檔之關鍵點作業間的作業事件數。預設值為 65,536。
