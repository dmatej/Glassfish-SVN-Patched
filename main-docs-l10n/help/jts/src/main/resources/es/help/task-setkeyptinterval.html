Para definir el intervalo de punto clave

Las operaciones de puntos clave comprimen el archivo log de transacciones. El intervalo de punto clave es el número de transacciones entre las operaciones de puntos clave en el log. Las operaciones de punto clave pueden reducir el tamaño de los archivos log de transacciones. Un número elevado de intervalos de puntos clave puede tener como resultado archivos log de transacciones mayores pero menos operaciones de puntos clave y potencialmente un rendimiento mejor. Un intervalo de punto clave más pequeño (por ejemplo, 256) originará archivos log más pequeños, pero se verá afectado el rendimiento debido a la mayor frecuencia de operaciones de puntos clave.

  1. En el árbol de navegación, amplíe el nodo Configuración.

  2. Seleccione la instancia que va a configurar:

    • Para configurar una instancia determinada, seleccione el nodo de configuración de la instancia. Por ejemplo, para la instancia por defecto server, seleccione el nodo server-config.

    • Para configurar los valores por defecto para todas las instancias, seleccione el nodo default-config.

  3. Seleccione el nodo Servicio de Transacción.

    Se abrirá la página Servicio de Transacción.

  4. En el campo Intervalo de Punto Clave, escriba el número de transacciones entre las operaciones de puntos clave.

    El valor por defecto es 65536.

  5. Haga clic en Guardar.
