设置关键点间隔

关键点操作可以压缩事务处理日志文件。关键点间隔是日志中关键点操作之间的事务处理数量。关键点操作可以减小事务处理日志文件的大小。关键点间隔数越大，事务处理日志文件也越大，但关键点操作较少，性能可能更佳。关键点间隔越小 (例如，256)，日志文件也越小，而同时由于关键点操作较为频繁，性能会略微降低。

  1. 在导航树中，展开“配置”节点。

  2. 选择要配置的实例：

    • 要配置特定的实例，请选择该实例的配置节点。例如，对于默认实例 server，请选择 server-config 节点。

    • 要配置所有实例的默认设置，请选择 default-config 节点。

  3. 选择“事务处理服务”节点。

    此时将打开“事务处理服务”页。

  4. 在“关键点间隔”字段中，键入关键点操作之间的事务处理数量。

    默认值为 65,536。

  5. 单击“保存”。
