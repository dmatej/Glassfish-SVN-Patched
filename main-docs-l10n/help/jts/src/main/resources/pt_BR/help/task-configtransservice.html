Para Configurar o Serviço de Transação

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Selecione a instância que será configurada:

    • Para configurar uma instância específica, selecione o nó de configuração da instância. Por exemplo, na instância default, server, selecione o nó server-config.

    • Para definir as definições default de todas as instâncias, selecione o nó default-config.

  3. Selecione o nó Serviço de Transação.

    A página Serviço de Transação é aberta.

  4. Na página Serviço de Transação, marque a caixa de seleção Ativado ao Reiniciar para ativar a recuperação de transações incompletas quando o servidor é reiniciado.

    Por default, essa opção está desativada.

  5. No campo Time-out da Transação, digite o número de segundos que o servidor aguarda antes de reverter uma transação que não foi concluída.

    O valor 0 desativa os time-outs da transação. O valor default é 0, significando que o servidor aguarda indefinidamente que uma transação seja concluída.

  6. No campo Time-out para Novas Tentativas, digite o número de segundos que o GlassFish Server tenta se conectar a um servidor inacessível.

    O valor default é 600 (10 minutos).

  7. No campo Localização do Log de Transações, digite o diretório em que os logs do servidor são mantidos.

    Os logs de transação são mantidos no subdiretório tx do diretório especificado neste campo. O valor default é o diretório especificado pelo campo Raiz de Logs da página Atributos de Domínio, que está localizada na guia Avançado do nó do GlassFish Server.

  8. Na lista drop-down Decisão heurística, selecione a política a ser seguida se o resultado de uma transação distribuída não puder ser determinado pelo fato de outros participantes estarem inacessíveis.

    A menos que haja um bom motivo para selecionar Confirmar, deixe a Decisão Heurística definida como Reversão (o default). A confirmação de transações indeterminadas pode comprometer a integridade dos dados da aplicação.

  9. No campo Intervalo de Keypoint, digite o número de transações entre as operações de keypoint.

    As operações de keypoint compactam o arquivo de log da transação. O valor default é 65536.

  10. Na seção Propriedades Adicionais, especifique as propriedades adicionais.

    Para obter uma descrição das propriedades disponíveis, consulte Propriedades Específicas para o Serviço de Transação.

    • Para adicionar uma propriedade, clique no botão Adicionar Propriedade. Na linha em branco que for exibida, digite o nome da propriedade no campo Nome e digite o valor da propriedade no campo Valor.

    • Para modificar uma propriedade, edite o campo Valor da propriedade.

    • Para deletar uma propriedade, marque a caixa de seleção à esquerda do campo Nome da Propriedade a ser deletada. Clique no botão Deletar Propriedades.

  11. Clique em Salvar.

    Dica:

    Para voltar às definições default, clique no botão Carregar Defaults e depois em Salvar.

  12. Reinicie o GlassFish Server.
