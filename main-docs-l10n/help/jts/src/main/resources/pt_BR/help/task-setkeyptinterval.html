Para Definir o Intervalo de Keypoint

As operações de keypoint compactam o arquivo de log da transação. O intervalo de keypoint é o número de transações entre as operações de keypoint no log. As operações de ponto-chave podem reduzir o tamanho dos arquivos de log da transação. Um número maior de intervalos de keypoint resulta em arquivos maiores de log de transação, mas em menos operações de ponto-chave e desempenho potencialmente melhor. Um intervalo menor de keypoint (por exemplo, 256) resulta em arquivos de log menores, mas em desempenho ligeiramente reduzido devido à frequência maior de operações de ponto-chave.

  1. Na árvore de navegação, expanda o nó Configuração.

  2. Selecione a instância que será configurada:

    • Para configurar uma instância específica, selecione o nó de configuração da instância. Por exemplo, na instância default, server, selecione o nó server-config.

    • Para definir as definições default de todas as instâncias, selecione o nó default-config.

  3. Selecione o nó Serviço de Transação.

    A página Serviço de Transação é aberta.

  4. No campo Intervalo de Keypoint, digite o número de transações entre as operações de keypoint.

    O valor default é 65536.

  5. Clique em Salvar.
